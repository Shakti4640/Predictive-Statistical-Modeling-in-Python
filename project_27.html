Project 27: Interpreting a Black Box — SHAP Values on a Churn Model

SHAP Shapley Values Interpretability Waterfall Plot Beeswarm Plot Global vs Local

Difficulty: Medium  |  Builds on: Projects 23, 21

1 · Hypothetical Problem

  • Your Gradient Boosting churn model (Project 23) achieves 92% accuracy — management is impressed
  • But the VP asks: "Why does the model predict THIS customer will churn? What should we do to retain them?"
  • Feature importance (Project 21) tells you which features matter globally — but not for a specific customer
  • You need per-prediction explanations: "Customer #4521 will churn because their monthly charge is $95 (pushes toward churn) and they have no tech support (pushes toward churn)"
  • Goal → use SHAP values to explain individual predictions AND understand global model behavior

2 · How to Approach It

  1. From Project 21: feature_importances_ gives global importance but no direction (does high tenure increase or decrease churn?)
  2. From Project 23: Gradient Boosting is a black box — hundreds of trees combined
  3. SHAP (SHapley Additive exPlanations) assigns each feature a contribution to each prediction
  4. SHAP values are grounded in game theory — they fairly distribute the prediction among features
  5. Use waterfall plots for individual explanations and beeswarm plots for global patterns

3 · How Can We Solve It

  1. Train a Gradient Boosting model on churn data (same setup as Project 23)
  2. Compute SHAP values for all predictions
  3. Create waterfall plots — explain why a specific customer is predicted to churn
  4. Create beeswarm plots — see how each feature affects predictions across all customers
  5. Compare SHAP-based importance with the built-in feature_importances_ from Project 21
  6. Show how SHAP reveals feature interactions that simple importance misses

4 · What Should Do

  • ✅ Use TreeExplainer for tree-based models — it's exact and fast (no approximation needed)
  • ✅ Always show the base value (expected prediction) alongside SHAP values — they're relative to it
  • ✅ Use waterfall plots for individual explanations — stakeholders understand "this pushed churn up"
  • ✅ Use beeswarm plots for global understanding — shows both importance AND direction
  • ✅ Validate SHAP explanations against domain knowledge — if they don't make sense, the model may be wrong

5 · What Should NOT Do

  • ❌ Do NOT confuse SHAP importance with causation — "high monthly charge pushes toward churn" doesn't mean lowering it will prevent churn
  • ❌ Do NOT use KernelExplainer when TreeExplainer is available — KernelExplainer is slow and approximate
  • ❌ Do NOT ignore feature interactions — SHAP can reveal that two features together have a different effect than individually
  • ❌ Do NOT present SHAP values without the base value — SHAP values are deviations from the average prediction
  • ❌ Do NOT assume SHAP replaces model validation — a well-explained bad model is still a bad model (Project 15)

6 · When to Use What

  • 🔹 SHAP TreeExplainer → for tree-based models (Random Forest, Gradient Boosting, XGBoost) — fast and exact
  • 🔹 SHAP KernelExplainer → for any model (logistic regression, SVM, neural nets) — slow but model-agnostic
  • 🔹 Feature importance (Project 21) → quick global overview, but no direction or per-prediction detail
  • 🔹 Coefficients (Project 10) → for linear/logistic models — already interpretable, SHAP adds little
  • 🔹 SHAP interaction values → when you suspect features interact (e.g., tenure × contract type)

7 · When NOT to Use

  • 🚫 When the model is already interpretable — logistic regression coefficients (Project 10) are self-explanatory
  • 🚫 When you need causal explanations — SHAP shows correlation-based contributions, not causal effects
  • 🚫 When computation time is critical — KernelExplainer on large datasets can take hours
  • 🚫 When features are highly correlated — SHAP can distribute importance arbitrarily among correlated features

8 · How It Only Works — Internal Mechanics

Shapley Values — The Game Theory Foundation

  • From cooperative game theory: how to fairly divide a payout among players
  • Each feature is a "player" — the "payout" is the prediction minus the average prediction
  • Shapley value of feature i = average marginal contribution of feature i across all possible feature coalitions
  • For n features, there are 2ⁿ coalitions — exact computation is exponential

The SHAP Equation

  • f(x) = base_value + SHAP₁ + SHAP₂ + ... + SHAPₙ
  • base_value = E[f(X)] = average model prediction across all training data
  • Each SHAP value tells you how much that feature pushed the prediction away from the average
  • Positive SHAP → pushes prediction higher. Negative SHAP → pushes prediction lower
  • They always sum exactly to f(x) - base_value — this is the "additive" property

TreeExplainer — Why It's Fast

  • For tree-based models, SHAP values can be computed exactly in O(TLD²) time
  • T = number of trees, L = max leaves, D = max depth
  • It exploits the tree structure — no need to enumerate all 2ⁿ coalitions
  • This is why TreeExplainer is preferred for Random Forest (Project 21) and Gradient Boosting (Project 23)

Global vs Local Explanations

  • Local: "Why did the model predict churn for THIS customer?" → waterfall plot for one prediction
  • Global: "What features matter most across ALL customers?" → beeswarm plot or mean |SHAP|
  • SHAP bridges both — compute local explanations, then aggregate for global understanding
  • This is more informative than feature_importances_ (Project 21) which only gives global importance without direction

SHAP vs Feature Importance (Project 21)

  • Feature importance: how much a feature reduces impurity on average — no direction, no per-prediction detail
  • SHAP: how much each feature contributes to each prediction — direction + magnitude + per-prediction
  • Feature importance can be biased toward high-cardinality features — SHAP is theoretically fair
  • SHAP is slower but much more informative

9 · Common Pitfalls and Failure Modes

  • ⚠️ Confusing SHAP with causation — "high monthly charge → churn" is correlation, not causation
  • ⚠️ Using KernelExplainer on tree models — TreeExplainer is exact and 100x faster
  • ⚠️ Ignoring correlated features — SHAP may split importance between correlated features unpredictably
  • ⚠️ Showing raw SHAP values to non-technical stakeholders — use waterfall plots instead
  • ⚠️ Computing SHAP on the entire dataset — for large datasets, use a representative sample

10 · Scalability Considerations

  • TreeExplainer: O(TLD²) per prediction — fast for typical models (hundreds of trees, depth 3-8)
  • KernelExplainer: O(2ⁿ) approximated via sampling — very slow for many features
  • For large datasets, compute SHAP on a sample (e.g., 1000 rows) — global patterns are stable
  • SHAP values can be precomputed and cached — compute once, visualize many times
  • For production, consider computing SHAP only for flagged predictions (e.g., high-risk customers)

11 · Interpretability Considerations

  • ✅ Waterfall plots are intuitive: "starting from average, monthly_charge pushed +0.15, tenure pushed -0.08"
  • ✅ Beeswarm plots show both importance AND direction — superior to bar charts
  • ✅ SHAP values are additive — they sum to the prediction, which is mathematically satisfying
  • ⚠️ Stakeholders may interpret SHAP as causal — always clarify it shows model behavior, not real-world causation
  • ⚠️ For regulated industries (finance, healthcare), SHAP explanations may need legal review

12 · Complete Working Code

import numpy as np
import pandas as pd
from sklearn.ensemble import GradientBoostingClassifier
from sklearn.model_selection import train_test_split
from sklearn.metrics import classification_report
from sklearn.datasets import make_classification
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt
import warnings
warnings.filterwarnings('ignore')

# ─── Step 1: Customer churn data (same scenario as Project 23) ───
np.random.seed(42)
X, y = make_classification(n_samples=2000, n_features=10, n_informative=7,
                           n_redundant=2, flip_y=0.05, random_state=42)
feature_names = ['monthly_charge', 'tenure', 'contract_length', 'tech_support_calls',
                 'internet_speed', 'payment_method', 'total_charges', 'num_products',
                 'online_security', 'paperless_billing']
X_df = pd.DataFrame(X, columns=feature_names)

X_train, X_test, y_train, y_test = train_test_split(
    X_df, y, test_size=0.25, stratify=y, random_state=42)

print("=== Churn Model for SHAP Analysis ===")
print(f"  Train: {len(y_train)}, Test: {len(y_test)}")
print(f"  Churn rate: {np.mean(y):.1%}")

# ─── Step 2: Train Gradient Boosting (from Project 23) ───
gb = GradientBoostingClassifier(n_estimators=200, learning_rate=0.1,
                                 max_depth=4, random_state=42)
gb.fit(X_train, y_train)
y_pred = gb.predict(X_test)
print(f"\n{classification_report(y_test, y_pred, target_names=['Retained', 'Churned'])}")

# ─── Step 3: Built-in feature importance (from Project 21) ───
print("=== Built-in Feature Importance (Project 21 style) ===")
importances = gb.feature_importances_
sorted_idx = np.argsort(importances)[::-1]
for i in sorted_idx:
    bar = "█" * int(importances[i] * 50)
    print(f"  {feature_names[i]:>22}: {importances[i]:.4f} {bar}")
print("  → Shows importance but NOT direction (does high tenure increase or decrease churn?)")

# ─── Step 4: SHAP values ───
try:
    import shap

    explainer = shap.TreeExplainer(gb)
    shap_values = explainer.shap_values(X_test)

    print(f"\n=== SHAP Values Computed ===")
    print(f"  Shape: {shap_values.shape} (samples × features)")
    print(f"  Base value (E[f(x)]): {explainer.expected_value:.4f}")
    print(f"  This is the average model output — SHAP values are deviations from this")

    # ─── Step 5: Global importance — mean |SHAP| ───
    mean_abs_shap = np.abs(shap_values).mean(axis=0)
    shap_sorted = np.argsort(mean_abs_shap)[::-1]

    print(f"\n=== SHAP-based Global Feature Importance ===")
    for i in shap_sorted:
        bar = "█" * int(mean_abs_shap[i] * 30)
        print(f"  {feature_names[i]:>22}: {mean_abs_shap[i]:.4f} {bar}")

    # ─── Step 6: Compare built-in vs SHAP importance ───
    fig, axes = plt.subplots(1, 2, figsize=(14, 5))

    axes[0].barh(range(len(feature_names)), importances[sorted_idx], color='#58a6ff', alpha=0.8)
    axes[0].set_yticks(range(len(feature_names)))
    axes[0].set_yticklabels([feature_names[i] for i in sorted_idx])
    axes[0].set_xlabel('Importance')
    axes[0].set_title('Built-in Feature Importance\n(no direction info)')
    axes[0].grid(True, alpha=0.2, axis='x')

    axes[1].barh(range(len(feature_names)), mean_abs_shap[shap_sorted], color='#f0883e', alpha=0.8)
    axes[1].set_yticks(range(len(feature_names)))
    axes[1].set_yticklabels([feature_names[i] for i in shap_sorted])
    axes[1].set_xlabel('Mean |SHAP Value|')
    axes[1].set_title('SHAP Feature Importance\n(with direction via beeswarm)')
    axes[1].grid(True, alpha=0.2, axis='x')

    plt.tight_layout()
    plt.savefig('project_27_importance_comparison.png', dpi=100, bbox_inches='tight')
    print("\n✅ Importance comparison saved as project_27_importance_comparison.png")

    # ─── Step 7: Beeswarm plot — global explanation with direction ───
    fig, ax = plt.subplots(figsize=(10, 6))
    shap.summary_plot(shap_values, X_test, feature_names=feature_names, show=False)
    plt.tight_layout()
    plt.savefig('project_27_beeswarm.png', dpi=100, bbox_inches='tight')
    plt.close()
    print("✅ Beeswarm plot saved as project_27_beeswarm.png")

    # ─── Step 8: Waterfall plot — local explanation for one customer ───
    customer_idx = 0
    print(f"\n=== Local Explanation: Customer #{customer_idx} ===")
    print(f"  Actual: {'Churned' if y_test.iloc[customer_idx] else 'Retained'}")
    print(f"  Predicted probability: {gb.predict_proba(X_test.iloc[[customer_idx]])[0][1]:.3f}")
    print(f"  Base value: {explainer.expected_value:.4f}")
    print(f"  SHAP breakdown:")

    customer_shap = shap_values[customer_idx]
    feature_order = np.argsort(np.abs(customer_shap))[::-1]
    for i in feature_order:
        direction = "↑ churn" if customer_shap[i] > 0 else "↓ retain"
        print(f"    {feature_names[i]:>22}: {customer_shap[i]:>+.4f} ({direction})")
    print(f"    {'Sum':>22}: {customer_shap.sum():>+.4f}")
    print(f"    {'Final prediction':>22}: {explainer.expected_value + customer_shap.sum():.4f}")

    # Waterfall plot
    fig, ax = plt.subplots(figsize=(10, 6))
    shap_explanation = shap.Explanation(
        values=shap_values[customer_idx],
        base_values=explainer.expected_value,
        data=X_test.iloc[customer_idx].values,
        feature_names=feature_names
    )
    shap.plots.waterfall(shap_explanation, show=False)
    plt.tight_layout()
    plt.savefig('project_27_waterfall.png', dpi=100, bbox_inches='tight')
    plt.close()
    print("✅ Waterfall plot saved as project_27_waterfall.png")

    # ─── Step 9: Compare two customers — why different predictions? ───
    churned_idx = np.where(y_test.values == 1)[0][0]
    retained_idx = np.where(y_test.values == 0)[0][0]

    print(f"\n=== Comparing Two Customers ===")
    for label, idx in [("Churned", churned_idx), ("Retained", retained_idx)]:
        print(f"\n  Customer #{idx} ({label}):")
        cust_shap = shap_values[idx]
        top3 = np.argsort(np.abs(cust_shap))[::-1][:3]
        for i in top3:
            print(f"    {feature_names[i]:>22}: SHAP={cust_shap[i]:>+.4f}, "
                  f"Value={X_test.iloc[idx, i]:.2f}")

    # ─── Step 10: SHAP dependence plot — feature interactions ───
    fig, axes = plt.subplots(1, 2, figsize=(14, 5))

    shap.dependence_plot(feature_names[0], shap_values, X_test,
                          feature_names=feature_names, ax=axes[0], show=False)
    axes[0].set_title(f'SHAP Dependence: {feature_names[0]}')
    axes[0].grid(True, alpha=0.2)

    shap.dependence_plot(feature_names[1], shap_values, X_test,
                          feature_names=feature_names, ax=axes[1], show=False)
    axes[1].set_title(f'SHAP Dependence: {feature_names[1]}')
    axes[1].grid(True, alpha=0.2)

    plt.tight_layout()
    plt.savefig('project_27_dependence.png', dpi=100, bbox_inches='tight')
    plt.close()
    print("✅ Dependence plots saved as project_27_dependence.png")

except ImportError:
    print("\n⚠️ shap not installed. Install with: pip install shap")
    print("Showing manual SHAP-like analysis using permutation importance instead...")

    from sklearn.inspection import permutation_importance
    perm_imp = permutation_importance(gb, X_test, y_test, n_repeats=10, random_state=42)

    print("\n=== Permutation Importance (SHAP alternative) ===")
    sorted_idx = perm_imp.importances_mean.argsort()[::-1]
    for i in sorted_idx:
        print(f"  {feature_names[i]:>22}: {perm_imp.importances_mean[i]:.4f} "
              f"± {perm_imp.importances_std[i]:.4f}")

    fig, ax = plt.subplots(figsize=(8, 5))
    ax.barh(range(len(feature_names)),
            perm_imp.importances_mean[sorted_idx], color='#f0883e', alpha=0.8)
    ax.set_yticks(range(len(feature_names)))
    ax.set_yticklabels([feature_names[i] for i in sorted_idx])
    ax.set_xlabel('Permutation Importance')
    ax.set_title('Feature Importance (Permutation-based)')
    ax.grid(True, alpha=0.2, axis='x')
    plt.tight_layout()
    plt.savefig('project_27_permutation_importance.png', dpi=100, bbox_inches='tight')
    print("✅ Permutation importance saved as project_27_permutation_importance.png")

13 · Key Takeaway + Bridge to Next Project

Key Takeaway

  • SHAP values explain individual predictions: "this customer churns because of X, Y, Z"
  • They're grounded in game theory — each feature gets a fair share of the prediction
  • SHAP values sum to the prediction minus the base value — mathematically consistent
  • TreeExplainer is fast and exact for tree-based models (Projects 21, 23)
  • Waterfall plots explain one prediction; beeswarm plots explain the whole model
  • SHAP shows correlation, not causation — always clarify this to stakeholders
