Project 34: Anomaly Detection in Server Logs — Isolation Forest

Anomaly Detection Isolation Forest Contamination One-Class SVM Outlier Detection

Difficulty: Hard  |  Builds on: Projects 14, 24

1 · Hypothetical Problem

  • You're a DevOps engineer monitoring 50 servers generating millions of log entries per day
  • Each log entry has features: response time, CPU usage, memory usage, request count, error rate
  • 99% of entries are normal — but the 1% that are anomalous signal potential outages, attacks, or misconfigurations
  • You have no labeled anomalies — nobody has manually tagged which entries are "bad"
  • Goal → automatically detect unusual server behavior without labeled training data

2 · How to Approach It

  1. Recall from Project 14: K-Means finds clusters of similar data — but anomalies are points that don't fit any cluster
  2. Anomaly detection is fundamentally different from classification — we model "normal" and flag deviations
  3. Isolation Forest takes a unique approach: anomalies are easier to isolate (separate) than normal points
  4. No labels needed — this is unsupervised, like K-Means (Project 14) and PCA (Project 24)
  5. Compare Isolation Forest with One-Class SVM and statistical approaches to understand tradeoffs

3 · How Can We Solve It

  1. Generate synthetic server log data with injected anomalies
  2. Train Isolation Forest on the data (without using anomaly labels)
  3. Evaluate using the known anomaly labels — precision, recall, and the anomaly score distribution
  4. Compare with One-Class SVM and Local Outlier Factor
  5. Visualize the decision boundary and anomaly scores
  6. Tune the contamination parameter and analyze its effect

4 · What Should Do

  • ✅ Set contamination to your best estimate of the anomaly rate — this controls the threshold
  • ✅ Scale features before One-Class SVM (distance-based) — same principle as Project 13
  • ✅ Use decision_function() to get continuous anomaly scores, not just binary labels
  • ✅ Visualize anomaly score distributions — the separation between normal and anomalous scores tells you model quality
  • ✅ Validate with domain experts — anomaly detection is subjective; what's "anomalous" depends on context

5 · What Should NOT Do

  • ❌ Do NOT treat anomaly detection as binary classification — you rarely have balanced labeled data
  • ❌ Do NOT use accuracy as a metric — with 1% anomalies, predicting "all normal" gives 99% accuracy
  • ❌ Do NOT assume Isolation Forest needs scaling — it's tree-based and scale-invariant (unlike One-Class SVM)
  • ❌ Do NOT set contamination=0.5 — anomalies are rare by definition; typical values are 0.01-0.1
  • ❌ Do NOT deploy without a feedback loop — anomaly detectors need periodic retraining as "normal" evolves

6 · When to Use What

  • 🔹 Isolation Forest → general-purpose, fast, handles high dimensions well, no scaling needed
  • 🔹 One-Class SVM → when normal data forms a tight cluster; needs scaling; slower on large data
  • 🔹 Local Outlier Factor (LOF) → when anomalies are defined by local density, not global isolation
  • 🔹 Statistical methods (Z-score) → when features are univariate and approximately Gaussian
  • 🔹 Supervised classification → when you have labeled anomalies — treat as imbalanced classification (Project 19)

7 · When NOT to Use

  • 🚫 When you have plenty of labeled anomalies — use supervised learning with class imbalance handling (Project 19)
  • 🚫 When "normal" behavior changes frequently — the model's definition of normal becomes stale
  • 🚫 When anomalies are subtle and context-dependent — Isolation Forest detects global outliers, not contextual ones
  • 🚫 When you need to explain WHY something is anomalous — Isolation Forest gives scores, not reasons

8 · How It Only Works — Internal Mechanics

Isolation Forest — Core Idea

  • Key insight: anomalies are few and different — they're easier to isolate with random splits
  • Build a random tree: pick a random feature, pick a random split value, partition the data
  • Normal points need many splits to be isolated (they're surrounded by similar points)
  • Anomalies need few splits to be isolated (they're far from the crowd)
  • The path length (number of splits to isolate a point) is the anomaly score

Building an Isolation Tree

  • Step 1: Randomly select a feature
  • Step 2: Randomly select a split value between the feature's min and max
  • Step 3: Partition data into left (≤ split) and right (> split)
  • Step 4: Recurse until each point is isolated or max depth is reached
  • Step 5: Record the path length for each point
  • Build many trees (forest) and average the path lengths — shorter average = more anomalous

Anomaly Score Normalization

  • Raw path length depends on dataset size — normalize using the expected path length of a BST
  • score(x) = 2^(-E[h(x)] / c(n)) where c(n) is the average path length of unsuccessful search in BST
  • Score close to 1 → anomaly (short path); score close to 0.5 → normal; score close to 0 → very normal
  • The contamination parameter sets the threshold on this score to classify as anomaly

One-Class SVM — Comparison

  • Learns a boundary around the normal data in a high-dimensional kernel space
  • Points outside the boundary are anomalies
  • Uses the RBF kernel (like SVM in classification) — sensitive to the gamma parameter
  • Slower than Isolation Forest: O(n² to n³) vs O(n log n)
  • Needs feature scaling (distance-based) — Isolation Forest doesn't (tree-based)

decision_function() — Continuous Scores

  • predict() returns -1 (anomaly) or 1 (normal) — binary, uses the contamination threshold
  • decision_function() returns continuous scores — more negative = more anomalous
  • Use continuous scores to rank anomalies by severity — more useful than binary labels
  • You can set your own threshold on decision_function scores for custom sensitivity

9 · Common Pitfalls and Failure Modes

  • ⚠️ Wrong contamination — too high flags normal data; too low misses real anomalies; estimate from domain knowledge
  • ⚠️ Training on data with anomalies — Isolation Forest is somewhat robust, but heavy contamination in training data degrades performance
  • ⚠️ High-dimensional irrelevant features — random splits on noise features dilute the signal; use PCA (Project 24) first
  • ⚠️ Evaluating without labels — without ground truth, you can't compute precision/recall; rely on domain expert review
  • ⚠️ Treating all anomalies equally — use decision_function scores to prioritize the most extreme anomalies

10 · Scalability Considerations

  • Isolation Forest: O(n × t × log(ψ)) where t = number of trees, ψ = subsample size — very fast
  • Default subsample size is 256 — Isolation Forest doesn't need all data; anomalies are isolated quickly regardless
  • One-Class SVM: O(n²) to O(n³) — impractical for datasets over 100K rows
  • LOF: O(n² × k) for k neighbors — also slow on large data; use approximate nearest neighbors
  • For streaming data, consider incremental anomaly detection or sliding window approaches

11 · Interpretability Considerations

  • ✅ Anomaly scores are intuitive: "this server's behavior is in the 99th percentile of abnormality"
  • ✅ You can inspect which features have extreme values for flagged anomalies
  • ⚠️ Isolation Forest doesn't tell you WHY a point is anomalous — only that it's easy to isolate
  • ⚠️ For explainability, compute feature-wise z-scores for flagged anomalies to identify the unusual dimensions
  • ⚠️ SHAP (Project 27) can be applied to Isolation Forest but interpretation is less straightforward than for classifiers

12 · Complete Working Code

import numpy as np
from sklearn.ensemble import IsolationForest
from sklearn.svm import OneClassSVM
from sklearn.neighbors import LocalOutlierFactor
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import precision_score, recall_score, f1_score, confusion_matrix
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

# ─── Step 1: Simulate server log data ───
np.random.seed(42)
n_normal = 5000
n_anomaly = 100

# Normal server behavior
normal_data = np.column_stack([
    np.random.normal(200, 30, n_normal),     # response_time_ms
    np.random.normal(45, 10, n_normal),      # cpu_percent
    np.random.normal(60, 8, n_normal),       # memory_percent
    np.random.normal(500, 100, n_normal),    # requests_per_min
    np.random.normal(0.02, 0.01, n_normal),  # error_rate
])

# Anomalous behavior (various types)
anomaly_data = np.column_stack([
    np.random.normal(800, 200, n_anomaly),   # high response time
    np.random.normal(92, 5, n_anomaly),      # high CPU
    np.random.normal(88, 5, n_anomaly),      # high memory
    np.random.normal(50, 30, n_anomaly),     # low requests (possible outage)
    np.random.normal(0.15, 0.05, n_anomaly), # high error rate
])

X = np.vstack([normal_data, anomaly_data])
y_true = np.array([1] * n_normal + [-1] * n_anomaly)  # 1=normal, -1=anomaly

# Shuffle
shuffle_idx = np.random.permutation(len(X))
X, y_true = X[shuffle_idx], y_true[shuffle_idx]

feature_names = ['response_time', 'cpu_pct', 'memory_pct', 'req_per_min', 'error_rate']
print(f"=== Server Log Dataset ===")
print(f"  Total: {len(X)}, Normal: {n_normal}, Anomalies: {n_anomaly}")
print(f"  Anomaly rate: {n_anomaly / len(X):.1%}")

# ─── Step 2: Isolation Forest ───
iso_forest = IsolationForest(
    n_estimators=200, contamination=n_anomaly / len(X),
    random_state=42, n_jobs=-1
)
iso_pred = iso_forest.fit_predict(X)
iso_scores = iso_forest.decision_function(X)

print(f"\n=== Isolation Forest ===")
print(f"  Precision: {precision_score(y_true, iso_pred, pos_label=-1):.4f}")
print(f"  Recall:    {recall_score(y_true, iso_pred, pos_label=-1):.4f}")
print(f"  F1:        {f1_score(y_true, iso_pred, pos_label=-1):.4f}")

# ─── Step 3: One-Class SVM ───
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

oc_svm = OneClassSVM(kernel='rbf', gamma='scale', nu=0.02)
svm_pred = oc_svm.fit_predict(X_scaled)
svm_scores = oc_svm.decision_function(X_scaled)

print(f"\n=== One-Class SVM ===")
print(f"  Precision: {precision_score(y_true, svm_pred, pos_label=-1):.4f}")
print(f"  Recall:    {recall_score(y_true, svm_pred, pos_label=-1):.4f}")
print(f"  F1:        {f1_score(y_true, svm_pred, pos_label=-1):.4f}")

# ─── Step 4: Local Outlier Factor ───
lof = LocalOutlierFactor(n_neighbors=20, contamination=n_anomaly / len(X))
lof_pred = lof.fit_predict(X)
lof_scores = lof.negative_outlier_factor_

print(f"\n=== Local Outlier Factor ===")
print(f"  Precision: {precision_score(y_true, lof_pred, pos_label=-1):.4f}")
print(f"  Recall:    {recall_score(y_true, lof_pred, pos_label=-1):.4f}")
print(f"  F1:        {f1_score(y_true, lof_pred, pos_label=-1):.4f}")

# ─── Step 5: Visualization ───
fig, axes = plt.subplots(2, 2, figsize=(14, 12))

# Anomaly score distribution (Isolation Forest)
ax = axes[0, 0]
normal_scores = iso_scores[y_true == 1]
anomaly_scores_vals = iso_scores[y_true == -1]
ax.hist(normal_scores, bins=50, alpha=0.7, color='#3fb950', label='Normal', density=True)
ax.hist(anomaly_scores_vals, bins=30, alpha=0.7, color='#f85149', label='Anomaly', density=True)
ax.set_xlabel('Isolation Forest Decision Score')
ax.set_ylabel('Density')
ax.set_title('Anomaly Score Distribution')
ax.legend()
ax.grid(True, alpha=0.2)

# 2D scatter: response_time vs error_rate
ax = axes[0, 1]
normal_mask = y_true == 1
anomaly_mask = y_true == -1
ax.scatter(X[normal_mask, 0], X[normal_mask, 4], c='#3fb950', alpha=0.3, s=10, label='Normal')
ax.scatter(X[anomaly_mask, 0], X[anomaly_mask, 4], c='#f85149', alpha=0.8, s=30, marker='x', label='Anomaly')
ax.set_xlabel('Response Time (ms)')
ax.set_ylabel('Error Rate')
ax.set_title('Server Logs — True Labels')
ax.legend()
ax.grid(True, alpha=0.2)

# Isolation Forest predictions
ax = axes[1, 0]
pred_normal = iso_pred == 1
pred_anomaly = iso_pred == -1
ax.scatter(X[pred_normal, 0], X[pred_normal, 4], c='#3fb950', alpha=0.3, s=10, label='Predicted Normal')
ax.scatter(X[pred_anomaly, 0], X[pred_anomaly, 4], c='#f85149', alpha=0.8, s=30, marker='x', label='Predicted Anomaly')
ax.set_xlabel('Response Time (ms)')
ax.set_ylabel('Error Rate')
ax.set_title('Isolation Forest Predictions')
ax.legend()
ax.grid(True, alpha=0.2)

# Contamination sensitivity
ax = axes[1, 1]
contamination_values = [0.005, 0.01, 0.02, 0.03, 0.05, 0.08, 0.1, 0.15]
precisions, recalls, f1s = [], [], []

for cont in contamination_values:
    iso_temp = IsolationForest(n_estimators=200, contamination=cont, random_state=42)
    pred_temp = iso_temp.fit_predict(X)
    precisions.append(precision_score(y_true, pred_temp, pos_label=-1))
    recalls.append(recall_score(y_true, pred_temp, pos_label=-1))
    f1s.append(f1_score(y_true, pred_temp, pos_label=-1))

ax.plot(contamination_values, precisions, 'o-', color='#58a6ff', label='Precision')
ax.plot(contamination_values, recalls, 's-', color='#3fb950', label='Recall')
ax.plot(contamination_values, f1s, '^-', color='#f0883e', label='F1')
ax.axvline(x=n_anomaly/len(X), color='#f85149', linestyle='--', alpha=0.5, label=f'True rate ({n_anomaly/len(X):.3f})')
ax.set_xlabel('Contamination Parameter')
ax.set_ylabel('Score')
ax.set_title('Effect of Contamination on Performance')
ax.legend()
ax.grid(True, alpha=0.2)

plt.tight_layout()
plt.savefig('project_34_anomaly_detection.png', dpi=100, bbox_inches='tight')
print(f"\n✅ Anomaly detection plots saved as project_34_anomaly_detection.png")

# ─── Step 6: Feature analysis of detected anomalies ───
print(f"\n=== Feature Profile of Detected Anomalies ===")
detected = X[iso_pred == -1]
normal = X[iso_pred == 1]
print(f"{'Feature':<15} {'Normal Mean':>12} {'Anomaly Mean':>14} {'Z-Score':>10}")
print(f"{'-'*55}")
for i, name in enumerate(feature_names):
    n_mean = normal[:, i].mean()
    n_std = normal[:, i].std()
    a_mean = detected[:, i].mean()
    z = (a_mean - n_mean) / n_std if n_std > 0 else 0
    print(f"{name:<15} {n_mean:>12.2f} {a_mean:>14.2f} {z:>10.2f}")
