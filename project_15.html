Project 15: Cross-Validating a Model — Getting a Trustworthy Score

K-Fold CV StratifiedKFold cross_val_score why single split fails

Difficulty: Easy  |  Builds on: Projects 8, 10, 12

1 · Hypothetical Problem

  • You've built a model and your manager asks: "What's the model's accuracy?"
  • You report 85% from a single train/test split
  • Your colleague runs the same code with a different random seed and gets 78%
  • Which number is right? Neither — a single split is unreliable
  • Goal → get a TRUSTWORTHY performance estimate using cross-validation

2 · How to Approach It

  1. Instead of one train/test split, use K different splits
  2. Each split uses a different portion as the test set
  3. Average the K scores → more reliable estimate
  4. The standard deviation of the K scores tells you how stable the model is
  5. Use StratifiedKFold for classification to maintain class proportions in each fold

3 · How Can We Solve It

  1. Show the instability of single train/test splits (different seeds → different scores)
  2. Implement K-Fold CV manually to understand the mechanics
  3. Use sklearn's cross_val_score for the clean version
  4. Compare KFold vs StratifiedKFold
  5. Use CV to compare multiple models fairly

4 · What Should Do

  • ✅ Use cross-validation for ALL model evaluations — not just a single split
  • ✅ Report mean ± std of CV scores — the std tells you about stability
  • ✅ Use StratifiedKFold for classification — preserves class balance in each fold
  • ✅ Use K=5 or K=10 as default — good balance between bias and variance of the estimate
  • ✅ Use Pipeline inside CV to prevent data leakage from preprocessing (Project 13)

5 · What Should NOT Do

  • ❌ Do NOT report a single train/test split as the final score — it's unreliable
  • ❌ Do NOT use K=n (Leave-One-Out) for large datasets — too slow and high variance
  • ❌ Do NOT use regular KFold for imbalanced classification — use StratifiedKFold
  • ❌ Do NOT tune hyperparameters using CV scores and then report those same scores as final performance
  • ❌ Do NOT forget that CV is for EVALUATION, not for training the final model

6 · When to Use What

  • 🔹 K-Fold CV (K=5) → default for most problems
  • 🔹 StratifiedKFold → for classification, especially with imbalanced classes
  • 🔹 K=10 → when you want a more precise estimate and have enough data
  • 🔹 RepeatedKFold → for even more reliable estimates (repeats the entire CV process)
  • 🔹 Leave-One-Out → only for very small datasets (<50 samples)

7 · When NOT to Use

  • 🚫 Leave-One-Out is too slow for large datasets and has high variance
  • 🚫 Regular KFold can create folds with no positive examples in imbalanced data
  • 🚫 CV doesn't work well for time series — use TimeSeriesSplit instead (Project 25)
  • 🚫 CV scores are for evaluation — the final model should be trained on ALL data

8 · How It Only Works — Internal Mechanics

K-Fold Cross-Validation

  1. Split data into K equal-sized folds
  2. For each fold i (i = 1 to K):
    • Use fold i as the test set
    • Use all other folds as the training set
    • Train the model, evaluate on fold i, record the score
  3. Final score = mean of K scores
  4. Uncertainty = standard deviation of K scores

Why K=5 or K=10?

  • K=2: each fold uses only 50% for training — high bias
  • K=5: each fold uses 80% for training — good balance
  • K=10: each fold uses 90% for training — lower bias, more computation
  • K=n (LOO): each fold uses n-1 for training — lowest bias, highest variance, very slow

StratifiedKFold

  • Ensures each fold has the same class proportions as the full dataset
  • Critical for imbalanced data: if 10% positive, each fold should have ~10% positive
  • Regular KFold might create a fold with 0% positive — useless for evaluation

CV for Model Comparison

  • Run CV on Model A and Model B with the SAME folds
  • Compare mean scores — the model with higher mean CV score is better
  • Check if the difference is larger than the standard deviations — statistical significance

9 · Common Pitfalls and Failure Modes

  • ⚠️ Data leakage in CV — scaling before CV leaks information; use Pipeline
  • ⚠️ Reporting CV score as final performance after tuning — need a held-out test set
  • ⚠️ Using CV for time series — future data leaks into training folds
  • ⚠️ Ignoring the standard deviation — mean=85% ± 10% is very different from mean=85% ± 1%
  • ⚠️ Training final model on only one fold — after CV, retrain on ALL data

10 · Scalability Considerations

  • K-Fold CV trains the model K times — K× the cost of a single train
  • For expensive models, K=5 is preferred over K=10
  • Can parallelize folds with n_jobs=-1 in cross_val_score
  • For very large datasets, a single 80/20 split may be sufficient (the variance is low)

11 · Interpretability Considerations

  • ✅ "85% ± 2%" is more honest and informative than just "85%"
  • ✅ High std means the model is sensitive to which data it sees — a red flag
  • ✅ CV enables fair model comparison — same data, same folds
  • ✅ Stakeholders can understand: "we tested the model 5 different ways and it consistently scored 83-87%"

12 · Complete Working Code

import numpy as np
from sklearn.datasets import load_iris
from sklearn.model_selection import (cross_val_score, KFold, StratifiedKFold,
                                     train_test_split)
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.neighbors import KNeighborsClassifier
from sklearn.tree import DecisionTreeClassifier

# ─── Step 1: Show instability of single splits ───
iris = load_iris()
X, y = iris.data, iris.target

print("=== Instability of Single Train/Test Split ===")
for seed in range(10):
    X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.25, random_state=seed)
    model = LogisticRegression(max_iter=1000)
    model.fit(X_tr, y_tr)
    acc = model.score(X_te, y_te)
    print(f"  Seed={seed}: Accuracy={acc:.2%}")

# ─── Step 2: Manual K-Fold CV ───
print(f"\n=== Manual 5-Fold CV ===")
kf = KFold(n_splits=5, shuffle=True, random_state=42)
fold_scores = []

for fold, (train_idx, test_idx) in enumerate(kf.split(X)):
    X_tr, X_te = X[train_idx], X[test_idx]
    y_tr, y_te = y[train_idx], y[test_idx]

    model = LogisticRegression(max_iter=1000)
    model.fit(X_tr, y_tr)
    score = model.score(X_te, y_te)
    fold_scores.append(score)
    print(f"  Fold {fold+1}: train={len(train_idx)}, test={len(test_idx)}, accuracy={score:.2%}")

print(f"  Mean: {np.mean(fold_scores):.2%} ± {np.std(fold_scores):.2%}")

# ─── Step 3: cross_val_score (the clean way) ───
print(f"\n=== cross_val_score ===")
scores = cross_val_score(
    LogisticRegression(max_iter=1000), X, y,
    cv=StratifiedKFold(n_splits=5, shuffle=True, random_state=42),
    scoring='accuracy'
)
print(f"  Scores: {scores.round(4)}")
print(f"  Mean: {scores.mean():.2%} ± {scores.std():.2%}")

# ─── Step 4: KFold vs StratifiedKFold ───
print(f"\n=== KFold vs StratifiedKFold ===")
for cv_name, cv in [('KFold', KFold(5, shuffle=True, random_state=42)),
                     ('StratifiedKFold', StratifiedKFold(5, shuffle=True, random_state=42))]:
    scores = cross_val_score(LogisticRegression(max_iter=1000), X, y, cv=cv)
    print(f"  {cv_name:>16}: {scores.mean():.2%} ± {scores.std():.2%}")

# ─── Step 5: Fair model comparison using CV ───
print(f"\n=== Model Comparison (5-Fold Stratified CV) ===")
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

models = {
    'Logistic Reg': Pipeline([('scaler', StandardScaler()),
                               ('model', LogisticRegression(max_iter=1000))]),
    'KNN (k=5)': Pipeline([('scaler', StandardScaler()),
                            ('model', KNeighborsClassifier(n_neighbors=5))]),
    'Decision Tree': DecisionTreeClassifier(max_depth=3, random_state=42),
}

print(f"{'Model':>15} {'Mean':>8} {'Std':>8} {'Scores':>30}")
print("-" * 65)

for name, model in models.items():
    scores = cross_val_score(model, X, y, cv=cv, scoring='accuracy')
    scores_str = " ".join(f"{s:.2f}" for s in scores)
    print(f"{name:>15} {scores.mean():>8.2%} {scores.std():>8.2%} [{scores_str}]")

# ─── Step 6: Pipeline inside CV (preventing leakage) ───
print(f"\n=== Pipeline + CV (no data leakage) ===")
pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('model', KNeighborsClassifier(n_neighbors=5))
])
scores = cross_val_score(pipe, X, y, cv=cv, scoring='accuracy')
print(f"  KNN with Pipeline CV: {scores.mean():.2%} ± {scores.std():.2%}")
print(f"  ✅ Scaler is fit INSIDE each fold — no leakage!")

# ─── Step 7: Different K values for CV ───
print(f"\n=== Effect of Number of Folds ===")
for k in [3, 5, 10]:
    cv_k = StratifiedKFold(n_splits=k, shuffle=True, random_state=42)
    scores = cross_val_score(
        LogisticRegression(max_iter=1000), X, y, cv=cv_k)
    print(f"  {k}-Fold: {scores.mean():.2%} ± {scores.std():.2%}")

13 · Key Takeaway + Bridge to Phase 3

Key Takeaway

  • A single train/test split is unreliable — cross-validation gives a trustworthy estimate
  • Report mean ± std of CV scores — the std reveals model stability
  • Use StratifiedKFold for classification to maintain class balance
  • Always use Pipeline inside CV to prevent data leakage from preprocessing
  • CV is for evaluation — the final model should be trained on ALL data
