1 · Hypothetical Problem

  • An energy company needs to forecast hourly electricity demand 24 hours ahead for grid management
  • ARIMA (Project 26) handles single-step forecasting well but multi-step forecasts degrade rapidly
  • Prophet (Project 44) captures seasonality but doesn't learn complex temporal dependencies
  • The demand pattern has daily cycles, weekly patterns, and nonlinear interactions with temperature
  • Goal → build an LSTM-style recurrent model from scratch that captures long-range temporal dependencies for multi-step forecasting

2 · How to Approach It

  1. From Project 47: we know how to build neural networks with backpropagation — now we add recurrence
  2. From Project 26: ARIMA uses autoregressive terms — LSTM generalizes this with learned memory
  3. From Project 44: time series have seasonality and trends — LSTM can learn these patterns from data
  4. Standard RNNs suffer from vanishing gradients — LSTM adds gates to control information flow
  5. We'll implement LSTM gates from scratch and use sliding windows for multi-step prediction

3 · How Can We Solve It

  1. Generate synthetic hourly electricity demand with daily/weekly seasonality and noise
  2. Create sliding window sequences: use past 48 hours to predict next 24 hours
  3. Implement LSTM cell with forget, input, and output gates using numpy
  4. Train with backpropagation through time (BPTT) and gradient clipping
  5. Compare multi-step forecasting: LSTM vs. simple RNN vs. linear baseline
  6. Visualize gate activations, forecasts, and error accumulation over forecast horizon

4 · What Should Do

  • ✅ Normalize time series before feeding to LSTM — neural nets need scaled inputs
  • ✅ Use sliding windows with proper train/test split — never leak future data into training
  • ✅ Clip gradients during BPTT — prevents exploding gradients in long sequences
  • ✅ Monitor validation loss for early stopping — LSTMs overfit easily on small time series
  • ✅ Evaluate at each forecast horizon separately — error at h=1 vs h=24 tells different stories

5 · What Should NOT Do

  • ❌ Do NOT shuffle time series data — temporal order must be preserved
  • ❌ Do NOT use future values as features — this is data leakage in time series
  • ❌ Do NOT ignore stationarity — LSTM works better on stationary or differenced series
  • ❌ Do NOT use very long sequences without truncated BPTT — memory and gradient issues
  • ❌ Do NOT compare LSTM on tiny datasets — it needs thousands of sequences to learn patterns

6 · When to Use What

  • 🔹 LSTM / Recurrent Models → long sequences, complex temporal dependencies, multi-step forecasting
  • 🔹 ARIMA (Project 26) → short series, single-step, when stationarity assumptions hold
  • 🔹 Prophet (Project 44) → business time series with strong seasonality, quick baseline
  • 🔹 XGBoost with lag features → tabular approach to time series, often competitive with LSTM
  • 🔹 Transformers → very long sequences where attention beats recurrence

7 · When NOT to Use

  • 🚫 Short time series (< 500 points) — not enough data to learn temporal patterns
  • 🚫 Simple seasonal patterns — ARIMA or Prophet are simpler and often sufficient
  • 🚫 When interpretability is required — LSTM gates are hard to explain to stakeholders
  • 🚫 Real-time inference with strict latency — recurrent computation is sequential, not parallelizable

8 · How It Only Works — Internal Mechanics

LSTM Cell — The Four Gates

  • Forget gate: f_t = σ(W_f · [h_{t-1}, x_t] + b_f) — what to forget from cell state
  • Input gate: i_t = σ(W_i · [h_{t-1}, x_t] + b_i) — what new info to store
  • Candidate: c̃_t = tanh(W_c · [h_{t-1}, x_t] + b_c) — new candidate values
  • Cell update: c_t = f_t ⊙ c_{t-1} + i_t ⊙ c̃_t — selective memory update
  • Output gate: o_t = σ(W_o · [h_{t-1}, x_t] + b_o) — what to output
  • Hidden state: h_t = o_t ⊙ tanh(c_t) — filtered cell state as output

Why Gates Solve Vanishing Gradients

  • Standard RNN: gradient flows through repeated matrix multiplications → vanishes or explodes
  • LSTM cell state: gradient flows through additions (c_t = f_t ⊙ c_{t-1} + ...) → gradient highway
  • Forget gate near 1.0: information persists indefinitely — long-range memory
  • Forget gate near 0.0: old information is discarded — the network can "reset"

Sliding Window for Multi-Step Forecasting

  • Input: [x_{t-L+1}, ..., x_t] — L past observations (lookback window)
  • Output: [x_{t+1}, ..., x_{t+H}] — H future values (forecast horizon)
  • Direct strategy: predict all H steps at once from the final hidden state
  • Recursive strategy: predict one step, feed it back as input for the next
  • Teacher forcing (training): use true values as inputs; at inference, use predictions

Backpropagation Through Time (BPTT)

  • Unroll the LSTM across time steps — treat as a very deep feedforward network
  • Compute gradients for each time step and sum them
  • Truncated BPTT: limit backprop to k steps — trades accuracy for memory/speed
  • Gradient clipping: if ||g|| > threshold, scale g down — prevents exploding gradients

9 · Common Pitfalls and Failure Modes

  • ⚠️ Error accumulation — recursive multi-step forecasts compound errors; direct strategy avoids this
  • ⚠️ Exposure bias — teacher forcing uses true inputs during training but predictions during inference
  • ⚠️ Overfitting on small series — LSTM has many parameters; use dropout and regularization
  • ⚠️ Wrong lookback window — too short misses patterns; too long adds noise and slows training
  • ⚠️ Not differencing non-stationary data — LSTM struggles with trends; difference or normalize first

10 · Scalability Considerations

  • LSTM is sequential — can't parallelize across time steps (unlike Transformers)
  • Training: O(T × h²) per sample where T=sequence length, h=hidden size
  • For very long sequences (>1000 steps): use truncated BPTT or switch to Transformers
  • Mini-batch training: process multiple sequences in parallel — essential for speed

11 · Interpretability Considerations

  • ✅ Gate activations reveal what the LSTM remembers and forgets — visualize forget gate over time
  • ✅ Cell state tracks long-term memory — plot it to see what information persists
  • ⚠️ Individual weights are uninterpretable — use attention mechanisms on top for interpretability
  • ⚠️ Multi-step forecasts are harder to explain — "why did you predict a spike at hour 18?"
  • ✅ Forecast uncertainty can be estimated via dropout at inference (MC Dropout)

# ─── Step 3: LSTM cell from scratch ───
class LSTMCell:
    """Single LSTM cell implemented from scratch."""

    def __init__(self, input_dim, hidden_dim):
        self.hidden_dim = hidden_dim
        scale = np.sqrt(2.0 / (input_dim + hidden_dim))

        # Forget gate
        self.Wf = np.random.randn(input_dim + hidden_dim, hidden_dim) * scale
        self.bf = np.zeros(hidden_dim)
        # Input gate
        self.Wi = np.random.randn(input_dim + hidden_dim, hidden_dim) * scale
        self.bi = np.zeros(hidden_dim)
        # Candidate
        self.Wc = np.random.randn(input_dim + hidden_dim, hidden_dim) * scale
        self.bc = np.zeros(hidden_dim)
        # Output gate
        self.Wo = np.random.randn(input_dim + hidden_dim, hidden_dim) * scale
        self.bo = np.zeros(hidden_dim)

        # Initialize forget gate bias to 1 (remember by default)
        self.bf = np.ones(hidden_dim)

    def forward(self, x_t, h_prev, c_prev):
        concat = np.concatenate([h_prev, x_t], axis=-1)

        f_t = self._sigmoid(concat @ self.Wf + self.bf)
        i_t = self._sigmoid(concat @ self.Wi + self.bi)
        c_tilde = np.tanh(concat @ self.Wc + self.bc)
        c_t = f_t * c_prev + i_t * c_tilde
        o_t = self._sigmoid(concat @ self.Wo + self.bo)
        h_t = o_t * np.tanh(c_t)

        cache = (x_t, h_prev, c_prev, f_t, i_t, c_tilde, c_t, o_t, concat)
        return h_t, c_t, cache

    def _sigmoid(self, z):
        z = np.clip(z, -500, 500)
        return 1.0 / (1.0 + np.exp(-z))

# ─── Step 4: LSTM-based forecasting model ───
class LSTMForecaster:
    """Multi-step forecaster using LSTM from scratch."""

    def __init__(self, input_dim, hidden_dim, output_horizon, lr=0.001):
        self.hidden_dim = hidden_dim
        self.output_horizon = output_horizon
        self.lr = lr
        self.lstm = LSTMCell(input_dim, hidden_dim)

        # Output projection: hidden state -> multi-step forecast
        self.W_out = np.random.randn(hidden_dim, output_horizon) * np.sqrt(2.0/hidden_dim)
        self.b_out = np.zeros(output_horizon)

    def forward(self, X_batch):
        """Process a batch of sequences. X_batch: (batch, seq_len)"""
        batch_size, seq_len = X_batch.shape
        h = np.zeros((batch_size, self.hidden_dim))
        c = np.zeros((batch_size, self.hidden_dim))
        caches = []

        for t in range(seq_len):
            x_t = X_batch[:, t:t+1]  # (batch, 1)
            h, c, cache = self.lstm.forward(x_t, h, c)
            caches.append(cache)

        # Direct multi-step: predict all horizons from final hidden state
        y_pred = h @ self.W_out + self.b_out  # (batch, horizon)
        return y_pred, h, caches

    def compute_loss(self, y_true, y_pred):
        return np.mean((y_true - y_pred) ** 2)

    def fit(self, X_train, y_train, epochs=50, batch_size=64, verbose=True):
        n = X_train.shape[0]
        losses = []

        for epoch in range(epochs):
            # Mini-batch training
            indices = np.random.permutation(n)
            epoch_loss = 0
            n_batches = 0

            for start in range(0, n, batch_size):
                end = min(start + batch_size, n)
                idx = indices[start:end]
                X_batch = X_train[idx]
                y_batch = y_train[idx]
                bs = X_batch.shape[0]

                # Forward
                y_pred, h_final, caches = self.forward(X_batch)
                loss = self.compute_loss(y_batch, y_pred)

                # Backward through output layer
                d_pred = 2 * (y_pred - y_batch) / (bs * self.output_horizon)
                d_W_out = h_final.T @ d_pred
                d_b_out = np.sum(d_pred, axis=0)
                d_h = d_pred @ self.W_out.T

                # Gradient clipping
                grad_norm = np.sqrt(np.sum(d_W_out**2) + np.sum(d_h**2))
                max_norm = 5.0
                if grad_norm > max_norm:
                    scale = max_norm / grad_norm
                    d_W_out *= scale
                    d_h *= scale

                # Update output weights
                self.W_out -= self.lr * d_W_out
                self.b_out -= self.lr * d_b_out

                # Simplified BPTT for last few steps
                d_c = np.zeros_like(d_h)
                for t in reversed(range(max(0, len(caches)-5), len(caches))):
                    cache = caches[t]
                    x_t, h_prev, c_prev, f_t, i_t, c_tilde, c_t, o_t, concat = cache

                    d_o = d_h * np.tanh(c_t)
                    d_c += d_h * o_t * (1 - np.tanh(c_t)**2)

                    d_f = d_c * c_prev * f_t * (1 - f_t)
                    d_i = d_c * c_tilde * i_t * (1 - i_t)
                    d_c_tilde = d_c * i_t * (1 - c_tilde**2)
                    d_o_gate = d_o * o_t * (1 - o_t)

                    # Update LSTM weights (accumulated)
                    for d_gate, W, b in [
                        (d_f, self.lstm.Wf, self.lstm.bf),
                        (d_i, self.lstm.Wi, self.lstm.bi),
                        (d_c_tilde, self.lstm.Wc, self.lstm.bc),
                        (d_o_gate, self.lstm.Wo, self.lstm.bo)
                    ]:
                        dW = concat.T @ d_gate / bs
                        db = np.mean(d_gate, axis=0)
                        # Clip
                        dW = np.clip(dW, -1.0, 1.0)
                        W -= self.lr * dW
                        b -= self.lr * db

                    d_c = d_c * f_t
                    d_concat = (d_f @ self.lstm.Wf.T + d_i @ self.lstm.Wi.T +
                                d_c_tilde @ self.lstm.Wc.T + d_o_gate @ self.lstm.Wo.T)
                    d_h = d_concat[:, :self.hidden_dim]

                epoch_loss += loss
                n_batches += 1

            avg_loss = epoch_loss / n_batches
            losses.append(avg_loss)

            if verbose and (epoch + 1) % 10 == 0:
                print(f"  Epoch {epoch+1:4d}: MSE={avg_loss:.6f}")

        return losses

    def predict(self, X):
        y_pred, _, _ = self.forward(X)
        return y_pred

# ─── Step 5: Simple RNN baseline ───
class SimpleRNNForecaster:
    """Vanilla RNN for comparison — no gates."""

    def __init__(self, input_dim, hidden_dim, output_horizon, lr=0.001):
        self.hidden_dim = hidden_dim
        self.lr = lr
        scale = np.sqrt(2.0 / (input_dim + hidden_dim))
        self.Wh = np.random.randn(input_dim + hidden_dim, hidden_dim) * scale
        self.bh = np.zeros(hidden_dim)
        self.W_out = np.random.randn(hidden_dim, output_horizon) * np.sqrt(2.0/hidden_dim)
        self.b_out = np.zeros(output_horizon)

    def forward(self, X_batch):
        batch_size, seq_len = X_batch.shape
        h = np.zeros((batch_size, self.hidden_dim))
        for t in range(seq_len):
            x_t = X_batch[:, t:t+1]
            concat = np.concatenate([h, x_t], axis=-1)
            h = np.tanh(concat @ self.Wh + self.bh)
        return h @ self.W_out + self.b_out

    def fit(self, X_train, y_train, epochs=50, batch_size=64, verbose=True):
        n = X_train.shape[0]
        losses = []
        for epoch in range(epochs):
            indices = np.random.permutation(n)
            epoch_loss = 0
            n_batches = 0
            for start in range(0, n, batch_size):
                end = min(start + batch_size, n)
                idx = indices[start:end]
                X_b, y_b = X_train[idx], y_train[idx]
                bs = X_b.shape[0]
                y_pred = self.forward(X_b)
                loss = np.mean((y_b - y_pred)**2)
                d_pred = 2*(y_pred - y_b)/(bs * y_b.shape[1])
                # Simple output update only
                h = np.zeros((bs, self.hidden_dim))
                for t in range(X_b.shape[1]):
                    x_t = X_b[:, t:t+1]
                    concat = np.concatenate([h, x_t], axis=-1)
                    h = np.tanh(concat @ self.Wh + self.bh)
                self.W_out -= self.lr * (h.T @ d_pred)
                self.b_out -= self.lr * np.sum(d_pred, axis=0)
                epoch_loss += loss
                n_batches += 1
            losses.append(epoch_loss / n_batches)
            if verbose and (epoch+1) % 10 == 0:
                print(f"  Epoch {epoch+1:4d}: MSE={losses[-1]:.6f}")
        return losses

    def predict(self, X):
        return self.forward(X)

# ─── Step 6: Linear baseline ───
class LinearForecaster:
    """Simple linear model: flatten lookback -> predict horizon."""

    def __init__(self, lookback, horizon):
        self.W = np.random.randn(lookback, horizon) * 0.01
        self.b = np.zeros(horizon)

    def fit(self, X, y, epochs=100, lr=0.001):
        for _ in range(epochs):
            pred = X @ self.W + self.b
            grad = 2 * X.T @ (pred - y) / X.shape[0]
            self.W -= lr * np.clip(grad, -1, 1)
            self.b -= lr * np.mean(2*(pred - y), axis=0)

    def predict(self, X):
        return X @ self.W + self.b

ax.set_title('LSTM Gate Activations (Sample Sequence)')

Key Takeaway

  • LSTMs solve the vanishing gradient problem with gates — forget, input, and output gates control information flow
  • The cell state acts as a gradient highway — information can persist across hundreds of time steps
  • Multi-step forecasting is harder than single-step — errors accumulate with each horizon step
  • Direct strategy (predict all steps at once) avoids error accumulation but may miss temporal dependencies in the forecast
  • For tabular time series, XGBoost with lag features is often competitive — LSTM shines on long, complex sequences
