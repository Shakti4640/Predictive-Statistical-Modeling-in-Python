📋 Phase 5 — Very Hard

Project 50: Transfer Learning on Tabular Data — Embeddings From Pre-Trained Models

Entity Embeddings Pre-Training Fine-Tuning Categorical Embeddings

Difficulty: Very Hard  |  Builds on: Projects 48, 49

1 · Hypothetical Problem

  • A retail company launches a new product category (organic snacks) with only 500 sales records
  • They have 50,000 records from existing categories (beverages, dairy, frozen foods) with the same features: store, region, day-of-week, promotion type, shelf position
  • Training a model on 500 rows leads to severe overfitting — especially for high-cardinality categorical features like store ID (200 stores)
  • One-hot encoding store ID creates 200 sparse columns from just 500 rows — the model can't learn meaningful patterns
  • Goal → learn dense entity embeddings for categorical features on the large dataset, then transfer them to the small dataset

2 · How to Approach It

  1. From Project 48: TabNet uses attention on features — but entity embeddings are a simpler, more targeted approach
  2. From Project 49: we learned how neural networks can capture complex patterns — embeddings are learned representations
  3. Entity embeddings map each categorical value to a dense vector — "Store 42" becomes [0.3, -0.1, 0.8] instead of a 200-dim one-hot
  4. Pre-train embeddings on the large dataset where there's enough data to learn good representations
  5. Transfer the learned embeddings to the small dataset — the model starts with knowledge instead of random weights

3 · How Can We Solve It

  1. Generate a synthetic retail dataset: 50K rows (large) and 500 rows (small) with shared categorical features
  2. Implement entity embeddings using numpy — lookup tables that map category indices to dense vectors
  3. Train a neural network with embeddings on the large dataset
  4. Transfer the learned embeddings to a new model for the small dataset
  5. Compare: (a) one-hot encoding, (b) random embeddings, (c) pre-trained embeddings on the small dataset
  6. Visualize embedding spaces to show that similar categories cluster together

4 · What Should Do

  • ✅ Choose embedding dimensions based on cardinality — rule of thumb: min(50, cardinality // 2)
  • ✅ Freeze embeddings initially when fine-tuning on small data — prevents catastrophic forgetting
  • ✅ Validate that source and target tasks share the same categorical features and values
  • ✅ Combine embeddings with numerical features — concatenate embedded categoricals with scaled numericals
  • ✅ Visualize embeddings with t-SNE/PCA — verify that semantically similar categories are close

5 · What Should NOT Do

  • ❌ Do NOT use embeddings for low-cardinality features (< 5 categories) — one-hot is simpler and sufficient
  • ❌ Do NOT transfer embeddings between unrelated domains — store embeddings from retail won't help in healthcare
  • ❌ Do NOT use embedding dimensions larger than cardinality — wastes parameters and overfits
  • ❌ Do NOT skip handling unseen categories — new stores in the target data need a default embedding
  • ❌ Do NOT fine-tune all layers at once on small data — unfreeze gradually to prevent overfitting

6 · When to Use What

  • 🔹 Entity Embeddings + Transfer → high-cardinality categoricals, small target dataset, related source domain
  • 🔹 One-Hot Encoding → low cardinality (< 10), enough data, tree-based models
  • 🔹 Target Encoding (Project 41) → medium cardinality, when you want a single numeric feature per category
  • 🔹 TabNet (Project 48) → when you want end-to-end feature selection with deep learning
  • 🔹 Few-shot learning → when target data is extremely small (< 50 samples)

7 · When NOT to Use

  • 🚫 When source and target domains are unrelated — embeddings won't transfer meaningful knowledge
  • 🚫 When the target dataset is large enough — direct training works fine without transfer
  • 🚫 When all features are numerical — embeddings are specifically for categorical features
  • 🚫 When using tree-based models — they handle categoricals natively; embeddings add unnecessary complexity

8 · How It Only Works — Internal Mechanics

Entity Embeddings

  • Each categorical feature gets an embedding matrix: E ∈ ℝ^(cardinality × embedding_dim)
  • Lookup: for category index i, the embedding is E[i] — a dense vector
  • During training, E is updated via backpropagation — the network learns which categories are similar
  • After training, similar categories have similar embeddings (e.g., nearby stores cluster together)

Pre-Training Phase (Large Dataset)

  • Train a neural network: [embeddings + numerical] → hidden layers → prediction
  • The embedding matrices learn rich representations from 50K examples
  • Store 42 might learn embedding [0.3, -0.1, 0.8] meaning "urban, high-traffic, weekend-heavy"
  • These representations capture relationships that one-hot encoding misses entirely

Transfer Phase (Small Dataset)

  • Initialize the new model's embedding matrices with the pre-trained values
  • Freeze embeddings (don't update during training) — prevents overfitting on 500 rows
  • Train only the downstream layers on the small dataset
  • Optionally: unfreeze embeddings with a very small learning rate for fine-tuning

Why This Works

  • Embeddings capture general knowledge about categories — "Store 42 is similar to Store 17"
  • This knowledge transfers across tasks — store similarity is the same whether predicting snacks or beverages
  • The small dataset only needs to learn the task-specific mapping, not the category representations
  • Reduces effective number of parameters — from 200 one-hot dims to 8 embedding dims per store

9 · Common Pitfalls and Failure Modes

  • ⚠️ Catastrophic forgetting — fine-tuning too aggressively destroys pre-trained knowledge; freeze or use small LR
  • ⚠️ Domain mismatch — if source and target distributions differ too much, transfer hurts performance
  • ⚠️ Unseen categories — new categories in target data have no pre-trained embedding; use mean embedding as default
  • ⚠️ Embedding dimension too large — overfits on rare categories; keep dimensions small
  • ⚠️ Ignoring numerical features — embeddings only help categoricals; numerical features still need standard preprocessing

10 · Scalability Considerations

  • Embedding lookup is O(1) per sample — much faster than one-hot matrix multiplication
  • Memory: O(cardinality × embedding_dim) per feature — manageable even for millions of categories
  • Pre-training is a one-time cost — embeddings can be reused across many downstream tasks
  • For very high cardinality (>1M): use hashing tricks or frequency-based pruning

11 · Interpretability Considerations

  • ✅ Embedding visualization reveals category relationships — PCA/t-SNE shows clusters of similar stores
  • ✅ Embedding distances are meaningful — cosine similarity between store embeddings reflects actual similarity
  • ✅ Dimension analysis: each embedding dimension may capture an interpretable concept (urban vs rural, size, etc.)
  • ⚠️ Individual embedding values are not directly interpretable — need visualization tools
  • ⚠️ Transfer success depends on domain overlap — hard to predict a priori if transfer will help

# ─── Step 1: Generate synthetic retail datasets ───
n_stores = 200
n_regions = 10
n_dow = 7        # day of week
n_promos = 5

def generate_retail_data(n_samples, category_effect=0):
    """Generate retail sales data with categorical and numerical features."""
    store_id = np.random.randint(0, n_stores, n_samples)
    region_id = np.random.randint(0, n_regions, n_samples)
    dow = np.random.randint(0, n_dow, n_samples)
    promo_type = np.random.randint(0, n_promos, n_samples)
    shelf_pos = np.random.randint(0, n_shelves, n_samples)

    # Numerical features
    price = np.random.uniform(2, 20, n_samples)
    temperature = np.random.normal(70, 15, n_samples)

    # True store effects (some stores sell more)
    store_effects = np.sin(np.arange(n_stores) * 0.1) * 50 + np.random.randn(n_stores) * 10
    region_effects = np.array([20, -10, 30, 5, -5, 15, -20, 10, 25, -15])
    dow_effects = np.array([0, -5, -3, 0, 10, 25, 20])  # Weekend boost
    promo_effects = np.array([0, 15, 25, 10, 30])
    shelf_effects = np.array([0, 5, 10, -5])

    # Sales = base + effects + interactions + noise
    sales = (100 + category_effect
             + store_effects[store_id]
             + region_effects[region_id]
             + dow_effects[dow]
             + promo_effects[promo_type]
             + shelf_effects[shelf_pos]
             - 2 * price
             + 0.3 * temperature
             + np.random.normal(0, 15, n_samples))

    sales = np.maximum(sales, 10)  # Floor

    return {
        'store_id': store_id, 'region_id': region_id, 'dow': dow,
        'promo_type': promo_type, 'shelf_pos': shelf_pos,
        'price': price, 'temperature': temperature, 'sales': sales
    }

# Large dataset (existing categories: beverages, dairy, frozen)
large_data = generate_retail_data(50000, category_effect=0)
# Small dataset (new category: organic snacks)
small_data = generate_retail_data(500, category_effect=20)

print("=== Retail Transfer Learning Dataset ===")
print(f"  Large dataset (source): {len(large_data['sales'])} samples")
print(f"  Small dataset (target): {len(small_data['sales'])} samples")

# ─── Step 2: Entity embedding layer ───
class EmbeddingLayer:
    """Learnable embedding lookup table."""

    def __init__(self, num_categories, embedding_dim):
        self.num_categories = num_categories
        self.embedding_dim = embedding_dim
        self.W = np.random.randn(num_categories, embedding_dim) * 0.1
        self._indices = None

    def forward(self, indices):
        self._indices = indices
        return self.W[indices]  # (batch, embedding_dim)

    def backward(self, d_output, lr):
        # Only update embeddings for categories that appeared in the batch
        for i in range(len(self._indices)):
            self.W[self._indices[i]] -= lr * d_output[i]

    def get_weights(self):
        return self.W.copy()

    def set_weights(self, weights):
        self.W = weights.copy()

# ─── Step 3: Neural network with entity embeddings ───
class EmbeddingNetwork:
    """Neural network with entity embeddings for categorical features."""

    def __init__(self, cat_configs, num_features, hidden_dim=64, lr=0.01):
        self.lr = lr
        self.cat_configs = cat_configs  # [(num_categories, embedding_dim), ...]

        # Create embedding layers
        self.embeddings = [EmbeddingLayer(nc, ed) for nc, ed in cat_configs]

        # Total input dim = sum of embedding dims + numerical features
        total_emb_dim = sum(ed for _, ed in cat_configs)
        input_dim = total_emb_dim + num_features

        # Hidden layers
        self.W1 = np.random.randn(input_dim, hidden_dim) * np.sqrt(2.0 / input_dim)
        self.b1 = np.zeros(hidden_dim)
        self.W2 = np.random.randn(hidden_dim, hidden_dim // 2) * np.sqrt(2.0 / hidden_dim)
        self.b2 = np.zeros(hidden_dim // 2)
        self.W3 = np.random.randn(hidden_dim // 2, 1) * np.sqrt(2.0 / (hidden_dim // 2))
        self.b3 = np.zeros(1)

    def forward(self, cat_features, num_features):
        """cat_features: list of (batch,) index arrays. num_features: (batch, n_num)."""
        # Get embeddings
        emb_list = [emb.forward(cat) for emb, cat in zip(self.embeddings, cat_features)]
        emb_concat = np.concatenate(emb_list, axis=1)

        # Concatenate with numerical features
        self._input = np.concatenate([emb_concat, num_features], axis=1)

        # Forward through hidden layers
        self._z1 = self._input @ self.W1 + self.b1
        self._a1 = np.maximum(0, self._z1)  # ReLU
        self._z2 = self._a1 @ self.W2 + self.b2
        self._a2 = np.maximum(0, self._z2)  # ReLU
        self._output = self._a2 @ self.W3 + self.b3

        return self._output.ravel()

    def backward(self, y_true, y_pred, update_embeddings=True):
        m = len(y_true)
        d_out = (2 * (y_pred - y_true) / m).reshape(-1, 1)

        # Layer 3
        d_W3 = self._a2.T @ d_out
        d_b3 = np.sum(d_out, axis=0)
        d_a2 = d_out @ self.W3.T

        # Layer 2
        d_z2 = d_a2 * (self._z2 > 0).astype(float)
        d_W2 = self._a1.T @ d_z2
        d_b2 = np.sum(d_z2, axis=0)
        d_a1 = d_z2 @ self.W2.T

        # Layer 1
        d_z1 = d_a1 * (self._z1 > 0).astype(float)
        d_W1 = self._input.T @ d_z1
        d_b1 = np.sum(d_z1, axis=0)
        d_input = d_z1 @ self.W1.T

        # Clip gradients
        for g in [d_W1, d_W2, d_W3]:
            np.clip(g, -5, 5, out=g)

        # Update weights
        self.W3 -= self.lr * d_W3
        self.b3 -= self.lr * d_b3
        self.W2 -= self.lr * d_W2
        self.b2 -= self.lr * d_b2
        self.W1 -= self.lr * d_W1
        self.b1 -= self.lr * d_b1

        # Update embeddings
        if update_embeddings:
            offset = 0
            for emb in self.embeddings:
                d_emb = d_input[:, offset:offset + emb.embedding_dim]
                emb.backward(d_emb, self.lr)
                offset += emb.embedding_dim

    def fit(self, cat_features, num_features, y, epochs=100, verbose=True):
        losses = []
        for epoch in range(epochs):
            y_pred = self.forward(cat_features, num_features)
            loss = np.mean((y - y_pred) ** 2)
            self.backward(y, y_pred)
            losses.append(loss)
            if verbose and (epoch + 1) % 25 == 0:
                rmse = np.sqrt(loss)
                print(f"  Epoch {epoch+1:4d}: RMSE={rmse:.2f}")
        return losses

    def predict(self, cat_features, num_features):
        return self.forward(cat_features, num_features)

    def get_embedding_weights(self):
        return [emb.get_weights() for emb in self.embeddings]

    def set_embedding_weights(self, weights_list):
        for emb, w in zip(self.embeddings, weights_list):
            emb.set_weights(w)

# ─── Step 4: Prepare data ───
def prepare_data(data):
    cat_features = [data['store_id'], data['region_id'], data['dow'],
                    data['promo_type'], data['shelf_pos']]
    num_features = np.column_stack([data['price'], data['temperature']])
    y = data['sales']
    return cat_features, num_features, y

cat_configs = [
    (n_stores, 8),    # store: 200 categories -> 8-dim embedding
    (n_regions, 4),   # region: 10 categories -> 4-dim embedding
    (n_dow, 3),       # day of week: 7 -> 3-dim
    (n_promos, 3),    # promo: 5 -> 3-dim
    (n_shelves, 2),   # shelf: 4 -> 2-dim
]

# Scale numerical features
scaler_large = StandardScaler()
scaler_small = StandardScaler()

large_cat, large_num, large_y = prepare_data(large_data)
small_cat, small_num, small_y = prepare_data(small_data)

large_num = scaler_large.fit_transform(large_num)
small_num = scaler_small.fit_transform(small_num)

# Split small dataset
indices = np.arange(len(small_y))
train_idx, test_idx = train_test_split(indices, test_size=0.3, random_state=42)

small_cat_train = [c[train_idx] for c in small_cat]
small_cat_test = [c[test_idx] for c in small_cat]
small_num_train = small_num[train_idx]
small_num_test = small_num[test_idx]
small_y_train = small_y[train_idx]
small_y_test = small_y[test_idx]

print(f"\n  Small dataset split: {len(train_idx)} train, {len(test_idx)} test")

# ─── Step 5: Pre-train on large dataset ───
print(f"\n=== Phase 1: Pre-Training on Large Dataset (50K samples) ===")
pretrain_model = EmbeddingNetwork(cat_configs, num_features=2, hidden_dim=64, lr=0.001)
pretrain_losses = pretrain_model.fit(large_cat, large_num, large_y, epochs=100)

pretrained_embeddings = pretrain_model.get_embedding_weights()
print(f"  Pre-trained embedding shapes: {[w.shape for w in pretrained_embeddings]}")

# ─── Step 6: Approach A — One-hot encoding baseline ───
print(f"\n=== Approach A: One-Hot Encoding on Small Dataset ===")
def one_hot_encode(cat_features, configs):
    encoded = []
    for feat, (n_cat, _) in zip(cat_features, configs):
        ohe = np.zeros((len(feat), n_cat))
        ohe[np.arange(len(feat)), feat] = 1
        encoded.append(ohe)
    return np.concatenate(encoded, axis=1)

ohe_train = np.concatenate([one_hot_encode(small_cat_train, cat_configs), small_num_train], axis=1)
ohe_test = np.concatenate([one_hot_encode(small_cat_test, cat_configs), small_num_test], axis=1)

ridge = Ridge(alpha=10.0)
ridge.fit(ohe_train, small_y_train)
y_pred_ohe = ridge.predict(ohe_test)
rmse_ohe = np.sqrt(mean_squared_error(small_y_test, y_pred_ohe))
r2_ohe = r2_score(small_y_test, y_pred_ohe)
print(f"  One-Hot + Ridge: RMSE={rmse_ohe:.2f}, R²={r2_ohe:.4f}")
print(f"  Feature dimensions: {ohe_train.shape[1]}")

# ─── Step 7: Approach B — Random embeddings (no transfer) ───
print(f"\n=== Approach B: Random Embeddings (No Transfer) ===")
random_model = EmbeddingNetwork(cat_configs, num_features=2, hidden_dim=64, lr=0.005)
random_losses = random_model.fit(small_cat_train, small_num_train, small_y_train, epochs=200)

y_pred_random = random_model.predict(small_cat_test, small_num_test)
rmse_random = np.sqrt(mean_squared_error(small_y_test, y_pred_random))
r2_random = r2_score(small_y_test, y_pred_random)
print(f"  Random Embeddings: RMSE={rmse_random:.2f}, R²={r2_random:.4f}")

# ─── Step 8: Approach C — Pre-trained embeddings (transfer) ───
print(f"\n=== Approach C: Pre-Trained Embeddings (Transfer Learning) ===")
transfer_model = EmbeddingNetwork(cat_configs, num_features=2, hidden_dim=64, lr=0.005)
transfer_model.set_embedding_weights(pretrained_embeddings)

# Phase 1: Train with frozen embeddings
print("  Phase 1: Frozen embeddings (50 epochs)")
frozen_losses = []
for epoch in range(50):
    y_pred = transfer_model.forward(small_cat_train, small_num_train)
    loss = np.mean((small_y_train - y_pred) ** 2)
    transfer_model.backward(small_y_train, y_pred, update_embeddings=False)
    frozen_losses.append(loss)
    if (epoch + 1) % 25 == 0:
        print(f"    Epoch {epoch+1}: RMSE={np.sqrt(loss):.2f}")

# Phase 2: Fine-tune with small learning rate
print("  Phase 2: Fine-tuning embeddings (50 epochs, lr=0.001)")
transfer_model.lr = 0.001
finetune_losses = []
for epoch in range(50):
    y_pred = transfer_model.forward(small_cat_train, small_num_train)
    loss = np.mean((small_y_train - y_pred) ** 2)
    transfer_model.backward(small_y_train, y_pred, update_embeddings=True)
    finetune_losses.append(loss)
    if (epoch + 1) % 25 == 0:
        print(f"    Epoch {epoch+1}: RMSE={np.sqrt(loss):.2f}")

transfer_losses = frozen_losses + finetune_losses

y_pred_transfer = transfer_model.predict(small_cat_test, small_num_test)
rmse_transfer = np.sqrt(mean_squared_error(small_y_test, y_pred_transfer))
r2_transfer = r2_score(small_y_test, y_pred_transfer)
print(f"  Transfer Learning: RMSE={rmse_transfer:.2f}, R²={r2_transfer:.4f}")

13 · Key Takeaway + Bridge to Next Project

Key Takeaway

  • Entity embeddings map high-cardinality categoricals to dense vectors — capturing similarity that one-hot encoding misses
  • Pre-training on a large related dataset learns rich category representations that transfer to small datasets
  • Freeze-then-fine-tune strategy prevents catastrophic forgetting — start with frozen embeddings, then unfreeze with small LR
  • Transfer learning on tabular data is underutilized — most practitioners default to one-hot encoding even with small data
