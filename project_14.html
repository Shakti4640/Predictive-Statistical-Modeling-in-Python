Project 14: Finding Natural Groups in Customer Spending — K-Means Clustering

Unsupervised learning K-Means inertia elbow method cluster assignment

Difficulty: Easy  |  Builds on: Projects 8, 13

1 · Hypothetical Problem

  • You work in marketing at a retail company
  • The CMO asks: "Can we group our customers into segments based on their spending behavior?"
  • You have data: annual spending, purchase frequency, and average order value — but NO labels
  • This is fundamentally different from all previous projects — there's no "right answer" to learn from
  • Goal → discover natural groups (clusters) in the data without supervision

2 · How to Approach It

  1. This is UNSUPERVISED learning — no target variable, no labels
  2. K-Means finds K groups by minimizing the distance from each point to its cluster center
  3. Key question: how many clusters (K)? Use the elbow method
  4. Scale features first (Project 13) — K-Means uses distances

3 · How Can We Solve It

  1. Generate synthetic customer spending data with natural clusters
  2. Scale features (Project 13 — required for distance-based methods)
  3. Run K-Means for K=2 to K=10
  4. Plot the elbow curve (inertia vs K) to find optimal K
  5. Analyze cluster profiles — what makes each segment unique?

4 · What Should Do

  • ✅ Scale features before clustering — same reason as KNN (Project 11, 13)
  • ✅ Use the elbow method to choose K — look for the "bend" in the inertia curve
  • ✅ Profile each cluster after fitting — compute mean/median of each feature per cluster
  • ✅ Run K-Means multiple times (n_init) — it's sensitive to initialization
  • ✅ Visualize clusters in 2D (even if data has more dimensions — use PCA, Project 24)

5 · What Should NOT Do

  • ❌ Do NOT use K-Means without scaling — features on larger scales dominate
  • ❌ Do NOT assume the elbow is always clear — sometimes there's no obvious bend
  • ❌ Do NOT treat cluster labels as meaningful — cluster 0 vs cluster 1 is arbitrary
  • ❌ Do NOT assume K-Means finds the "true" clusters — it finds spherical clusters only
  • ❌ Do NOT use accuracy to evaluate clustering — there are no labels to compare against

6 · When to Use What

  • 🔹 K-Means → when you expect roughly spherical, equal-sized clusters
  • 🔹 Elbow method → to choose K when you don't know the number of groups
  • 🔹 Silhouette score → alternative to elbow method (measures cluster separation)
  • 🔹 Cluster profiling → to give business meaning to each cluster

7 · When NOT to Use

  • 🚫 K-Means assumes spherical clusters — fails with elongated or irregular shapes
  • 🚫 Sensitive to outliers — one extreme point can pull a centroid away
  • 🚫 Requires specifying K in advance — doesn't discover the number of clusters automatically
  • 🚫 For non-spherical clusters, use DBSCAN or hierarchical clustering

8 · How It Only Works — Internal Mechanics

The K-Means Algorithm

  1. Randomly initialize K centroids
  2. Assign: each point goes to the nearest centroid
  3. Update: move each centroid to the mean of its assigned points
  4. Repeat steps 2-3 until centroids stop moving (convergence)

Inertia (Within-Cluster Sum of Squares)

  • Inertia = Σ Σ ||xᵢ - cₖ||² (sum of squared distances to nearest centroid)
  • Lower inertia = tighter clusters
  • Inertia always decreases with more K — at K=n, inertia=0 (each point is its own cluster)
  • The elbow is where adding more K gives diminishing returns

Initialization Sensitivity

  • Bad initialization → bad clusters (local minimum)
  • sklearn uses "k-means++" initialization — smart initial placement
  • n_init=10 (default) → runs 10 times, keeps the best result

Supervised vs Unsupervised — The Paradigm Shift

  • Supervised (Projects 1-13): given (X, y), learn f(X) → y
  • Unsupervised (this project): given X only, find structure in X
  • No "right answer" → evaluation is subjective and domain-dependent

9 · Common Pitfalls and Failure Modes

  • ⚠️ Not scaling — income ($100K) dominates age (30) in distance calculations
  • ⚠️ Choosing K by gut feeling — use elbow method or silhouette score
  • ⚠️ Interpreting cluster numbers as meaningful — cluster 0 is not "worse" than cluster 2
  • ⚠️ Assuming clusters are real — K-Means ALWAYS finds K clusters, even in random data
  • ⚠️ Single run — always use n_init > 1 to avoid bad local minima

10 · Scalability Considerations

  • K-Means is O(n × K × d × iterations) — fast for moderate data
  • For very large n, use MiniBatchKMeans — processes random subsets
  • Memory: stores K centroids (K × d) — very compact
  • Scales well to millions of rows with MiniBatch variant

11 · Interpretability Considerations

  • ✅ Cluster centroids describe the "average" member of each group
  • ✅ Cluster profiles are intuitive: "Cluster A = high spenders, low frequency"
  • ✅ Marketing teams can create targeted campaigns per cluster
  • ⚠️ Cluster boundaries are hard to explain — they're Voronoi regions in high-dimensional space
  • ⚠️ No causal explanation — clusters describe WHAT, not WHY

12 · Complete Working Code

import numpy as np
import pandas as pd
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

# ─── Step 1: Synthetic customer data with natural clusters ───
np.random.seed(42)

# 3 natural customer segments
# Budget shoppers: low spending, high frequency
budget = np.column_stack([
    np.random.normal(200, 50, 80),    # annual spending
    np.random.normal(40, 10, 80),     # purchase frequency
    np.random.normal(5, 1.5, 80)      # avg order value
])

# Premium shoppers: high spending, medium frequency
premium = np.column_stack([
    np.random.normal(2000, 400, 60),
    np.random.normal(20, 5, 60),
    np.random.normal(100, 20, 60)
])

# Occasional big spenders: medium spending, low frequency
occasional = np.column_stack([
    np.random.normal(800, 200, 60),
    np.random.normal(5, 2, 60),
    np.random.normal(160, 30, 60)
])

X = np.vstack([budget, premium, occasional])
feature_names = ['annual_spending', 'purchase_freq', 'avg_order_value']
true_labels = np.array([0]*80 + [1]*60 + [2]*60)  # for validation only

# Shuffle
idx = np.random.permutation(len(X))
X = X[idx]
true_labels = true_labels[idx]

print(f"=== Data Summary ===")
print(f"  Customers: {len(X)}")
for i, name in enumerate(feature_names):
    print(f"  {name:>20}: min={X[:, i].min():.0f}, max={X[:, i].max():.0f}")

# ─── Step 2: Scale features (Project 13 — required!) ───
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# ─── Step 3: Elbow Method ───
inertias = []
K_range = range(2, 11)

for k in K_range:
    km = KMeans(n_clusters=k, n_init=10, random_state=42)
    km.fit(X_scaled)
    inertias.append(km.inertia_)

print(f"\n=== Elbow Method ===")
for k, inertia in zip(K_range, inertias):
    bar = "█" * int(inertia / max(inertias) * 30)
    print(f"  K={k:>2}: inertia={inertia:>8.1f} {bar}")

# ─── Step 4: Fit with optimal K=3 ───
kmeans = KMeans(n_clusters=3, n_init=10, random_state=42)
labels = kmeans.fit_predict(X_scaled)

# ─── Step 5: Cluster Profiling ───
df = pd.DataFrame(X, columns=feature_names)
df['cluster'] = labels

print(f"\n=== Cluster Profiles ===")
profile = df.groupby('cluster')[feature_names].mean()
print(profile.round(1).to_string())

print(f"\n=== Cluster Sizes ===")
for c in sorted(df['cluster'].unique()):
    n = (labels == c).sum()
    print(f"  Cluster {c}: {n} customers ({n/len(X):.1%})")

# ─── Step 6: Centroid analysis ───
centroids_original = scaler.inverse_transform(kmeans.cluster_centers_)
print(f"\n=== Centroids (original scale) ===")
for i, centroid in enumerate(centroids_original):
    print(f"  Cluster {i}: " + ", ".join(
        f"{name}={val:.0f}" for name, val in zip(feature_names, centroid)))

# ─── Step 7: Visualization ───
fig, axes = plt.subplots(1, 2, figsize=(12, 5))

# Elbow plot
axes[0].plot(list(K_range), inertias, 'o-', color='#58a6ff', linewidth=2)
axes[0].axvline(x=3, color='#f85149', linestyle='--', label='Chosen K=3')
axes[0].set_xlabel('Number of Clusters (K)')
axes[0].set_ylabel('Inertia')
axes[0].set_title('Elbow Method')
axes[0].legend()
axes[0].grid(True, alpha=0.2)

# Cluster scatter (2D projection: spending vs frequency)
colors = ['#58a6ff', '#3fb950', '#f0883e']
for c in range(3):
    mask = labels == c
    axes[1].scatter(X[mask, 0], X[mask, 1], c=colors[c],
                    alpha=0.6, label=f'Cluster {c}')
axes[1].scatter(centroids_original[:, 0], centroids_original[:, 1],
                c='#f85149', marker='X', s=200, edgecolors='white', linewidth=2,
                label='Centroids')
axes[1].set_xlabel('Annual Spending ($)')
axes[1].set_ylabel('Purchase Frequency')
axes[1].set_title('Customer Segments')
axes[1].legend()
axes[1].grid(True, alpha=0.2)

plt.tight_layout()
plt.savefig('project_14_clusters.png', dpi=100, bbox_inches='tight')
print(f"\n✅ Plots saved as project_14_clusters.png")
