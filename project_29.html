Project 29: Survival Analysis — When Will a Machine Fail?

Survival Analysis Kaplan-Meier Censored Data Log-Rank Test Lifelines Hazard Function

Difficulty: Medium  |  Builds on: Projects 10, 25

1 · Hypothetical Problem

  • You're a reliability engineer at a manufacturing plant with 500 industrial machines
  • Some machines have failed — you know exactly when. Others are still running — you only know they haven't failed YET
  • Management asks: "What's the probability a machine survives past 2 years? Do machines from Supplier A last longer than Supplier B?"
  • Standard classification (Project 10) can predict IF a machine will fail, but not WHEN
  • The "still running" machines are censored — dropping them wastes data; treating them as non-failures is wrong
  • Goal → use survival analysis to model time-to-failure while properly handling censored observations

2 · How to Approach It

  1. From Project 10: logistic regression predicts binary outcomes — but ignores the TIME dimension
  2. From Project 25: time series analysis respects temporal structure — survival analysis does too, but for event times
  3. The key challenge: censored data — some machines haven't failed yet, but we can't ignore them
  4. Use the Kaplan-Meier estimator to estimate the survival function nonparametrically
  5. Use the log-rank test to compare survival curves between groups (e.g., Supplier A vs B)
  6. Use the lifelines library — Python's go-to for survival analysis

3 · How Can We Solve It

  1. Generate machine failure data with censoring (some machines still running)
  2. Fit a Kaplan-Meier survival curve — estimate P(survival > t) for any time t
  3. Compare survival curves for different machine groups (suppliers, environments)
  4. Apply the log-rank test to determine if group differences are statistically significant
  5. Estimate median survival time and confidence intervals
  6. Show why ignoring censored data gives biased results

4 · What Should Do

  • ✅ Always account for censoring — censored observations carry information ("survived at least this long")
  • ✅ Use Kaplan-Meier for nonparametric survival estimation — no distributional assumptions needed
  • ✅ Plot survival curves with confidence intervals — they widen as fewer subjects remain at risk
  • ✅ Use the log-rank test to compare groups — it's the standard test for survival curve differences
  • ✅ Report median survival time — more intuitive than mean for skewed time-to-event data

5 · What Should NOT Do

  • ❌ Do NOT drop censored observations — you lose data AND introduce bias (only failures remain)
  • ❌ Do NOT treat censored as "no event" — a machine running for 6 months isn't the same as one that will never fail
  • ❌ Do NOT use mean time-to-failure without accounting for censoring — it underestimates true survival
  • ❌ Do NOT use standard regression (Project 2) for time-to-event — it ignores censoring
  • ❌ Do NOT compare survival curves visually without a statistical test — curves can look different by chance

6 · When to Use What

  • 🔹 Kaplan-Meier → nonparametric survival estimation — no assumptions about the failure distribution
  • 🔹 Log-rank test → compare survival curves between two or more groups
  • 🔹 Cox Proportional Hazards → when you have covariates (features) that affect survival — the "regression" of survival analysis
  • 🔹 Logistic Regression (Project 10) → when you only care about IF an event happens, not WHEN
  • 🔹 Time Series (Project 25) → when you're modeling a continuous measurement over time, not time-to-event

7 · When NOT to Use

  • 🚫 When there's no censoring — if all events are observed, simpler methods work fine
  • 🚫 When the outcome isn't time-to-event — survival analysis is for "how long until X happens"
  • 🚫 When you need to predict a continuous value — use regression (Project 9)
  • 🚫 When the event can happen multiple times — standard survival analysis assumes one event per subject

8 · How It Only Works — Internal Mechanics

Censoring — The Core Challenge

  • Right censoring: the subject hasn't experienced the event by the end of the study
  • Example: machine installed 6 months ago, still running → censored at 6 months
  • We know it survived AT LEAST 6 months — that's useful information
  • Dropping censored data biases results toward shorter survival times (only failures remain)

The Survival Function S(t)

  • S(t) = P(T > t) = probability of surviving beyond time t
  • S(0) = 1 (everyone starts alive), S(∞) = 0 (eventually everyone experiences the event)
  • S(t) is a decreasing step function — drops at each observed event time
  • Median survival = time t where S(t) = 0.5 — half the population has experienced the event

Kaplan-Meier Estimator

  • At each event time tᵢ: Ŝ(tᵢ) = Ŝ(tᵢ₋₁) × (1 - dᵢ/nᵢ)
  • dᵢ = number of events at time tᵢ, nᵢ = number at risk just before tᵢ
  • Censored subjects are "at risk" until their censoring time, then removed from the risk set
  • This is why censored data matters — they contribute to nᵢ (the denominator) until they're censored

The Hazard Function h(t)

  • h(t) = instantaneous rate of failure at time t, given survival up to t
  • h(t) = f(t) / S(t) where f(t) is the density of failure times
  • Constant hazard → exponential distribution (memoryless — machine doesn't age)
  • Increasing hazard → Weibull distribution (machine wears out over time)
  • The hazard function connects survival analysis to the concept of risk from Project 10

Log-Rank Test

  • H₀: the survival curves of two groups are identical
  • Compares observed vs expected events in each group at every event time
  • Test statistic follows a chi-squared distribution under H₀
  • p-value < 0.05 → the groups have significantly different survival

9 · Common Pitfalls and Failure Modes

  • ⚠️ Ignoring censoring — the #1 mistake; biases all estimates toward shorter survival
  • ⚠️ Informative censoring — if censoring is related to the outcome (sicker patients drop out), KM is biased
  • ⚠️ Small sample at late times — the survival curve becomes unreliable when few subjects remain at risk
  • ⚠️ Comparing curves without a test — visual differences can be due to chance; always use log-rank
  • ⚠️ Confusing survival analysis with time series — time series models continuous measurements; survival models time-to-event

10 · Scalability Considerations

  • Kaplan-Meier is O(n log n) — sorting event times is the bottleneck; very fast
  • Log-rank test is O(n) after sorting — trivial computation
  • Cox Proportional Hazards is O(n × p) per iteration — scales well for moderate feature counts
  • For millions of subjects, lifelines handles it efficiently with vectorized operations
  • Memory is rarely an issue — survival data is compact (time, event indicator, covariates)

11 · Interpretability Considerations

  • ✅ Kaplan-Meier curves are highly intuitive — "70% of machines survive past 1 year"
  • ✅ Median survival time is easy to communicate — "half the machines fail by month 18"
  • ✅ Comparing curves visually is powerful — stakeholders immediately see which group lasts longer
  • ✅ Confidence intervals show uncertainty — wider bands mean fewer observations
  • ⚠️ Hazard ratios (from Cox models) require more explanation — "1.5x the risk" is less intuitive than survival curves

12 · Complete Working Code

import numpy as np
import pandas as pd
from lifelines import KaplanMeierFitter, CoxPHFitter
from lifelines.statistics import logrank_test
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

# ─── Step 1: Generate machine failure data with censoring ───
np.random.seed(42)
n_machines = 500

# Two suppliers with different reliability
supplier = np.random.choice(['A', 'B'], size=n_machines, p=[0.6, 0.4])
# Supplier A: Weibull with shape=2 (increasing hazard), scale=24 months
# Supplier B: Weibull with shape=2, scale=18 months (fails sooner)
true_failure_time = np.where(
    supplier == 'A',
    np.random.weibull(2, n_machines) * 24,
    np.random.weibull(2, n_machines) * 18
)

# Censoring: study ends at 36 months, some machines installed later
study_duration = 36
install_time = np.random.uniform(0, 24, n_machines)  # installed between month 0-24
observation_time = study_duration - install_time  # max observable time

# Observed time = min(true failure, observation window)
observed_time = np.minimum(true_failure_time, observation_time)
event_observed = (true_failure_time <= observation_time).astype(int)

df = pd.DataFrame({
    'duration': observed_time,
    'event': event_observed,
    'supplier': supplier,
    'install_month': install_time.astype(int),
    'operating_temp': np.random.normal(70, 10, n_machines),  # Fahrenheit
    'maintenance_score': np.random.uniform(1, 10, n_machines)
})

print("=== Machine Failure Dataset ===")
print(f"  Total machines: {n_machines}")
print(f"  Failed (event=1): {event_observed.sum()} ({event_observed.mean():.1%})")
print(f"  Censored (event=0): {(1 - event_observed).sum()} ({1 - event_observed.mean():.1%})")
print(f"  Supplier A: {(supplier == 'A').sum()}, Supplier B: {(supplier == 'B').sum()}")
print(f"  Observed time range: {observed_time.min():.1f} – {observed_time.max():.1f} months")

# ─── Step 2: Why ignoring censoring is wrong ───
print("\n=== Why Censoring Matters ===")
# Wrong: mean of observed times (ignores censoring)
naive_mean = observed_time.mean()
# Wrong: mean of only failed machines
failed_only_mean = observed_time[event_observed == 1].mean()
print(f"  Naive mean (all observed times):  {naive_mean:.1f} months — BIASED")
print(f"  Failed-only mean:                 {failed_only_mean:.1f} months — BIASED (ignores survivors)")
print(f"  → Both underestimate true survival because they mishandle censored data")

# ─── Step 3: Kaplan-Meier survival curve ───
kmf = KaplanMeierFitter()
kmf.fit(df['duration'], event_observed=df['event'], label='All Machines')

print(f"\n=== Kaplan-Meier Estimates ===")
print(f"  Median survival: {kmf.median_survival_time_:.1f} months")
for t in [6, 12, 18, 24, 30]:
    surv = kmf.predict(t)
    print(f"  P(survive > {t:>2} months): {surv:.1%}")

fig, ax = plt.subplots(figsize=(10, 6))
kmf.plot_survival_function(ax=ax, color='#58a6ff', linewidth=2)
ax.set_title('Kaplan-Meier Survival Curve — All Machines')
ax.set_xlabel('Time (months)')
ax.set_ylabel('Survival Probability')
ax.grid(True, alpha=0.2)
ax.axhline(y=0.5, color='#f85149', linestyle='--', alpha=0.5, label='50% survival')
ax.legend()
plt.tight_layout()
plt.savefig('project_29_km_overall.png', dpi=100, bbox_inches='tight')
print("\n✅ Overall KM curve saved as project_29_km_overall.png")

# ─── Step 4: Compare suppliers — KM by group ───
fig, ax = plt.subplots(figsize=(10, 6))

for name, color in [('A', '#3fb950'), ('B', '#f85149')]:
    mask = df['supplier'] == name
    kmf_group = KaplanMeierFitter()
    kmf_group.fit(df.loc[mask, 'duration'], event_observed=df.loc[mask, 'event'],
                  label=f'Supplier {name}')
    kmf_group.plot_survival_function(ax=ax, color=color, linewidth=2)
    print(f"  Supplier {name}: median survival = {kmf_group.median_survival_time_:.1f} months")

ax.set_title('Survival Curves by Supplier')
ax.set_xlabel('Time (months)')
ax.set_ylabel('Survival Probability')
ax.grid(True, alpha=0.2)
ax.legend()
plt.tight_layout()
plt.savefig('project_29_km_by_supplier.png', dpi=100, bbox_inches='tight')
print("✅ KM by supplier saved as project_29_km_by_supplier.png")

# ─── Step 5: Log-rank test — is the difference significant? ───
mask_a = df['supplier'] == 'A'
mask_b = df['supplier'] == 'B'

result = logrank_test(
    df.loc[mask_a, 'duration'], df.loc[mask_b, 'duration'],
    event_observed_A=df.loc[mask_a, 'event'],
    event_observed_B=df.loc[mask_b, 'event']
)

print(f"\n=== Log-Rank Test: Supplier A vs B ===")
print(f"  Test statistic: {result.test_statistic:.4f}")
print(f"  p-value: {result.p_value:.6f}")
print(f"  Significant: {'YES ✅' if result.p_value < 0.05 else 'NO ❌'} (α=0.05)")

# ─── Step 6: Cox Proportional Hazards — survival regression ───
print(f"\n=== Cox Proportional Hazards Model ===")
df_cox = df.copy()
df_cox['supplier_B'] = (df_cox['supplier'] == 'B').astype(int)
df_cox = df_cox.drop(columns=['supplier', 'install_month'])

cph = CoxPHFitter()
cph.fit(df_cox, duration_col='duration', event_col='event')
cph.print_summary()

print(f"\nInterpretation:")
for col in ['supplier_B', 'operating_temp', 'maintenance_score']:
    hr = np.exp(cph.params_[col])
    direction = "increases" if hr > 1 else "decreases"
    print(f"  {col}: hazard ratio = {hr:.3f} — "
          f"a 1-unit increase {direction} failure risk by {abs(hr - 1) * 100:.1f}%")

# ─── Step 7: Survival curves from Cox model ───
fig, ax = plt.subplots(figsize=(10, 6))
cph.plot(ax=ax)
ax.set_title('Cox PH — Covariate Effects (Hazard Ratios)')
ax.grid(True, alpha=0.2)
plt.tight_layout()
plt.savefig('project_29_cox_coefficients.png', dpi=100, bbox_inches='tight')
print("\n✅ Cox coefficients saved as project_29_cox_coefficients.png")

# ─── Step 8: Predicted survival for specific machines ───
print(f"\n=== Predicted Survival for Specific Machines ===")
new_machines = pd.DataFrame({
    'supplier_B': [0, 1, 0],
    'operating_temp': [65, 80, 70],
    'maintenance_score': [8, 3, 5]
})

fig, ax = plt.subplots(figsize=(10, 6))
colors = ['#3fb950', '#f85149', '#58a6ff']
labels = ['Good (A, cool, maintained)', 'Bad (B, hot, neglected)', 'Average']

for i in range(len(new_machines)):
    surv_func = cph.predict_survival_function(new_machines.iloc[[i]])
    ax.plot(surv_func.index, surv_func.values, color=colors[i],
            linewidth=2, label=labels[i])
    median_t = surv_func.index[surv_func.values.flatten() <= 0.5]
    if len(median_t) > 0:
        print(f"  {labels[i]}: median survival = {median_t[0]:.1f} months")
    else:
        print(f"  {labels[i]}: median survival > {surv_func.index[-1]:.0f} months")

ax.set_title('Predicted Survival Curves for Specific Machines')
ax.set_xlabel('Time (months)')
ax.set_ylabel('Survival Probability')
ax.axhline(y=0.5, color='#c9d1d9', linestyle='--', alpha=0.3)
ax.legend()
ax.grid(True, alpha=0.2)
plt.tight_layout()
plt.savefig('project_29_predicted_survival.png', dpi=100, bbox_inches='tight')
print("✅ Predicted survival curves saved as project_29_predicted_survival.png")

# ─── Step 9: Number at risk table ───
print(f"\n=== Number at Risk Over Time ===")
for t in [0, 6, 12, 18, 24, 30, 36]:
    at_risk = (df['duration'] >= t).sum()
    events_by_t = ((df['duration'] <= t) & (df['event'] == 1)).sum()
    censored_by_t = ((df['duration'] <= t) & (df['event'] == 0)).sum()
    print(f"  t={t:>2} months: at risk={at_risk:>3}, "
          f"failed={events_by_t:>3}, censored={censored_by_t:>3}")

13 · Key Takeaway + Bridge to Next Project

Key Takeaway

  • Survival analysis models WHEN an event happens, not just IF — it answers "how long until failure?"
  • Censored data is the key challenge — some subjects haven't experienced the event yet, but they still carry information
  • Kaplan-Meier estimates the survival function nonparametrically — no distributional assumptions
  • The log-rank test compares survival curves between groups — the survival equivalent of a t-test
  • Cox Proportional Hazards adds covariates — the "regression" of survival analysis
  • Never ignore censoring — it biases all estimates toward shorter survival times
