📋 Phase 1 — Very Easy

Project 1: Will It Rain Tomorrow? Predicting Binary Weather Outcomes

numpy arrays input/output thinking train/test split what is a model

Difficulty: Very Easy  |  Dependency: None (Starting Point)

1 · Hypothetical Problem

  • You work at a small weather station in a rural town
  • Farmers call every evening asking: "Will it rain tomorrow?"
  • You have a notebook with 30 days of data: humidity (%) and whether it rained the next day (Yes=1, No=0)
  • Goal → build the simplest possible "model" that takes humidity as input and outputs a rain prediction
  • No libraries beyond numpy at first — understand what a model actually IS before using sklearn

2 · How to Approach It

  1. Frame the question as: Input → Process → Output
    • Input: a single number (humidity %)
    • Process: some rule or function
    • Output: 0 (no rain) or 1 (rain)
  2. Start with the dumbest possible model — a fixed threshold
    • If humidity > 70 → predict rain
    • Otherwise → predict no rain
  3. Measure how often this dumb model is correct → that's accuracy
  4. Then ask: can we find a BETTER threshold from the data itself?
  5. This is the birth of "learning from data" — the core of all modeling

3 · How Can We Solve It

  1. Create synthetic data with numpy (30 days of humidity + rain labels)
  2. Split data into two parts:
    • Training set (first 20 days) → used to find the best threshold
    • Test set (last 10 days) → used to check if the threshold works on unseen data
  3. Loop through candidate thresholds (50, 55, 60, 65, 70, 75, 80) on training data
  4. Pick the threshold that gives highest accuracy on training data
  5. Apply that threshold to test data → report test accuracy

4 · What Should Do

  • ✅ Always split data BEFORE looking at performance — never evaluate on training data alone
  • ✅ Start with the simplest model possible — a threshold is a valid model
  • ✅ Understand that a "model" is just a function: f(input) → output
  • ✅ Use numpy arrays instead of Python lists — they are faster and support vectorized operations
  • ✅ Print intermediate results to build intuition (predictions vs actuals)
  • ✅ Think about what "accuracy" actually means: correct predictions / total predictions
  • ✅ Recognize that even a coin flip gives 50% accuracy — your model must beat that

5 · What Should NOT Do

  • ❌ Do NOT jump to sklearn or complex algorithms — understand the fundamentals first
  • ❌ Do NOT evaluate your model on the same data you trained on — this gives a falsely optimistic score
  • ❌ Do NOT hardcode a threshold without checking alternatives — let the data guide you
  • ❌ Do NOT ignore the class distribution — if it rains 90% of days, always predicting "rain" gives 90% accuracy but is useless
  • ❌ Do NOT confuse correlation with a model — just because humidity and rain are related doesn't mean any threshold works well
  • ❌ Do NOT use pandas or complex structures yet — keep it raw numpy to see what's happening

6 · When to Use What

  • 🔹 Threshold-based model → when you have ONE numeric feature and a binary outcome
  • 🔹 numpy arrays → when you need fast, vectorized numeric operations
  • 🔹 Train/test split → ALWAYS, for any model, no exceptions
  • 🔹 Accuracy metric → when classes are roughly balanced (similar number of rain vs no-rain days)
  • 🔹 Manual threshold search → when you want to understand what "learning" means before automating it

7 · When NOT to Use

  • 🚫 Threshold model fails when the relationship is non-linear (e.g., rain happens at both very low AND very high humidity)
  • 🚫 Single-feature models fail when the outcome depends on multiple factors (humidity + wind + pressure)
  • 🚫 Accuracy is misleading when classes are imbalanced — if 95% of days are sunny, predicting "no rain" always gives 95% accuracy
  • 🚫 Manual threshold search doesn't scale — with 100 features, you can't loop through all combinations
  • 🚫 This approach assumes a sharp cutoff exists — real-world boundaries are often fuzzy

8 · How It Only Works — Internal Mechanics

What is a Model?

  • A model is a function that maps inputs to outputs
  • In this project: f(humidity) → 0 or 1
  • The "learning" part is finding the best parameter (threshold) from data

What is a Threshold Model?

  • The simplest possible classifier
  • Rule: if x > threshold → class 1, else → class 0
  • The threshold is the ONLY parameter — this model has exactly one knob to tune

What is Train/Test Split?

  • Training data → the model "sees" this data and learns from it
  • Test data → the model has NEVER seen this data — it's the honest exam
  • Why? → A model can memorize training data perfectly but fail on new data
  • Analogy: studying from a textbook (training) vs taking the actual exam (testing)

What is Accuracy?

  • accuracy = number of correct predictions / total predictions
  • Range: 0.0 (all wrong) to 1.0 (all correct)
  • Baseline: random guessing gives ~50% for balanced binary problems

How the Threshold Search Works

  1. Define candidate thresholds: [50, 55, 60, 65, 70, 75, 80]
  2. For each threshold, predict on training data
  3. Compare predictions to actual labels → compute accuracy
  4. Pick the threshold with highest training accuracy
  5. Apply that threshold to test data → get honest accuracy

9 · Common Pitfalls and Failure Modes

  • ⚠️ Data leakage — accidentally including test data when choosing the threshold → inflated accuracy
  • ⚠️ Too few candidates — testing only 3 thresholds might miss the optimal one
  • ⚠️ Overfitting to training data — the best training threshold might not be the best test threshold
  • ⚠️ Ignoring class balance — if 80% of days are "no rain", a model that always says "no rain" gets 80% accuracy
  • ⚠️ Confusing training accuracy with real performance — always report test accuracy
  • ⚠️ Not setting a random seed — results change every run, making debugging impossible

10 · Scalability Considerations

  • This threshold approach works for tiny datasets (30–1000 rows) with 1 feature
  • With 1 million rows → still fast because numpy vectorizes the comparison
  • With 100 features → this approach breaks down completely (need multi-dimensional models)
  • With 1000 candidate thresholds → still O(n × t) where n=rows, t=thresholds — manageable
  • Memory: numpy arrays of 1M floats ≈ 8 MB — no issue
  • This is a teaching model, not a production model — it shows the CONCEPT of learning

11 · Interpretability Considerations

  • ✅ Fully interpretable — you can explain it to anyone: "if humidity is above X%, I predict rain"
  • ✅ One parameter (threshold) → one sentence explanation
  • ✅ No black box — every prediction can be traced to the threshold rule
  • ✅ Stakeholders (farmers) can understand and trust this model
  • ⚠️ Interpretability comes at the cost of flexibility — this model can't capture complex patterns
  • ⚠️ As models get more powerful (later projects), interpretability decreases — this tradeoff is fundamental

12 · Complete Working Code

import numpy as np

# ─── Step 1: Create synthetic weather data ───
np.random.seed(42)

# 30 days of humidity readings (40% to 95%)
humidity = np.random.uniform(40, 95, size=30)

# Rain label: higher humidity → higher chance of rain
# Simple rule with noise: rain if humidity > 65 (with some randomness)
noise = np.random.normal(0, 10, size=30)
rain = (humidity + noise > 68).astype(int)

print("=== Raw Data (first 10 days) ===")
for i in range(10):
    print(f"  Day {i+1}: Humidity={humidity[i]:.1f}%  Rain={rain[i]}")

# ─── Step 2: Train/Test Split ───
# First 20 days → training
# Last 10 days → testing
X_train, y_train = humidity[:20], rain[:20]
X_test, y_test = humidity[20:], rain[20:]

print(f"\nTraining samples: {len(X_train)}")
print(f"Test samples:     {len(X_test)}")

# ─── Step 3: Find the best threshold on training data ───
candidate_thresholds = np.arange(45, 85, 5)  # [45, 50, 55, ..., 80]
best_threshold = None
best_train_accuracy = 0

print("\n=== Threshold Search (Training Data) ===")
for threshold in candidate_thresholds:
    # Predict: if humidity > threshold → rain (1), else → no rain (0)
    predictions = (X_train > threshold).astype(int)

    # Accuracy: how many predictions match actual labels
    accuracy = np.mean(predictions == y_train)

    print(f"  Threshold={threshold:.0f}%  →  Training Accuracy={accuracy:.2%}")

    if accuracy > best_train_accuracy:
        best_train_accuracy = accuracy
        best_threshold = threshold

print(f"\n✅ Best threshold: {best_threshold:.0f}%  (Training Accuracy: {best_train_accuracy:.2%})")

# ─── Step 4: Evaluate on test data (the honest exam) ───
test_predictions = (X_test > best_threshold).astype(int)
test_accuracy = np.mean(test_predictions == y_test)

print(f"\n=== Test Set Evaluation ===")
print(f"  Threshold used: {best_threshold:.0f}%")
print(f"  Test Accuracy:  {test_accuracy:.2%}")

# ─── Step 5: Show predictions vs actuals ───
print("\n=== Predictions vs Actuals (Test Set) ===")
for i in range(len(X_test)):
    status = "✓" if test_predictions[i] == y_test[i] else "✗"
    print(f"  Day {21+i}: Humidity={X_test[i]:.1f}%  Predicted={test_predictions[i]}  Actual={y_test[i]}  {status}")

# ─── Step 6: Baseline comparison ───
# What if we always predicted the most common class?
most_common = int(np.round(np.mean(y_train)))
baseline_accuracy = np.mean(most_common == y_test)
print(f"\n=== Baseline (always predict {most_common}) ===")
print(f"  Baseline Accuracy: {baseline_accuracy:.2%}")
print(f"  Our Model:         {test_accuracy:.2%}")
print(f"  Improvement:       {test_accuracy - baseline_accuracy:+.2%}")

13 · Key Takeaway + Bridge to Next Project

Key Takeaway

  • A model is just a function: f(input) → output
  • Learning = finding the best parameter(s) from data
  • Always split data into train/test BEFORE evaluating
  • Accuracy = correct / total — but only meaningful on unseen test data
  • Even the simplest threshold model teaches the entire modeling workflow: data → split → train → evaluate
