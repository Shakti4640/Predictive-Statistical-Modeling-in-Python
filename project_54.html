Difficulty: Very Hard  |  Builds on: Projects 37, 40, 24

1 · Hypothetical Problem

  • An HR analytics team wants to understand what drives employee performance
  • They measure observable variables: test scores, peer reviews, project completion rate, attendance, training hours, satisfaction surveys
  • But the real drivers are LATENT (unobservable): "competence," "motivation," and "work environment quality"
  • PCA (Project 24) can find latent dimensions — but doesn't model causal relationships between them
  • Causal inference (Project 37) handles causation — but assumes all variables are observed
  • Goal → use SEM to model how latent constructs cause observable measurements AND how latent constructs influence each other

2 · How to Approach It

  1. From Project 24: PCA extracts latent dimensions — SEM's measurement model does the same but with theory-driven structure
  2. From Project 37: causal inference estimates treatment effects — SEM models entire causal systems simultaneously
  3. From Project 40: mixed effects handle grouped data — SEM handles latent groupings
  4. SEM has two parts: (1) measurement model (latent → observed) and (2) structural model (latent → latent)
  5. Use the semopy library — Python's implementation of SEM with LISREL-style syntax

3 · How Can We Solve It

  1. Generate synthetic data where latent variables cause observed measurements
  2. Define the measurement model: which observed variables load onto which latent factors
  3. Define the structural model: how latent factors influence each other
  4. Fit the SEM using maximum likelihood estimation
  5. Evaluate model fit using CFI, RMSEA, and chi-square statistics
  6. Interpret path coefficients and compare with simple regression

4 · What Should Do

  • ✅ Start with a theory-driven model — SEM requires you to specify the structure BEFORE fitting
  • ✅ Check model fit indices: CFI > 0.95, RMSEA < 0.06, SRMR < 0.08
  • ✅ Verify factor loadings are significant and > 0.4 — weak loadings mean the indicator doesn't measure the construct
  • ✅ Report both standardized and unstandardized path coefficients
  • ✅ Use at least 3 indicators per latent variable — fewer leads to identification problems

5 · What Should NOT Do

  • ❌ Do NOT use SEM for data exploration — it's a confirmatory technique; use EFA/PCA (Project 24) for exploration
  • ❌ Do NOT add paths just to improve fit — every path must be theoretically justified
  • ❌ Do NOT ignore model fit — a poorly fitting model means your theory doesn't match the data
  • ❌ Do NOT use SEM with small samples — need at least 200 observations, ideally 10× the number of parameters
  • ❌ Do NOT confuse SEM with causal proof — SEM tests if data is CONSISTENT with a causal model, not that it's true

6 · When to Use What

  • 🔹 SEM → when you have latent variables AND causal paths between them; confirmatory analysis
  • 🔹 Confirmatory Factor Analysis (CFA) → SEM without structural paths; just the measurement model
  • 🔹 PCA (Project 24) → exploratory dimension reduction; no causal assumptions
  • 🔹 Path Analysis → SEM without latent variables; causal paths between observed variables only
  • 🔹 Causal Inference (Project 37) → when you need treatment effect estimates, not full system modeling
  • 🔹 Mixed Effects (Project 40) → when the grouping structure is observed (schools, hospitals)

7 · When NOT to Use

  • 🚫 When you don't have a theoretical model — SEM is confirmatory, not exploratory
  • 🚫 When sample size is small (< 200) — SEM requires large samples for stable estimates
  • 🚫 When variables are highly non-normal — standard SEM assumes multivariate normality
  • 🚫 When you just need prediction — SEM is for understanding causal structure, not prediction accuracy
  • 🚫 When the causal structure is unknown — use exploratory methods first, then confirm with SEM

8 · How It Only Works — Internal Mechanics

The Two Parts of SEM

  • Measurement Model: latent variable → observed indicators (like factor analysis)
  • Example: "Competence" (latent) → test_score, project_quality, peer_review (observed)
  • Structural Model: latent variable → latent variable (causal paths)
  • Example: "Motivation" → "Performance", "Work Environment" → "Motivation"

Path Diagram Notation

  • Circles/ovals = latent variables (unobserved constructs)
  • Rectangles = observed variables (measured indicators)
  • Single-headed arrows = causal/directional paths (regression)
  • Double-headed arrows = correlations (no causal direction)
  • Each arrow has a path coefficient (like a regression weight)

Estimation

  • SEM estimates all parameters simultaneously using Maximum Likelihood (ML)
  • ML finds parameters that make the model-implied covariance matrix as close as possible to the observed covariance matrix
  • Σ(θ) ≈ S where Σ(θ) is model-implied and S is observed covariance
  • The discrepancy between Σ(θ) and S is measured by the chi-square statistic

Model Fit Indices

  • Chi-square: tests if Σ(θ) = S exactly; sensitive to sample size; p > 0.05 desired but rarely achieved
  • CFI (Comparative Fit Index): compares model to null model; > 0.95 = good fit
  • RMSEA (Root Mean Square Error of Approximation): < 0.06 = good, < 0.08 = acceptable
  • SRMR (Standardized Root Mean Residual): < 0.08 = good fit

9 · Common Pitfalls and Failure Modes

  • ⚠️ Model not identified — too many parameters relative to observed covariances; need at least 3 indicators per latent
  • ⚠️ Heywood cases — negative variance estimates indicate model misspecification
  • ⚠️ Overfitting with modification indices — adding paths to improve fit without theory leads to capitalization on chance
  • ⚠️ Confusing fit with truth — a well-fitting model isn't necessarily the correct causal model; equivalent models exist
  • ⚠️ Non-normal data — standard ML estimation assumes normality; use robust estimators for skewed data

10 · Scalability Considerations

  • SEM estimation is O(p³) where p = number of observed variables — cubic in the number of indicators
  • Practical limit: ~50-100 observed variables; beyond that, use parceling or Bayesian SEM
  • Sample size: minimum 200, recommended 10-20 observations per estimated parameter
  • semopy handles moderate models efficiently; for large models, consider lavaan (R) via rpy2
  • Bayesian SEM (via PyMC) handles small samples better but is computationally expensive

11 · Interpretability Considerations

  • ✅ Path diagrams are highly visual — stakeholders can see the causal story at a glance
  • ✅ Standardized path coefficients are comparable — "motivation has a 0.6 effect on performance"
  • ✅ Direct, indirect, and total effects decompose complex causal chains
  • ⚠️ Latent variables are abstract — "what exactly IS competence?" requires careful definition
  • ⚠️ Equivalent models can fit equally well — the data alone can't distinguish between them

12 · Complete Working Code

import numpy as np
import pandas as pd
from sklearn.preprocessing import StandardScaler
from scipy import stats
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

np.random.seed(42)

# ═══════════════════════════════════════════════════════════════
# STEP 1: Generate Data From a Known SEM Structure
# ═══════════════════════════════════════════════════════════════
n = 800

# Latent variables (unobserved)
work_environment = np.random.normal(0, 1, n)
motivation = 0.5 * work_environment + np.random.normal(0, 0.7, n)
competence = 0.3 * motivation + np.random.normal(0, 0.8, n)
performance = 0.4 * competence + 0.35 * motivation + 0.2 * work_environment + np.random.normal(0, 0.5, n)

# Observed indicators of Work Environment (latent → observed)
office_quality = 0.8 * work_environment + np.random.normal(0, 0.4, n)
team_support = 0.7 * work_environment + np.random.normal(0, 0.5, n)
resource_access = 0.65 * work_environment + np.random.normal(0, 0.5, n)

# Observed indicators of Motivation
engagement_score = 0.85 * motivation + np.random.normal(0, 0.3, n)
initiative_rating = 0.7 * motivation + np.random.normal(0, 0.5, n)
goal_commitment = 0.75 * motivation + np.random.normal(0, 0.4, n)

# Observed indicators of Competence
test_score = 0.8 * competence + np.random.normal(0, 0.4, n)
skill_assessment = 0.75 * competence + np.random.normal(0, 0.45, n)
certification_level = 0.6 * competence + np.random.normal(0, 0.6, n)

# Observed indicators of Performance
project_completion = 0.85 * performance + np.random.normal(0, 0.3, n)
quality_rating = 0.8 * performance + np.random.normal(0, 0.35, n)
peer_review = 0.7 * performance + np.random.normal(0, 0.5, n)

df = pd.DataFrame({
    'office_quality': office_quality, 'team_support': team_support, 'resource_access': resource_access,
    'engagement': engagement_score, 'initiative': initiative_rating, 'goal_commit': goal_commitment,
    'test_score': test_score, 'skill_assess': skill_assessment, 'cert_level': certification_level,
    'proj_complete': project_completion, 'quality_rating': quality_rating, 'peer_review': peer_review
})

# Standardize
scaler = StandardScaler()
df_scaled = pd.DataFrame(scaler.fit_transform(df), columns=df.columns)

print("=== Employee Performance Data ===")
print(f"  Employees: {n}")
print(f"  Observed variables: {len(df.columns)}")
print(f"  Latent constructs: 4 (Work Environment, Motivation, Competence, Performance)")
print(f"\n  Correlation matrix (sample):")
corr = df_scaled.corr()
print(corr.iloc[:4, :4].round(3).to_string())

# ═══════════════════════════════════════════════════════════════
# STEP 2: SEM Specification (semopy syntax)
# ═══════════════════════════════════════════════════════════════
model_spec = """
# Measurement Model (latent =~ indicators)
WorkEnv =~ office_quality + team_support + resource_access
Motivation =~ engagement + initiative + goal_commit
Competence =~ test_score + skill_assess + cert_level
Performance =~ proj_complete + quality_rating + peer_review

# Structural Model (latent ~ latent)
Motivation ~ WorkEnv
Competence ~ Motivation
Performance ~ Competence + Motivation + WorkEnv
"""

print(f"\n=== SEM Model Specification ===")
print(model_spec)

# ═══════════════════════════════════════════════════════════════
# STEP 3: Fit SEM using semopy (if available) or manual approach
# ═══════════════════════════════════════════════════════════════
try:
    import semopy
    sem_model = semopy.Model(model_spec)
    result = sem_model.fit(df_scaled)

    print(f"\n=== SEM Results (semopy) ===")
    estimates = sem_model.inspect()
    print(estimates.to_string())

    # Extract fit indices
    fit_stats = semopy.calc_stats(sem_model)
    print(f"\n=== Model Fit Indices ===")
    for stat_name in ['chi2', 'dof', 'CFI', 'RMSEA', 'SRMR']:
        if stat_name in fit_stats.columns:
            print(f"  {stat_name}: {fit_stats[stat_name].values[0]:.4f}")

    sem_available = True
except ImportError:
    print(f"\n  semopy not installed — using manual factor analysis approach")
    sem_available = False

# ═══════════════════════════════════════════════════════════════
# STEP 4: Manual Factor Analysis (Confirmatory-style)
# ═══════════════════════════════════════════════════════════════
from numpy.linalg import svd

print(f"\n=== Manual Confirmatory Factor Analysis ===")

# Define factor structure
factors = {
    'WorkEnv': ['office_quality', 'team_support', 'resource_access'],
    'Motivation': ['engagement', 'initiative', 'goal_commit'],
    'Competence': ['test_score', 'skill_assess', 'cert_level'],
    'Performance': ['proj_complete', 'quality_rating', 'peer_review']
}

# Extract factor scores using indicator means (simple approach)
factor_scores = {}
for factor, indicators in factors.items():
    # Factor score = first principal component of indicators
    X_factor = df_scaled[indicators].values
    U, S, Vt = svd(X_factor, full_matrices=False)
    factor_scores[factor] = U[:, 0] * S[0]

    # Factor loadings
    loadings = np.corrcoef(factor_scores[factor], X_factor.T)[0, 1:]
    print(f"\n  {factor}:")
    for ind, loading in zip(indicators, loadings):
        strength = "strong" if abs(loading) > 0.7 else "moderate" if abs(loading) > 0.4 else "weak"
        print(f"    {ind:<18}: loading = {loading:.3f} ({strength})")

# ═══════════════════════════════════════════════════════════════
# STEP 5: Structural Path Analysis (regression between factors)
# ═══════════════════════════════════════════════════════════════
from sklearn.linear_model import LinearRegression

print(f"\n=== Structural Path Coefficients ===")

# Path: WorkEnv → Motivation
reg1 = LinearRegression().fit(
    np.column_stack([factor_scores['WorkEnv']]),
    factor_scores['Motivation'])
print(f"  WorkEnv → Motivation:  β = {reg1.coef_[0]:.3f}")

# Path: Motivation → Competence
reg2 = LinearRegression().fit(
    np.column_stack([factor_scores['Motivation']]),
    factor_scores['Competence'])
print(f"  Motivation → Competence: β = {reg2.coef_[0]:.3f}")

# Path: Competence + Motivation + WorkEnv → Performance
reg3 = LinearRegression().fit(
    np.column_stack([factor_scores['Competence'],
                     factor_scores['Motivation'],
                     factor_scores['WorkEnv']]),
    factor_scores['Performance'])
print(f"  Competence → Performance:  β = {reg3.coef_[0]:.3f}")
print(f"  Motivation → Performance:  β = {reg3.coef_[1]:.3f}")
print(f"  WorkEnv → Performance:     β = {reg3.coef_[2]:.3f}")

# ═══════════════════════════════════════════════════════════════
# STEP 6: Direct, Indirect, and Total Effects
# ═══════════════════════════════════════════════════════════════
print(f"\n=== Effect Decomposition: WorkEnv → Performance ===")
direct_we_perf = reg3.coef_[2]
indirect_via_mot = reg1.coef_[0] * reg3.coef_[1]
indirect_via_comp = reg1.coef_[0] * reg2.coef_[0] * reg3.coef_[0]
total = direct_we_perf + indirect_via_mot + indirect_via_comp

print(f"  Direct effect:                    {direct_we_perf:.3f}")
print(f"  Indirect via Motivation:          {indirect_via_mot:.3f}")
print(f"  Indirect via Motivation→Competence: {indirect_via_comp:.3f}")
print(f"  Total effect:                     {total:.3f}")

# ═══════════════════════════════════════════════════════════════
# STEP 7: Visualization
# ═══════════════════════════════════════════════════════════════
fig, axes = plt.subplots(2, 2, figsize=(14, 12))

# Plot 1: Factor loadings heatmap
ax = axes[0, 0]
loading_matrix = np.zeros((12, 4))
all_indicators = []
for j, (factor, indicators) in enumerate(factors.items()):
    for i, ind in enumerate(indicators):
        row = j * 3 + i
        loading_matrix[row, j] = np.corrcoef(factor_scores[factor], df_scaled[ind].values)[0, 1]
        all_indicators.append(ind)

im = ax.imshow(loading_matrix, cmap='RdBu_r', vmin=-1, vmax=1, aspect='auto')
ax.set_xticks(range(4))
ax.set_xticklabels(list(factors.keys()), rotation=45, ha='right', fontsize=8)
ax.set_yticks(range(12))
ax.set_yticklabels(all_indicators, fontsize=7)
ax.set_title('Factor Loading Matrix')
plt.colorbar(im, ax=ax, shrink=0.8)

# Plot 2: Path coefficients
ax = axes[0, 1]
paths = ['WE→Mot', 'Mot→Comp', 'Comp→Perf', 'Mot→Perf', 'WE→Perf']
coeffs = [reg1.coef_[0], reg2.coef_[0], reg3.coef_[0], reg3.coef_[1], reg3.coef_[2]]
colors = ['#3fb950' if c > 0 else '#f85149' for c in coeffs]
ax.barh(paths, coeffs, color=colors, alpha=0.8, edgecolor='white')
ax.axvline(0, color='white', linewidth=0.5)
ax.set_xlabel('Standardized Path Coefficient (β)')
ax.set_title('Structural Path Coefficients')
ax.grid(True, alpha=0.2, axis='x')

# Plot 3: Effect decomposition
ax = axes[1, 0]
effects = ['Direct', 'Via Motivation', 'Via Mot→Comp', 'Total']
values = [direct_we_perf, indirect_via_mot, indirect_via_comp, total]
colors_eff = ['#58a6ff', '#3fb950', '#f0883e', '#bc8cff']
ax.bar(effects, values, color=colors_eff, alpha=0.8, edgecolor='white')
ax.set_ylabel('Effect Size')
ax.set_title('WorkEnv → Performance: Effect Decomposition')
ax.grid(True, alpha=0.2, axis='y')

# Plot 4: Factor score correlations
ax = axes[1, 1]
factor_df = pd.DataFrame(factor_scores)
factor_corr = factor_df.corr()
im2 = ax.imshow(factor_corr.values, cmap='coolwarm', vmin=-1, vmax=1)
ax.set_xticks(range(4))
ax.set_xticklabels(factor_corr.columns, rotation=45, ha='right', fontsize=9)
ax.set_yticks(range(4))
ax.set_yticklabels(factor_corr.columns, fontsize=9)
for i in range(4):
    for j in range(4):
        ax.text(j, i, f'{factor_corr.values[i,j]:.2f}', ha='center', va='center',
                color='black' if abs(factor_corr.values[i,j]) < 0.5 else 'white', fontsize=9)
ax.set_title('Latent Factor Correlations')
plt.colorbar(im2, ax=ax, shrink=0.8)

for a in axes.flat:
    a.set_facecolor('#0d1117')
    a.tick_params(colors='white')
    a.xaxis.label.set_color('white')
    a.yaxis.label.set_color('white')
    a.title.set_color('white')
fig.patch.set_facecolor('#0d1117')
plt.tight_layout()
plt.savefig('project_54_sem.png', dpi=100, bbox_inches='tight', facecolor='#0d1117')
print(f"\n✅ SEM analysis saved as project_54_sem.png")

13 · Key Takeaway + Bridge to Next Project

Key Takeaway

  • SEM combines measurement models (latent → observed) with structural models (latent → latent) in one framework
  • It's confirmatory — you specify the theory first, then test if the data supports it
  • Direct, indirect, and total effects decompose complex causal chains through latent variables
  • Model fit indices (CFI, RMSEA, SRMR) tell you if your theoretical model matches the observed data
  • SEM doesn't prove causation — it tests whether data is consistent with a proposed causal structure
