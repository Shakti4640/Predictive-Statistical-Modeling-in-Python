Project 11: Classifying Iris Flowers — K-Nearest Neighbors

KNN algorithm distance metrics choosing K decision boundary

Difficulty: Easy  |  Builds on: Projects 4, 8

1 · Hypothetical Problem

  • You're a botanist cataloging flower species
  • Given measurements (petal length, petal width, sepal length, sepal width), classify the species
  • Three species: Setosa, Versicolor, Virginica
  • Logistic regression (Project 6, 10) assumes a linear boundary — what if species overlap in complex ways?
  • KNN makes NO assumptions about the boundary shape — it just looks at the nearest neighbors

2 · How to Approach It

  1. The idea: to classify a new flower, find the K most similar flowers in the training data
  2. The majority class among those K neighbors = the prediction
  3. Key decisions: what value of K? What distance metric?
  4. Feature scaling is CRITICAL — KNN uses distances, and unscaled features distort distances

3 · How Can We Solve It

  1. Load the Iris dataset (sklearn built-in)
  2. Scale features (essential for distance-based methods)
  3. Implement KNN from scratch to understand the algorithm
  4. Use sklearn's KNeighborsClassifier
  5. Try different K values and compare accuracy
  6. Visualize decision boundaries (2D projection)

4 · What Should Do

  • ✅ ALWAYS scale features before KNN — distances are meaningless on unscaled data
  • ✅ Try multiple K values — too small = overfitting, too large = underfitting (Project 8)
  • ✅ Use odd K for binary classification to avoid ties
  • ✅ Understand that KNN stores ALL training data — it's a "lazy learner"
  • ✅ Compare KNN with logistic regression on the same data — see where each excels

5 · What Should NOT Do

  • ❌ Do NOT use KNN without scaling — a feature in thousands will dominate one in single digits
  • ❌ Do NOT use K=1 in production — it overfits to individual training points
  • ❌ Do NOT use KNN with high-dimensional data (100+ features) — distances become meaningless (curse of dimensionality)
  • ❌ Do NOT expect KNN to be fast at prediction time — it computes distances to ALL training points

6 · When to Use What

  • 🔹 KNN → when the decision boundary is complex/nonlinear and you have moderate data
  • 🔹 Euclidean distance → default, works well for continuous features on similar scales
  • 🔹 Manhattan distance → more robust to outliers than Euclidean
  • 🔹 Small K (3-5) → captures local patterns, risk of overfitting
  • 🔹 Large K (15-30) → smoother boundaries, risk of underfitting

7 · When NOT to Use

  • 🚫 High-dimensional data → distances become meaningless (curse of dimensionality)
  • 🚫 Large training sets (1M+ rows) → prediction is slow (compute distance to every point)
  • 🚫 When you need interpretability → KNN can't explain WHY it classified something
  • 🚫 When features are on very different scales and you can't scale them

8 · How It Only Works — Internal Mechanics

The KNN Algorithm

  • Training phase: just store the data (no computation)
  • Prediction phase for a new point x:
    1. Compute distance from x to EVERY training point
    2. Sort by distance, pick the K closest
    3. Count the class labels among those K neighbors
    4. Predict the majority class

Distance Metrics

  • Euclidean: d = √(Σ(xᵢ - yᵢ)²) — straight-line distance
  • Manhattan: d = Σ|xᵢ - yᵢ| — city-block distance
  • Euclidean is sensitive to outliers (squared terms); Manhattan is more robust

K as a Complexity Knob

  • K=1 → most complex boundary, highest variance (overfitting)
  • K=n → simplest boundary (always predict majority class), highest bias (underfitting)
  • This is the bias-variance tradeoff from Project 8 in action

Why Scaling Matters

  • If feature A ranges 0-1000 and feature B ranges 0-1, distance is dominated by A
  • StandardScaler (mean=0, std=1) puts all features on equal footing
  • This is the first time scaling is REQUIRED — for linear models it's optional

9 · Common Pitfalls and Failure Modes

  • ⚠️ Forgetting to scale — the #1 KNN mistake
  • ⚠️ Using K=1 — memorizes training data, terrible generalization
  • ⚠️ Even K for binary classification — can result in ties
  • ⚠️ Curse of dimensionality — with 100 features, all points are roughly equidistant
  • ⚠️ Slow prediction — KNN is O(n×d) per prediction where n=training size, d=features

10 · Scalability Considerations

  • Training: O(1) — just store the data
  • Prediction: O(n × d) per sample — slow for large n
  • KD-trees and Ball-trees speed up neighbor search to O(d × log n) for low dimensions
  • For n > 100K, consider approximate nearest neighbors (FAISS, Annoy)
  • Memory: stores entire training set — O(n × d)

11 · Interpretability Considerations

  • ✅ Intuitive concept: "this flower is most similar to these K flowers, which are all Setosa"
  • ✅ Can show the actual neighbors as explanation
  • ❌ No global model — can't say "petal length is the most important feature" directly
  • ❌ Decision boundary is implicit — can't write it as an equation
  • ⚠️ Less interpretable than logistic regression, but more flexible

12 · Complete Working Code

import numpy as np
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.neighbors import KNeighborsClassifier
from sklearn.metrics import classification_report, accuracy_score
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

# ─── Step 1: Load Iris dataset ───
iris = load_iris()
X, y = iris.data, iris.target
feature_names = iris.feature_names
class_names = iris.target_names

print(f"=== Iris Dataset ===")
print(f"  Samples: {X.shape[0]} | Features: {X.shape[1]}")
print(f"  Classes: {list(class_names)}")
print(f"  Features: {feature_names}")

# ─── Step 2: Scale features (CRITICAL for KNN) ───
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, random_state=42, stratify=y)

scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s = scaler.transform(X_test)

print(f"\n=== Scaling Effect ===")
print(f"  Before: mean={X_train.mean(axis=0).round(2)}, std={X_train.std(axis=0).round(2)}")
print(f"  After:  mean={X_train_s.mean(axis=0).round(2)}, std={X_train_s.std(axis=0).round(2)}")

# ─── Step 3: KNN from scratch (for understanding) ───
def knn_predict_one(X_train, y_train, x_new, k=5):
    distances = np.sqrt(np.sum((X_train - x_new) ** 2, axis=1))
    nearest_idx = np.argsort(distances)[:k]
    nearest_labels = y_train[nearest_idx]
    prediction = np.bincount(nearest_labels).argmax()
    return prediction, distances[nearest_idx]

# Demo on one test point
pred, dists = knn_predict_one(X_train_s, y_train, X_test_s[0], k=5)
print(f"\n=== Manual KNN (K=5) for first test point ===")
print(f"  Predicted: {class_names[pred]} | Actual: {class_names[y_test[0]]}")
print(f"  Neighbor distances: {dists.round(3)}")

# ─── Step 4: Try different K values ───
k_values = [1, 3, 5, 7, 9, 15, 25]
print(f"\n=== K Value Comparison ===")
print(f"{'K':>5} {'Train Acc':>10} {'Test Acc':>10} {'Gap':>8} {'Status':>12}")
print("-" * 48)

best_k, best_acc = 1, 0
for k in k_values:
    knn = KNeighborsClassifier(n_neighbors=k)
    knn.fit(X_train_s, y_train)
    train_acc = knn.score(X_train_s, y_train)
    test_acc = knn.score(X_test_s, y_test)
    gap = train_acc - test_acc

    if gap > 0.05:
        status = "OVERFITTING"
    elif train_acc < 0.85:
        status = "UNDERFITTING"
    else:
        status = "GOOD"

    print(f"{k:>5} {train_acc:>10.2%} {test_acc:>10.2%} {gap:>+8.2%} {status:>12}")

    if test_acc > best_acc:
        best_acc = test_acc
        best_k = k

print(f"\n✅ Best K={best_k} (Test Accuracy={best_acc:.2%})")

# ─── Step 5: Final model with best K ───
final_model = KNeighborsClassifier(n_neighbors=best_k)
final_model.fit(X_train_s, y_train)
y_pred = final_model.predict(X_test_s)

print(f"\n=== Classification Report (K={best_k}) ===")
print(classification_report(y_test, y_pred, target_names=class_names))

# ─── Step 6: Distance metric comparison ───
print(f"=== Distance Metric Comparison ===")
for metric in ['euclidean', 'manhattan']:
    knn_m = KNeighborsClassifier(n_neighbors=best_k, metric=metric)
    knn_m.fit(X_train_s, y_train)
    acc = knn_m.score(X_test_s, y_test)
    print(f"  {metric:>12}: {acc:.2%}")

# ─── Step 7: Comparison with Logistic Regression ───
from sklearn.linear_model import LogisticRegression
lr = LogisticRegression(max_iter=1000, random_state=42)
lr.fit(X_train_s, y_train)
lr_acc = lr.score(X_test_s, y_test)
print(f"\n=== KNN vs Logistic Regression ===")
print(f"  KNN (K={best_k}):          {best_acc:.2%}")
print(f"  Logistic Regression: {lr_acc:.2%}")

13 · Key Takeaway + Bridge to Next Project

Key Takeaway

  • KNN classifies by majority vote of K nearest neighbors — no training phase
  • Feature scaling is MANDATORY — distances are meaningless without it
  • K controls the bias-variance tradeoff: small K = complex boundary, large K = smooth boundary
  • KNN makes no assumptions about boundary shape — more flexible than logistic regression
  • Tradeoff: flexibility comes at the cost of interpretability and prediction speed
