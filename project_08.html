Project 8: Diagnosing a Model — Is It Learning or Memorizing?

Overfitting vs underfitting bias-variance tradeoff learning curves validation set

Difficulty: Very Easy  |  Builds on: Projects 2–7

1 · Hypothetical Problem

  • You've built several models in Projects 1-7 — some worked well, some didn't
  • Your manager asks: "How do I know if this model will work on NEW data we haven't seen yet?"
  • A model that scores 99% on training data but 60% on test data is MEMORIZING, not learning
  • A model that scores 55% on both training and test data is too SIMPLE — it's not capturing the pattern
  • Goal → learn to diagnose these two failure modes and understand the fundamental tradeoff

2 · How to Approach It

  1. Create a dataset where we can control complexity
  2. Fit models of increasing complexity (polynomial degree 1, 3, 10, 20)
  3. Track training error AND validation error for each complexity level
  4. Plot learning curves: error vs training set size
  5. Identify the sweet spot between underfitting and overfitting
  6. Introduce the validation set as a tool for model selection

3 · How Can We Solve It

  1. Generate data with a known nonlinear pattern
  2. Split into train / validation / test (three-way split)
  3. Fit polynomial models of degree 1, 3, 10, 20
  4. Compare train vs validation error for each
  5. Plot learning curves using sklearn's learning_curve()
  6. Diagnose: which model underfits? Which overfits? Which is just right?

4 · What Should Do

  • ✅ Always compare training error vs validation/test error — the GAP tells the story
  • ✅ Use a validation set for model selection, test set for final evaluation only
  • ✅ Plot learning curves — they show if more data would help
  • ✅ Start simple, increase complexity gradually — don't jump to the most complex model
  • ✅ Understand that some error is irreducible (noise in the data)
  • ✅ Use this diagnostic framework for EVERY model in future projects

5 · What Should NOT Do

  • ❌ Do NOT use the test set for model selection — that's data leakage
  • ❌ Do NOT celebrate high training accuracy — it means nothing without validation accuracy
  • ❌ Do NOT assume more complex = better — complexity without data leads to overfitting
  • ❌ Do NOT ignore the gap between train and validation error — it's the overfitting signal
  • ❌ Do NOT use the same data for training, validation, AND testing — each has a distinct role

6 · When to Use What

  • 🔹 Learning curves → to diagnose if the model needs more data or more complexity
  • 🔹 Validation set → for choosing between models (hyperparameter tuning, model selection)
  • 🔹 Test set → for final, unbiased performance estimate (use ONCE)
  • 🔹 Train/val error gap → large gap = overfitting, both high = underfitting
  • 🔹 Simpler model → when you have limited data or need interpretability
  • 🔹 More complex model → when you have lots of data and the simple model underfits

7 · When NOT to Use

  • 🚫 Learning curves are less useful with very small datasets (<50 samples) — too noisy
  • 🚫 A single train/val split can be unreliable — cross-validation (Project 15) is more robust
  • 🚫 The bias-variance tradeoff doesn't tell you WHICH model to use — only whether to go simpler or more complex
  • 🚫 Overfitting diagnosis doesn't help if the features are fundamentally uninformative

8 · How It Only Works — Internal Mechanics

Overfitting — The Model Memorizes

  • The model fits the training data TOO well — including the noise
  • Training error: very low (near zero)
  • Validation error: much higher than training error
  • The model has learned patterns that don't generalize
  • Analogy: memorizing exam answers vs understanding the subject

Underfitting — The Model Is Too Simple

  • The model can't capture the underlying pattern
  • Training error: high
  • Validation error: also high (and close to training error)
  • The model is too constrained to learn the relationship
  • Analogy: trying to draw a curve with a straight ruler

Bias-Variance Tradeoff

  • Bias = error from wrong assumptions (underfitting) → model is too simple
  • Variance = error from sensitivity to training data (overfitting) → model is too complex
  • Total error = Bias² + Variance + Irreducible noise
  • As complexity increases: bias decreases, variance increases
  • The sweet spot minimizes total error

Learning Curves — What They Tell You

  • Plot training error and validation error vs training set size
  • Underfitting: both curves plateau at a HIGH error → more data won't help, need more complexity
  • Overfitting: training error is low, validation error is high → more data WILL help
  • Good fit: both curves converge at a LOW error

Three-Way Split: Train / Validation / Test

  • Training set → model learns from this
  • Validation set → used to choose between models (tune hyperparameters)
  • Test set → used ONCE at the end for final performance estimate
  • Typical split: 60% train / 20% validation / 20% test

9 · Common Pitfalls and Failure Modes

  • ⚠️ Using test set for model selection — the most common and most damaging mistake
  • ⚠️ Ignoring the train-val gap — reporting only test accuracy without diagnosing
  • ⚠️ Adding complexity to fix underfitting when the features are bad — no model can learn from uninformative features
  • ⚠️ Assuming overfitting is always bad — slight overfitting is normal; massive overfitting is the problem
  • ⚠️ Not enough data for three-way split — with 100 samples, 20% validation = only 20 points (noisy estimate)

10 · Scalability Considerations

  • Learning curves require training the model multiple times → can be slow for complex models
  • For large datasets, subsample for learning curve computation
  • The diagnostic concepts (train vs val error) are O(1) to compute once you have predictions
  • Three-way split wastes data — cross-validation (Project 15) is more data-efficient
  • These diagnostic principles apply to models of ANY complexity — from linear to deep learning

11 · Interpretability Considerations

  • ✅ Learning curves are visual and intuitive — even non-technical people can see the gap
  • ✅ "The model memorizes" vs "the model generalizes" is a clear explanation
  • ✅ The bias-variance tradeoff provides a framework for ALL modeling decisions
  • ✅ Stakeholders can understand: "we need more data" or "we need a simpler model"
  • ⚠️ The tradeoff is conceptual — you can't directly measure bias and variance separately in practice

12 · Complete Working Code

import numpy as np
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error
from sklearn.model_selection import learning_curve
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

# ─── Step 1: Generate data with a known nonlinear pattern ───
np.random.seed(42)
n = 100
X = np.random.uniform(0, 10, n)
# True function: y = sin(x) + 0.5x (nonlinear)
y = np.sin(X) + 0.5 * X + np.random.normal(0, 0.5, n)

# ─── Step 2: Three-way split ───
X_train, y_train = X[:60], y[:60]
X_val, y_val = X[60:80], y[60:80]
X_test, y_test = X[80:], y[80:]
print(f"Train: {len(X_train)} | Validation: {len(X_val)} | Test: {len(X_test)}")

# ─── Step 3: Fit models of increasing complexity ───
degrees = [1, 3, 10, 20]
results = []

print(f"\n=== Model Complexity Comparison ===")
print(f"{'Degree':>8} {'Train RMSE':>12} {'Val RMSE':>12} {'Gap':>10} {'Diagnosis':>15}")
print("-" * 60)

for d in degrees:
    # Create polynomial features
    poly = PolynomialFeatures(degree=d, include_bias=False)
    X_train_poly = poly.fit_transform(X_train.reshape(-1, 1))
    X_val_poly = poly.transform(X_val.reshape(-1, 1))
    X_test_poly = poly.transform(X_test.reshape(-1, 1))

    # Fit
    model = LinearRegression()
    model.fit(X_train_poly, y_train)

    # Evaluate
    train_rmse = np.sqrt(mean_squared_error(y_train, model.predict(X_train_poly)))
    val_rmse = np.sqrt(mean_squared_error(y_val, model.predict(X_val_poly)))
    gap = val_rmse - train_rmse

    # Diagnose
    if train_rmse > 0.8:
        diagnosis = "UNDERFITTING"
    elif gap > 0.5:
        diagnosis = "OVERFITTING"
    else:
        diagnosis = "GOOD FIT"

    results.append({
        'degree': d, 'train_rmse': train_rmse,
        'val_rmse': val_rmse, 'diagnosis': diagnosis
    })

    print(f"{d:>8} {train_rmse:>12.4f} {val_rmse:>12.4f} {gap:>+10.4f} {diagnosis:>15}")

# ─── Step 4: Pick the best model using validation set ───
best = min(results, key=lambda r: r['val_rmse'])
print(f"\n✅ Best model: degree={best['degree']} (Val RMSE={best['val_rmse']:.4f})")

# Final evaluation on test set (ONCE)
poly_best = PolynomialFeatures(degree=best['degree'], include_bias=False)
X_train_best = poly_best.fit_transform(X_train.reshape(-1, 1))
X_test_best = poly_best.transform(X_test.reshape(-1, 1))
model_best = LinearRegression().fit(X_train_best, y_train)
test_rmse = np.sqrt(mean_squared_error(y_test, model_best.predict(X_test_best)))
print(f"   Test RMSE (final, unbiased): {test_rmse:.4f}")

# ─── Step 5: Learning Curves ───
from sklearn.pipeline import make_pipeline

fig, axes = plt.subplots(2, 2, figsize=(12, 10))

for idx, d in enumerate(degrees):
    ax = axes[idx // 2][idx % 2]
    pipe = make_pipeline(PolynomialFeatures(d, include_bias=False), LinearRegression())

    train_sizes, train_scores, val_scores = learning_curve(
        pipe, X.reshape(-1, 1), y,
        train_sizes=np.linspace(0.2, 1.0, 8),
        cv=5, scoring='neg_mean_squared_error'
    )

    train_rmse_lc = np.sqrt(-train_scores.mean(axis=1))
    val_rmse_lc = np.sqrt(-val_scores.mean(axis=1))

    ax.plot(train_sizes, train_rmse_lc, 'o-', color='#58a6ff', label='Training')
    ax.plot(train_sizes, val_rmse_lc, 'o-', color='#f0883e', label='Validation')
    ax.set_xlabel('Training Set Size')
    ax.set_ylabel('RMSE')
    ax.set_title(f'Degree {d} — {results[idx]["diagnosis"]}')
    ax.legend()
    ax.grid(True, alpha=0.2)
    ax.set_ylim(0, 3)

plt.tight_layout()
plt.savefig('project_08_learning_curves.png', dpi=100, bbox_inches='tight')
print(f"\n✅ Learning curves saved as project_08_learning_curves.png")

# ─── Step 6: Visual comparison of fits ───
fig2, axes2 = plt.subplots(1, 4, figsize=(16, 4))
x_plot = np.linspace(0, 10, 200)

for idx, d in enumerate(degrees):
    ax = axes2[idx]
    poly = PolynomialFeatures(degree=d, include_bias=False)
    X_tr_p = poly.fit_transform(X_train.reshape(-1, 1))
    m = LinearRegression().fit(X_tr_p, y_train)

    y_plot = m.predict(poly.transform(x_plot.reshape(-1, 1)))

    ax.scatter(X_train, y_train, color='#58a6ff', alpha=0.4, s=15)
    ax.plot(x_plot, y_plot, color='#3fb950', linewidth=2)
    ax.set_title(f'Degree {d}\n{results[idx]["diagnosis"]}')
    ax.set_ylim(-2, 8)
    ax.grid(True, alpha=0.2)

plt.tight_layout()
plt.savefig('project_08_fits.png', dpi=100, bbox_inches='tight')
print(f"✅ Model fits saved as project_08_fits.png")

13 · Key Takeaway + Bridge to Next Project

Key Takeaway

  • Overfitting = low training error, high validation error (model memorizes)
  • Underfitting = high training error, high validation error (model too simple)
  • The bias-variance tradeoff is the central tension in ALL of machine learning
  • Learning curves diagnose whether you need more data or more complexity
  • Use validation set for model selection, test set for final evaluation ONLY
