Difficulty: Medium  |  Builds on: Projects 16, 19

1 · Hypothetical Problem

  • You work at an e-commerce company — sellers upload products with free-text descriptions but often miscategorize them
  • Dataset: 5,000 product descriptions labeled as Electronics, Clothing, Home & Kitchen, or Sports
  • Problem: the input is raw text, not numeric features — every model we've built so far (Projects 1–27) expects numbers
  • Some categories have far more products than others — class imbalance (Project 19)
  • Goal → convert text into numeric features and classify products into the correct category

2 · How to Approach It

  1. From Project 16: Pipeline and ColumnTransformer handle preprocessing — we'll build a text pipeline
  2. From Project 19: class imbalance affects precision/recall — we'll monitor per-class metrics
  3. Step 1: Convert text to numbers using Bag of Words (CountVectorizer) or TF-IDF
  4. Step 2: Train a Naive Bayes classifier — it's fast, works well with sparse text features, and handles high dimensions
  5. Step 3: Compare CountVectorizer vs TF-IDF and tune the text preprocessing

3 · How Can We Solve It

  1. Generate realistic product descriptions for 4 categories
  2. Apply CountVectorizer to create a Bag of Words matrix
  3. Apply TfidfVectorizer as an alternative — it downweights common words
  4. Train MultinomialNB on both representations and compare
  5. Build a complete Pipeline (vectorizer → classifier) for clean, reproducible workflow
  6. Evaluate with per-class precision/recall (Project 19 mindset)

4 · What Should Do

  • ✅ Use a Pipeline to combine vectorization and classification — prevents data leakage and simplifies deployment
  • ✅ Try both CountVectorizer and TfidfVectorizer — TF-IDF often works better by downweighting common words
  • ✅ Set max_features to limit vocabulary size — reduces noise from rare words
  • ✅ Use stop_words='english' to remove common words (the, is, and) that don't help classification
  • ✅ Check per-class metrics (Project 19) — overall accuracy can hide poor performance on minority classes

5 · What Should NOT Do

  • ❌ Do NOT fit the vectorizer on test data — fit on train, transform both (same principle as scaling in Project 13)
  • ❌ Do NOT ignore preprocessing — lowercasing, removing punctuation, and stop words matter
  • ❌ Do NOT use tree-based models (Project 21) directly on raw text — they need numeric features first
  • ❌ Do NOT assume Naive Bayes is always best for text — it's a strong baseline, but Logistic Regression and SVM often beat it
  • ❌ Do NOT skip n-grams — bigrams ("not good") capture meaning that unigrams ("not", "good") miss

6 · When to Use What

  • 🔹 MultinomialNB + CountVectorizer → fast baseline for text classification — start here
  • 🔹 MultinomialNB + TF-IDF → better when some words appear in many documents (downweights them)
  • 🔹 Logistic Regression + TF-IDF → often beats Naive Bayes; good next step
  • 🔹 SVM + TF-IDF → strong performer on text; handles high-dimensional sparse data well
  • 🔹 Pipeline → always wrap vectorizer + classifier in a Pipeline (Project 16)

7 · When NOT to Use

  • 🚫 When word order matters — Bag of Words ignores order; "dog bites man" = "man bites dog"
  • 🚫 When you need semantic understanding — BoW/TF-IDF don't understand meaning, just word frequency
  • 🚫 When the text is very short (1-2 words) — not enough signal for BoW features
  • 🚫 When you have very few labeled examples — Naive Bayes needs enough examples per class

8 · How It Only Works — Internal Mechanics

Bag of Words (CountVectorizer)

  • Each document becomes a vector of word counts: [0, 3, 0, 1, 2, 0, ...] — one dimension per word in the vocabulary
  • Vocabulary is built from the training data — typically thousands of unique words
  • The result is a sparse matrix — most entries are 0 (a document uses only a fraction of the vocabulary)
  • Order is lost: "the cat sat on the mat" → {cat: 1, sat: 1, mat: 1, the: 2, on: 1}

TF-IDF (Term Frequency–Inverse Document Frequency)

  • TF: how often a word appears in THIS document (same as CountVectorizer)
  • IDF: log(total documents / documents containing this word) — rare words get higher weight
  • TF-IDF = TF × IDF — words that are frequent in one document but rare overall get the highest score
  • "the" appears everywhere → low IDF → low TF-IDF. "bluetooth" appears only in Electronics → high IDF → high TF-IDF

Naive Bayes — The "Naive" Assumption

  • Bayes' theorem: P(class|features) ∝ P(features|class) × P(class)
  • The "naive" part: assumes all features (words) are independent given the class
  • This is obviously wrong — "machine" and "learning" are correlated — but it works surprisingly well
  • MultinomialNB: assumes features are word counts (or TF-IDF scores) — perfect for text
  • Training is just counting: P(word|class) = (count of word in class + α) / (total words in class + α × vocab_size)
  • α (alpha) is Laplace smoothing — prevents zero probabilities for unseen words

Why Naive Bayes Works for Text

  • Text data is high-dimensional (thousands of features) and sparse — Naive Bayes handles this naturally
  • Training is O(n × d) — just counting; no iterative optimization needed
  • The independence assumption is wrong but the resulting classifier is still good — it gets the ranking right even if probabilities are off
  • With enough data, the word frequency patterns are strong enough to overcome the naive assumption

N-grams — Capturing Word Pairs

  • Unigrams: individual words — "not", "good" are separate features
  • Bigrams: word pairs — "not good" is one feature (captures negation!)
  • ngram_range=(1,2): use both unigrams and bigrams
  • More n-grams = more features = more memory — balance with max_features

9 · Common Pitfalls and Failure Modes

  • ⚠️ Fitting vectorizer on all data — fit on train only, transform both (data leakage, same as Project 13)
  • ⚠️ Too large vocabulary — 100K features mostly noise; use max_features=5000-10000
  • ⚠️ Ignoring stop words — "the", "is", "and" add noise without information
  • ⚠️ No n-grams — unigrams miss "not good", "high quality", "low battery"
  • ⚠️ Evaluating only accuracy — if 60% of products are Electronics, a dummy classifier gets 60% (Project 19)

10 · Scalability Considerations

  • Naive Bayes training is O(n × d) — extremely fast even on millions of documents
  • TF-IDF matrices are sparse — scipy sparse matrices use minimal memory
  • Vocabulary size is the main memory concern — use max_features to cap it
  • For very large corpora, use HashingVectorizer — fixed-size feature space, no vocabulary storage
  • Naive Bayes supports partial_fit — can train incrementally on data that doesn't fit in memory

11 · Interpretability Considerations

  • ✅ Naive Bayes is inherently interpretable — you can inspect P(word|class) for each word
  • ✅ Top words per class are intuitive: Electronics → "battery", "screen", "wireless"
  • ✅ TF-IDF weights show which words are most distinctive for each document
  • ⚠️ High-dimensional feature space makes global visualization difficult
  • ⚠️ Misclassifications often reveal ambiguous products — "wireless speaker" could be Electronics or Home

import numpy as np
import pandas as pd
from sklearn.feature_extraction.text import CountVectorizer, TfidfVectorizer
from sklearn.naive_bayes import MultinomialNB
from sklearn.linear_model import LogisticRegression
from sklearn.pipeline import Pipeline
from sklearn.model_selection import train_test_split, cross_val_score, StratifiedKFold
from sklearn.metrics import classification_report, confusion_matrix
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

# ─── Step 1: Generate product descriptions ───
np.random.seed(42)

electronics_words = [
    "wireless bluetooth headphones noise cancelling battery life",
    "laptop computer processor RAM SSD storage screen display",
    "smartphone camera megapixel battery charging fast screen",
    "tablet touchscreen wifi bluetooth portable lightweight",
    "smart watch fitness tracker heart rate GPS waterproof",
    "USB cable charger adapter fast charging compatible",
    "portable speaker bluetooth waterproof bass sound quality",
    "keyboard mechanical RGB backlit gaming wireless",
    "mouse optical wireless ergonomic DPI gaming sensor",
    "monitor LED display resolution refresh rate HDMI",
]

clothing_words = [
    "cotton t-shirt comfortable breathable casual summer wear",
    "denim jeans stretch slim fit comfortable everyday",
    "winter jacket warm insulated waterproof hood zipper",
    "running shoes lightweight cushioned breathable mesh sole",
    "dress formal elegant evening party occasion fitted",
    "sweater knit wool warm cozy pullover crew neck",
    "shorts athletic quick dry comfortable elastic waist",
    "socks cotton moisture wicking comfortable ankle pack",
    "hat cap adjustable sun protection outdoor casual",
    "scarf wool warm winter soft wrap fashion accessory",
]

home_words = [
    "stainless steel kitchen knife set sharp cutting board",
    "bedding set cotton sheets pillow comfortable soft",
    "vacuum cleaner cordless powerful suction lightweight",
    "coffee maker programmable brew carafe filter drip",
    "towel set bath cotton absorbent soft plush",
    "candle scented soy wax aromatherapy relaxing jar",
    "storage organizer shelf closet bins containers",
    "curtains blackout window thermal insulated panel",
    "blender smoothie mixer powerful motor pitcher ice",
    "lamp desk LED adjustable brightness reading light",
]

sports_words = [
    "yoga mat non slip exercise fitness thick cushion",
    "dumbbell weight set adjustable strength training home",
    "resistance bands elastic workout fitness stretching",
    "water bottle insulated stainless steel sports gym",
    "jump rope speed cardio workout fitness training",
    "foam roller muscle recovery massage exercise therapy",
    "cycling gloves padded grip breathable mountain bike",
    "tennis racket lightweight grip control power spin",
    "basketball indoor outdoor rubber grip durable bounce",
    "hiking backpack waterproof lightweight trail camping",
]

def generate_descriptions(templates, category, n=300):
    descriptions = []
    for _ in range(n):
        template = np.random.choice(templates)
        words = template.split()
        n_words = np.random.randint(4, len(words) + 1)
        selected = np.random.choice(words, size=n_words, replace=False)
        noise_words = np.random.choice(['great', 'new', 'best', 'quality', 'premium',
                                         'sale', 'free', 'shipping', 'brand', 'original'],
                                        size=np.random.randint(1, 4))
        desc = ' '.join(list(selected) + list(noise_words))
        descriptions.append(desc)
    return descriptions

texts, labels = [], []
for words, label, n in [(electronics_words, 'Electronics', 400),
                         (clothing_words, 'Clothing', 350),
                         (home_words, 'Home & Kitchen', 300),
                         (sports_words, 'Sports', 250)]:
    texts.extend(generate_descriptions(words, label, n))
    labels.extend([label] * n)

texts = np.array(texts)
labels = np.array(labels)

# Shuffle
idx = np.random.permutation(len(texts))
texts, labels = texts[idx], labels[idx]

X_train, X_test, y_train, y_test = train_test_split(
    texts, labels, test_size=0.25, stratify=labels, random_state=42)

print("=== Product Description Dataset ===")
print(f"  Total: {len(texts)} products")
print(f"  Train: {len(y_train)}, Test: {len(y_test)}")
print(f"  Categories: {dict(zip(*np.unique(labels, return_counts=True)))}")
print(f"\n  Sample descriptions:")
for i in range(3):
    print(f"    [{y_train[i]}] {X_train[i][:60]}...")

# ─── Step 2: CountVectorizer — Bag of Words ───
print("\n=== CountVectorizer (Bag of Words) ===")
count_vec = CountVectorizer(stop_words='english', max_features=5000)
X_train_counts = count_vec.fit_transform(X_train)
X_test_counts = count_vec.transform(X_test)

print(f"  Vocabulary size: {len(count_vec.vocabulary_)}")
print(f"  Feature matrix shape: {X_train_counts.shape}")
print(f"  Sparsity: {1 - X_train_counts.nnz / (X_train_counts.shape[0] * X_train_counts.shape[1]):.1%}")

# ─── Step 3: TfidfVectorizer ───
print("\n=== TfidfVectorizer ===")
tfidf_vec = TfidfVectorizer(stop_words='english', max_features=5000)
X_train_tfidf = tfidf_vec.fit_transform(X_train)
X_test_tfidf = tfidf_vec.transform(X_test)

# Show top TF-IDF words for a sample document
sample_idx = 0
sample_tfidf = X_train_tfidf[sample_idx].toarray().flatten()
top_word_idx = sample_tfidf.argsort()[::-1][:5]
vocab_inv = {v: k for k, v in tfidf_vec.vocabulary_.items()}
print(f"  Top TF-IDF words in sample: {[vocab_inv[i] for i in top_word_idx]}")

# ─── Step 4: Naive Bayes — CountVectorizer vs TF-IDF ───
print("\n=== Model Comparison ===")
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

pipelines = {
    'NB + CountVec': Pipeline([
        ('vec', CountVectorizer(stop_words='english', max_features=5000)),
        ('clf', MultinomialNB())
    ]),
    'NB + TF-IDF': Pipeline([
        ('vec', TfidfVectorizer(stop_words='english', max_features=5000)),
        ('clf', MultinomialNB())
    ]),
    'NB + TF-IDF + bigrams': Pipeline([
        ('vec', TfidfVectorizer(stop_words='english', max_features=5000, ngram_range=(1, 2))),
        ('clf', MultinomialNB())
    ]),
    'LogReg + TF-IDF': Pipeline([
        ('vec', TfidfVectorizer(stop_words='english', max_features=5000)),
        ('clf', LogisticRegression(max_iter=1000, random_state=42))
    ]),
}

results = {}
for name, pipe in pipelines.items():
    scores = cross_val_score(pipe, X_train, y_train, cv=cv, scoring='f1_macro')
    results[name] = scores
    print(f"  {name:>25}: F1-macro = {scores.mean():.4f} ± {scores.std():.4f}")

# ─── Step 5: Best model — detailed evaluation ───
best_name = max(results, key=lambda k: results[k].mean())
best_pipe = pipelines[best_name]
best_pipe.fit(X_train, y_train)
y_pred = best_pipe.predict(X_test)

print(f"\n=== Best Model: {best_name} ===")
print(classification_report(y_test, y_pred))

# ─── Step 6: Confusion matrix ───
cm = confusion_matrix(y_test, y_pred, labels=best_pipe.classes_)
fig, ax = plt.subplots(figsize=(8, 6))
im = ax.imshow(cm, cmap='Blues')
ax.set_xticks(range(len(best_pipe.classes_)))
ax.set_yticks(range(len(best_pipe.classes_)))
ax.set_xticklabels(best_pipe.classes_, rotation=45, ha='right')
ax.set_yticklabels(best_pipe.classes_)
ax.set_xlabel('Predicted')
ax.set_ylabel('Actual')
ax.set_title(f'Confusion Matrix — {best_name}')
for i in range(len(best_pipe.classes_)):
    for j in range(len(best_pipe.classes_)):
        ax.text(j, i, str(cm[i, j]), ha='center', va='center',
                color='white' if cm[i, j] > cm.max() / 2 else 'black')
plt.colorbar(im)
plt.tight_layout()
plt.savefig('project_28_confusion_matrix.png', dpi=100, bbox_inches='tight')
print("✅ Confusion matrix saved as project_28_confusion_matrix.png")

# ─── Step 7: Top words per category ───
print("\n=== Top Discriminative Words Per Category ===")
nb_pipe = pipelines['NB + TF-IDF']
nb_pipe.fit(X_train, y_train)
vectorizer = nb_pipe.named_steps['vec']
classifier = nb_pipe.named_steps['clf']
feature_names_arr = np.array(vectorizer.get_feature_names_out())

for i, category in enumerate(classifier.classes_):
    log_probs = classifier.feature_log_prob_[i]
    top_idx = log_probs.argsort()[::-1][:8]
    top_words = feature_names_arr[top_idx]
    print(f"  {category:>15}: {', '.join(top_words)}")

# ─── Step 8: Effect of max_features and alpha ───
print("\n=== Tuning: max_features and alpha ===")
fig, axes = plt.subplots(1, 2, figsize=(14, 5))

# max_features
max_features_range = [100, 500, 1000, 2000, 5000, 10000]
mf_scores = []
for mf in max_features_range:
    pipe = Pipeline([
        ('vec', TfidfVectorizer(stop_words='english', max_features=mf)),
        ('clf', MultinomialNB())
    ])
    scores = cross_val_score(pipe, X_train, y_train, cv=cv, scoring='f1_macro')
    mf_scores.append(scores.mean())

axes[0].plot(max_features_range, mf_scores, 'o-', color='#58a6ff', linewidth=2)
axes[0].set_xlabel('max_features')
axes[0].set_ylabel('CV F1-macro')
axes[0].set_title('Effect of Vocabulary Size')
axes[0].set_xscale('log')
axes[0].grid(True, alpha=0.2)

# alpha (smoothing)
alpha_range = [0.001, 0.01, 0.1, 0.5, 1.0, 2.0, 5.0]
alpha_scores = []
for alpha in alpha_range:
    pipe = Pipeline([
        ('vec', TfidfVectorizer(stop_words='english', max_features=5000)),
        ('clf', MultinomialNB(alpha=alpha))
    ])
    scores = cross_val_score(pipe, X_train, y_train, cv=cv, scoring='f1_macro')
    alpha_scores.append(scores.mean())

axes[1].plot(alpha_range, alpha_scores, 'o-', color='#f0883e', linewidth=2)
axes[1].set_xlabel('Alpha (Laplace Smoothing)')
axes[1].set_ylabel('CV F1-macro')
axes[1].set_title('Effect of Smoothing Parameter')
axes[1].set_xscale('log')
axes[1].grid(True, alpha=0.2)

plt.tight_layout()
plt.savefig('project_28_tuning.png', dpi=100, bbox_inches='tight')

Key Takeaway

  • Text must be converted to numbers before modeling — CountVectorizer (word counts) or TF-IDF (weighted counts)
  • Naive Bayes is a fast, effective baseline for text classification — it handles sparse, high-dimensional data naturally
  • TF-IDF usually outperforms raw counts by downweighting common words
  • Always use a Pipeline (Project 16) to prevent data leakage in the vectorization step
  • N-grams capture word pairs that unigrams miss — "not good" vs "not" + "good"
  • Check per-class metrics (Project 19) — overall accuracy hides category-level problems
