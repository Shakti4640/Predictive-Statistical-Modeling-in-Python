Project 16: Surviving the Titanic — End-to-End Feature Engineering Pipeline

Difficulty: Medium  |  Builds on: Projects 13, 15

1 · Hypothetical Problem

  • You're given Titanic passenger data: age, sex, class, fare, embarked port — predict survival
  • Real-world data is MESSY: missing ages, categorical features (sex, embarked), mixed types
  • Previous projects used clean numeric data — now we handle the full preprocessing pipeline
  • Goal → build a complete, reproducible preprocessing + modeling pipeline

2 · How to Approach It

  1. Inspect the data: what's missing? What's numeric vs categorical?
  2. Handle missing values: impute with median (numeric) or mode (categorical)
  3. Encode categorical features: one-hot for nominal (sex, embarked), ordinal for ordered (class)
  4. Scale numeric features (Project 13)
  5. Combine everything into a ColumnTransformer + Pipeline
  6. Evaluate with cross-validation (Project 15)

3 · How Can We Solve It

  1. Create a Titanic-like synthetic dataset with missing values and mixed types
  2. Build separate transformers for numeric and categorical columns
  3. Combine with ColumnTransformer
  4. Chain with a classifier in a Pipeline
  5. Cross-validate the entire pipeline (no leakage)

4 · What Should Do

  • ✅ Use ColumnTransformer to apply different preprocessing to different column types
  • ✅ Impute missing values INSIDE the pipeline — not before splitting
  • ✅ Use one-hot encoding for nominal categories (no natural order)
  • ✅ Use ordinal encoding for ordered categories (1st class > 2nd class > 3rd class)
  • ✅ The entire pipeline should be one object: preprocess + model

5 · What Should NOT Do

  • ❌ Do NOT drop rows with missing values — you lose data and it doesn't work at prediction time
  • ❌ Do NOT impute before splitting — test data statistics leak into imputation
  • ❌ Do NOT one-hot encode ordinal features — you lose the ordering information
  • ❌ Do NOT label-encode nominal features for linear models — "male=0, female=1" implies an order
  • ❌ Do NOT preprocess manually outside the pipeline — it breaks reproducibility

6 · When to Use What

  • 🔹 SimpleImputer(strategy='median') → for numeric features with missing values
  • 🔹 SimpleImputer(strategy='most_frequent') → for categorical features with missing values
  • 🔹 OneHotEncoder → for nominal categories (city, color, sex)
  • 🔹 OrdinalEncoder → for ordered categories (low/medium/high, class 1/2/3)
  • 🔹 ColumnTransformer → to apply different transformations to different columns
  • 🔹 Pipeline → to chain preprocessing + model into one reproducible object

7 · When NOT to Use

  • 🚫 Median imputation assumes missing-at-random — if missingness is informative, add a "is_missing" indicator
  • 🚫 One-hot encoding creates many columns with high-cardinality features (1000 cities → 1000 columns)
  • 🚫 Pipeline can't handle feature engineering that requires domain knowledge (e.g., creating "family_size" from siblings + parents)

8 · How It Only Works — Internal Mechanics

ColumnTransformer

  • Takes a list of (name, transformer, columns) tuples
  • Applies each transformer to its specified columns independently
  • Concatenates the results into a single feature matrix
  • Handles the complexity of mixed-type data in one clean object

OneHotEncoder

  • Converts each category into a binary column: male→[1,0], female→[0,1]
  • No implied ordering — each category is equidistant
  • Use handle_unknown='ignore' for new categories at prediction time
  • Use drop='first' to avoid multicollinearity (the "dummy variable trap")

Imputation Strategies

  • Mean: sensitive to outliers, assumes symmetric distribution
  • Median: robust to outliers, good default for numeric
  • Mode (most_frequent): for categorical features
  • Constant: fill with a specific value (e.g., "Unknown")

Pipeline Execution Flow

  1. pipe.fit(X_train, y_train): fits imputer → fits encoder → fits scaler → fits model
  2. pipe.predict(X_test): transforms X_test through each step → predicts
  3. Each step only sees training statistics — no leakage

9 · Common Pitfalls and Failure Modes

  • ⚠️ Imputing before splitting — test data mean/median leaks into training imputation
  • ⚠️ Forgetting handle_unknown='ignore' — new categories at prediction time cause errors
  • ⚠️ One-hot encoding high-cardinality features — 1000 categories = 1000 new columns
  • ⚠️ Not checking what ColumnTransformer outputs — column order may change
  • ⚠️ Manual preprocessing outside pipeline — works in notebook, breaks in production

10 · Scalability Considerations

  • Imputation: O(n) — trivial
  • One-hot encoding: creates p_new = Σ(categories per feature) columns — can explode with high cardinality
  • Pipeline overhead is minimal — it's organizational, not computational
  • For very high cardinality, use target encoding or hashing (advanced topics)

11 · Interpretability Considerations

  • ✅ Pipeline makes the entire workflow transparent and reproducible
  • ✅ One-hot encoded features are interpretable: "being female increases survival odds"
  • ⚠️ After one-hot encoding, feature names become less intuitive (sex_female, sex_male)
  • ⚠️ Imputed values are synthetic — they can affect model interpretation

12 · Complete Working Code

import numpy as np
import pandas as pd
from sklearn.compose import ColumnTransformer
from sklearn.pipeline import Pipeline
from sklearn.impute import SimpleImputer
from sklearn.preprocessing import StandardScaler, OneHotEncoder, OrdinalEncoder
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import cross_val_score, StratifiedKFold

# ─── Step 1: Synthetic Titanic-like data ───
np.random.seed(42)
n = 600

pclass = np.random.choice([1, 2, 3], n, p=[0.2, 0.3, 0.5])
sex = np.random.choice(['male', 'female'], n, p=[0.6, 0.4])
age = np.random.uniform(1, 80, n)
fare = np.where(pclass == 1, np.random.uniform(50, 500, n),
         np.where(pclass == 2, np.random.uniform(15, 80, n),
                  np.random.uniform(5, 30, n)))
embarked = np.random.choice(['S', 'C', 'Q'], n, p=[0.7, 0.2, 0.1])

# Survival depends on class, sex, age
log_odds = (-1
            - 0.5 * pclass
            + 1.5 * (sex == 'female').astype(float)
            - 0.02 * age
            + 0.005 * fare
            + np.random.normal(0, 0.3, n))
survived = (1 / (1 + np.exp(-log_odds)) > 0.5).astype(int)

df = pd.DataFrame({
    'pclass': pclass, 'sex': sex, 'age': age,
    'fare': fare, 'embarked': embarked, 'survived': survived
})

# Introduce missing values (realistic)
missing_age = np.random.choice(n, size=int(n * 0.2), replace=False)
missing_embarked = np.random.choice(n, size=int(n * 0.05), replace=False)
df.loc[missing_age, 'age'] = np.nan
df.loc[missing_embarked, 'embarked'] = np.nan

print("=== Data Summary ===")
print(f"  Shape: {df.shape}")
print(f"  Survived: {survived.mean():.1%}")
print(f"\n  Missing values:")
print(df.isnull().sum().to_string())

# ─── Step 2: Define column groups ───
numeric_features = ['age', 'fare']
categorical_features = ['sex', 'embarked']
ordinal_features = ['pclass']

# ─── Step 3: Build transformers ───
numeric_transformer = Pipeline([
    ('imputer', SimpleImputer(strategy='median')),
    ('scaler', StandardScaler())
])

categorical_transformer = Pipeline([
    ('imputer', SimpleImputer(strategy='most_frequent')),
    ('encoder', OneHotEncoder(handle_unknown='ignore', drop='first'))
])

ordinal_transformer = Pipeline([
    ('encoder', OrdinalEncoder(categories=[[1, 2, 3]]))
])

# ─── Step 4: ColumnTransformer ───
preprocessor = ColumnTransformer(transformers=[
    ('num', numeric_transformer, numeric_features),
    ('cat', categorical_transformer, categorical_features),
    ('ord', ordinal_transformer, ordinal_features)
])

# ─── Step 5: Full Pipeline ───
full_pipeline = Pipeline([
    ('preprocessor', preprocessor),
    ('classifier', LogisticRegression(max_iter=1000, random_state=42))
])

# ─── Step 6: Cross-validate (Project 15) ───
X = df.drop('survived', axis=1)
y = df['survived']

cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(full_pipeline, X, y, cv=cv, scoring='accuracy')

print(f"\n=== Cross-Validation Results ===")
print(f"  Scores: {scores.round(4)}")
print(f"  Mean:   {scores.mean():.2%} ± {scores.std():.2%}")

# ─── Step 7: Inspect the pipeline ───
full_pipeline.fit(X, y)
print(f"\n=== Pipeline Steps ===")
for name, step in full_pipeline.steps:
    print(f"  {name}: {type(step).__name__}")

# Show transformed feature names
print(f"\n=== Transformed Features ===")
transformed = preprocessor.transform(X)
print(f"  Original columns: {list(X.columns)}")
print(f"  Transformed shape: {transformed.shape}")

# ─── Step 8: Coefficient analysis ───
coefs = full_pipeline.named_steps['classifier'].coef_[0]
feature_labels = (numeric_features +
                  ['sex_male', 'embarked_Q', 'embarked_S'] +
                  ['pclass_ordinal'])
print(f"\n=== Feature Coefficients ===")
for name, coef in sorted(zip(feature_labels, coefs), key=lambda x: -abs(x[1])):
    direction = "↑ survival" if coef > 0 else "↓ survival"
    print(f"  {name:>18}: {coef:+.4f}  ({direction})")

13 · Key Takeaway + Bridge to Next Project

Key Takeaway

  • Real data has missing values and mixed types — Pipeline + ColumnTransformer handles it all
  • Impute inside the pipeline to prevent data leakage
  • One-hot encode nominal features, ordinal encode ordered features
  • The entire preprocessing + model is one reproducible object
  • This pipeline pattern is used in EVERY professional ML project
