Project 3: House Price From Square Footage — One Feature, One Output

Difficulty: Very Easy  |  Builds on: Project 2

1 · Hypothetical Problem

  • You're a junior data analyst at a real estate agency
  • Agents ask: "Given a house with X square feet, what price should we list it at?"
  • You have data from 50 recent sales: square footage and sale price
  • In Project 2 we learned to fit a line — now we focus on understanding the error
  • Goal → fit a linear model, then deeply analyze the cost function, residuals, and what "good fit" really means

2 · How to Approach It

  1. Same linear regression setup as Project 2: price = slope × sqft + intercept
  2. But now we ask deeper questions:
    • What exactly is the cost function measuring?
    • What do the residuals (errors) look like?
    • How do we visualize "goodness of fit" beyond a single number?
  3. Plot the cost function landscape — see how MSE changes with different slopes
  4. Plot residuals — they should look random if the model is appropriate
  5. Introduce the concept of underfitting through residual patterns

3 · How Can We Solve It

  1. Generate synthetic house data (sqft → price with realistic noise)
  2. Train/test split (Project 1 concept)
  3. Fit linear regression (Project 2 concept)
  4. Compute residuals: residual = actual - predicted
  5. Plot residuals vs predicted values — check for patterns
  6. Plot the MSE cost surface by varying the slope — see the "bowl" shape
  7. Compare prediction vs actual in a 45-degree line plot

4 · What Should Do

  • ✅ Always plot residuals after fitting — they reveal problems the MSE number hides
  • ✅ Residuals should be randomly scattered around zero — any pattern means the model is missing something
  • ✅ Use a predicted-vs-actual plot — points should cluster around the 45° diagonal line
  • ✅ Understand MSE as a landscape — OLS finds the bottom of the bowl
  • ✅ Report RMSE in the target's units — "predictions are off by ~$X on average"
  • ✅ Check if residuals grow with the input (heteroscedasticity) — this violates OLS assumptions

5 · What Should NOT Do

  • ❌ Do NOT just report MSE and move on — the number alone doesn't tell you WHERE the model fails
  • ❌ Do NOT ignore residual patterns — a U-shaped residual plot screams "nonlinear relationship"
  • ❌ Do NOT assume low MSE means a good model — MSE depends on the scale of the target
  • ❌ Do NOT forget to check if the linear assumption holds — plot first, model second
  • ❌ Do NOT compare MSE across different datasets — MSE of $1000 on house prices vs $1000 on car prices means different things

6 · When to Use What

  • 🔹 Residual plot → after EVERY regression model — it's the diagnostic X-ray
  • 🔹 Predicted vs Actual plot → to visually assess overall model quality
  • 🔹 Cost function visualization → when teaching/learning optimization concepts
  • 🔹 RMSE over MSE → when communicating results to non-technical stakeholders (same units as target)
  • 🔹 Scatter plot with fitted line → for single-feature regression (same as Project 2)

7 · When NOT to Use

  • 🚫 If residuals show a clear curve → linear model is wrong, need polynomial (Project 20) or other nonlinear model
  • 🚫 If residuals fan out (grow larger for bigger predictions) → OLS assumptions violated, need transformation or weighted regression
  • 🚫 If there are clusters in the residual plot → there might be hidden subgroups in the data
  • 🚫 Cost function visualization doesn't scale beyond 2-3 parameters — useful for teaching, not for high-dimensional models

8 · How It Only Works — Internal Mechanics

The Cost Function (MSE) as a Landscape

  • For a single-feature linear model, MSE is a function of two parameters: slope and intercept
  • If you fix the intercept and vary the slope, MSE traces a parabola (U-shape)
  • The bottom of the parabola = the optimal slope = what OLS finds
  • With both parameters free, the cost surface is a 3D bowl — OLS finds the lowest point

Residuals — The Model's Confession

  • Residual = actual value − predicted value
  • Positive residual → model underestimated
  • Negative residual → model overestimated
  • If residuals are random (no pattern) → the model captured all the learnable signal
  • If residuals show a pattern → the model missed something systematic

Underfitting Signal in Residuals

  • A curved pattern in residuals means the true relationship is nonlinear
  • The linear model is too simple — it "underfits" the data
  • This connects to the bias-variance tradeoff (Project 8)

Predicted vs Actual Plot

  • Plot predicted (x-axis) vs actual (y-axis)
  • Perfect model → all points on the 45° diagonal
  • Spread around the diagonal = prediction error
  • Systematic deviation from diagonal = bias in the model

9 · Common Pitfalls and Failure Modes

  • ⚠️ Ignoring residual plots — the single most common beginner mistake in regression
  • ⚠️ Confusing low MSE with good model — MSE of 100 is great for house prices ($100K scale) but terrible for exam scores (0-100 scale)
  • ⚠️ Not checking for outliers in residuals — a few extreme residuals can inflate MSE dramatically
  • ⚠️ Plotting residuals vs X instead of vs predicted — for simple regression they look similar, but for multiple regression (Project 9) only residuals vs predicted is correct
  • ⚠️ Assuming residuals should be exactly zero — noise is expected; the goal is no PATTERN, not no error

10 · Scalability Considerations

  • Residual computation is O(n) — trivial even for millions of rows
  • Plotting millions of residuals → use alpha transparency or hexbin plots
  • Cost function visualization is only practical for 1-2 parameters — not scalable to high dimensions
  • For large datasets, sample residuals for plotting (e.g., random 10,000 points)
  • The diagnostic concepts (residual analysis, predicted vs actual) scale to ANY regression model in later projects

11 · Interpretability Considerations

  • ✅ Residual plots make the model's failures visible — even non-technical people can see patterns
  • ✅ Predicted vs actual plot is intuitive: "how close are we to perfect?"
  • ✅ RMSE in dollars: "our predictions are off by about $15,000 on average" — actionable
  • ✅ The cost function bowl visualization builds intuition for optimization (used in every ML algorithm)
  • ⚠️ Residual analysis is a skill — it takes practice to distinguish real patterns from random noise

12 · Complete Working Code

import numpy as np
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

# ─── Step 1: Synthetic house data ───
np.random.seed(42)
sqft = np.random.uniform(600, 3000, size=50)
# True relationship: price = 50 * sqft + 30000 + noise
price = 50 * sqft + 30000 + np.random.normal(0, 15000, size=50)

print("=== Sample Data (first 8 houses) ===")
for i in range(8):
    print(f"  House {i+1}: {sqft[i]:.0f} sqft → ${price[i]:,.0f}")

# ─── Step 2: Train/Test Split ───
X_train, y_train = sqft[:35], price[:35]
X_test, y_test = sqft[35:], price[35:]
print(f"\nTrain: {len(X_train)} houses | Test: {len(X_test)} houses")

# ─── Step 3: Fit Linear Regression (Project 2 skill) ───
model = LinearRegression()
model.fit(X_train.reshape(-1, 1), y_train)
print(f"\nModel: price = {model.coef_[0]:.2f} × sqft + {model.intercept_:,.2f}")

# ─── Step 4: Predictions ───
y_train_pred = model.predict(X_train.reshape(-1, 1))
y_test_pred = model.predict(X_test.reshape(-1, 1))

train_rmse = np.sqrt(mean_squared_error(y_train, y_train_pred))
test_rmse = np.sqrt(mean_squared_error(y_test, y_test_pred))
print(f"Train RMSE: ${train_rmse:,.0f}")
print(f"Test RMSE:  ${test_rmse:,.0f}")

# ─── Step 5: Residual Analysis ───
train_residuals = y_train - y_train_pred
test_residuals = y_test - y_test_pred

print(f"\n=== Residual Statistics (Test Set) ===")
print(f"  Mean residual:   ${np.mean(test_residuals):,.0f}  (should be near 0)")
print(f"  Std of residuals: ${np.std(test_residuals):,.0f}")
print(f"  Max overestimate: ${np.min(test_residuals):,.0f}")
print(f"  Max underestimate: ${np.max(test_residuals):,.0f}")

# ─── Step 6: Cost Function Landscape ───
# Vary the slope while keeping intercept fixed → see how MSE changes
slopes = np.linspace(0, 100, 200)
mse_values = []
for s in slopes:
    preds = s * X_train + model.intercept_
    mse_values.append(np.mean((y_train - preds) ** 2))

# ─── Step 7: Four-Panel Diagnostic Plot ───
fig, axes = plt.subplots(2, 2, figsize=(12, 10))

# Panel 1: Scatter + fitted line
ax1 = axes[0, 0]
ax1.scatter(X_train, y_train, color='#58a6ff', alpha=0.6, label='Train')
ax1.scatter(X_test, y_test, color='#f0883e', alpha=0.6, label='Test')
x_line = np.linspace(500, 3100, 100)
ax1.plot(x_line, model.predict(x_line.reshape(-1, 1)), color='#3fb950', linewidth=2)
ax1.set_xlabel('Square Footage')
ax1.set_ylabel('Price ($)')
ax1.set_title('Data + Fitted Line')
ax1.legend()
ax1.grid(True, alpha=0.2)

# Panel 2: Residuals vs Predicted
ax2 = axes[0, 1]
ax2.scatter(y_train_pred, train_residuals, color='#58a6ff', alpha=0.6, label='Train')
ax2.scatter(y_test_pred, test_residuals, color='#f0883e', alpha=0.6, label='Test')
ax2.axhline(y=0, color='#3fb950', linestyle='--', linewidth=1)
ax2.set_xlabel('Predicted Price ($)')
ax2.set_ylabel('Residual ($)')
ax2.set_title('Residuals vs Predicted (check for patterns)')
ax2.legend()
ax2.grid(True, alpha=0.2)

# Panel 3: Predicted vs Actual
ax3 = axes[1, 0]
ax3.scatter(y_test_pred, y_test, color='#f0883e', alpha=0.7)
min_val = min(y_test.min(), y_test_pred.min())
max_val = max(y_test.max(), y_test_pred.max())
ax3.plot([min_val, max_val], [min_val, max_val], color='#3fb950', linestyle='--', linewidth=2, label='Perfect prediction')
ax3.set_xlabel('Predicted Price ($)')
ax3.set_ylabel('Actual Price ($)')
ax3.set_title('Predicted vs Actual (closer to diagonal = better)')
ax3.legend()
ax3.grid(True, alpha=0.2)

# Panel 4: Cost Function Landscape
ax4 = axes[1, 1]
ax4.plot(slopes, mse_values, color='#bc8cff', linewidth=2)
ax4.axvline(x=model.coef_[0], color='#f85149', linestyle='--', label=f'OLS slope={model.coef_[0]:.1f}')
ax4.set_xlabel('Slope Value')
ax4.set_ylabel('MSE')
ax4.set_title('Cost Function — MSE vs Slope')
ax4.legend()
ax4.grid(True, alpha=0.2)

plt.tight_layout()
plt.savefig('project_03_diagnostics.png', dpi=100, bbox_inches='tight')
print("\n✅ Diagnostic plots saved as project_03_diagnostics.png")

13 · Key Takeaway + Bridge to Next Project

Key Takeaway

  • MSE is a cost function — OLS minimizes it, and you can visualize it as a bowl
  • Residuals are the model's confession — always plot them after fitting
  • Random residuals = good model; patterned residuals = model is missing something
  • Predicted vs actual plot gives an instant visual of model quality
  • These diagnostic tools apply to EVERY regression model you'll build in later projects
