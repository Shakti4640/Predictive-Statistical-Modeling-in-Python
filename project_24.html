Project 24: Reducing Dimensions in High-Dimensional Gene Data — PCA

PCA Explained Variance Eigenvectors Scree Plot Dimensionality Reduction

Difficulty: Medium  |  Builds on: Projects 13, 14

1 · Hypothetical Problem

  • You're a bioinformatics researcher analyzing gene expression data to classify tumor types
  • Dataset: 200 patient samples, each with 5,000 gene expression measurements
  • Problem: 5,000 features but only 200 samples — models overfit instantly (curse of dimensionality)
  • Many genes are correlated — groups of genes activate together
  • Goal → reduce 5,000 dimensions to a manageable number while preserving the important patterns

2 · How to Approach It

  1. Recall from Project 13: features must be scaled before distance-based or variance-based methods
  2. PCA finds the directions (principal components) along which the data varies most
  3. The first few components capture most of the variance — the rest is noise
  4. Use the scree plot (explained variance) to decide how many components to keep
  5. Feed the reduced data into a classifier and compare with the original high-dimensional data

3 · How Can We Solve It

  1. Generate high-dimensional gene expression data with underlying structure
  2. Apply PCA and examine explained variance ratios
  3. Plot the scree plot — find the "elbow" where adding components gives diminishing returns
  4. Visualize data in 2D using the first two principal components
  5. Compare classifier performance: original features vs PCA-reduced features
  6. Show how PCA helps when p >> n (more features than samples)

4 · What Should Do

  • ✅ Scale features before PCA — PCA is based on variance, so features must be on the same scale (Project 13)
  • ✅ Use the scree plot to choose the number of components — look for the elbow
  • ✅ Check cumulative explained variance — aim for 90-95% of total variance
  • ✅ Fit PCA on training data only, then transform both train and test
  • ✅ Use PCA as a preprocessing step in a Pipeline

5 · What Should NOT Do

  • ❌ Do NOT apply PCA without scaling — features with larger variance dominate the components
  • ❌ Do NOT fit PCA on the entire dataset (train + test) — that's data leakage
  • ❌ Do NOT always use PCA — tree-based models (Projects 21, 23) handle high dimensions natively
  • ❌ Do NOT interpret principal components as original features — they're linear combinations
  • ❌ Do NOT use PCA to reduce from 10 to 8 features — the overhead isn't worth it; PCA shines with massive reduction

6 · When to Use What

  • 🔹 PCA → when you have many correlated features and want to reduce dimensions
  • 🔹 Lasso → when you want to select original features, not create new ones (Project 18)
  • 🔹 K-Means on PCA → reduce dimensions first, then cluster (Project 14)
  • 🔹 PCA + Logistic Regression → when p >> n and you need a simple classifier
  • 🔹 No PCA → when using tree-based models that handle high dimensions well

7 · When NOT to Use

  • 🚫 When feature interpretability is critical — PCA components are abstract combinations
  • 🚫 With tree-based models — they don't benefit from PCA and may perform worse
  • 🚫 When features are already uncorrelated — PCA can't compress uncorrelated features
  • 🚫 When the important signal is in low-variance directions — PCA might discard it

8 · How It Only Works — Internal Mechanics

What PCA Does — Step by Step

  • Step 1: Center the data (subtract the mean of each feature)
  • Step 2: Compute the covariance matrix (p × p)
  • Step 3: Find eigenvectors and eigenvalues of the covariance matrix
  • Step 4: Sort eigenvectors by eigenvalue (largest first)
  • Step 5: Project data onto the top-k eigenvectors → k-dimensional representation

Eigenvectors and Eigenvalues — Intuition

  • Each eigenvector is a direction in feature space
  • The corresponding eigenvalue tells you how much variance lies along that direction
  • The first eigenvector points in the direction of maximum variance
  • The second eigenvector is perpendicular to the first and captures the next most variance
  • Each component is uncorrelated with all others — PCA decorrelates the data

Explained Variance Ratio

  • explained_variance_ratio_[i] = eigenvalue_i / sum(all eigenvalues)
  • Tells you what fraction of total variance each component captures
  • Cumulative sum tells you how much variance is retained with k components
  • If 3 components capture 95% of variance, the other 4,997 features are mostly noise

The Scree Plot

  • Plot explained variance ratio vs component number
  • Look for the "elbow" — where the curve flattens
  • Components after the elbow contribute little — safe to discard
  • Similar to the elbow method in K-Means (Project 14) — same intuition

PCA vs Feature Selection

  • PCA creates NEW features (linear combinations of originals) — dimensionality reduction
  • Lasso (Project 18) selects EXISTING features — feature selection
  • PCA is unsupervised — doesn't use the target variable
  • Lasso is supervised — selects features that predict the target

9 · Common Pitfalls and Failure Modes

  • ⚠️ Not scaling before PCA — a feature measured in thousands dominates one measured in decimals
  • ⚠️ Keeping too few components — losing important signal; check explained variance
  • ⚠️ Keeping too many components — defeats the purpose; you're not reducing enough
  • ⚠️ Fitting PCA on test data — fit on train, transform both (same as scaling in Project 13)
  • ⚠️ Using PCA with tree-based models — trees handle high dimensions natively; PCA may hurt

10 · Scalability Considerations

  • Full PCA: O(min(n²p, np²)) — expensive when both n and p are large
  • For very large datasets, use IncrementalPCA (processes data in batches)
  • For very high p, use TruncatedSVD (doesn't need to center data, works on sparse matrices)
  • randomized PCA (svd_solver='randomized') is much faster for large p with few components
  • PCA reduces downstream model training time — fewer features = faster training

11 · Interpretability Considerations

  • ✅ Explained variance ratio is intuitive: "3 components capture 92% of the information"
  • ✅ 2D PCA plots are excellent for visualization and presentations
  • ⚠️ Components are abstract — "PC1 = 0.3×gene_A + 0.2×gene_B - 0.1×gene_C" is hard to explain
  • ⚠️ You can inspect components_ to see which original features load heavily on each component
  • ⚠️ PCA is unsupervised — the components that explain variance may not be the ones that predict the target

12 · Complete Working Code

import numpy as np
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
from sklearn.model_selection import cross_val_score, StratifiedKFold
from sklearn.linear_model import LogisticRegression
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import make_classification
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

# ─── Step 1: High-dimensional gene expression data ───
np.random.seed(42)
n_samples = 200
n_features = 500  # simulating high-dimensional gene data
n_informative = 20

X, y = make_classification(n_samples=n_samples, n_features=n_features,
                           n_informative=n_informative, n_redundant=30,
                           n_clusters_per_class=2, flip_y=0.05,
                           random_state=42)

print(f"=== Gene Expression Dataset ===")
print(f"  Samples: {n_samples}, Features: {n_features}")
print(f"  Informative: {n_informative}, Redundant: 30, Noise: {n_features - n_informative - 30}")
print(f"  Class balance: {np.mean(y):.1%} positive")
print(f"  Problem: p ({n_features}) >> n ({n_samples}) — curse of dimensionality")

# ─── Step 2: PCA — explained variance analysis ───
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

pca_full = PCA().fit(X_scaled)
cumulative_var = np.cumsum(pca_full.explained_variance_ratio_)

print(f"\n=== Explained Variance ===")
for n_comp in [1, 2, 5, 10, 20, 50, 100]:
    print(f"  {n_comp:>3} components: {cumulative_var[n_comp-1]:.1%} variance explained")

# Find 90% and 95% thresholds
n_90 = np.argmax(cumulative_var >= 0.90) + 1
n_95 = np.argmax(cumulative_var >= 0.95) + 1
print(f"\n  Components for 90% variance: {n_90}")
print(f"  Components for 95% variance: {n_95}")

# ─── Step 3: Scree plot ───
fig, axes = plt.subplots(1, 3, figsize=(18, 5))

# Individual explained variance
axes[0].bar(range(1, 51), pca_full.explained_variance_ratio_[:50], color='#58a6ff', alpha=0.7)
axes[0].set_xlabel('Principal Component')
axes[0].set_ylabel('Explained Variance Ratio')
axes[0].set_title('Scree Plot (first 50 components)')
axes[0].grid(True, alpha=0.2, axis='y')

# Cumulative explained variance
axes[1].plot(range(1, n_features + 1), cumulative_var, color='#f0883e', linewidth=2)
axes[1].axhline(y=0.90, color='#3fb950', linestyle='--', label='90% variance')
axes[1].axhline(y=0.95, color='#f85149', linestyle='--', label='95% variance')
axes[1].axvline(x=n_90, color='#3fb950', linestyle=':', alpha=0.5)
axes[1].axvline(x=n_95, color='#f85149', linestyle=':', alpha=0.5)
axes[1].set_xlabel('Number of Components')
axes[1].set_ylabel('Cumulative Explained Variance')
axes[1].set_title('Cumulative Variance')
axes[1].legend()
axes[1].grid(True, alpha=0.2)

# 2D PCA visualization
pca_2d = PCA(n_components=2).fit_transform(X_scaled)
scatter = axes[2].scatter(pca_2d[:, 0], pca_2d[:, 1], c=y, cmap='coolwarm',
                           alpha=0.6, s=30, edgecolors='none')
axes[2].set_xlabel(f'PC1 ({pca_full.explained_variance_ratio_[0]:.1%} variance)')
axes[2].set_ylabel(f'PC2 ({pca_full.explained_variance_ratio_[1]:.1%} variance)')
axes[2].set_title('2D PCA Projection')
plt.colorbar(scatter, ax=axes[2], label='Class')
axes[2].grid(True, alpha=0.2)

plt.tight_layout()
plt.savefig('project_24_pca_analysis.png', dpi=100, bbox_inches='tight')
print(f"\n✅ PCA analysis saved as project_24_pca_analysis.png")

# ─── Step 4: Before/after PCA — classifier comparison ───
print(f"\n=== Classifier Performance: Original vs PCA-Reduced ===")
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

configs = [
    ('LogReg — All 500 features', Pipeline([
        ('scaler', StandardScaler()),
        ('clf', LogisticRegression(max_iter=5000, random_state=42))
    ])),
    ('LogReg — PCA (20 comp)', Pipeline([
        ('scaler', StandardScaler()),
        ('pca', PCA(n_components=20)),
        ('clf', LogisticRegression(max_iter=5000, random_state=42))
    ])),
    ('LogReg — PCA (50 comp)', Pipeline([
        ('scaler', StandardScaler()),
        ('pca', PCA(n_components=50)),
        ('clf', LogisticRegression(max_iter=5000, random_state=42))
    ])),
    ('LogReg — PCA (95% var)', Pipeline([
        ('scaler', StandardScaler()),
        ('pca', PCA(n_components=0.95)),
        ('clf', LogisticRegression(max_iter=5000, random_state=42))
    ])),
    ('RandomForest — All 500', Pipeline([
        ('clf', RandomForestClassifier(n_estimators=200, random_state=42, n_jobs=-1))
    ])),
    ('RandomForest — PCA (50)', Pipeline([
        ('scaler', StandardScaler()),
        ('pca', PCA(n_components=50)),
        ('clf', RandomForestClassifier(n_estimators=200, random_state=42, n_jobs=-1))
    ])),
]

for name, pipe in configs:
    scores = cross_val_score(pipe, X, y, cv=cv, scoring='f1')
    print(f"  {name:>30}: F1 = {scores.mean():.4f} ± {scores.std():.4f}")

# ─── Step 5: Component loadings — what do PCs mean? ───
pca_10 = PCA(n_components=10).fit(X_scaled)

print(f"\n=== Top Feature Loadings for PC1 and PC2 ===")
for pc_idx in [0, 1]:
    loadings = np.abs(pca_10.components_[pc_idx])
    top_features = np.argsort(loadings)[::-1][:5]
    print(f"  PC{pc_idx+1} top features: {top_features.tolist()}")
    print(f"    Loadings: {[f'{loadings[i]:.3f}' for i in top_features]}")

# ─── Step 6: Optimal number of components via CV ───
print(f"\n=== Finding Optimal n_components via CV ===")
n_components_range = [2, 5, 10, 20, 30, 50, 75, 100, 150]
cv_scores = []

for n_comp in n_components_range:
    pipe = Pipeline([
        ('scaler', StandardScaler()),
        ('pca', PCA(n_components=n_comp)),
        ('clf', LogisticRegression(max_iter=5000, random_state=42))
    ])
    scores = cross_val_score(pipe, X, y, cv=cv, scoring='f1')
    cv_scores.append(scores.mean())
    print(f"  n_components={n_comp:>3}: F1 = {scores.mean():.4f}")

best_n = n_components_range[np.argmax(cv_scores)]
print(f"\n  → Best n_components: {best_n} (F1 = {max(cv_scores):.4f})")

# ─── Step 7: Plot optimal components ───
fig, ax = plt.subplots(figsize=(8, 5))
ax.plot(n_components_range, cv_scores, 'o-', color='#58a6ff', linewidth=2)
ax.axvline(x=best_n, color='#3fb950', linestyle='--', label=f'Best: {best_n} components')
ax.set_xlabel('Number of PCA Components')
ax.set_ylabel('CV F1 Score')
ax.set_title('Optimal Number of PCA Components')
ax.legend()
ax.grid(True, alpha=0.2)
plt.tight_layout()
plt.savefig('project_24_optimal_components.png', dpi=100, bbox_inches='tight')
print(f"✅ Optimal components plot saved as project_24_optimal_components.png")

13 · Key Takeaway + Bridge to Next Project

Key Takeaway

  • PCA finds the directions of maximum variance and projects data onto them
  • The scree plot and cumulative variance help choose how many components to keep
  • Always scale before PCA — it's variance-based (Project 13)
  • PCA creates new abstract features — not the same as feature selection (Lasso, Project 18)
  • PCA helps most when p >> n (many features, few samples) — reduces overfitting
  • Tree-based models (Projects 21, 23) often don't benefit from PCA — they handle high dimensions natively
