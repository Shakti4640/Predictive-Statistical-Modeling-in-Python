Project 26: Forecasting Monthly Sales With ARIMA

ARIMA Stationarity ADF Test Differencing auto_arima Forecasting

Difficulty: Medium  |  Builds on: Project 25

1 · Hypothetical Problem

  • Same retail chain from Project 25 — you've decomposed the sales series and understand the trend and seasonality
  • Now management asks: "What will sales be for the next 12 months?"
  • Decomposition (Project 25) described the past but can't forecast — you need a predictive model
  • The ACF from Project 25 showed strong autocorrelation — today's sales depend on recent past sales
  • Goal → build an ARIMA model that captures the autocorrelation structure and produces forecasts with confidence intervals

2 · How to Approach It

  1. From Project 25: the ACF showed the series is autocorrelated and has seasonality — ARIMA can model this
  2. ARIMA requires stationarity — the mean and variance must be constant over time
  3. Use the ADF (Augmented Dickey-Fuller) test to check stationarity
  4. If non-stationary, apply differencing (subtract each value from the previous) until stationary
  5. Choose AR (autoregressive), I (integrated/differencing), and MA (moving average) orders
  6. Use pmdarima's auto_arima to automatically find the best (p, d, q) parameters

3 · How Can We Solve It

  1. Generate monthly sales data with trend and seasonality (same structure as Project 25)
  2. Test for stationarity using the ADF test
  3. Apply differencing and re-test until stationary
  4. Fit ARIMA manually and with auto_arima
  5. Produce 12-month forecasts with confidence intervals
  6. Evaluate forecast accuracy on a held-out test period

4 · What Should Do

  • ✅ Always test for stationarity before fitting ARIMA — use the ADF test
  • ✅ Difference the series if non-stationary — that's what the "I" in ARIMA does
  • ✅ Use ACF and PACF plots (from Project 25) to guide AR and MA order selection
  • ✅ Use auto_arima for automatic order selection — it searches over (p, d, q) combinations
  • ✅ Hold out the last 12 months for evaluation — never evaluate on training data
  • ✅ Plot forecasts with confidence intervals — point forecasts alone are misleading

5 · What Should NOT Do

  • ❌ Do NOT fit ARIMA on non-stationary data — the model assumptions are violated
  • ❌ Do NOT over-difference — differencing twice when once is enough adds noise
  • ❌ Do NOT use high-order ARIMA (e.g., ARIMA(10,2,10)) — overfitting; keep it simple
  • ❌ Do NOT ignore seasonal patterns — use SARIMA (seasonal ARIMA) when seasonality exists
  • ❌ Do NOT trust long-horizon forecasts blindly — confidence intervals widen rapidly; 12 months is reasonable, 5 years is not

6 · When to Use What

  • 🔹 ARIMA → univariate time series with trend but no strong seasonality
  • 🔹 SARIMA → univariate time series with both trend and seasonality (most business data)
  • 🔹 auto_arima → when you don't want to manually select (p, d, q) — good starting point
  • 🔹 Decomposition (Project 25) → for understanding the series, not forecasting
  • 🔹 Exponential Smoothing → simpler alternative to ARIMA for seasonal data

7 · When NOT to Use

  • 🚫 When you have external predictors (weather, promotions) — ARIMA is univariate; use ARIMAX or regression
  • 🚫 When the series has multiple seasonal periods (daily + weekly + yearly) — ARIMA handles one seasonal period
  • 🚫 When the pattern is highly nonlinear — ARIMA is a linear model
  • 🚫 When you have very short series (<30 points) — not enough data to estimate parameters reliably

8 · How It Only Works — Internal Mechanics

The Three Components of ARIMA(p, d, q)

  • AR(p) — Autoregressive: predict using p past values. Y(t) = φ₁Y(t-1) + φ₂Y(t-2) + ... + φₚY(t-p)
  • I(d) — Integrated: difference the series d times to make it stationary. d=1 means Y'(t) = Y(t) - Y(t-1)
  • MA(q) — Moving Average: predict using q past forecast errors. Y(t) = θ₁ε(t-1) + θ₂ε(t-2) + ... + θ_qε(t-q)
  • Combined: ARIMA models the differenced series as a function of its own past values AND past errors

Stationarity and the ADF Test

  • Stationary: constant mean, constant variance, autocorrelation depends only on lag (not time)
  • ADF test: H₀ = series has a unit root (non-stationary). If p-value < 0.05 → reject → series is stationary
  • Differencing removes trends: if Y(t) has a linear trend, Y(t) - Y(t-1) ≈ constant
  • Seasonal differencing: Y(t) - Y(t-12) removes yearly seasonality in monthly data

ACF and PACF for Order Selection

  • ACF (Project 25): shows total correlation at each lag — used to identify MA order (q)
  • PACF: shows direct correlation at each lag (removing intermediate effects) — used to identify AR order (p)
  • ACF cuts off after lag q → MA(q). PACF cuts off after lag p → AR(p)
  • Both decay gradually → mixed ARMA model — use auto_arima

SARIMA — Seasonal Extension

  • SARIMA(p,d,q)(P,D,Q,m): adds seasonal AR, differencing, and MA terms with period m
  • P = seasonal AR order, D = seasonal differencing, Q = seasonal MA order, m = seasonal period
  • Example: SARIMA(1,1,1)(1,1,1,12) for monthly data with yearly seasonality
  • auto_arima with seasonal=True searches over both non-seasonal and seasonal parameters

Information Criteria — AIC and BIC

  • AIC (Akaike Information Criterion): balances model fit vs complexity — lower is better
  • BIC (Bayesian Information Criterion): penalizes complexity more than AIC
  • auto_arima uses AIC by default to select the best (p, d, q) combination
  • Similar to cross-validation (Project 15) — prevents overfitting by penalizing model complexity

9 · Common Pitfalls and Failure Modes

  • ⚠️ Fitting ARIMA without checking stationarity — the model will produce garbage forecasts
  • ⚠️ Over-differencing — d=2 when d=1 is enough introduces artificial patterns in residuals
  • ⚠️ Ignoring seasonality — using ARIMA instead of SARIMA on seasonal data misses the pattern
  • ⚠️ Trusting long-horizon forecasts — ARIMA forecasts revert to the mean; confidence intervals explode
  • ⚠️ Not checking residuals — residuals should be white noise; patterns mean the model missed something

10 · Scalability Considerations

  • ARIMA fitting is O(n × p²) — fast for typical business time series (hundreds to thousands of points)
  • auto_arima searches over many (p,d,q) combinations — can be slow with large search spaces
  • For thousands of individual series (e.g., per-product forecasts), parallelize with joblib
  • For very long series (100K+ points), consider downsampling or using faster implementations
  • SARIMA is slower than ARIMA — seasonal parameters multiply the search space

11 · Interpretability Considerations

  • ✅ ARIMA parameters are interpretable: "sales depend on the last 2 months (AR=2) with a correction for recent errors (MA=1)"
  • ✅ Confidence intervals communicate uncertainty — wider = less certain
  • ✅ Residual diagnostics tell you if the model captured all the signal
  • ⚠️ SARIMA parameters are harder to explain to non-technical stakeholders
  • ⚠️ The "integrated" (differencing) step means the model works on changes, not levels — can confuse people

12 · Complete Working Code

import numpy as np
import pandas as pd
from statsmodels.tsa.stattools import adfuller
from statsmodels.tsa.arima.model import ARIMA
from statsmodels.graphics.tsaplots import plot_acf, plot_pacf
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt
import warnings
warnings.filterwarnings('ignore')

# ─── Step 1: Generate monthly sales with trend + seasonality ───
np.random.seed(42)
n_months = 60  # 5 years
dates = pd.date_range(start='2019-01-01', periods=n_months, freq='MS')

trend = np.linspace(10000, 20000, n_months)
seasonal_pattern = [0, -500, 200, 400, 300, -200, -400, -300, 100, 500, 800, 2000]
seasonality = np.tile(seasonal_pattern, n_months // 12)
noise = np.random.normal(0, 300, n_months)

sales = trend + seasonality + noise
ts = pd.Series(sales, index=dates, name='sales')

# Train/test split — hold out last 12 months
train = ts[:-12]
test = ts[-12:]
print("=== Monthly Sales Data ===")
print(f"  Total: {len(ts)} months")
print(f"  Train: {len(train)} months ({train.index[0].strftime('%Y-%m')} to {train.index[-1].strftime('%Y-%m')})")
print(f"  Test:  {len(test)} months ({test.index[0].strftime('%Y-%m')} to {test.index[-1].strftime('%Y-%m')})")

# ─── Step 2: Stationarity test — ADF ───
def adf_test(series, name="Series"):
    result = adfuller(series, autolag='AIC')
    print(f"\n  ADF Test on {name}:")
    print(f"    Test Statistic: {result[0]:.4f}")
    print(f"    p-value:        {result[1]:.4f}")
    print(f"    Lags Used:      {result[2]}")
    print(f"    Stationary:     {'YES ✅' if result[1] < 0.05 else 'NO ❌'}")
    return result[1] < 0.05

print("\n=== Stationarity Testing ===")
is_stationary = adf_test(train, "Original Series")

# First differencing
train_diff1 = train.diff().dropna()
is_stationary_d1 = adf_test(train_diff1, "After 1st Differencing")

# Seasonal differencing
train_seasonal_diff = (train - train.shift(12)).dropna()
is_stationary_sd = adf_test(train_seasonal_diff, "After Seasonal Differencing (lag=12)")

# ─── Step 3: ACF and PACF on differenced series ───
fig, axes = plt.subplots(2, 2, figsize=(14, 8))

plot_acf(train, lags=30, ax=axes[0, 0], color='#58a6ff')
axes[0, 0].set_title('ACF — Original')

plot_acf(train_diff1, lags=30, ax=axes[0, 1], color='#f0883e')
axes[0, 1].set_title('ACF — After 1st Differencing')

plot_pacf(train_diff1, lags=30, ax=axes[1, 0], color='#3fb950', method='ywm')
axes[1, 0].set_title('PACF — After 1st Differencing')

plot_acf(train_seasonal_diff, lags=30, ax=axes[1, 1], color='#bc8cff')
axes[1, 1].set_title('ACF — After Seasonal Differencing')

for ax in axes.ravel():
    ax.grid(True, alpha=0.2)
plt.tight_layout()
plt.savefig('project_26_acf_pacf.png', dpi=100, bbox_inches='tight')
print("\n✅ ACF/PACF plots saved as project_26_acf_pacf.png")

# ─── Step 4: Manual ARIMA fitting ───
print("\n=== Manual ARIMA Models ===")
results = []
for order in [(1,1,0), (0,1,1), (1,1,1), (2,1,1), (1,1,2)]:
    try:
        model = ARIMA(train, order=order)
        fit = model.fit()
        forecast = fit.forecast(steps=12)
        mae = np.mean(np.abs(test.values - forecast.values))
        results.append((order, fit.aic, fit.bic, mae))
        print(f"  ARIMA{order}: AIC={fit.aic:.1f}, BIC={fit.bic:.1f}, Test MAE=${mae:,.0f}")
    except Exception as e:
        print(f"  ARIMA{order}: Failed — {e}")

best = min(results, key=lambda x: x[3])
print(f"\n  → Best by Test MAE: ARIMA{best[0]} (MAE=${best[3]:,.0f})")

# ─── Step 5: auto_arima for automatic selection ───
print("\n=== auto_arima (Automatic Order Selection) ===")
try:
    from pmdarima import auto_arima
    auto_model = auto_arima(train, seasonal=True, m=12,
                            start_p=0, start_q=0, max_p=3, max_q=3,
                            d=None, D=None, max_d=2, max_D=1,
                            trace=False, error_action='ignore',
                            suppress_warnings=True, stepwise=True)
    print(f"  Best model: {auto_model.summary().tables[0]}")
    print(f"  Order: {auto_model.order}")
    print(f"  Seasonal order: {auto_model.seasonal_order}")
    print(f"  AIC: {auto_model.aic():.1f}")

    auto_forecast = auto_model.predict(n_periods=12)
    auto_mae = np.mean(np.abs(test.values - auto_forecast))
    print(f"  Test MAE: ${auto_mae:,.0f}")
    use_auto = True
except ImportError:
    print("  pmdarima not installed — using best manual ARIMA instead")
    print("  Install with: pip install pmdarima")
    use_auto = False

# ─── Step 6: Forecast with confidence intervals ───
best_order = best[0]
model_final = ARIMA(train, order=best_order)
fit_final = model_final.fit()

forecast_result = fit_final.get_forecast(steps=12)
forecast_mean = forecast_result.predicted_mean
forecast_ci = forecast_result.conf_int(alpha=0.05)

print(f"\n=== 12-Month Forecast (ARIMA{best_order}) ===")
for i in range(12):
    date = test.index[i].strftime('%Y-%m')
    actual = test.values[i]
    pred = forecast_mean.values[i]
    lo = forecast_ci.iloc[i, 0]
    hi = forecast_ci.iloc[i, 1]
    print(f"  {date}: Actual=${actual:>8,.0f}  Forecast=${pred:>8,.0f}  "
          f"95% CI=[${lo:>8,.0f}, ${hi:>8,.0f}]  Error=${actual-pred:>+7,.0f}")

# ─── Step 7: Forecast visualization ───
fig, axes = plt.subplots(2, 1, figsize=(14, 8))

# Full series with forecast
axes[0].plot(train.index, train.values, color='#58a6ff', label='Training Data')
axes[0].plot(test.index, test.values, color='#3fb950', linewidth=2, label='Actual (Test)')
axes[0].plot(test.index, forecast_mean.values, color='#f0883e', linewidth=2,
             linestyle='--', label=f'Forecast ARIMA{best_order}')
axes[0].fill_between(test.index, forecast_ci.iloc[:, 0], forecast_ci.iloc[:, 1],
                      color='#f0883e', alpha=0.15, label='95% CI')
axes[0].set_title('ARIMA Forecast vs Actual')
axes[0].set_ylabel('Sales ($)')
axes[0].legend()
axes[0].grid(True, alpha=0.2)

# Zoomed into forecast period
axes[1].plot(test.index, test.values, 'o-', color='#3fb950', linewidth=2, label='Actual')
axes[1].plot(test.index, forecast_mean.values, 's--', color='#f0883e', linewidth=2, label='Forecast')
axes[1].fill_between(test.index, forecast_ci.iloc[:, 0], forecast_ci.iloc[:, 1],
                      color='#f0883e', alpha=0.15)
axes[1].set_title('Forecast Period — Zoomed In')
axes[1].set_ylabel('Sales ($)')
axes[1].legend()
axes[1].grid(True, alpha=0.2)

plt.tight_layout()
plt.savefig('project_26_forecast.png', dpi=100, bbox_inches='tight')
print("\n✅ Forecast plot saved as project_26_forecast.png")

# ─── Step 8: Residual diagnostics ───
residuals = fit_final.resid

fig, axes = plt.subplots(2, 2, figsize=(12, 8))

axes[0, 0].plot(residuals, color='#58a6ff', linewidth=0.8)
axes[0, 0].axhline(y=0, color='#f85149', linestyle='--')
axes[0, 0].set_title('Residuals Over Time')
axes[0, 0].grid(True, alpha=0.2)

axes[0, 1].hist(residuals, bins=20, color='#f0883e', alpha=0.7, edgecolor='#30363d')
axes[0, 1].set_title('Residual Distribution')
axes[0, 1].grid(True, alpha=0.2)

plot_acf(residuals, lags=24, ax=axes[1, 0], color='#3fb950')
axes[1, 0].set_title('ACF of Residuals')
axes[1, 0].grid(True, alpha=0.2)

from scipy import stats
stats.probplot(residuals, dist="norm", plot=axes[1, 1])
axes[1, 1].set_title('Q-Q Plot')
axes[1, 1].grid(True, alpha=0.2)

plt.suptitle(f'Residual Diagnostics — ARIMA{best_order}', fontsize=13, color='#c9d1d9')
plt.tight_layout()
plt.savefig('project_26_residuals.png', dpi=100, bbox_inches='tight')
print("✅ Residual diagnostics saved as project_26_residuals.png")

# ─── Step 9: Forecast accuracy metrics ───
from sklearn.metrics import mean_absolute_error, mean_squared_error

mae = mean_absolute_error(test, forecast_mean)
rmse = np.sqrt(mean_squared_error(test, forecast_mean))
mape = np.mean(np.abs((test.values - forecast_mean.values) / test.values)) * 100

print(f"\n=== Forecast Accuracy ===")
print(f"  MAE:  ${mae:,.0f}")
print(f"  RMSE: ${rmse:,.0f}")
print(f"  MAPE: {mape:.1f}%")
print(f"  (MAPE < 10% is generally considered good for business forecasting)")
