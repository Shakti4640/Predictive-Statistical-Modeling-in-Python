Project 18: Sparse Feature Selection in Housing Data With Lasso

Lasso (L1) Feature Selection Sparsity ElasticNet

Difficulty: Medium  |  Builds on: Project 17

1 · Hypothetical Problem

  • You're a data scientist at a real estate analytics firm predicting house prices
  • The dataset has 30 features: square footage, lot size, number of rooms, garage area, pool area, year built, neighborhood code, and 23 more
  • You suspect many features are irrelevant — pool area barely varies, some neighborhood codes have no signal
  • Ridge (Project 17) kept all 30 features — coefficients shrank but none went to zero
  • Goal → use Lasso to automatically identify which features actually matter and drop the rest

2 · How to Approach It

  1. Recall from Project 17: Ridge shrinks coefficients but keeps all features
  2. Lasso uses L1 penalty: Loss = MSE + α × Σ|wᵢ| — absolute values instead of squares
  3. The L1 penalty can push coefficients exactly to zero — automatic feature selection
  4. Use LassoCV to find the best alpha, then inspect which coefficients survived
  5. Compare with Ridge to see the difference in sparsity behavior

3 · How Can We Solve It

  1. Generate housing data with 30 features — some relevant, some noise
  2. Fit Lasso and observe which coefficients become exactly zero
  3. Plot coefficient paths as alpha increases — watch features drop out
  4. Compare Lasso vs Ridge coefficient behavior side by side
  5. Introduce ElasticNet as a hybrid of both penalties
  6. Use LassoCV for automatic alpha selection with cross-validation

4 · What Should Do

  • ✅ Scale features before Lasso — same reason as Ridge (Project 17): penalty treats all coefficients equally
  • ✅ Use LassoCV to automatically find the best alpha
  • ✅ Inspect which coefficients are zero — those features are eliminated
  • ✅ Compare the number of surviving features at different alpha values
  • ✅ Consider ElasticNet when features are correlated AND you want sparsity

5 · What Should NOT Do

  • ❌ Do NOT use Lasso without scaling — same pitfall as Ridge (Project 17)
  • ❌ Do NOT assume zero coefficient means the feature is truly useless — with correlated features, Lasso arbitrarily picks one and drops the other
  • ❌ Do NOT set alpha too high — all coefficients go to zero (model predicts the mean)
  • ❌ Do NOT use Lasso when you want to keep all correlated features — use Ridge instead
  • ❌ Do NOT compare raw coefficient magnitudes between Lasso and Ridge — different penalties, different scales

6 · When to Use What

  • 🔹 Lasso (L1) → when you suspect many features are irrelevant and want automatic selection
  • 🔹 Ridge (L2) → when all features are potentially useful and correlated (Project 17)
  • 🔹 ElasticNet → when you want sparsity (like Lasso) but also stability with correlated features (like Ridge)
  • 🔹 LassoCV → automatic alpha selection via cross-validation
  • 🔹 OLS → when you have few features with no multicollinearity (Project 9)

7 · When NOT to Use

  • 🚫 Lasso with highly correlated features — it randomly picks one and drops the rest (unstable selection)
  • 🚫 When you need all features for interpretability — Lasso eliminates some
  • 🚫 When the true model uses all features equally — Ridge is more appropriate
  • 🚫 For nonlinear relationships — Lasso is still a linear model

8 · How It Only Works — Internal Mechanics

Lasso Loss Function

  • Loss = Σ(yᵢ - ŷᵢ)² + α × Σ|wⱼ|
  • First term: MSE (fit the data) — same as Ridge
  • Second term: L1 penalty (absolute values) — this is what creates sparsity

Why L1 Creates Zeros But L2 Doesn't

  • L2 penalty (Ridge): the gradient of w² is 2w — gets smaller as w approaches 0, never pushes to exactly 0
  • L1 penalty (Lasso): the gradient of |w| is ±1 — constant push toward 0 regardless of coefficient size
  • Geometrically: L1 constraint region is a diamond with corners on axes — optimal point often lands on a corner where some coefficients are exactly 0
  • L2 constraint region is a circle — optimal point rarely lands exactly on an axis

Coordinate Descent — How Lasso Is Solved

  • Unlike Ridge, Lasso has no closed-form solution (the absolute value is not differentiable at 0)
  • Solved via coordinate descent: optimize one coefficient at a time, cycling through all
  • For each coefficient, apply soft-thresholding: if the update is small enough, set it to exactly 0

ElasticNet — Best of Both

  • Loss = MSE + α × [ρ × Σ|wⱼ| + (1-ρ) × Σwⱼ²]
  • ρ (l1_ratio) controls the mix: ρ=1 is pure Lasso, ρ=0 is pure Ridge
  • ElasticNet handles correlated features better than Lasso — tends to keep groups of correlated features together

9 · Common Pitfalls and Failure Modes

  • ⚠️ Correlated feature instability — Lasso picks one from a correlated group arbitrarily; run it multiple times with different seeds to check stability
  • ⚠️ Too aggressive alpha — all features eliminated, model predicts the mean
  • ⚠️ Interpreting zero coefficients as "feature doesn't matter" — it might be correlated with a surviving feature
  • ⚠️ Not scaling features — same issue as Ridge (Project 17)
  • ⚠️ Using Lasso for prediction only — its real power is feature selection; if you only care about prediction, Ridge or ElasticNet may perform better

10 · Scalability Considerations

  • Lasso via coordinate descent: efficient for sparse solutions — skips zero coefficients
  • Scales well to high-dimensional data (p >> n) — Lasso can handle more features than samples
  • LassoCV uses warm starts — each alpha starts from the previous solution, making the path efficient
  • For very large datasets, use SGD with L1 penalty (SGDRegressor with penalty='l1')
  • ElasticNet is slightly more expensive than pure Lasso but still efficient

11 · Interpretability Considerations

  • ✅ Lasso produces sparse models — easier to explain ("only these 8 out of 30 features matter")
  • ✅ Non-zero coefficients indicate feature importance (with caveats about correlation)
  • ⚠️ Zero coefficient ≠ "feature is useless" — it might be redundant with a surviving feature
  • ⚠️ Surviving coefficients are biased (shrunk) — they underestimate the true effect
  • ⚠️ Feature selection depends on alpha — different alphas give different selected features

12 · Complete Working Code

import numpy as np
from sklearn.linear_model import Lasso, LassoCV, Ridge, ElasticNet, ElasticNetCV
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import cross_val_score, KFold
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

# ─── Step 1: Synthetic housing data — 30 features, only ~8 relevant ───
np.random.seed(42)
n = 400
p_total = 30
p_relevant = 8

X = np.random.randn(n, p_total)
# Make some features correlated
X[:, 1] = X[:, 0] * 0.9 + np.random.normal(0, 0.3, n)  # correlated with feature 0
X[:, 3] = X[:, 2] * 0.85 + np.random.normal(0, 0.4, n)  # correlated with feature 2

# True coefficients: only first 8 features matter
true_coefs = np.zeros(p_total)
true_coefs[:p_relevant] = [15, -8, 12, -5, 10, 7, -6, 4]

y = X @ true_coefs + np.random.normal(0, 10, n)
feature_names = [f'feat_{i}' for i in range(p_total)]

# ─── Step 2: Scale features ───
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# ─── Step 3: Lasso — watch coefficients go to zero ───
print("=== Lasso vs Ridge: Coefficient Comparison ===")
lasso = Lasso(alpha=1.0).fit(X_scaled, y)
ridge = Ridge(alpha=1.0).fit(X_scaled, y)

nonzero_lasso = np.sum(lasso.coef_ != 0)
print(f"\nLasso (α=1.0): {nonzero_lasso}/{p_total} features kept")
print(f"Ridge (α=1.0): {np.sum(ridge.coef_ != 0)}/{p_total} features kept (Ridge keeps all)")

print(f"\n{'Feature':>10} {'True':>8} {'Lasso':>8} {'Ridge':>8}")
print("-" * 40)
for i in range(p_total):
    marker = " ← zero" if lasso.coef_[i] == 0 else ""
    print(f"{feature_names[i]:>10} {true_coefs[i]:>8.1f} {lasso.coef_[i]:>8.3f} "
          f"{ridge.coef_[i]:>8.3f}{marker}")

# ─── Step 4: LassoCV — automatic alpha selection ───
alphas = np.logspace(-3, 2, 100)
lasso_cv = LassoCV(alphas=alphas, cv=5, random_state=42)
lasso_cv.fit(X_scaled, y)

print(f"\n=== LassoCV Results ===")
print(f"  Best alpha: {lasso_cv.alpha_:.4f}")
print(f"  Non-zero coefficients: {np.sum(lasso_cv.coef_ != 0)}/{p_total}")
print(f"  R² score: {lasso_cv.score(X_scaled, y):.4f}")

selected = [feature_names[i] for i in range(p_total) if lasso_cv.coef_[i] != 0]
print(f"  Selected features: {selected}")

# ─── Step 5: ElasticNet — hybrid L1 + L2 ───
enet_cv = ElasticNetCV(l1_ratio=[0.1, 0.5, 0.7, 0.9, 0.95, 1.0],
                        alphas=alphas, cv=5, random_state=42)
enet_cv.fit(X_scaled, y)

print(f"\n=== ElasticNetCV Results ===")
print(f"  Best alpha: {enet_cv.alpha_:.4f}")
print(f"  Best l1_ratio: {enet_cv.l1_ratio_:.2f}")
print(f"  Non-zero coefficients: {np.sum(enet_cv.coef_ != 0)}/{p_total}")
print(f"  R² score: {enet_cv.score(X_scaled, y):.4f}")

# ─── Step 6: Cross-validated comparison ───
print(f"\n=== CV Comparison: OLS vs Ridge vs Lasso vs ElasticNet ===")
cv = KFold(n_splits=5, shuffle=True, random_state=42)
from sklearn.linear_model import LinearRegression
from sklearn.pipeline import Pipeline

models = [
    ('OLS', Pipeline([('s', StandardScaler()), ('m', LinearRegression())])),
    ('Ridge', Pipeline([('s', StandardScaler()), ('m', Ridge(alpha=1.0))])),
    ('Lasso', Pipeline([('s', StandardScaler()), ('m', Lasso(alpha=lasso_cv.alpha_))])),
    ('ElasticNet', Pipeline([('s', StandardScaler()),
                              ('m', ElasticNet(alpha=enet_cv.alpha_,
                                               l1_ratio=enet_cv.l1_ratio_))])),
]
for name, model in models:
    scores = cross_val_score(model, X, y, cv=cv, scoring='neg_root_mean_squared_error')
    print(f"  {name:>12}: RMSE = {-scores.mean():.2f} ± {scores.std():.2f}")

# ─── Step 7: Coefficient path plot — Lasso vs Ridge ───
fig, axes = plt.subplots(1, 2, figsize=(14, 5))

# Lasso path
alphas_path = np.logspace(-3, 2, 100)
lasso_paths = []
for a in alphas_path:
    m = Lasso(alpha=a, max_iter=10000).fit(X_scaled, y)
    lasso_paths.append(m.coef_)
lasso_paths = np.array(lasso_paths)

for i in range(p_total):
    color = '#3fb950' if i < p_relevant else '#30363d'
    alpha_line = 0.9 if i < p_relevant else 0.3
    axes[0].plot(alphas_path, lasso_paths[:, i], color=color, alpha=alpha_line,
                 label=feature_names[i] if i < p_relevant else None)
axes[0].set_xscale('log')
axes[0].axvline(x=lasso_cv.alpha_, color='#f85149', linestyle='--', label=f'Best α={lasso_cv.alpha_:.3f}')
axes[0].set_xlabel('Alpha')
axes[0].set_ylabel('Coefficient value')
axes[0].set_title('Lasso Coefficient Paths (L1)')
axes[0].legend(fontsize=7, loc='upper right')
axes[0].grid(True, alpha=0.2)

# Ridge path for comparison
ridge_paths = []
for a in alphas_path:
    m = Ridge(alpha=a).fit(X_scaled, y)
    ridge_paths.append(m.coef_)
ridge_paths = np.array(ridge_paths)

for i in range(p_total):
    color = '#3fb950' if i < p_relevant else '#30363d'
    alpha_line = 0.9 if i < p_relevant else 0.3
    axes[1].plot(alphas_path, ridge_paths[:, i], color=color, alpha=alpha_line)
axes[1].set_xscale('log')
axes[1].set_xlabel('Alpha')
axes[1].set_ylabel('Coefficient value')
axes[1].set_title('Ridge Coefficient Paths (L2) — No Zeros')
axes[1].grid(True, alpha=0.2)

plt.tight_layout()
plt.savefig('project_18_lasso_vs_ridge_paths.png', dpi=100, bbox_inches='tight')
print(f"\n✅ Coefficient paths saved as project_18_lasso_vs_ridge_paths.png")

13 · Key Takeaway + Bridge to Next Project

Key Takeaway

  • Lasso (L1) pushes coefficients to exactly zero — automatic feature selection
  • Ridge (L2) shrinks but never eliminates — keeps all features (Project 17)
  • ElasticNet combines both: sparsity from L1 + stability from L2
  • With correlated features, Lasso arbitrarily picks one — ElasticNet is more stable
  • LassoCV finds the best alpha automatically via cross-validation
