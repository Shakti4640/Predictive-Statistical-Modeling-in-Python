Project 39: Generalized Linear Models — Count Data and Poisson Regression

GLM Poisson Regression Link Function Overdispersion Negative Binomial

Difficulty: Hard  |  Builds on: Projects 9, 17

1 · Hypothetical Problem

  • A hospital wants to predict the number of ER visits per day based on weather, day of week, and flu season
  • The outcome is a COUNT (0, 1, 2, 3, ...) — not continuous, not binary
  • Linear regression (Project 9) can predict negative counts — that's impossible
  • Logistic regression (Project 6) handles binary — but counts aren't binary
  • Goal → use Poisson regression (a GLM) to properly model count data

2 · How to Approach It

  1. From Project 9: linear regression assumes y is continuous and normally distributed
  2. From Project 17: Ridge adds regularization but doesn't change the distributional assumption
  3. GLMs generalize linear regression by allowing different distributions (Poisson, Binomial, Gamma)
  4. The link function connects the linear predictor to the expected value of y
  5. For counts: Poisson distribution with log link → ensures predictions are always non-negative

3 · How Can We Solve It

  1. Generate synthetic ER visit count data with known relationships
  2. Show why linear regression fails for count data (negative predictions, wrong distribution)
  3. Fit a Poisson GLM using statsmodels
  4. Check for overdispersion — when variance > mean, Poisson is too restrictive
  5. Fit a Negative Binomial model to handle overdispersion
  6. Compare models using deviance, AIC, and residual analysis

4 · What Should Do

  • ✅ Check if variance ≈ mean — Poisson assumes this (equidispersion)
  • ✅ Use the log link function for count data — ensures non-negative predictions
  • ✅ Check for overdispersion — if variance >> mean, use Negative Binomial instead
  • ✅ Interpret coefficients as multiplicative effects: exp(β) = rate ratio
  • ✅ Use deviance residuals to assess model fit — they should be roughly standard normal

5 · What Should NOT Do

  • ❌ Do NOT use linear regression for count data — it predicts negative values and assumes normality
  • ❌ Do NOT ignore overdispersion — Poisson with overdispersed data gives wrong standard errors
  • ❌ Do NOT interpret Poisson coefficients like linear regression — they're on the log scale
  • ❌ Do NOT forget about zero-inflation — if there are excess zeros, use zero-inflated models
  • ❌ Do NOT use R² for GLMs — use deviance, AIC, or pseudo-R² instead

6 · When to Use What

  • 🔹 Poisson regression → count data where variance ≈ mean
  • 🔹 Negative Binomial → count data with overdispersion (variance > mean)
  • 🔹 Zero-inflated Poisson/NB → when there are more zeros than the model predicts
  • 🔹 Linear regression (Project 9) → continuous outcomes, approximately normal
  • 🔹 Logistic regression (Project 6) → binary outcomes (a GLM with binomial family)
  • 🔹 Gamma GLM → positive continuous outcomes with right skew (e.g., insurance claims)

7 · When NOT to Use

  • 🚫 When the outcome is continuous and approximately normal — use linear regression
  • 🚫 When counts are very large (>100) — normal approximation works fine, use linear regression
  • 🚫 When the relationship is highly nonlinear — GLMs are still linear in the link space
  • 🚫 When you need complex interactions — tree-based models (Project 21) may be better

8 · How It Only Works — Internal Mechanics

The GLM Framework

  • Three components: (1) Random component (distribution), (2) Systematic component (linear predictor), (3) Link function
  • Linear regression: Normal distribution + identity link → E[y] = Xβ
  • Logistic regression: Binomial distribution + logit link → log(p/(1-p)) = Xβ
  • Poisson regression: Poisson distribution + log link → log(E[y]) = Xβ → E[y] = exp(Xβ)

Poisson Distribution

  • P(Y=k) = λ^k × e^(-λ) / k! for k = 0, 1, 2, ...
  • Mean = λ, Variance = λ (equidispersion property)
  • The log link ensures λ = exp(Xβ) > 0 always

Coefficient Interpretation

  • log(E[y]) = β₀ + β₁x₁ → E[y] = exp(β₀) × exp(β₁x₁)
  • A 1-unit increase in x₁ multiplies the expected count by exp(β₁)
  • exp(β₁) is the Incidence Rate Ratio (IRR)
  • Example: β₁ = 0.3 → exp(0.3) = 1.35 → 35% increase in expected count per unit increase

Overdispersion

  • Poisson assumes Var(Y) = E[Y] — but real data often has Var(Y) > E[Y]
  • Overdispersion: extra variability not captured by the Poisson model
  • Causes: unobserved heterogeneity, clustering, missing covariates
  • Solution: Negative Binomial adds a dispersion parameter → Var(Y) = μ + μ²/α

9 · Common Pitfalls and Failure Modes

  • ⚠️ Ignoring overdispersion — Poisson with overdispersed data gives too-narrow confidence intervals
  • ⚠️ Interpreting coefficients linearly — β=0.5 doesn't mean "0.5 more counts"; it means exp(0.5)=1.65× more
  • ⚠️ Negative predictions from OLS — linear regression on counts can predict -3 visits; Poisson can't
  • ⚠️ Zero-inflation — if 60% of days have zero ER visits, standard Poisson underestimates zeros
  • ⚠️ Exposure/offset — comparing counts across different time periods without an offset is wrong

10 · Scalability Considerations

  • GLM fitting uses IRLS (Iteratively Reweighted Least Squares) — O(n × p²) per iteration
  • Converges in 5–25 iterations typically — very fast for moderate data
  • statsmodels handles millions of rows efficiently
  • For very high-dimensional data, use regularized GLMs (glmnet equivalent in Python)

11 · Interpretability Considerations

  • ✅ Rate ratios (exp(β)) are intuitive: "flu season increases ER visits by 40%"
  • ✅ GLMs are parametric — each coefficient has a clear meaning
  • ✅ Confidence intervals on rate ratios quantify uncertainty
  • ⚠️ The log link makes raw coefficients less intuitive — always exponentiate for interpretation
  • ⚠️ Interaction terms in GLMs are multiplicative on the response scale — harder to explain

12 · Complete Working Code

import numpy as np
import pandas as pd
import statsmodels.api as sm
from sklearn.linear_model import LinearRegression
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

np.random.seed(42)

# ─── Step 1: Generate ER visit count data ───
n = 1000
temperature = np.random.normal(60, 15, n)
is_weekend = np.random.binomial(1, 2/7, n)
flu_season = np.random.binomial(1, 0.25, n)
humidity = np.random.uniform(30, 90, n)

# True Poisson model: log(λ) = β₀ + β₁*temp + β₂*weekend + β₃*flu + β₄*humidity
log_lambda = (2.5 - 0.01 * temperature + 0.3 * is_weekend
              + 0.5 * flu_season + 0.005 * humidity)
lambda_true = np.exp(log_lambda)
er_visits = np.random.poisson(lambda_true)

df = pd.DataFrame({
    'temperature': temperature, 'is_weekend': is_weekend,
    'flu_season': flu_season, 'humidity': humidity, 'er_visits': er_visits
})

print("=== ER Visit Data ===")
print(f"  Days: {n}")
print(f"  Mean visits: {er_visits.mean():.1f}, Variance: {er_visits.var():.1f}")
print(f"  Ratio Var/Mean: {er_visits.var()/er_visits.mean():.2f} (1.0 = equidispersion)")
print(f"  Range: {er_visits.min()} – {er_visits.max()}")
print(f"  Zero days: {(er_visits == 0).sum()} ({(er_visits == 0).mean():.1%})")

# ─── Step 2: Why linear regression fails ───
X = df[['temperature', 'is_weekend', 'flu_season', 'humidity']]
y = df['er_visits']

lr = LinearRegression()
lr.fit(X, y)
lr_pred = lr.predict(X)

print(f"\n=== Linear Regression (Wrong Model) ===")
print(f"  Negative predictions: {(lr_pred < 0).sum()} ({(lr_pred < 0).mean():.1%})")
print(f"  Min prediction: {lr_pred.min():.1f} (impossible for counts!)")
print(f"  R²: {lr.score(X, y):.4f}")

# ─── Step 3: Poisson GLM ───
X_sm = sm.add_constant(X)
poisson_model = sm.GLM(y, X_sm, family=sm.families.Poisson())
poisson_result = poisson_model.fit()

print(f"\n=== Poisson GLM ===")
print(poisson_result.summary2().tables[1].to_string())

print(f"\n  Rate Ratios (exp(β)):")
for name, coef in zip(['const'] + list(X.columns), poisson_result.params):
    print(f"    {name:<15}: β={coef:.4f}, IRR={np.exp(coef):.4f}")

print(f"\n  Deviance: {poisson_result.deviance:.1f}")
print(f"  Pearson chi2: {poisson_result.pearson_chi2:.1f}")
print(f"  AIC: {poisson_result.aic:.1f}")

# ─── Step 4: Check for overdispersion ───
pearson_chi2 = poisson_result.pearson_chi2
dof = poisson_result.df_resid
dispersion = pearson_chi2 / dof

print(f"\n=== Overdispersion Check ===")
print(f"  Pearson chi2 / df = {dispersion:.3f}")
if dispersion > 1.5:
    print(f"  ⚠️ Overdispersion detected (ratio >> 1) — consider Negative Binomial")
else:
    print(f"  ✅ No significant overdispersion (ratio ≈ 1)")

# ─── Step 5: Negative Binomial for comparison ───
nb_model = sm.GLM(y, X_sm, family=sm.families.NegativeBinomial(alpha=1.0))
nb_result = nb_model.fit()

print(f"\n=== Negative Binomial GLM ===")
print(f"  AIC (Poisson):  {poisson_result.aic:.1f}")
print(f"  AIC (NegBin):   {nb_result.aic:.1f}")
print(f"  Better model:   {'Negative Binomial' if nb_result.aic < poisson_result.aic else 'Poisson'}")

# ─── Step 6: Predictions and comparison ───
poisson_pred = poisson_result.predict(X_sm)
nb_pred = nb_result.predict(X_sm)

print(f"\n=== Prediction Comparison ===")
print(f"  {'Model':<15} {'Min':>8} {'Max':>8} {'Mean':>8} {'RMSE':>8}")
for name, pred in [('Linear', lr_pred), ('Poisson', poisson_pred), ('NegBin', nb_pred)]:
    rmse = np.sqrt(np.mean((y - pred)**2))
    print(f"  {name:<15} {pred.min():>8.1f} {pred.max():>8.1f} {pred.mean():>8.1f} {rmse:>8.2f}")

# ─── Step 7: Interpretation example ───
print(f"\n=== Interpretation ===")
flu_coef = poisson_result.params['flu_season']
flu_irr = np.exp(flu_coef)
print(f"  Flu season coefficient: {flu_coef:.4f}")
print(f"  Flu season IRR: {flu_irr:.4f}")
print(f"  → During flu season, ER visits increase by {(flu_irr-1)*100:.1f}%")

weekend_coef = poisson_result.params['is_weekend']
weekend_irr = np.exp(weekend_coef)
print(f"  Weekend IRR: {weekend_irr:.4f}")
print(f"  → Weekends see {(weekend_irr-1)*100:.1f}% more ER visits")

# ─── Step 8: Visualization ───
fig, axes = plt.subplots(2, 2, figsize=(14, 12))

# Actual vs predicted
axes[0, 0].scatter(y, poisson_pred, alpha=0.3, color='#58a6ff', s=10, label='Poisson')
axes[0, 0].scatter(y, lr_pred, alpha=0.3, color='#f85149', s=10, label='Linear')
max_val = max(y.max(), poisson_pred.max())
axes[0, 0].plot([0, max_val], [0, max_val], '--', color='white', alpha=0.5)
axes[0, 0].set_xlabel('Actual ER Visits')
axes[0, 0].set_ylabel('Predicted')
axes[0, 0].set_title('Actual vs Predicted')
axes[0, 0].legend()
axes[0, 0].grid(True, alpha=0.2)

# Distribution of residuals
poisson_resid = poisson_result.resid_deviance
axes[0, 1].hist(poisson_resid, bins=40, color='#3fb950', alpha=0.7, density=True, label='Deviance residuals')
x_norm = np.linspace(-4, 4, 100)
axes[0, 1].plot(x_norm, np.exp(-x_norm**2/2)/np.sqrt(2*np.pi), '--', color='white', label='Standard Normal')
axes[0, 1].set_xlabel('Deviance Residual')
axes[0, 1].set_ylabel('Density')
axes[0, 1].set_title('Deviance Residuals (should be ~Normal)')
axes[0, 1].legend()
axes[0, 1].grid(True, alpha=0.2)

# Effect of flu season
axes[1, 0].boxplot([df[df.flu_season==0].er_visits, df[df.flu_season==1].er_visits],
                    labels=['No Flu Season', 'Flu Season'])
axes[1, 0].set_ylabel('ER Visits')
axes[1, 0].set_title(f'Flu Season Effect (IRR={flu_irr:.2f})')
axes[1, 0].grid(True, alpha=0.2)

# Rate ratios with confidence intervals
params = poisson_result.params[1:]
conf = poisson_result.conf_int().iloc[1:]
irr = np.exp(params)
irr_lower = np.exp(conf[0])
irr_upper = np.exp(conf[1])

y_pos = range(len(params))
axes[1, 1].errorbar(irr, y_pos,
                     xerr=[irr - irr_lower, irr_upper - irr],
                     fmt='o', color='#58a6ff', capsize=5, markersize=8)
axes[1, 1].axvline(1.0, color='#f85149', linestyle='--', label='No effect (IRR=1)')
axes[1, 1].set_yticks(list(y_pos))
axes[1, 1].set_yticklabels(params.index)
axes[1, 1].set_xlabel('Incidence Rate Ratio')
axes[1, 1].set_title('Rate Ratios with 95% CI')
axes[1, 1].legend()
axes[1, 1].grid(True, alpha=0.2)

plt.tight_layout()
plt.savefig('project_39_glm_poisson.png', dpi=100, bbox_inches='tight')
print(f"\n✅ GLM analysis saved as project_39_glm_poisson.png")

13 · Key Takeaway + Bridge to Next Project

Key Takeaway

  • GLMs extend linear regression to non-normal outcomes through link functions and distribution families
  • Poisson regression is the go-to for count data — log link ensures non-negative predictions
  • Always check for overdispersion — if Var > Mean, use Negative Binomial instead
  • Coefficients are on the log scale — exponentiate to get interpretable rate ratios
  • Linear regression, logistic regression, and Poisson regression are all GLMs — same framework, different families
