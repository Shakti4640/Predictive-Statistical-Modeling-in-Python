Project 12: Predicting Loan Default With a Decision Tree

Decision Tree Gini impurity entropy max_depth tree visualization

Difficulty: Easy  |  Builds on: Projects 7, 10

1 · Hypothetical Problem

  • You work at a bank's risk department
  • Loan officers ask: "Given an applicant's income, credit score, debt ratio, and employment years, will they default?"
  • The bank needs EXPLAINABLE decisions — regulators require it
  • Logistic regression (Project 10) gives probabilities but the decision boundary is linear
  • Decision trees create IF-THEN rules that anyone can follow and audit

2 · How to Approach It

  1. A decision tree splits the data into groups using yes/no questions
  2. At each split, it picks the feature and threshold that best separates the classes
  3. "Best" is measured by Gini impurity or entropy — both measure class mixing
  4. The tree grows until it perfectly classifies training data (overfitting!) or hits a depth limit
  5. max_depth is the key hyperparameter — controls the bias-variance tradeoff (Project 8)

3 · How Can We Solve It

  1. Generate synthetic loan applicant data
  2. Fit a DecisionTreeClassifier with different max_depth values
  3. Visualize the tree structure
  4. Compare Gini vs entropy splitting criteria
  5. Show overfitting with unlimited depth vs controlled depth

4 · What Should Do

  • ✅ Control tree depth with max_depth — unlimited trees overfit dramatically
  • ✅ Visualize the tree — it's one of the few models you can literally draw
  • ✅ Use feature_importances_ to see which features the tree relies on most
  • ✅ Compare train vs test accuracy at different depths (Project 8 diagnostic)
  • ✅ Decision trees don't need feature scaling — they only use thresholds, not distances

5 · What Should NOT Do

  • ❌ Do NOT use unlimited depth — the tree will memorize every training point
  • ❌ Do NOT trust feature importance from a single tree — it's unstable (Random Forest in Project 21 fixes this)
  • ❌ Do NOT expect smooth decision boundaries — trees create axis-aligned rectangular regions
  • ❌ Do NOT use a single decision tree for critical applications — they're high variance (Project 21)

6 · When to Use What

  • 🔹 Decision Tree → when interpretability is paramount (regulatory, medical)
  • 🔹 Gini impurity → default, slightly faster to compute
  • 🔹 Entropy → slightly different splits, often similar performance
  • 🔹 max_depth=3-5 → interpretable tree that fits on a page
  • 🔹 Feature importance → quick feature ranking (but use with caution)

7 · When NOT to Use

  • 🚫 Single trees are unstable — small data changes can produce completely different trees
  • 🚫 Trees can't extrapolate — they predict within the range of training data only
  • 🚫 Axis-aligned splits struggle with diagonal decision boundaries
  • 🚫 For best accuracy, use ensembles of trees (Random Forest Project 21, Gradient Boosting Project 23)

8 · How It Only Works — Internal Mechanics

Gini Impurity

  • Gini = 1 - Σ(pᵢ²) where pᵢ = proportion of class i in the node
  • Gini = 0 → pure node (all one class)
  • Gini = 0.5 → maximum impurity for binary (50/50 split)
  • The tree picks the split that reduces Gini the most

Entropy

  • Entropy = -Σ(pᵢ × log₂(pᵢ))
  • Entropy = 0 → pure node
  • Entropy = 1 → maximum uncertainty for binary
  • Information Gain = parent entropy - weighted child entropy

How the Tree Grows

  1. Start with all data at the root
  2. For each feature, try every possible threshold
  3. Pick the (feature, threshold) pair that maximizes impurity reduction
  4. Split the data into two child nodes
  5. Repeat recursively until stopping criteria (max_depth, min_samples, pure nodes)

max_depth as Bias-Variance Control

  • depth=1 → "decision stump" — one split, high bias, low variance
  • depth=∞ → fully grown tree — low bias, high variance (overfits)
  • Optimal depth is found via validation (Project 8) or cross-validation (Project 15)

9 · Common Pitfalls and Failure Modes

  • ⚠️ Unlimited depth — tree memorizes training data, 100% train accuracy, poor test accuracy
  • ⚠️ Instability — removing one training point can change the entire tree structure
  • ⚠️ Biased toward features with many unique values — continuous features get more split opportunities
  • ⚠️ Overly complex trees are uninterpretable — a tree with 50 levels defeats the purpose

10 · Scalability Considerations

  • Training: O(n × p × log n) per split — fast for moderate data
  • Prediction: O(depth) per sample — very fast
  • No feature scaling needed — saves preprocessing time
  • Memory: stores the tree structure, not the data — compact
  • For large datasets, consider histogram-based trees (LightGBM, Project 31)

11 · Interpretability Considerations

  • ✅ The most interpretable ML model — literally a flowchart
  • ✅ Each prediction can be traced through the tree: "income > $50K AND credit > 700 → low risk"
  • ✅ Feature importance shows which features drive decisions
  • ✅ Regulators and auditors can inspect every decision path
  • ⚠️ Deep trees lose interpretability — keep depth ≤ 5 for human readability

12 · Complete Working Code

import numpy as np
import pandas as pd
from sklearn.tree import DecisionTreeClassifier, export_text
from sklearn.model_selection import train_test_split
from sklearn.metrics import classification_report, accuracy_score

# ─── Step 1: Synthetic loan data ───
np.random.seed(42)
n = 500

income = np.random.uniform(20000, 150000, n)
credit_score = np.random.uniform(300, 850, n)
debt_ratio = np.random.uniform(0, 0.8, n)
employment_years = np.random.uniform(0, 30, n)

# Default probability depends on features
log_odds = (-3
            - 0.00003 * income
            - 0.005 * credit_score
            + 4 * debt_ratio
            - 0.05 * employment_years
            + np.random.normal(0, 0.5, n))
prob = 1 / (1 + np.exp(-log_odds))
default = (np.random.random(n) < prob).astype(int)

df = pd.DataFrame({
    'income': income, 'credit_score': credit_score,
    'debt_ratio': debt_ratio, 'employment_years': employment_years,
    'default': default
})

print(f"=== Data Summary ===")
print(f"  Applicants: {n} | Defaults: {default.sum()} ({default.mean():.1%})")

# ─── Step 2: Split ───
features = ['income', 'credit_score', 'debt_ratio', 'employment_years']
X = df[features].values
y = df['default'].values
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, random_state=42, stratify=y)

# ─── Step 3: Compare different max_depth values ───
print(f"\n=== Depth Comparison (Bias-Variance from Project 8) ===")
print(f"{'Depth':>6} {'Train Acc':>10} {'Test Acc':>10} {'Gap':>8} {'Nodes':>7}")
print("-" * 45)

for depth in [1, 2, 3, 5, 10, None]:
    tree = DecisionTreeClassifier(max_depth=depth, random_state=42)
    tree.fit(X_train, y_train)
    train_acc = tree.score(X_train, y_train)
    test_acc = tree.score(X_test, y_test)
    n_nodes = tree.tree_.node_count
    depth_str = str(depth) if depth else "None"
    print(f"{depth_str:>6} {train_acc:>10.2%} {test_acc:>10.2%} "
          f"{train_acc-test_acc:>+8.2%} {n_nodes:>7}")

# ─── Step 4: Best model (depth=3 for interpretability) ───
best_tree = DecisionTreeClassifier(max_depth=3, random_state=42)
best_tree.fit(X_train, y_train)
y_pred = best_tree.predict(X_test)

print(f"\n=== Tree Structure (max_depth=3) ===")
tree_text = export_text(best_tree, feature_names=features, max_depth=3)
print(tree_text)

# ─── Step 5: Feature Importance ───
print(f"=== Feature Importance ===")
for feat, imp in sorted(zip(features, best_tree.feature_importances_),
                         key=lambda x: -x[1]):
    bar = "█" * int(imp * 40)
    print(f"  {feat:>18}: {imp:.4f} {bar}")

# ─── Step 6: Classification Report ───
print(f"\n=== Classification Report ===")
print(classification_report(y_test, y_pred, target_names=['No Default', 'Default']))

# ─── Step 7: Gini vs Entropy comparison ───
print(f"=== Gini vs Entropy ===")
for criterion in ['gini', 'entropy']:
    tree_c = DecisionTreeClassifier(max_depth=3, criterion=criterion, random_state=42)
    tree_c.fit(X_train, y_train)
    acc = tree_c.score(X_test, y_test)
    print(f"  {criterion:>8}: Test Accuracy={acc:.2%}")

# ─── Step 8: Trace a prediction ───
print(f"\n=== Tracing a Prediction ===")
sample = X_test[0]
pred = best_tree.predict([sample])[0]
prob = best_tree.predict_proba([sample])[0]
print(f"  Applicant: income=${sample[0]:,.0f}, credit={sample[1]:.0f}, "
      f"debt_ratio={sample[2]:.2f}, years={sample[3]:.1f}")
print(f"  Prediction: {'Default' if pred else 'No Default'} "
      f"(confidence: {max(prob):.1%})")
print(f"  Follow the tree above to trace this decision!")
