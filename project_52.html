Project 52: Online Learning — Models That Update With Every New Transaction

Online Learning SGDClassifier Concept Drift Partial Fit Sliding Window

Difficulty: Very Hard  |  Builds on: Projects 26, 44, 42

1 · Hypothetical Problem

  • A payment processor handles 10 million transactions per day and needs real-time fraud detection
  • Fraud patterns evolve constantly — last month's top attack was card-not-present fraud; this month it's account takeover with synthetic identities
  • Their batch-trained Random Forest (Project 26) achieved 98% recall at deployment — but after 3 weeks, recall dropped to 82% as fraudsters adapted
  • Retraining the full model takes 6 hours on the complete dataset — during which new fraud patterns go undetected
  • They need a model that updates incrementally with each new labeled transaction, adapting to concept drift without full retraining
  • Goal → implement online learning that maintains high fraud detection as attack patterns shift over time

2 · How to Approach It

  1. From Project 26 (Random Forest): batch models are powerful but static — they can't adapt between retraining cycles
  2. From Project 44 (Imbalanced Learning): fraud is rare (~0.5%) — online learning must handle class imbalance in streaming data
  3. From Project 42 (Pipeline): production models need robust preprocessing — online learning must standardize features incrementally
  4. The key idea: use models with a partial_fit() method that updates weights with each new mini-batch instead of retraining from scratch
  5. Combine with drift detection — monitor performance metrics over sliding windows to detect when the data distribution changes

3 · How Can We Solve It

  1. Generate a synthetic transaction stream with deliberate concept drift — fraud patterns change at known time points
  2. Implement online learning with SGDClassifier using partial_fit — update the model with each mini-batch of transactions
  3. Build a sliding window performance tracker — monitor precision, recall, and F1 over recent windows
  4. Implement concept drift detection using the ADWIN-inspired approach — detect when error rate changes significantly
  5. Compare online SGD vs periodically retrained batch model vs static (never updated) model
  6. Show how online learning maintains performance through drift while the static model degrades

4 · What Should Do

  • ✅ Use partial_fit() with explicit classes parameter on the first call — sklearn needs to know all possible classes upfront
  • ✅ Scale features incrementally — use running mean/variance or sklearn's StandardScaler.partial_fit()
  • ✅ Monitor performance on a sliding window — detect drift before it causes major damage
  • ✅ Use class weights or oversampling in the stream to handle imbalanced fraud data
  • ✅ Keep a buffer of recent data for evaluation — you can't evaluate on data you just trained on

5 · What Should NOT Do

  • ❌ Do NOT call fit() instead of partial_fit() — fit() resets the model, losing all previously learned patterns
  • ❌ Do NOT use a learning rate that's too high — the model will oscillate and forget stable patterns
  • ❌ Do NOT ignore class imbalance in the stream — if 99.5% of transactions are legitimate, the model will learn to predict "not fraud" always
  • ❌ Do NOT evaluate on the same data you just trained on — this gives optimistically biased metrics
  • ❌ Do NOT assume all sklearn models support partial_fit — only SGDClassifier, MultinomialNB, MiniBatchKMeans, and a few others do

6 · When to Use What

  • 🔹 Online SGD (partial_fit) → streaming data, concept drift, need to update without full retraining
  • 🔹 Sliding Window Retraining → moderate drift, enough compute to retrain on recent window periodically
  • 🔹 Batch Retraining (Project 26) → stable distributions, can afford periodic full retraining (e.g., weekly)
  • 🔹 Ensemble of Windows → combine models trained on different time windows for robustness
  • 🔹 Adaptive Learning Rate → when drift speed varies — fast adaptation during drift, stability otherwise

7 · When NOT to Use

  • 🚫 When the data distribution is truly static — batch models with full data access will outperform online models
  • 🚫 When labels arrive with long delays — online learning needs timely labels to update; if fraud labels take months, batch is better
  • 🚫 When the model is complex (deep ensemble, stacked models) — partial_fit is limited to simple model families
  • 🚫 When you need reproducibility — online learning results depend on data ordering, making exact reproduction difficult

8 · How It Only Works — Internal Mechanics

Stochastic Gradient Descent (Online)

  • Standard gradient descent: w ← w - η · ∇L(w; X_all, y_all) — requires full dataset
  • Online SGD: w ← w - η · ∇L(w; xᵢ, yᵢ) — updates with single sample or mini-batch
  • Each partial_fit call computes the gradient on the new batch and takes one step
  • With appropriate learning rate schedule (e.g., η_t = η₀ / (1 + η₀ · λ · t)), SGD converges to the optimum

Concept Drift Types

  • Sudden drift: distribution changes abruptly (e.g., new fraud technique appears overnight)
  • Gradual drift: old and new distributions overlap during transition period
  • Incremental drift: distribution shifts slowly and continuously
  • Recurring drift: distribution cycles between known states (e.g., seasonal fraud patterns)

Sliding Window Strategy

  • Maintain a window of the most recent W samples
  • Retrain the model on the window periodically — forgets old patterns, adapts to new ones
  • Window size trades off: too small → high variance, too large → slow adaptation
  • Can combine with online learning: partial_fit on new data + periodic window retraining as a reset

Drift Detection

  • Monitor error rate over a sliding window — if it increases significantly, drift has occurred
  • Page-Hinkley test: accumulate deviations from the mean error; trigger when cumulative sum exceeds threshold
  • ADWIN (Adaptive Windowing): automatically shrinks the window when drift is detected
  • When drift is detected: increase learning rate temporarily, or reset the model and retrain on recent data

9 · Common Pitfalls and Failure Modes

  • ⚠️ Catastrophic forgetting — learning rate too high causes the model to forget stable patterns when adapting to drift
  • ⚠️ Label delay — in fraud detection, true labels may arrive days or weeks later; the model trains on stale feedback
  • ⚠️ Class imbalance in stream — if mini-batches contain no fraud examples, the model drifts toward "always predict legitimate"
  • ⚠️ Feature distribution shift — if you standardized features on old data, new data may have different scales; use incremental scaling
  • ⚠️ Evaluation bias — evaluating on the training stream gives misleading metrics; always use a held-out evaluation window

10 · Scalability Considerations

  • Online SGD is O(d) per sample (d = number of features) — can handle millions of transactions per second
  • Memory is O(d) for model weights — no need to store the full dataset
  • Sliding window retraining is O(W × d) where W is window size — choose W based on compute budget
  • For very high throughput: batch partial_fit calls (e.g., 1000 samples at a time) for better vectorization
  • Drift detection adds minimal overhead — O(1) per sample for running statistics

11 · Interpretability Considerations

  • ✅ SGD coefficients are interpretable — positive weight on "transaction_amount" means higher amounts increase fraud probability
  • ✅ Coefficient drift over time shows which features become more/less important — track weight trajectories
  • ✅ Sliding window metrics show exactly when performance degrades — pinpoint drift timing
  • ⚠️ Online models change constantly — explanations for the same input may differ between yesterday and today
  • ⚠️ Hard to audit — no single "model version" to inspect; need to log model snapshots periodically

12 · Complete Working Code

import numpy as np
import pandas as pd
from sklearn.linear_model import SGDClassifier
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import accuracy_score, f1_score, precision_score, recall_score
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

np.random.seed(42)

# ═══════════════════════════════════════════════════════════════
# STEP 1: Generate Streaming Transaction Data With Concept Drift
# ═══════════════════════════════════════════════════════════════
def generate_transaction_batch(n, fraud_pattern='A', fraud_rate=0.02):
    """Generate a batch of transactions with a specific fraud pattern."""
    amount = np.random.exponential(50, n)
    hour = np.random.randint(0, 24, n)
    is_international = np.random.binomial(1, 0.15, n)
    merchant_risk = np.random.uniform(0, 1, n)
    velocity = np.random.poisson(2, n)
    device_age_days = np.random.exponential(180, n)

    X = np.column_stack([amount, hour, is_international, merchant_risk, velocity, device_age_days])

    if fraud_pattern == 'A':
        fraud_score = (0.01 * amount + 0.5 * is_international + 0.3 * merchant_risk
                       + 0.1 * velocity - 0.001 * device_age_days)
    elif fraud_pattern == 'B':
        fraud_score = (0.005 * amount + 0.1 * is_international + 0.6 * merchant_risk
                       + 0.4 * velocity + 0.3 * (hour > 22).astype(float))
    else:
        fraud_score = (0.02 * amount + 0.3 * is_international + 0.2 * merchant_risk
                       + 0.05 * velocity + 0.5 * (device_age_days < 30).astype(float))

    threshold = np.percentile(fraud_score, 100 * (1 - fraud_rate))
    y = (fraud_score > threshold).astype(int)
    return X, y

# Create a stream with concept drift at known points
stream_config = [
    ('A', 3000), ('A', 2000),  # Phase 1: pattern A (5000 transactions)
    ('B', 2000), ('B', 3000),  # Phase 2: drift to pattern B (5000 transactions)
    ('C', 2000), ('C', 3000),  # Phase 3: drift to pattern C (5000 transactions)
]

X_stream, y_stream = [], []
drift_points = []
total = 0
for pattern, n in stream_config:
    X_batch, y_batch = generate_transaction_batch(n, fraud_pattern=pattern)
    X_stream.append(X_batch)
    y_stream.append(y_batch)
    total += n
    drift_points.append(total)

X_stream = np.vstack(X_stream)
y_stream = np.concatenate(y_stream)

print(f"=== Transaction Stream ===")
print(f"  Total transactions: {len(y_stream)}")
print(f"  Fraud rate: {y_stream.mean():.2%}")
print(f"  Drift points: {drift_points[1::2]}")

# ═══════════════════════════════════════════════════════════════
# STEP 2: Online Learning with SGDClassifier
# ═══════════════════════════════════════════════════════════════
batch_size = 200
n_batches = len(y_stream) // batch_size

# Incremental scaler
scaler = StandardScaler()

# Online model
online_model = SGDClassifier(loss='log_loss', learning_rate='optimal',
                              class_weight={0: 1, 1: 50}, random_state=42)

# Static model (trained once on first 1000 samples)
static_model = SGDClassifier(loss='log_loss', learning_rate='optimal',
                              class_weight={0: 1, 1: 50}, random_state=42)

# Metrics tracking
online_metrics = {'f1': [], 'precision': [], 'recall': [], 'accuracy': []}
static_metrics = {'f1': [], 'precision': [], 'recall': [], 'accuracy': []}
batch_indices = []

# Initial training on first 5 batches
init_batches = 5
X_init = X_stream[:init_batches * batch_size]
y_init = y_stream[:init_batches * batch_size]
scaler.partial_fit(X_init)
X_init_scaled = scaler.transform(X_init)

online_model.partial_fit(X_init_scaled, y_init, classes=[0, 1])
static_model.fit(X_init_scaled, y_init)

print(f"\n=== Online Learning Progress ===")

# Process remaining stream
eval_window = 500  # evaluate on last 500 predictions
predictions_online = []
predictions_static = []
actuals = []

for i in range(init_batches, n_batches):
    start = i * batch_size
    end = start + batch_size
    X_batch = X_stream[start:end]
    y_batch = y_stream[start:end]

    # Scale incrementally
    scaler.partial_fit(X_batch)
    X_batch_scaled = scaler.transform(X_batch)

    # Predict BEFORE updating (honest evaluation)
    pred_online = online_model.predict(X_batch_scaled)
    pred_static = static_model.predict(X_batch_scaled)

    predictions_online.extend(pred_online)
    predictions_static.extend(pred_static)
    actuals.extend(y_batch)

    # Update online model
    online_model.partial_fit(X_batch_scaled, y_batch)

    # Evaluate on sliding window
    if len(actuals) >= eval_window:
        window_actual = np.array(actuals[-eval_window:])
        window_online = np.array(predictions_online[-eval_window:])
        window_static = np.array(predictions_static[-eval_window:])

        if len(np.unique(window_actual)) > 1:
            online_metrics['f1'].append(f1_score(window_actual, window_online, zero_division=0))
            online_metrics['recall'].append(recall_score(window_actual, window_online, zero_division=0))
            static_metrics['f1'].append(f1_score(window_actual, window_static, zero_division=0))
            static_metrics['recall'].append(recall_score(window_actual, window_static, zero_division=0))
            batch_indices.append(i * batch_size)

    if (i - init_batches) % 15 == 0 and len(online_metrics['f1']) > 0:
        print(f"  Batch {i:>3} (t={i*batch_size:>5}): "
              f"Online F1={online_metrics['f1'][-1]:.3f}, "
              f"Static F1={static_metrics['f1'][-1]:.3f}")

# ═══════════════════════════════════════════════════════════════
# STEP 3: Drift Detection (Simple Error Rate Monitor)
# ═══════════════════════════════════════════════════════════════
print(f"\n=== Drift Detection ===")
window_size = 500
error_rates = []
drift_detected = []

for i in range(window_size, len(actuals)):
    window = np.array(actuals[i-window_size:i])
    preds = np.array(predictions_online[i-window_size:i])
    error_rate = 1 - accuracy_score(window, preds)
    error_rates.append(error_rate)

    if len(error_rates) > 100:
        recent_mean = np.mean(error_rates[-50:])
        historical_mean = np.mean(error_rates[-100:-50])
        if recent_mean > historical_mean * 1.5 and recent_mean > 0.05:
            if not drift_detected or (i - drift_detected[-1]) > 500:
                drift_detected.append(i)
                print(f"  ⚠️ Drift detected at t={i + init_batches*batch_size}: "
                      f"error rate jumped from {historical_mean:.3f} to {recent_mean:.3f}")

# ═══════════════════════════════════════════════════════════════
# STEP 4: Final Comparison
# ═══════════════════════════════════════════════════════════════
print(f"\n=== Final Performance Summary ===")
final_actual = np.array(actuals[-2000:])
final_online = np.array(predictions_online[-2000:])
final_static = np.array(predictions_static[-2000:])

print(f"  {'Metric':<15} {'Online SGD':>12} {'Static Model':>14}")
print(f"  {'-'*43}")
for metric_name, metric_fn in [('F1', f1_score), ('Precision', precision_score),
                                 ('Recall', recall_score), ('Accuracy', accuracy_score)]:
    o = metric_fn(final_actual, final_online, zero_division=0)
    s = metric_fn(final_actual, final_static, zero_division=0)
    print(f"  {metric_name:<15} {o:>12.4f} {s:>14.4f}")

# ═══════════════════════════════════════════════════════════════
# STEP 5: Visualization
# ═══════════════════════════════════════════════════════════════
fig, axes = plt.subplots(2, 2, figsize=(14, 12))

# F1 over time
ax = axes[0, 0]
ax.plot(batch_indices, online_metrics['f1'], color='#3fb950', linewidth=1.5, label='Online SGD', alpha=0.8)
ax.plot(batch_indices, static_metrics['f1'], color='#f85149', linewidth=1.5, label='Static Model', alpha=0.8)
for dp in [5000, 10000]:
    ax.axvline(x=dp, color='#f0883e', linestyle='--', alpha=0.7, label='Drift point' if dp == 5000 else '')
ax.set_xlabel('Transaction #')
ax.set_ylabel('F1 Score (sliding window)')
ax.set_title('F1 Score Over Time — Online vs Static')
ax.legend(fontsize=8)
ax.grid(True, alpha=0.2)

# Recall over time
ax = axes[0, 1]
ax.plot(batch_indices, online_metrics['recall'], color='#3fb950', linewidth=1.5, label='Online SGD')
ax.plot(batch_indices, static_metrics['recall'], color='#f85149', linewidth=1.5, label='Static Model')
for dp in [5000, 10000]:
    ax.axvline(x=dp, color='#f0883e', linestyle='--', alpha=0.7)
ax.set_xlabel('Transaction #')
ax.set_ylabel('Recall (sliding window)')
ax.set_title('Fraud Recall Over Time')
ax.legend(fontsize=8)
ax.grid(True, alpha=0.2)

# Error rate with drift detection
ax = axes[1, 0]
if error_rates:
    ax.plot(range(len(error_rates)), error_rates, color='#58a6ff', linewidth=1, alpha=0.7)
    for dd in drift_detected:
        ax.axvline(x=dd - window_size, color='#f85149', linestyle='--', alpha=0.8)
ax.set_xlabel('Time Step')
ax.set_ylabel('Error Rate')
ax.set_title('Error Rate — Drift Detection')
ax.grid(True, alpha=0.2)

# Cumulative performance
ax = axes[1, 1]
cum_correct_online = np.cumsum(np.array(actuals) == np.array(predictions_online))
cum_correct_static = np.cumsum(np.array(actuals) == np.array(predictions_static))
x_range = range(len(actuals))
ax.plot(x_range, cum_correct_online / (np.arange(len(actuals)) + 1), color='#3fb950', label='Online SGD')
ax.plot(x_range, cum_correct_static / (np.arange(len(actuals)) + 1), color='#f85149', label='Static')
ax.set_xlabel('Transaction #')
ax.set_ylabel('Cumulative Accuracy')
ax.set_title('Cumulative Accuracy Over Stream')
ax.legend(fontsize=8)
ax.grid(True, alpha=0.2)

for a in axes.flat:
    a.set_facecolor('#0d1117')
    a.tick_params(colors='white')
    a.xaxis.label.set_color('white')
    a.yaxis.label.set_color('white')
    a.title.set_color('white')
fig.patch.set_facecolor('#0d1117')
plt.tight_layout()
plt.savefig('project_52_online_learning.png', dpi=100, bbox_inches='tight', facecolor='#0d1117')
print(f"\n✅ Online learning analysis saved as project_52_online_learning.png")

13 · Key Takeaway + Bridge to Next Project

Key Takeaway

  • Online learning updates models incrementally — no full retraining needed when data streams continuously
  • SGDClassifier.partial_fit() is the workhorse — O(d) per sample, handles millions of transactions
  • Concept drift degrades static models — online learning adapts, maintaining performance through distribution shifts
  • Drift detection monitors error rates over sliding windows — triggers alerts when patterns change
  • Always evaluate on held-out windows, not the data you just trained on
