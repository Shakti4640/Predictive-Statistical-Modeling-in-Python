1 · Hypothetical Problem

  • An insurance company has 100 features for predicting whether a claim is fraudulent
  • XGBoost (Project 31) works well and gives feature importance, but the company wants to explore deep learning for potential gains
  • Standard neural networks (Project 47) treat all features equally — no built-in feature selection
  • TabNet promises the best of both worlds: deep learning performance with attention-based feature selection
  • Goal → implement a simplified attention-based tabular network and compare with XGBoost on feature selection and accuracy

2 · How to Approach It

  1. From Project 47: we know how to build neural networks — but they don't naturally select features
  2. From Project 31: XGBoost gives feature importance via split counts — but it's not differentiable end-to-end
  3. From Project 41: feature selection matters — irrelevant features hurt performance
  4. TabNet uses sequential attention to decide which features to focus on at each decision step
  5. We'll implement the core attention mechanism from scratch using numpy/sklearn

3 · How Can We Solve It

  1. Generate a synthetic insurance dataset with 100 features (only ~15 truly informative)
  2. Build a simplified attention-based tabular network using numpy
  3. Implement soft attention masks that learn which features matter per decision step
  4. Train with gradient descent and extract attention weights as feature importance
  5. Compare accuracy and feature selection with XGBoost
  6. Visualize attention masks and feature importance rankings

4 · What Should Do

  • ✅ Use attention mechanisms for interpretable feature selection — each step focuses on different features
  • ✅ Compare with strong baselines (XGBoost, Random Forest) — TabNet should match or beat them
  • ✅ Analyze attention weights — they reveal which features the model considers important per sample
  • ✅ Use sparsity regularization on attention — encourages the model to use fewer features per step
  • ✅ Pre-train on unlabeled data when available — TabNet supports self-supervised pre-training

5 · What Should NOT Do

  • ❌ Do NOT assume TabNet always beats GBMs — on many tabular benchmarks, XGBoost still wins
  • ❌ Do NOT skip feature preprocessing — TabNet still benefits from normalization and encoding
  • ❌ Do NOT use TabNet on tiny datasets — it needs sufficient data to learn attention patterns
  • ❌ Do NOT ignore the sparsity coefficient — too low means no feature selection; too high means underfitting
  • ❌ Do NOT treat attention weights as causal — they show correlation, not causation

6 · When to Use What

  • 🔹 TabNet / Attention-based DL → large tabular data, need built-in feature selection, want end-to-end differentiable model
  • 🔹 XGBoost (Project 31) → tabular data default, fast training, robust feature importance
  • 🔹 Neural Networks (Project 47) → when you'll add custom layers or multi-modal inputs later
  • 🔹 Lasso/ElasticNet (Project 41) → linear feature selection, small data, interpretability required
  • 🔹 Random Forest (Project 21) → quick baseline, handles mixed feature types well

7 · When NOT to Use

  • 🚫 Small datasets (< 5K rows) — attention mechanisms need data to learn meaningful patterns
  • 🚫 When training speed matters — TabNet is slower than XGBoost by 5-10×
  • 🚫 When you need a simple, explainable model — logistic regression or decision trees are better
  • 🚫 When features are already well-selected — attention adds complexity without benefit

8 · How It Only Works — Internal Mechanics

TabNet Architecture Overview

  • TabNet processes data in sequential decision steps (like boosting rounds)
  • At each step, an attention mechanism selects a subset of features to focus on
  • The selected features are processed through a shared network to produce a partial output
  • Final prediction is the sum of all partial outputs across steps

Attention Mechanism (Sparsemax)

  • At step i: M[i] = sparsemax(P[i-1] · h(a[i-1])) — attention mask
  • P[i] = P[i-1] · (γ - M[i]) — prior scales track which features have been used
  • sparsemax produces sparse outputs (many exact zeros) unlike softmax
  • γ controls feature reuse — γ=1 means each feature used once; γ>1 allows reuse

Feature Processing

  • Masked features: x̃[i] = M[i] ⊙ x — element-wise masking of input
  • Each step transforms masked features through shared + step-specific layers
  • Batch normalization applied at each step for stability

Sparsity Regularization

  • L_sparse = -Σ M[i] · log(M[i] + ε) — entropy of attention masks
  • Encourages the model to make decisive feature selections (low entropy = sparse)
  • Controlled by coefficient λ_sparse — higher means sparser attention

9 · Common Pitfalls and Failure Modes

  • ⚠️ Overfitting on small data — TabNet has many parameters; use dropout and early stopping
  • ⚠️ Attention collapse — all steps focus on the same features; increase sparsity regularization
  • ⚠️ Slow convergence — TabNet needs more epochs than GBMs; patience is required
  • ⚠️ Hyperparameter sensitivity — number of steps, attention width, sparsity coefficient all matter
  • ⚠️ Misleading feature importance — attention weights ≠ causal importance; validate with permutation importance

10 · Scalability Considerations

  • TabNet scales linearly with number of features and samples — better than quadratic attention in transformers
  • GPU acceleration is essential for large datasets — CPU training is 10-50× slower
  • Memory: O(n × d × n_steps) — manageable for most tabular datasets
  • For very wide data (>1000 features): pre-filter with variance threshold before TabNet

11 · Interpretability Considerations

  • ✅ Attention masks provide per-sample feature importance — more granular than global importance
  • ✅ Aggregate attention across samples gives global feature ranking — comparable to XGBoost importance
  • ✅ Sequential steps show how the model builds its decision — step 1 might check income, step 2 checks credit
  • ⚠️ Attention ≠ explanation — the model may attend to correlated proxies, not true causes
  • ⚠️ Harder to explain to non-technical stakeholders than tree-based feature importance

# ─── Step 2: Simplified attention-based tabular network ───
def softmax(x, axis=-1):
    e_x = np.exp(x - np.max(x, axis=axis, keepdims=True))
    return e_x / e_x.sum(axis=axis, keepdims=True)

def sparsemax_1d(z):
    """Sparsemax activation — produces sparse probability distributions."""
    sorted_z = np.sort(z)[::-1]
    k = np.arange(1, len(z) + 1)
    cumsum = np.cumsum(sorted_z)
    support = sorted_z - (cumsum - 1) / k > 0
    k_z = np.sum(support)
    tau = (np.sum(sorted_z[:k_z]) - 1) / k_z
    return np.maximum(z - tau, 0)

def sparsemax_batch(Z):
    """Apply sparsemax row-wise."""
    return np.array([sparsemax_1d(z) for z in Z])

def sigmoid(z):
    z = np.clip(z, -500, 500)
    return 1.0 / (1.0 + np.exp(-z))

def relu(z):
    return np.maximum(0, z)

class SimpleTabNet:
    """Simplified TabNet-inspired attention network."""

    def __init__(self, input_dim, hidden_dim=64, n_steps=3, lr=0.01, sparsity_coef=1e-3, gamma=1.3):
        self.input_dim = input_dim
        self.hidden_dim = hidden_dim
        self.n_steps = n_steps
        self.lr = lr
        self.sparsity_coef = sparsity_coef
        self.gamma = gamma

        # Attention network weights (per step)
        self.W_attn = [np.random.randn(input_dim, input_dim) * 0.05 for _ in range(n_steps)]
        self.b_attn = [np.zeros(input_dim) for _ in range(n_steps)]

        # Feature processing weights (per step)
        self.W_feat = [np.random.randn(input_dim, hidden_dim) * np.sqrt(2.0/input_dim) for _ in range(n_steps)]
        self.b_feat = [np.zeros(hidden_dim) for _ in range(n_steps)]

        # Output aggregation
        self.W_out = np.random.randn(hidden_dim, 1) * np.sqrt(1.0/hidden_dim)
        self.b_out = np.zeros(1)

        # Store attention masks for interpretability
        self.attention_masks = []

    def forward(self, X, store_attention=False):
        m = X.shape[0]
        prior_scales = np.ones((m, self.input_dim))
        aggregated = np.zeros((m, self.hidden_dim))
        self.attention_masks = []
        self._cache = {'X': X, 'steps': []}

        for step in range(self.n_steps):
            # Attention: which features to focus on
            attn_logits = X @ self.W_attn[step] + self.b_attn[step]
            attn_logits = attn_logits * prior_scales
            mask = sparsemax_batch(attn_logits)

            # Update prior scales (reduce attention on already-used features)
            prior_scales = prior_scales * (self.gamma - mask)

            # Masked feature selection
            masked_x = mask * X

            # Feature processing
            z = masked_x @ self.W_feat[step] + self.b_feat[step]
            h = relu(z)

            aggregated += h

            if store_attention:
                self.attention_masks.append(mask.copy())

            self._cache['steps'].append({
                'mask': mask, 'masked_x': masked_x,
                'z': z, 'h': h, 'attn_logits': attn_logits,
                'prior_scales_before': prior_scales.copy()
            })

        # Output
        logits = aggregated @ self.W_out + self.b_out
        probs = sigmoid(logits)
        self._cache['aggregated'] = aggregated
        self._cache['probs'] = probs
        return probs

    def compute_loss(self, y_true, y_pred):
        eps = 1e-8
        y_pred = np.clip(y_pred, eps, 1 - eps)
        bce = -np.mean(y_true * np.log(y_pred) + (1 - y_true) * np.log(1 - y_pred))

        # Sparsity regularization on attention masks
        sparsity_loss = 0
        for step_cache in self._cache['steps']:
            mask = step_cache['mask']
            sparsity_loss += np.mean(np.sum(-mask * np.log(mask + 1e-15), axis=1))
        sparsity_loss *= self.sparsity_coef

        return bce + sparsity_loss

    def backward(self, y_true):
        m = y_true.shape[0]
        probs = self._cache['probs']
        aggregated = self._cache['aggregated']

        # Output gradient
        d_logits = (probs - y_true) / m
        d_W_out = aggregated.T @ d_logits
        d_b_out = np.sum(d_logits, axis=0)
        d_aggregated = d_logits @ self.W_out.T

        self.W_out -= self.lr * d_W_out
        self.b_out -= self.lr * d_b_out

        # Per-step gradients
        for step in range(self.n_steps):
            cache = self._cache['steps'][step]
            d_h = d_aggregated.copy()
            d_z = d_h * (cache['z'] > 0).astype(float)

            d_W_feat = cache['masked_x'].T @ d_z / m
            d_b_feat = np.mean(d_z, axis=0)

            self.W_feat[step] -= self.lr * d_W_feat
            self.b_feat[step] -= self.lr * d_b_feat

            d_masked_x = d_z @ self.W_feat[step].T
            d_attn = d_masked_x * self._cache['X']

            d_W_attn = self._cache['X'].T @ d_attn / m
            d_b_attn = np.mean(d_attn, axis=0)

            self.W_attn[step] -= self.lr * d_W_attn
            self.b_attn[step] -= self.lr * d_b_attn

    def fit(self, X, y, epochs=200, verbose=True):
        y_col = y.reshape(-1, 1)
        losses = []
        for epoch in range(epochs):
            probs = self.forward(X)
            loss = self.compute_loss(y_col, probs)
            self.backward(y_col)
            losses.append(loss)
            if verbose and (epoch + 1) % 50 == 0:
                preds = (probs >= 0.5).astype(int).ravel()
                acc = accuracy_score(y, preds)
                print(f"  Epoch {epoch+1:4d}: loss={loss:.4f}, acc={acc:.4f}")
        return losses

    def predict_proba(self, X):
        return self.forward(X, store_attention=True).ravel()

    def predict(self, X):
        return (self.predict_proba(X) >= 0.5).astype(int)

    def get_feature_importance(self, X):
        """Aggregate attention masks across steps and samples."""
        self.forward(X, store_attention=True)
        total_attention = np.zeros(self.input_dim)
        for mask in self.attention_masks:
            total_attention += np.mean(mask, axis=0)
        total_attention /= self.n_steps
        return total_attention
