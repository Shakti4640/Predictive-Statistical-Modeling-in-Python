Project 7: Predicting Employee Salary Band — Discrete Buckets

Ordinal target binning label encoding pandas cut classification vs regression

Difficulty: Very Easy  |  Builds on: Projects 2, 6

1 · Hypothetical Problem

  • You work in HR at a mid-size company
  • Management asks: "Given an employee's years of experience and education level, which salary band will they fall into?"
  • Salary bands: Band 1 (Junior: $30K-$50K), Band 2 (Mid: $50K-$80K), Band 3 (Senior: $80K-$120K)
  • This is NOT regression (predicting exact salary) — it's classification into ordered categories
  • The target is ordinal: Band 1 < Band 2 < Band 3 (there's a natural order)
  • This project clarifies WHEN to use regression vs classification

2 · How to Approach It

  1. First decision: regression or classification?
    • If the business needs an exact salary → regression (Project 2)
    • If the business needs a salary BAND → classification (this project)
  2. Create salary bands from continuous salary data using pandas.cut()
  3. Encode the ordinal labels as integers: Band 1=0, Band 2=1, Band 3=2
  4. Use logistic regression (Project 6) extended to multi-class
  5. Evaluate with a multi-class confusion matrix

3 · How Can We Solve It

  1. Generate synthetic employee data (experience, education → salary)
  2. Use pd.cut() to bin continuous salary into 3 bands
  3. Encode bands as integers with label encoding
  4. Train/test split
  5. Fit LogisticRegression with multi_class='multinomial'
  6. Evaluate with multi-class confusion matrix and classification report

4 · What Should Do

  • ✅ Ask the business question first: do they need an exact number or a category?
  • ✅ Use pd.cut() for equal-width bins or pd.qcut() for equal-frequency bins
  • ✅ Understand ordinal vs nominal: salary bands have order, colors don't
  • ✅ Use label encoding for ordinal targets (preserves order: 0 < 1 < 2)
  • ✅ Check class distribution after binning — imbalanced bins cause problems
  • ✅ Use classification_report for per-class precision/recall (extends Project 4)

5 · What Should NOT Do

  • ❌ Do NOT use one-hot encoding for ordinal targets — it destroys the order information
  • ❌ Do NOT use regression when the business needs categories — predicting $67,432 when they want "Mid-level" is unhelpful
  • ❌ Do NOT create too many bins — with 10 bins and 100 samples, each bin has only 10 samples
  • ❌ Do NOT ignore the bin boundaries — they're arbitrary choices that affect model performance
  • ❌ Do NOT treat ordinal classification the same as nominal — the order matters for evaluation

6 · When to Use What

  • 🔹 Regression → when the target is a continuous number (exact salary, exact temperature)
  • 🔹 Binary classification → when the target has 2 categories (Projects 1, 4, 6)
  • 🔹 Multi-class classification → when the target has 3+ categories (this project)
  • 🔹 pd.cut() → to create equal-width bins from continuous data
  • 🔹 pd.qcut() → to create equal-frequency bins (each bin has ~same number of samples)
  • 🔹 Label encoding → for ordinal categories (Low=0, Medium=1, High=2)

7 · When NOT to Use

  • 🚫 Don't bin continuous targets unless the business specifically needs categories — you lose information
  • 🚫 Label encoding is wrong for nominal categories (Red=0, Blue=1, Green=2 implies Blue is "between" Red and Green)
  • 🚫 Multi-class logistic regression struggles with many classes (20+) — need different approaches
  • 🚫 Ordinal regression (specialized method) is better than treating ordinal as nominal — but sklearn doesn't have it built-in

8 · How It Only Works — Internal Mechanics

Binning — Converting Continuous to Categorical

  • pd.cut(salary, bins=[0, 50000, 80000, 120000]) → creates 3 bins with fixed boundaries
  • pd.qcut(salary, q=3) → creates 3 bins with equal number of samples each
  • Binning is a lossy transformation — $51K and $79K both become "Mid" band
  • The choice of bin boundaries is a design decision, not a statistical one

Label Encoding vs One-Hot Encoding

  • Label encoding: Band 1→0, Band 2→1, Band 3→2 (preserves order, single column)
  • One-hot encoding: Band 1→[1,0,0], Band 2→[0,1,0], Band 3→[0,0,1] (no order, multiple columns)
  • For ordinal targets → label encoding is correct
  • For nominal features (e.g., city names) → one-hot encoding is correct (Project 16)

Multi-Class Logistic Regression

  • Extends binary logistic regression (Project 6) to K classes
  • One-vs-Rest (OvR): train K binary classifiers, each separating one class from the rest
  • Multinomial (Softmax): train one model that outputs K probabilities summing to 1
  • sklearn default: multi_class='auto' → chooses based on solver

Classification vs Regression — The Decision Framework

  • Target is a number on a continuous scale → regression
  • Target is one of a fixed set of categories → classification
  • Target is a number but business wants categories → bin first, then classify
  • Target is ordinal → classification with label encoding (or specialized ordinal regression)

9 · Common Pitfalls and Failure Modes

  • ⚠️ Arbitrary bin boundaries — moving the boundary by $5K can change many labels
  • ⚠️ Imbalanced bins — if 80% of employees are in Band 2, the model just predicts Band 2 for everyone
  • ⚠️ Information loss from binning — two employees earning $50,001 and $79,999 are treated identically
  • ⚠️ Using one-hot encoding for ordinal targets — the model loses the ordering information
  • ⚠️ Confusing feature encoding with target encoding — features and targets may need different encoding strategies

10 · Scalability Considerations

  • Binning with pd.cut() is O(n) — trivial
  • Multi-class logistic regression: OvR trains K models, multinomial trains 1 model with K outputs
  • With 3 classes and 100K samples → both approaches are fast
  • With 100+ classes → multinomial is more efficient than OvR
  • Label encoding adds no computational cost — it's just integer mapping

11 · Interpretability Considerations

  • ✅ Salary bands are intuitive: "this employee is predicted to be in the Senior band"
  • ✅ Probabilities per band: "60% Senior, 30% Mid, 10% Junior" — actionable
  • ✅ Coefficients still interpretable (per-class log-odds changes)
  • ✅ Confusion matrix shows which bands get confused with each other
  • ⚠️ Binning introduces artificial boundaries — employees near the boundary are hard to classify
  • ⚠️ Stakeholders may question why $50,001 is "Mid" but $49,999 is "Junior"

12 · Complete Working Code

import numpy as np
import pandas as pd
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import classification_report, confusion_matrix, accuracy_score
from sklearn.model_selection import train_test_split

# ─── Step 1: Synthetic employee data ───
np.random.seed(42)
n = 200

experience = np.random.uniform(0, 25, n)          # years
education = np.random.choice([0, 1, 2], n,        # 0=Bachelor, 1=Master, 2=PhD
                              p=[0.5, 0.35, 0.15])

# Salary depends on experience + education + noise
salary = (30000
          + 3000 * experience
          + 10000 * education
          + np.random.normal(0, 8000, n))
salary = np.maximum(salary, 25000)

df = pd.DataFrame({
    'experience': experience,
    'education': education,
    'salary': salary
})

print("=== Sample Data ===")
print(df.head(8).to_string(index=False))

# ─── Step 2: Create salary bands with pd.cut() ───
bins = [0, 50000, 80000, 200000]
labels_text = ['Junior', 'Mid', 'Senior']
df['band'] = pd.cut(df['salary'], bins=bins, labels=labels_text)
df['band_code'] = df['band'].cat.codes  # Label encoding: Junior=0, Mid=1, Senior=2

print(f"\n=== Salary Band Distribution ===")
print(df['band'].value_counts().sort_index())

# Show binning effect
print(f"\n=== Binning Examples ===")
for band in labels_text:
    subset = df[df['band'] == band]['salary']
    print(f"  {band}: ${subset.min():,.0f} — ${subset.max():,.0f} (n={len(subset)})")

# ─── Step 3: Prepare features and target ───
X = df[['experience', 'education']].values
y = df['band_code'].values

X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, random_state=42, stratify=y
)
print(f"\nTrain: {len(X_train)} | Test: {len(X_test)}")
print(f"Train class distribution: {np.bincount(y_train)}")

# ─── Step 4: Multi-class Logistic Regression ───
model = LogisticRegression(multi_class='multinomial', max_iter=1000, random_state=42)
model.fit(X_train, y_train)

print(f"\n=== Model Coefficients ===")
for i, band in enumerate(labels_text):
    print(f"  {band}: experience={model.coef_[i][0]:+.4f}, "
          f"education={model.coef_[i][1]:+.4f}, "
          f"intercept={model.intercept_[i]:+.4f}")

# ─── Step 5: Predictions with probabilities ───
y_pred = model.predict(X_test)
y_proba = model.predict_proba(X_test)

print(f"\n=== Sample Predictions ===")
for i in range(8):
    probs_str = " | ".join([f"{labels_text[j]}:{y_proba[i][j]:.1%}" for j in range(3)])
    actual = labels_text[y_test[i]]
    predicted = labels_text[y_pred[i]]
    status = "✓" if y_pred[i] == y_test[i] else "✗"
    print(f"  Exp={X_test[i][0]:.1f}yr Edu={int(X_test[i][1])} → [{probs_str}] "
          f"→ Pred={predicted} Actual={actual} {status}")

# ─── Step 6: Evaluation ───
acc = accuracy_score(y_test, y_pred)
print(f"\n=== Evaluation ===")
print(f"  Accuracy: {acc:.2%}")
print(f"\n  Classification Report:")
print(classification_report(y_test, y_pred, target_names=labels_text))

print(f"  Confusion Matrix (rows=actual, cols=predicted):")
cm = confusion_matrix(y_test, y_pred)
print(f"  {'':>10} {'Junior':>8} {'Mid':>8} {'Senior':>8}")
for i, band in enumerate(labels_text):
    print(f"  {band:>10} {cm[i][0]:>8} {cm[i][1]:>8} {cm[i][2]:>8}")

# ─── Step 7: Classification vs Regression comparison ───
from sklearn.linear_model import LinearRegression as LinReg
from sklearn.metrics import mean_squared_error

reg_model = LinReg()
reg_model.fit(X_train, df.loc[X_train[:, 0].argsort()[:len(X_train)], 'salary'].values[:len(X_train)])
# Simpler: just show the conceptual difference
print(f"\n=== Classification vs Regression ===")
print(f"  Classification output: 'Mid' (a category)")
print(f"  Regression output:     '$67,432' (a number)")
print(f"  Choose based on what the business NEEDS, not what's easier")
