Project 13: Scaling and Normalizing Features — Why Raw Numbers Lie

StandardScaler MinMaxScaler RobustScaler pipeline concept

Difficulty: Easy  |  Builds on: Projects 9, 11

1 · Hypothetical Problem

  • You're building a model to predict customer lifetime value using: age (20-70), annual income ($20K-$200K), and number of purchases (1-50)
  • Income is in thousands, age in decades, purchases in single digits — wildly different scales
  • KNN (Project 11) gave terrible results until you scaled — but WHY?
  • Decision trees (Project 12) worked fine without scaling — WHY?
  • Goal → understand WHEN and HOW to scale, and which scaler to use

2 · How to Approach It

  1. Compare model performance with and without scaling for different algorithms
  2. Test three scalers: StandardScaler, MinMaxScaler, RobustScaler
  3. Introduce sklearn Pipeline to prevent data leakage during scaling
  4. Show that tree-based models are scale-invariant, but distance/gradient-based models are not

3 · How Can We Solve It

  1. Generate data with features on very different scales
  2. Run KNN, Logistic Regression, and Decision Tree with and without scaling
  3. Compare all three scalers on each model
  4. Build a Pipeline that combines scaling + model in one object
  5. Show the data leakage problem when scaling before splitting

4 · What Should Do

  • ✅ Use Pipeline to combine preprocessing and modeling — prevents data leakage
  • ✅ Fit the scaler on training data ONLY, then transform both train and test
  • ✅ Use StandardScaler as the default — works well for most algorithms
  • ✅ Use RobustScaler when data has outliers — it uses median/IQR instead of mean/std
  • ✅ Use MinMaxScaler when you need values in a specific range (e.g., 0-1 for neural networks)

5 · What Should NOT Do

  • ❌ Do NOT fit the scaler on the entire dataset (train+test) — this leaks test information
  • ❌ Do NOT scale tree-based models — it's unnecessary and adds complexity
  • ❌ Do NOT use MinMaxScaler with outliers — one extreme value compresses all others
  • ❌ Do NOT forget to apply the SAME scaler to new data at prediction time

6 · When to Use What

  • 🔹 StandardScaler → default choice; mean=0, std=1; works for most algorithms
  • 🔹 MinMaxScaler → when you need bounded range [0,1]; good for neural networks
  • 🔹 RobustScaler → when data has outliers; uses median and IQR
  • 🔹 No scaling needed → tree-based models (Decision Tree, Random Forest, XGBoost)
  • 🔹 Scaling required → KNN, SVM, Logistic Regression, Neural Networks, PCA

7 · When NOT to Use

  • 🚫 Don't scale binary/indicator features (0/1) — they're already on a natural scale
  • 🚫 Don't scale ordinal encoded features if using tree-based models
  • 🚫 MinMaxScaler is fragile with outliers — one extreme value ruins the scaling
  • 🚫 Scaling doesn't fix fundamental data quality issues — garbage in, scaled garbage out

8 · How It Only Works — Internal Mechanics

StandardScaler

  • x_scaled = (x - mean) / std
  • Result: mean=0, std=1 for each feature
  • Assumes roughly Gaussian distribution

MinMaxScaler

  • x_scaled = (x - min) / (max - min)
  • Result: values in [0, 1]
  • Preserves the shape of the distribution

RobustScaler

  • x_scaled = (x - median) / IQR
  • IQR = Q3 - Q1 (interquartile range)
  • Robust to outliers because median and IQR are not affected by extreme values

Why Trees Don't Need Scaling

  • Trees split on thresholds: "is income > $50K?"
  • Scaling changes the numbers but not the ORDER — the same split is found
  • Distance-based models (KNN) and gradient-based models (logistic regression) ARE affected by scale

Pipeline — Preventing Data Leakage

  • Pipeline chains preprocessing + model into one object
  • pipe.fit(X_train, y_train) → fits scaler on X_train, then fits model on scaled X_train
  • pipe.predict(X_test) → scales X_test using training statistics, then predicts
  • This guarantees no information from test data leaks into scaling

9 · Common Pitfalls and Failure Modes

  • ⚠️ Scaling before splitting — test data statistics leak into the scaler
  • ⚠️ Forgetting to scale at prediction time — model receives unscaled data, predictions are wrong
  • ⚠️ Using MinMaxScaler with outliers — one outlier at 1,000,000 compresses all normal values near 0
  • ⚠️ Scaling tree-based models unnecessarily — adds complexity without benefit
  • ⚠️ Not using Pipeline — manual scaling is error-prone, especially with cross-validation

10 · Scalability Considerations

  • All scalers are O(n × p) — trivial computation
  • Pipeline adds no overhead — it's just organizational
  • For streaming data, use partial_fit() on scalers (incremental scaling)
  • Scaling is a one-time cost — negligible compared to model training

11 · Interpretability Considerations

  • ✅ After StandardScaler, coefficients are comparable: "feature A has 2x the effect of feature B"
  • ⚠️ Scaled values lose their original meaning: "0.73" doesn't mean anything to a stakeholder
  • ⚠️ Always keep the inverse transform available to convert back to original units
  • ✅ Pipeline makes the entire workflow reproducible and auditable

12 · Complete Working Code

import numpy as np
from sklearn.preprocessing import StandardScaler, MinMaxScaler, RobustScaler
from sklearn.pipeline import Pipeline
from sklearn.neighbors import KNeighborsClassifier
from sklearn.linear_model import LogisticRegression
from sklearn.tree import DecisionTreeClassifier
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score

# ─── Step 1: Data with very different scales ───
np.random.seed(42)
n = 300

age = np.random.uniform(20, 70, n)
income = np.random.uniform(20000, 200000, n)
purchases = np.random.uniform(1, 50, n)

# Add some outliers to income
income[0:5] = [500000, 600000, 450000, 550000, 700000]

# Target: high-value customer
log_odds = -5 + 0.03 * age + 0.00002 * income + 0.08 * purchases
prob = 1 / (1 + np.exp(-log_odds))
target = (np.random.random(n) < prob).astype(int)

X = np.column_stack([age, income, purchases])
feature_names = ['age', 'income', 'purchases']

X_train, X_test, y_train, y_test = train_test_split(
    X, target, test_size=0.25, random_state=42, stratify=target)

print(f"=== Feature Ranges (unscaled) ===")
for i, name in enumerate(feature_names):
    print(f"  {name:>12}: min={X_train[:, i].min():.0f}, max={X_train[:, i].max():.0f}, "
          f"mean={X_train[:, i].mean():.0f}")

# ─── Step 2: Compare scaled vs unscaled for different models ───
models = {
    'KNN (k=5)': KNeighborsClassifier(n_neighbors=5),
    'Logistic Reg': LogisticRegression(max_iter=1000, random_state=42),
    'Decision Tree': DecisionTreeClassifier(max_depth=3, random_state=42),
}

scalers = {
    'No Scaling': None,
    'StandardScaler': StandardScaler(),
    'MinMaxScaler': MinMaxScaler(),
    'RobustScaler': RobustScaler(),
}

print(f"\n=== Scaling Impact on Model Accuracy ===")
header = f"{'Model':>15}" + "".join(f"{s:>16}" for s in scalers.keys())
print(header)
print("-" * len(header))

for model_name, model in models.items():
    row = f"{model_name:>15}"
    for scaler_name, scaler in scalers.items():
        if scaler is None:
            m = type(model)(**model.get_params())
            m.fit(X_train, y_train)
            acc = m.score(X_test, y_test)
        else:
            pipe = Pipeline([
                ('scaler', type(scaler)()),
                ('model', type(model)(**model.get_params()))
            ])
            pipe.fit(X_train, y_train)
            acc = pipe.score(X_test, y_test)
        row += f"{acc:>16.2%}"
    print(row)

# ─── Step 3: Show what each scaler does ───
print(f"\n=== Scaler Comparison (income feature) ===")
income_train = X_train[:, 1].reshape(-1, 1)

for name, scaler in [('Standard', StandardScaler()),
                      ('MinMax', MinMaxScaler()),
                      ('Robust', RobustScaler())]:
    scaled = scaler.fit_transform(income_train)
    print(f"  {name:>10}: min={scaled.min():.3f}, max={scaled.max():.3f}, "
          f"mean={scaled.mean():.3f}, std={scaled.std():.3f}")

# ─── Step 4: Data leakage demonstration ───
print(f"\n=== Data Leakage Demo ===")

# WRONG: scale on all data
scaler_wrong = StandardScaler()
X_all_scaled = scaler_wrong.fit_transform(X)  # fits on ALL data including test
X_tr_wrong = X_all_scaled[:len(X_train)]
X_te_wrong = X_all_scaled[len(X_train):]

# RIGHT: scale on train only
scaler_right = StandardScaler()
X_tr_right = scaler_right.fit_transform(X_train)
X_te_right = scaler_right.transform(X_test)

knn = KNeighborsClassifier(n_neighbors=5)
knn.fit(X_tr_wrong, y_train)
acc_wrong = knn.score(X_te_wrong, y_test)

knn2 = KNeighborsClassifier(n_neighbors=5)
knn2.fit(X_tr_right, y_train)
acc_right = knn2.score(X_te_right, y_test)

print(f"  Leaky scaling:  {acc_wrong:.2%} (optimistic!)")
print(f"  Proper scaling: {acc_right:.2%} (honest)")

# ─── Step 5: Pipeline — the right way ───
print(f"\n=== Pipeline Example ===")
pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('model', KNeighborsClassifier(n_neighbors=5))
])
pipe.fit(X_train, y_train)
print(f"  Pipeline accuracy: {pipe.score(X_test, y_test):.2%}")
print(f"  Pipeline steps: {[step[0] for step in pipe.steps]}")

13 · Key Takeaway + Bridge to Next Project

Key Takeaway

  • Distance-based models (KNN, SVM) and gradient-based models (logistic regression) NEED scaling
  • Tree-based models (Decision Tree, Random Forest) do NOT need scaling
  • StandardScaler is the default; RobustScaler for outliers; MinMaxScaler for bounded ranges
  • Always use Pipeline to prevent data leakage — fit scaler on train only
  • Scaling is a preprocessing step, not a modeling step — but it dramatically affects results
