📋 Phase 1 — Very Easy

Project 4: Classifying Spam vs Not-Spam Using One Threshold

Binary classification decision threshold confusion matrix accuracy

Difficulty: Very Easy  |  Builds on: Project 1

1 · Hypothetical Problem

  • You manage an email system for a small company
  • Users complain about spam flooding their inbox
  • You have data: each email has a "spam score" (0-100) computed from keyword frequency, and a label (spam=1, not-spam=0)
  • Goal → pick a threshold on the spam score to classify emails
  • In Project 1 we used a threshold on humidity — same idea, but now we measure performance with a confusion matrix
  • Accuracy alone (Project 1) isn't enough — we need to know WHAT KIND of mistakes the model makes

2 · How to Approach It

  1. Recall from Project 1: threshold model → if score > T → spam, else → not-spam
  2. But now we ask: what happens when we get it wrong?
    • Marking a real email as spam (false positive) → user misses important email
    • Letting spam through (false negative) → user gets annoyed
  3. These two types of errors have DIFFERENT costs — accuracy treats them equally
  4. The confusion matrix breaks down all four outcomes: TP, FP, TN, FN
  5. Changing the threshold shifts the balance between these errors

3 · How Can We Solve It

  1. Generate synthetic email data (spam scores + labels)
  2. Train/test split (Project 1)
  3. Try multiple thresholds on training data
  4. For each threshold, compute the full confusion matrix
  5. Pick the threshold that balances false positives and false negatives
  6. Evaluate on test data with the confusion matrix

4 · What Should Do

  • ✅ Always look at the confusion matrix — not just accuracy
  • ✅ Understand the four cells: True Positive, False Positive, True Negative, False Negative
  • ✅ Think about the COST of each error type — in spam filtering, false positives are worse than false negatives
  • ✅ Try multiple thresholds and see how the confusion matrix changes
  • ✅ Report accuracy, but also precision and recall (even if informally at this stage)
  • ✅ Visualize the threshold's effect on the confusion matrix

5 · What Should NOT Do

  • ❌ Do NOT rely on accuracy alone — 95% accuracy can mean you're missing all the spam
  • ❌ Do NOT pick the threshold that maximizes accuracy without considering error types
  • ❌ Do NOT treat false positives and false negatives as equally bad — they rarely are in practice
  • ❌ Do NOT forget that the threshold is a CHOICE — different business needs → different thresholds
  • ❌ Do NOT confuse the confusion matrix rows and columns — always label them clearly

6 · When to Use What

  • 🔹 Confusion matrix → for ANY binary classification problem — it's the foundation
  • 🔹 Accuracy → only when classes are balanced AND both error types are equally costly
  • 🔹 Threshold tuning → when you have a continuous score and need to decide a cutoff
  • 🔹 Precision focus → when false positives are expensive (spam filter, medical screening)
  • 🔹 Recall focus → when false negatives are expensive (disease detection, fraud detection)

7 · When NOT to Use

  • 🚫 Accuracy is misleading with imbalanced classes — if 99% of emails are not-spam, always predicting "not-spam" gives 99% accuracy
  • 🚫 A single threshold fails when the score distributions overlap heavily — need probabilistic models (Project 6)
  • 🚫 Confusion matrix alone doesn't tell you how to IMPROVE — it diagnoses, but doesn't prescribe
  • 🚫 For multi-class problems (more than 2 categories), the 2×2 confusion matrix extends to N×N (Project 7)

8 · How It Only Works — Internal Mechanics

The Confusion Matrix — Four Outcomes

  • True Positive (TP): predicted spam, actually spam → correct catch
  • False Positive (FP): predicted spam, actually not-spam → wrongly blocked
  • True Negative (TN): predicted not-spam, actually not-spam → correct pass
  • False Negative (FN): predicted not-spam, actually spam → missed spam

Derived Metrics

  • Accuracy = (TP + TN) / (TP + FP + TN + FN) → overall correctness
  • Precision = TP / (TP + FP) → "of all emails I flagged as spam, how many actually were?"
  • Recall = TP / (TP + FN) → "of all actual spam, how many did I catch?"
  • Precision and recall are in tension — raising the threshold increases precision but decreases recall

Threshold as a Knob

  • Low threshold → flag more emails as spam → high recall, low precision (catch all spam, but block some real emails)
  • High threshold → flag fewer emails as spam → low recall, high precision (miss some spam, but never block real emails)
  • The "right" threshold depends on the business cost of each error type

9 · Common Pitfalls and Failure Modes

  • ⚠️ Accuracy paradox — high accuracy with imbalanced data is meaningless
  • ⚠️ Swapping rows/columns in confusion matrix — always use a consistent convention (sklearn: rows=actual, columns=predicted)
  • ⚠️ Optimizing for accuracy when you should optimize for precision or recall
  • ⚠️ Forgetting that threshold choice is a business decision, not a purely statistical one
  • ⚠️ Not reporting the threshold used — results are meaningless without knowing the threshold

10 · Scalability Considerations

  • Threshold comparison is O(n × t) — same as Project 1, scales easily
  • Confusion matrix computation is O(n) — trivial
  • With millions of emails, the threshold search is still fast
  • The real scalability challenge comes from computing the spam score itself (not covered here)
  • Confusion matrix concepts scale to any classifier in later projects

11 · Interpretability Considerations

  • ✅ Confusion matrix is highly interpretable — anyone can understand "we caught 90 out of 100 spam emails"
  • ✅ Precision and recall translate directly to business language
  • ✅ The threshold is a single number that can be explained and adjusted
  • ✅ Stakeholders can make informed decisions: "do we want to catch more spam at the risk of blocking real emails?"
  • ⚠️ The confusion matrix shows WHAT happened, not WHY — for that, you need feature analysis (later projects)

12 · Complete Working Code

import numpy as np
from sklearn.metrics import confusion_matrix, accuracy_score

# ─── Step 1: Synthetic email data ───
np.random.seed(42)
n_emails = 200

# Spam emails tend to have higher spam scores
spam_scores_spam = np.random.normal(65, 15, size=80)      # 80 spam emails
spam_scores_legit = np.random.normal(35, 15, size=120)     # 120 legit emails

scores = np.concatenate([spam_scores_spam, spam_scores_legit])
scores = np.clip(scores, 0, 100)  # keep in 0-100 range
labels = np.concatenate([np.ones(80), np.zeros(120)])

# Shuffle
shuffle_idx = np.random.permutation(n_emails)
scores = scores[shuffle_idx]
labels = labels[shuffle_idx]

print("=== Data Summary ===")
print(f"  Total emails: {n_emails}")
print(f"  Spam: {int(labels.sum())} | Legit: {int((1-labels).sum())}")
print(f"  Spam score range: {scores.min():.1f} to {scores.max():.1f}")

# ─── Step 2: Train/Test Split ───
X_train, y_train = scores[:140], labels[:140]
X_test, y_test = scores[140:], labels[140:]
print(f"\nTrain: {len(X_train)} | Test: {len(X_test)}")

# ─── Step 3: Threshold Search with Confusion Matrix ───
thresholds = np.arange(30, 75, 5)
print(f"\n=== Threshold Search ===")
print(f"{'Threshold':>10} {'Accuracy':>10} {'TP':>5} {'FP':>5} {'TN':>5} {'FN':>5} {'Precision':>10} {'Recall':>10}")
print("-" * 70)

best_threshold = None
best_f1 = 0

for t in thresholds:
    preds = (X_train > t).astype(int)
    tn, fp, fn, tp = confusion_matrix(y_train, preds).ravel()
    acc = accuracy_score(y_train, preds)
    precision = tp / (tp + fp) if (tp + fp) > 0 else 0
    recall = tp / (tp + fn) if (tp + fn) > 0 else 0
    f1 = 2 * precision * recall / (precision + recall) if (precision + recall) > 0 else 0

    print(f"{t:>10.0f} {acc:>10.2%} {tp:>5} {fp:>5} {tn:>5} {fn:>5} {precision:>10.2%} {recall:>10.2%}")

    if f1 > best_f1:
        best_f1 = f1
        best_threshold = t

print(f"\n✅ Best threshold (by F1): {best_threshold:.0f}")

# ─── Step 4: Evaluate on Test Set ───
test_preds = (X_test > best_threshold).astype(int)
tn, fp, fn, tp = confusion_matrix(y_test, test_preds).ravel()
test_acc = accuracy_score(y_test, test_preds)
test_precision = tp / (tp + fp) if (tp + fp) > 0 else 0
test_recall = tp / (tp + fn) if (tp + fn) > 0 else 0

print(f"\n=== Test Set Results (threshold={best_threshold:.0f}) ===")
print(f"  Accuracy:  {test_acc:.2%}")
print(f"  Precision: {test_precision:.2%}")
print(f"  Recall:    {test_recall:.2%}")
print(f"\n  Confusion Matrix:")
print(f"                  Predicted Not-Spam  Predicted Spam")
print(f"  Actual Not-Spam       {tn:>5}            {fp:>5}")
print(f"  Actual Spam           {fn:>5}            {tp:>5}")

# ─── Step 5: Interpret the errors ───
print(f"\n=== Error Interpretation ===")
print(f"  ✓ True Positives  (caught spam):       {tp}")
print(f"  ✗ False Positives (blocked real email): {fp}  ← users miss emails!")
print(f"  ✓ True Negatives  (passed real email):  {tn}")
print(f"  ✗ False Negatives (missed spam):        {fn}  ← spam gets through!")

# ─── Step 6: Show threshold tradeoff ───
print(f"\n=== Threshold Tradeoff Demo ===")
for t in [30, 50, 70]:
    p = (X_test > t).astype(int)
    tn_t, fp_t, fn_t, tp_t = confusion_matrix(y_test, p).ravel()
    prec = tp_t / (tp_t + fp_t) if (tp_t + fp_t) > 0 else 0
    rec = tp_t / (tp_t + fn_t) if (tp_t + fn_t) > 0 else 0
    print(f"  Threshold={t}: Precision={prec:.2%}, Recall={rec:.2%}, "
          f"Blocked legit={fp_t}, Missed spam={fn_t}")
