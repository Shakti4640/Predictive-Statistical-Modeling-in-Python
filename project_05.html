Project 5: Estimating Ice Cream Sales From Temperature

Correlation Pearson r R² score residuals underfitting

Difficulty: Very Easy  |  Builds on: Projects 2, 3

1 · Hypothetical Problem

  • You run a small ice cream shop and want to predict daily sales based on temperature
  • You have 60 days of data: daily high temperature (°F) and number of ice cream units sold
  • Before building a model, you want to know: how strongly are temperature and sales related?
  • In Projects 2-3 we jumped straight to fitting a line — now we step back and measure the relationship FIRST
  • Goal → learn correlation (Pearson r), R² score, and what "underfitting" looks like

2 · How to Approach It

  1. Before modeling, compute the Pearson correlation coefficient (r) between temperature and sales
  2. r tells you the strength and direction of the LINEAR relationship
    • r = +1 → perfect positive linear relationship
    • r = 0 → no linear relationship
    • r = -1 → perfect negative linear relationship
  3. Fit a linear regression (Project 2 skill) and compute R²
  4. R² = proportion of variance in sales explained by temperature
  5. Analyze residuals (Project 3 skill) to check if the linear model is sufficient
  6. Introduce the concept of underfitting: when the model is too simple for the data

3 · How Can We Solve It

  1. Generate synthetic ice cream data with a slightly nonlinear relationship
  2. Compute Pearson r manually and with numpy
  3. Fit linear regression and get R²
  4. Plot residuals — look for the curve that signals underfitting
  5. Compare R² with the residual pattern — high R² doesn't always mean good fit

4 · What Should Do

  • ✅ Always compute correlation BEFORE building a model — it's a sanity check
  • ✅ Understand that r measures LINEAR association only — a curved relationship can have r ≈ 0
  • ✅ Use R² to quantify how much variance the model explains (0% to 100%)
  • ✅ Plot the scatter plot first — your eyes catch patterns that numbers miss
  • ✅ Check residuals even when R² looks good — residual patterns reveal model inadequacy
  • ✅ Recognize underfitting: the model is too simple, residuals show a systematic pattern

5 · What Should NOT Do

  • ❌ Do NOT assume high correlation means causation — ice cream sales and drowning deaths are correlated (both driven by summer)
  • ❌ Do NOT use Pearson r for nonlinear relationships — it only measures linear association
  • ❌ Do NOT treat R² as the only quality metric — always pair it with residual analysis (Project 3)
  • ❌ Do NOT ignore underfitting — a model that's too simple leaves predictable patterns in the residuals
  • ❌ Do NOT confuse r and R² — for simple regression R² = r², but this breaks for multiple regression (Project 9)

6 · When to Use What

  • 🔹 Pearson r → quick check of linear association between two continuous variables
  • 🔹 R² score → after fitting a regression model, to quantify explained variance
  • 🔹 Scatter plot → always first — before computing any statistic
  • 🔹 Residual plot → after fitting, to check if the model form is appropriate (Project 3)
  • 🔹 Spearman rank correlation → when the relationship is monotonic but not linear (mentioned here, used later)

7 · When NOT to Use

  • 🚫 Pearson r is meaningless for U-shaped or circular relationships
  • 🚫 R² can be misleadingly high even when the model is wrong (e.g., fitting a line to a curve that happens to trend upward)
  • 🚫 Correlation doesn't work well with categorical variables — need different measures
  • 🚫 A single outlier can dramatically change r — always check for outliers visually
  • 🚫 R² doesn't tell you if predictions are accurate in absolute terms — only relative to the variance

8 · How It Only Works — Internal Mechanics

Pearson Correlation Coefficient (r)

  • Formula: r = Σ((xᵢ - x̄)(yᵢ - ȳ)) / √(Σ(xᵢ - x̄)² × Σ(yᵢ - ȳ)²)
  • Numerator = covariance (how x and y move together)
  • Denominator = product of standard deviations (normalizes to [-1, +1])
  • r is dimensionless — doesn't depend on the units of x or y

R² — Coefficient of Determination

  • R² = 1 - (SS_res / SS_tot)
  • SS_res = Σ(yᵢ - ŷᵢ)² → residual sum of squares (error left after modeling)
  • SS_tot = Σ(yᵢ - ȳ)² → total sum of squares (total variance in y)
  • R² = 0.85 means the model explains 85% of the variance in y
  • For simple linear regression: R² = r² (the square of Pearson r)

Underfitting — The Model Is Too Simple

  • If the true relationship is curved but you fit a straight line → underfitting
  • Symptoms: residuals show a systematic pattern (e.g., U-shape)
  • R² might still be decent (e.g., 0.75) but the model is missing structure
  • Fix: use a more flexible model (polynomial regression → Project 20)
  • Underfitting = high bias, low variance (Project 8 will formalize this)

9 · Common Pitfalls and Failure Modes

  • ⚠️ Anscombe's quartet — four datasets with identical r and R² but wildly different patterns → always plot
  • ⚠️ Outlier sensitivity — one extreme point can push r from 0.3 to 0.9
  • ⚠️ Confusing r with R² — r = 0.7 means R² = 0.49, so only 49% of variance is explained
  • ⚠️ Ignoring nonlinearity — a strong curved relationship can have r ≈ 0
  • ⚠️ Ecological fallacy — correlation at group level doesn't imply correlation at individual level

10 · Scalability Considerations

  • Pearson r computation is O(n) — handles millions of rows instantly
  • R² computation is O(n) after model fitting
  • For many features, computing pairwise correlations → O(n × p²) where p = number of features
  • Correlation matrix visualization (heatmap) becomes useful with many features (Project 9)
  • These concepts are lightweight — the real cost is in the model fitting, not the diagnostics

11 · Interpretability Considerations

  • ✅ r is intuitive: "temperature and sales have a strong positive relationship (r = 0.88)"
  • ✅ R² is intuitive: "temperature explains 77% of the variation in sales"
  • ✅ Both are single numbers — easy to communicate to stakeholders
  • ✅ Scatter plots make the relationship visible to anyone
  • ⚠️ r and R² don't tell you the DIRECTION of causation — only association
  • ⚠️ Underfitting is harder to explain to non-technical people — residual plots help

12 · Complete Working Code

import numpy as np
from sklearn.linear_model import LinearRegression
from sklearn.metrics import r2_score, mean_squared_error
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

# ─── Step 1: Synthetic ice cream data (slightly nonlinear) ───
np.random.seed(42)
temperature = np.random.uniform(55, 100, size=60)  # °F
# True relationship: slightly quadratic (sales accelerate at high temps)
sales = 0.05 * (temperature - 50) ** 2 + 2 * temperature - 80 + np.random.normal(0, 15, 60)
sales = np.maximum(sales, 0)  # no negative sales

print("=== Sample Data (first 8 days) ===")
for i in range(8):
    print(f"  Day {i+1}: Temp={temperature[i]:.1f}°F  Sales={sales[i]:.0f} units")

# ─── Step 2: Compute Pearson r ───
# Manual calculation
x_mean, y_mean = temperature.mean(), sales.mean()
numerator = np.sum((temperature - x_mean) * (sales - y_mean))
denominator = np.sqrt(np.sum((temperature - x_mean)**2) * np.sum((sales - y_mean)**2))
r_manual = numerator / denominator

# numpy shortcut
r_numpy = np.corrcoef(temperature, sales)[0, 1]

print(f"\n=== Pearson Correlation ===")
print(f"  r (manual): {r_manual:.4f}")
print(f"  r (numpy):  {r_numpy:.4f}")
print(f"  Interpretation: {'Strong' if abs(r_numpy) > 0.7 else 'Moderate' if abs(r_numpy) > 0.4 else 'Weak'} "
      f"{'positive' if r_numpy > 0 else 'negative'} linear relationship")

# ─── Step 3: Train/Test Split + Linear Regression ───
X_train, y_train = temperature[:42], sales[:42]
X_test, y_test = temperature[42:], sales[42:]

model = LinearRegression()
model.fit(X_train.reshape(-1, 1), y_train)

y_train_pred = model.predict(X_train.reshape(-1, 1))
y_test_pred = model.predict(X_test.reshape(-1, 1))

# ─── Step 4: R² Score ───
r2_train = r2_score(y_train, y_train_pred)
r2_test = r2_score(y_test, y_test_pred)
rmse_test = np.sqrt(mean_squared_error(y_test, y_test_pred))

print(f"\n=== Model Performance ===")
print(f"  R² (train): {r2_train:.4f} → model explains {r2_train*100:.1f}% of training variance")
print(f"  R² (test):  {r2_test:.4f} → model explains {r2_test*100:.1f}% of test variance")
print(f"  Test RMSE:  {rmse_test:.1f} units")
print(f"  r² check:   {r_numpy**2:.4f} (should ≈ R² for simple regression)")

# ─── Step 5: Residual Analysis — Detecting Underfitting ───
train_residuals = y_train - y_train_pred
test_residuals = y_test - y_test_pred

print(f"\n=== Residual Analysis ===")
print(f"  Mean residual (train): {np.mean(train_residuals):.2f} (should be ~0)")
print(f"  Std residual (train):  {np.std(train_residuals):.2f}")

# Check for pattern: sort by temperature and see if residuals trend
sorted_idx = np.argsort(X_train)
sorted_resid = train_residuals[sorted_idx]
first_third = sorted_resid[:14].mean()
middle_third = sorted_resid[14:28].mean()
last_third = sorted_resid[28:].mean()
print(f"\n  Residual means by temperature tercile:")
print(f"    Low temps:  {first_third:+.1f}")
print(f"    Mid temps:  {middle_third:+.1f}")
print(f"    High temps: {last_third:+.1f}")
print(f"  → {'Pattern detected! Model may be underfitting.' if abs(last_third - middle_third) > 5 else 'No strong pattern.'}")

# ─── Step 6: Visualization ───
fig, axes = plt.subplots(1, 3, figsize=(15, 5))

# Scatter + fit
ax1 = axes[0]
ax1.scatter(temperature, sales, color='#58a6ff', alpha=0.6)
x_line = np.linspace(50, 105, 100)
ax1.plot(x_line, model.predict(x_line.reshape(-1, 1)), color='#3fb950', linewidth=2)
ax1.set_xlabel('Temperature (°F)')
ax1.set_ylabel('Ice Cream Sales')
ax1.set_title(f'Linear Fit (R²={r2_train:.3f}, r={r_numpy:.3f})')
ax1.grid(True, alpha=0.2)

# Residuals vs predicted
ax2 = axes[1]
ax2.scatter(y_train_pred, train_residuals, color='#58a6ff', alpha=0.6)
ax2.axhline(y=0, color='#f85149', linestyle='--')
ax2.set_xlabel('Predicted Sales')
ax2.set_ylabel('Residual')
ax2.set_title('Residuals — Look for Patterns!')
ax2.grid(True, alpha=0.2)

# Predicted vs Actual
ax3 = axes[2]
ax3.scatter(y_test_pred, y_test, color='#f0883e', alpha=0.7)
lims = [min(y_test.min(), y_test_pred.min()), max(y_test.max(), y_test_pred.max())]
ax3.plot(lims, lims, color='#3fb950', linestyle='--', linewidth=2)
ax3.set_xlabel('Predicted')
ax3.set_ylabel('Actual')
ax3.set_title('Predicted vs Actual')
ax3.grid(True, alpha=0.2)

plt.tight_layout()
plt.savefig('project_05_correlation.png', dpi=100, bbox_inches='tight')
print("\n✅ Plots saved as project_05_correlation.png")

Key Takeaway

  • Pearson r measures the strength and direction of LINEAR association (-1 to +1)
  • R² tells you what fraction of the target's variance the model explains
  • Always plot before computing — Anscombe's quartet proves numbers can lie
  • Underfitting = model too simple → systematic residual patterns
  • Correlation ≠ causation — the most important statistical lesson
