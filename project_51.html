Project 51: Conformal Prediction — Guaranteed Coverage Intervals on Any Model

Conformal Prediction Coverage Guarantee Nonconformity Score MAPIE

Difficulty: Very Hard  |  Builds on: Projects 33, 38, 46

1 · Hypothetical Problem

  • A pharmaceutical company develops a machine learning model to predict optimal drug dosage (mg) based on patient weight, age, kidney function, and genetic markers
  • Regulators require not just a point prediction but a prediction interval with guaranteed 95% coverage — if the model says "50–70 mg", at least 95% of true dosages must fall within that range
  • The team tried Bayesian linear regression (Project 38) — but it assumes Gaussian residuals, which is wrong for this skewed dosage distribution
  • They tried Gaussian Process regression (Project 46) — but GP assumes a specific kernel structure and scales poorly to their 100K patient dataset
  • Both methods produce intervals that claim 95% coverage but actually cover only 88–91% because the distributional assumptions are violated
  • Goal → produce prediction intervals that provably contain the true value at least 95% of the time, regardless of the model or data distribution

2 · How to Approach It

  1. From Project 33 (Cross-Validation): we learned to hold out data for honest evaluation — conformal prediction uses a calibration set the same way
  2. From Project 38 (Bayesian Regression): we saw that parametric intervals depend on distributional assumptions — conformal prediction is distribution-free
  3. From Project 46 (Gaussian Processes): we learned about uncertainty quantification — but GP intervals assume Gaussian posteriors
  • The key insight: split your data into training and calibration sets. Train any model on the training set. Compute residuals on the calibration set. Use the distribution of residuals to build intervals
  • The coverage guarantee comes from a simple statistical fact: the quantile of calibration residuals bounds future residuals with known probability

    • 3 · How Can We Solve It

    1. Generate a synthetic drug dosage dataset with non-Gaussian, heteroscedastic noise (violating Bayesian assumptions)
    2. Implement split conformal prediction from scratch — train model, compute calibration residuals, derive interval width
    3. Wrap a Random Forest with conformal intervals — show that coverage holds even for a non-parametric model
    4. Implement conformalized quantile regression for adaptive-width intervals (narrower where the model is confident)
    5. Compare conformal intervals with Bayesian credible intervals — show conformal achieves target coverage while Bayesian may not
    6. Verify the coverage guarantee empirically across multiple random splits

    4 · What Should Do

    • ✅ Always use a held-out calibration set that the model has never seen — this is what provides the guarantee
    • ✅ Choose the nonconformity score carefully — absolute residual |y - ŷ| is simplest, but normalized residuals adapt to heteroscedasticity
    • ✅ Use the corrected quantile: ceil((n_cal + 1) × (1 - α)) / n_cal — the +1 ensures finite-sample validity
    • ✅ Report both marginal coverage (overall) and conditional coverage (per subgroup) — marginal is guaranteed, conditional is not
    • ✅ Combine conformal prediction with any model — Random Forest, gradient boosting, neural networks all work

    5 · What Should NOT Do

    • ❌ Do NOT use the training set for calibration — this violates exchangeability and destroys the coverage guarantee
    • ❌ Do NOT assume conformal intervals are optimal in width — they guarantee coverage but may be wider than necessary
    • ❌ Do NOT confuse marginal and conditional coverage — conformal guarantees marginal (average over all x), not conditional (for each specific x)
    • ❌ Do NOT use too small a calibration set — fewer than 100 calibration points leads to very loose intervals
    • ❌ Do NOT ignore the exchangeability assumption — if data is time-ordered or has distribution shift, standard conformal breaks

    6 · When to Use What

    • 🔹 Split Conformal Prediction → when you need guaranteed coverage with any black-box model, simplest to implement
    • 🔹 Conformalized Quantile Regression → when you want adaptive-width intervals (narrower in easy regions, wider in hard regions)
    • 🔹 Bayesian Credible Intervals (Project 38) → when distributional assumptions hold and you want posterior interpretation
    • 🔹 Gaussian Process (Project 46) → small datasets where kernel assumptions are reasonable and you want smooth uncertainty
    • 🔹 Bootstrap Prediction Intervals → quick approximation, but no formal coverage guarantee

    7 · When NOT to Use

    • 🚫 When data is non-exchangeable (e.g., time series with trend) — standard conformal prediction assumes exchangeability
    • 🚫 When you need tight conditional coverage for specific subgroups — conformal only guarantees marginal coverage
    • 🚫 When calibration data is extremely scarce (< 30 points) — intervals become too wide to be useful
    • 🚫 When you need the full predictive distribution, not just intervals — conformal gives intervals, not densities

    8 · How It Only Works — Internal Mechanics

    Split Conformal Prediction

    • Split data into three parts: training set, calibration set, test set
    • Train model f̂ on training set only
    • Compute nonconformity scores on calibration set: sᵢ = |yᵢ - f̂(xᵢ)| for i = 1, ..., n_cal
    • Find the quantile q̂ = Quantile(s₁, ..., s_n_cal, level = ceil((n_cal + 1)(1 - α)) / n_cal)
    • Prediction interval for new x: [f̂(x) - q̂, f̂(x) + q̂]

    Why the Guarantee Holds

    • Under exchangeability, the calibration scores and the new test score are equally likely to be in any rank order
    • P(s_new ≤ q̂) ≥ 1 - α by construction — the quantile of n exchangeable values bounds the (n+1)-th
    • This is a finite-sample guarantee — it holds for any n_cal, any model, any distribution
    • No asymptotics needed — unlike Bayesian intervals which rely on posterior convergence

    Conformalized Quantile Regression (CQR)

    • Train two quantile regressors: q̂_lo(x) targeting α/2 and q̂_hi(x) targeting 1 - α/2
    • Nonconformity score: sᵢ = max(q̂_lo(xᵢ) - yᵢ, yᵢ - q̂_hi(xᵢ)) — how much the true value exceeds the predicted interval
    • Calibrate: find Q = quantile of calibration scores at level (1 - α)(1 + 1/n_cal)
    • Final interval: [q̂_lo(x) - Q, q̂_hi(x) + Q] — adapts width to local difficulty

    Comparison with Bayesian Intervals

    • Bayesian: P(y ∈ CI | data, model) ≥ 1 - α — conditional on the model being correct
    • Conformal: P(y ∈ CI) ≥ 1 - α — unconditional, holds even if the model is wrong
    • Bayesian intervals can be too narrow when assumptions are violated → under-coverage
    • Conformal intervals are always valid but may be wider than necessary → conservative but safe

    9 · Common Pitfalls and Failure Modes

    • ⚠️ Calibration on training data — using training residuals gives optimistically narrow intervals; always use held-out calibration data
    • ⚠️ Too small calibration set — with n_cal = 20, the 95th percentile is just the maximum residual; need n_cal ≥ 100 for useful intervals
    • ⚠️ Distribution shift — if test distribution differs from calibration, exchangeability breaks and coverage is not guaranteed
    • ⚠️ Confusing coverage levels — 95% marginal coverage means 95% overall, but some subgroups may have 80% and others 100%
    • ⚠️ Constant-width intervals — basic split conformal gives the same width everywhere; use CQR for adaptive widths

    10 · Scalability Considerations

    • Split conformal is extremely efficient — O(n_cal log n_cal) for sorting residuals, then O(1) per prediction
    • No retraining needed — once calibrated, intervals are just ŷ ± q̂, same cost as a point prediction
    • Calibration set size trades off with training set size — larger calibration gives tighter intervals but less training data
    • For very large datasets (>1M): use a small calibration fraction (5–10%) — even 50K calibration points give very precise quantiles
    • CQR requires training two quantile models — doubles training cost but still O(1) at prediction time

    11 · Interpretability Considerations

    • ✅ Interval width is directly interpretable — wider intervals mean more uncertainty for that prediction
    • ✅ Coverage guarantee is easy to explain to stakeholders — "95% of true values will fall in our interval"
    • ✅ CQR intervals vary by input — can show which patients have more dosage uncertainty
    • ⚠️ Conformal intervals don't explain why uncertainty is high — combine with SHAP (Project 27) for feature-level explanations
    • ⚠️ Marginal vs conditional coverage distinction is subtle — stakeholders may expect per-patient guarantees that conformal doesn't provide

# ─── Step 3: Implement split conformal prediction from scratch ───
class SplitConformalPredictor:
    """Split conformal prediction — wraps any point predictor with
    guaranteed coverage intervals."""

    def __init__(self, model, alpha=0.05):
        self.model = model
        self.alpha = alpha
        self.q_hat = None
        self.cal_scores = None

    def fit(self, X_train, y_train):
        """Train the underlying model."""
        self.model.fit(X_train, y_train)
        return self

    def calibrate(self, X_cal, y_cal):
        """Compute nonconformity scores on calibration set."""
        y_pred_cal = self.model.predict(X_cal)
        self.cal_scores = np.abs(y_cal - y_pred_cal)

        n = len(self.cal_scores)
        # Corrected quantile level for finite-sample validity
        level = np.ceil((n + 1) * (1 - self.alpha)) / n
        level = min(level, 1.0)
        self.q_hat = np.quantile(self.cal_scores, level)

        print(f"  Calibration scores: mean={self.cal_scores.mean():.2f}, "
              f"median={np.median(self.cal_scores):.2f}, "
              f"q_hat={self.q_hat:.2f}")
        return self

    def predict(self, X):
        """Return point prediction and prediction interval."""
        y_pred = self.model.predict(X)
        lower = y_pred - self.q_hat
        upper = y_pred + self.q_hat
        return y_pred, lower, upper

    def evaluate_coverage(self, X_test, y_test):
        """Check empirical coverage on test set."""
        y_pred, lower, upper = self.predict(X_test)
        covered = (y_test >= lower) & (y_test <= upper)
        coverage = covered.mean()
        avg_width = (upper - lower).mean()
        return coverage, avg_width

# ─── Step 6: Conformalized Quantile Regression (CQR) ───
print("\n=== Conformalized Quantile Regression (Adaptive Widths) ===")

class CQRPredictor:
    """Conformalized Quantile Regression — adaptive-width intervals."""

    def __init__(self, alpha=0.05):
        self.alpha = alpha
        self.model_lo = None
        self.model_hi = None
        self.Q = None

    def fit(self, X_train, y_train):
        """Train lower and upper quantile regressors using GBR."""
        self.model_lo = GradientBoostingRegressor(
            n_estimators=100, max_depth=4, loss='quantile',
            alpha=self.alpha / 2, random_state=42
        )
        self.model_hi = GradientBoostingRegressor(
            n_estimators=100, max_depth=4, loss='quantile',
            alpha=1 - self.alpha / 2, random_state=42
        )
        self.model_lo.fit(X_train, y_train)
        self.model_hi.fit(X_train, y_train)
        return self

    def calibrate(self, X_cal, y_cal):
        """Compute conformity scores and calibration quantile."""
        lo_cal = self.model_lo.predict(X_cal)
        hi_cal = self.model_hi.predict(X_cal)

        scores = np.maximum(lo_cal - y_cal, y_cal - hi_cal)
        n = len(scores)
        level = np.ceil((n + 1) * (1 - self.alpha)) / n
        level = min(level, 1.0)
        self.Q = np.quantile(scores, level)

        print(f"  CQR calibration: Q={self.Q:.2f}")
        return self

    def predict(self, X):
        """Return adaptive prediction intervals."""
        lo = self.model_lo.predict(X) - self.Q
        hi = self.model_hi.predict(X) + self.Q
        point = (lo + hi) / 2
        return point, lo, hi

    def evaluate_coverage(self, X_test, y_test):
        _, lo, hi = self.predict(X_test)
        covered = (y_test >= lo) & (y_test <= hi)
        widths = hi - lo
        return covered.mean(), widths.mean(), widths

    13 · Key Takeaway + Bridge to Next Project

    Key Takeaway

    • Conformal prediction provides distribution-free prediction intervals with finite-sample coverage guarantees — no assumptions about the data or model
    • Split conformal is dead simple: train model, compute calibration residuals, take a quantile — that's the interval width
    • The coverage guarantee holds for any model (linear, tree, neural net) and any data distribution (skewed, heavy-tailed, heteroscedastic)
    • Conformalized Quantile Regression (CQR) gives adaptive-width intervals — narrower where the model is confident, wider where it's uncertain
    • Bayesian intervals can under-cover when distributional assumptions are violated — conformal intervals never do
