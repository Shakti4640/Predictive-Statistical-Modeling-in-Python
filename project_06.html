Project 6: Will a Customer Buy? Logistic Regression From Scratch

Difficulty: Very Easy  |  Builds on: Projects 1, 4

1 · Hypothetical Problem

  • You work at an e-commerce company
  • Marketing asks: "Given how long a customer browses, what's the PROBABILITY they'll buy?"
  • Not just "buy or not" (Project 4) — they want a probability: "this customer has a 73% chance of buying"
  • You have data: browsing time (minutes) and whether the customer purchased (1) or not (0)
  • Goal → build a model that outputs probabilities between 0 and 1
  • Linear regression (Project 2) can't do this — it outputs any number, including negatives and values above 1

2 · How to Approach It

  1. The problem with linear regression for classification:
    • y = slope × time + intercept can give y = -0.3 or y = 1.5 — not valid probabilities
  2. Solution: wrap the linear output in a sigmoid function
    • Sigmoid squashes any number into the range (0, 1)
    • σ(z) = 1 / (1 + e^(-z))
  3. The model becomes: P(buy) = sigmoid(slope × time + intercept)
  4. This is Logistic Regression — the most fundamental classification algorithm
  5. We can't use MSE as the cost function (Project 3) — we need binary cross-entropy

3 · How Can We Solve It

  1. Generate synthetic browsing data (time → purchase label)
  2. Implement the sigmoid function from scratch
  3. Understand why MSE doesn't work for classification → introduce log loss
  4. Use sklearn's LogisticRegression to fit the model
  5. Extract probabilities with predict_proba()
  6. Apply a threshold (Project 4 concept) to convert probabilities to predictions
  7. Evaluate with confusion matrix (Project 4) and accuracy

4 · What Should Do

  • ✅ Understand the sigmoid function — it's the bridge between linear models and probabilities
  • ✅ Use predict_proba() to get probabilities, not just predict()
  • ✅ Remember that the 0.5 threshold is a DEFAULT — you can change it (Project 4 lesson)
  • ✅ Use log loss (binary cross-entropy) as the cost function — not MSE
  • ✅ Interpret coefficients as log-odds: "each extra minute of browsing increases the log-odds of buying by X"
  • ✅ Combine with confusion matrix analysis (Project 4) for full evaluation

5 · What Should NOT Do

  • ❌ Do NOT use linear regression for binary classification — predictions go outside [0, 1]
  • ❌ Do NOT use MSE for classification — it creates a non-convex optimization landscape
  • ❌ Do NOT interpret logistic regression coefficients as "change in probability" — they're change in LOG-ODDS
  • ❌ Do NOT assume 0.5 is always the right threshold — it depends on the problem (Project 4)
  • ❌ Do NOT forget that logistic regression assumes a linear decision boundary — it can't capture complex patterns

6 · When to Use What

  • 🔹 Logistic Regression → binary classification with probability output, interpretable coefficients
  • 🔹 Sigmoid function → whenever you need to squash a real number into (0, 1)
  • 🔹 Log loss / cross-entropy → the standard loss function for classification
  • 🔹 predict_proba() → when you need probabilities, not just class labels
  • 🔹 Threshold tuning → after getting probabilities, choose cutoff based on business needs (Project 4)

7 · When NOT to Use

  • 🚫 Logistic regression fails when the decision boundary is nonlinear (e.g., XOR pattern)
  • 🚫 Fails with highly correlated features — coefficients become unstable (addressed in Project 17)
  • 🚫 Not suitable for multi-class problems without modification (one-vs-rest or softmax)
  • 🚫 Assumes features contribute independently to the log-odds — no interaction effects by default
  • 🚫 Sensitive to outliers in the feature space — extreme values can dominate the fit

8 · How It Only Works — Internal Mechanics

The Sigmoid Function

  • σ(z) = 1 / (1 + e^(-z))
  • When z = 0 → σ = 0.5 (the decision boundary)
  • When z → +∞ → σ → 1
  • When z → -∞ → σ → 0
  • The S-curve smoothly maps any real number to (0, 1)

Log-Odds (Logit)

  • Logistic regression models the LOG-ODDS: log(p / (1-p)) = w₁x + w₀
  • Odds = p / (1-p) → if p = 0.75, odds = 3:1
  • Log-odds can be any real number → that's what the linear part produces
  • Sigmoid converts log-odds back to probability

Binary Cross-Entropy (Log Loss)

  • Loss = -[y·log(p) + (1-y)·log(1-p)]
  • When y=1: loss = -log(p) → penalizes low probability for positive class
  • When y=0: loss = -log(1-p) → penalizes high probability for negative class
  • This is convex → gradient descent finds the global minimum
  • MSE for classification is NOT convex → can get stuck in local minima

Why Not MSE for Classification?

  • MSE + sigmoid creates a non-convex loss surface with multiple local minima
  • Log loss + sigmoid creates a convex loss surface → guaranteed to find the best solution
  • Log loss also has a probabilistic interpretation (maximum likelihood estimation)

9 · Common Pitfalls and Failure Modes

  • ⚠️ Using linear regression for classification — the most common beginner mistake
  • ⚠️ Interpreting coefficients as probability changes — they're log-odds changes
  • ⚠️ Forgetting to reshape X for sklearn — same issue as Project 2
  • ⚠️ Not checking class balance — logistic regression can be biased toward the majority class
  • ⚠️ Ignoring the probability output — using only predict() when predict_proba() gives richer information
  • ⚠️ Perfect separation — when features perfectly separate classes, coefficients explode to infinity

10 · Scalability Considerations

  • Logistic regression uses iterative optimization (not closed-form like OLS in Project 2)
  • sklearn uses L-BFGS or liblinear solver — efficient for up to ~100K samples
  • For millions of rows → use SGDClassifier with log loss (stochastic gradient descent)
  • Memory: same as linear regression — stores only the coefficients
  • Prediction is O(1) per sample — just a dot product + sigmoid
  • One of the most scalable classification algorithms

11 · Interpretability Considerations

  • ✅ Coefficients have meaning: "each extra minute increases log-odds of buying by 0.3"
  • ✅ Can convert to odds ratio: e^(coefficient) → "each extra minute multiplies the odds by 1.35"
  • ✅ Probability output is intuitive: "73% chance of buying"
  • ✅ One of the most interpretable classification models
  • ⚠️ Log-odds are less intuitive than raw probabilities — need to convert for stakeholders
  • ⚠️ Interpretability decreases with many features (Project 10 will address this)

12 · Complete Working Code

import numpy as np
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import confusion_matrix, accuracy_score, log_loss

# ─── Step 1: Sigmoid function from scratch ───
def sigmoid(z):
    return 1 / (1 + np.exp(-z))

# Demonstrate sigmoid behavior
print("=== Sigmoid Function Demo ===")
for z in [-5, -2, 0, 2, 5]:
    print(f"  sigmoid({z:+d}) = {sigmoid(z):.4f}")

# ─── Step 2: Synthetic customer data ───
np.random.seed(42)
n = 150
browsing_time = np.random.uniform(1, 30, size=n)  # minutes

# True relationship: longer browsing → higher chance of buying
# Logistic: P(buy) = sigmoid(0.3 * time - 5)
true_probs = sigmoid(0.3 * browsing_time - 5)
purchased = (np.random.random(n) < true_probs).astype(int)

print(f"\n=== Data Summary ===")
print(f"  Customers: {n}")
print(f"  Purchased: {purchased.sum()} ({purchased.mean():.1%})")
print(f"  Avg browsing (buyers):     {browsing_time[purchased==1].mean():.1f} min")
print(f"  Avg browsing (non-buyers): {browsing_time[purchased==0].mean():.1f} min")

# ─── Step 3: Train/Test Split ───
X_train, y_train = browsing_time[:105], purchased[:105]
X_test, y_test = browsing_time[105:], purchased[105:]

# ─── Step 4: Why NOT linear regression? ───
from sklearn.linear_model import LinearRegression
bad_model = LinearRegression()
bad_model.fit(X_train.reshape(-1, 1), y_train)
bad_preds = bad_model.predict(X_test.reshape(-1, 1))
print(f"\n=== Why Linear Regression Fails ===")
print(f"  Min prediction: {bad_preds.min():.3f}  (should be ≥ 0)")
print(f"  Max prediction: {bad_preds.max():.3f}  (should be ≤ 1)")
print(f"  Predictions outside [0,1]: {((bad_preds < 0) | (bad_preds > 1)).sum()}")

# ─── Step 5: Logistic Regression ───
model = LogisticRegression(random_state=42)
model.fit(X_train.reshape(-1, 1), y_train)

print(f"\n=== Logistic Regression ===")
print(f"  Coefficient (w1): {model.coef_[0][0]:.4f}")
print(f"  Intercept (w0):   {model.intercept_[0]:.4f}")
print(f"  Model: log-odds = {model.coef_[0][0]:.3f} × time + ({model.intercept_[0]:.3f})")
print(f"  Odds ratio: e^{model.coef_[0][0]:.3f} = {np.exp(model.coef_[0][0]):.3f}")
print(f"  → Each extra minute multiplies buying odds by {np.exp(model.coef_[0][0]):.2f}x")

# ─── Step 6: Probability Predictions ───
probs = model.predict_proba(X_test.reshape(-1, 1))[:, 1]  # P(buy)
preds = model.predict(X_test.reshape(-1, 1))

print(f"\n=== Sample Predictions ===")
for i in range(10):
    print(f"  Customer: {X_test[i]:.1f} min → P(buy)={probs[i]:.2%} → "
          f"Predict={'Buy' if preds[i]==1 else 'No'} | Actual={'Buy' if y_test[i]==1 else 'No'} "
          f"{'✓' if preds[i]==y_test[i] else '✗'}")

# ─── Step 7: Evaluation ───
acc = accuracy_score(y_test, preds)
logloss = log_loss(y_test, probs)
tn, fp, fn, tp = confusion_matrix(y_test, preds).ravel()

print(f"\n=== Test Evaluation ===")
print(f"  Accuracy: {acc:.2%}")
print(f"  Log Loss: {logloss:.4f}")
print(f"  Confusion Matrix:")
print(f"    TP={tp}  FP={fp}")
print(f"    FN={fn}  TN={tn}")

# ─── Step 8: Sigmoid curve visualization data ───
print(f"\n=== Decision Boundary ===")
# Find where P(buy) = 0.5 → sigmoid(w1*x + w0) = 0.5 → w1*x + w0 = 0
boundary = -model.intercept_[0] / model.coef_[0][0]
print(f"  At {boundary:.1f} minutes, P(buy) = 50%")
print(f"  Below {boundary:.1f} min → more likely NOT to buy")
print(f"  Above {boundary:.1f} min → more likely to buy")
