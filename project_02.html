Project 2: Predicting a Student's Score From Study Hours

Simple Linear Regression OLS slope/intercept sklearn LinearRegression

Difficulty: Very Easy  |  Builds on: Project 1

1 · Hypothetical Problem

  • You're a teaching assistant at a university
  • Students ask: "If I study X hours, what score can I expect?"
  • You have data from 25 past students: hours studied and their exam scores
  • Goal → build a model that takes hours as input and outputs a predicted score (a continuous number, not a category)
  • In Project 1 we predicted a category (rain/no rain) — now we predict a number
  • This shift from classification to regression is fundamental

2 · How to Approach It

  1. Recall from Project 1: a model is f(input) → output
    • Project 1: f(humidity) → 0 or 1 (category)
    • This project: f(hours) → score (continuous number)
  2. Plot the data — if hours and scores form a roughly straight line, a linear model fits
  3. A linear model: score = slope × hours + intercept
    • slope → how much the score increases per extra hour
    • intercept → the predicted score with zero hours of study
  4. Find the slope and intercept that make the line "best fit" the data
  5. "Best fit" = minimize the total error between predicted and actual scores

3 · How Can We Solve It

  1. Generate synthetic student data (hours → scores with some noise)
  2. Split into train/test (same concept from Project 1)
  3. First: compute slope and intercept manually using the OLS formulas
  4. Then: use sklearn's LinearRegression to verify
  5. Evaluate using Mean Squared Error (MSE) — the regression equivalent of accuracy
  6. Visualize the fitted line against the data points

4 · What Should Do

  • ✅ Understand the OLS formula before using sklearn — know what the library does under the hood
  • ✅ Always visualize the data first — a scatter plot reveals if a linear model makes sense
  • ✅ Use train/test split (learned in Project 1) — evaluate on unseen data
  • ✅ Check both training error and test error — compare them
  • ✅ Interpret the slope: "each additional hour of study adds ~X points to the score"
  • ✅ Interpret the intercept: "with zero hours, the expected score is ~Y" (may or may not make real-world sense)
  • ✅ Use MSE and RMSE — they tell you the average prediction error in the same units as the target

5 · What Should NOT Do

  • ❌ Do NOT use accuracy for regression — accuracy is for classification (Project 1)
  • ❌ Do NOT skip the scatter plot — fitting a line to non-linear data is a common mistake
  • ❌ Do NOT extrapolate wildly — predicting the score for 100 hours of study is meaningless if your data only goes up to 10 hours
  • ❌ Do NOT ignore the intercept — it anchors the line and affects all predictions
  • ❌ Do NOT assume a good training fit means a good model — always check test performance
  • ❌ Do NOT confuse correlation with causation — the model shows association, not that studying CAUSES higher scores

6 · When to Use What

  • 🔹 Simple Linear Regression → when you have ONE numeric input and ONE continuous output, and the relationship looks linear
  • 🔹 OLS (Ordinary Least Squares) → the standard method to find the best-fit line — minimizes squared errors
  • 🔹 MSE / RMSE → standard regression metrics — RMSE is in the same units as the target (easier to interpret)
  • 🔹 Scatter plot + fitted line → always the first visualization for single-feature regression
  • 🔹 sklearn LinearRegression → when you want a clean, tested implementation after understanding the math

7 · When NOT to Use

  • 🚫 Linear regression fails when the relationship is curved (e.g., diminishing returns after 8 hours of study)
  • 🚫 Fails when the output is a category (use classification from Project 1 instead)
  • 🚫 Fails when there are multiple features that interact — need Multiple Regression (Project 9)
  • 🚫 Sensitive to outliers — one student who studied 2 hours and scored 100 will distort the line
  • 🚫 Assumes constant variance of errors (homoscedasticity) — if prediction errors grow with the input, the model is unreliable

8 · How It Only Works — Internal Mechanics

The Linear Model

  • Equation: ŷ = w₁·x + w₀
  • w₁ = slope (weight) — how much y changes per unit change in x
  • w₀ = intercept (bias) — the y-value when x = 0
  • Two parameters to learn — that's it

OLS — Ordinary Least Squares

  • Goal: minimize the sum of squared errors → Σ(yᵢ - ŷᵢ)²
  • Why squared? → penalizes large errors more than small ones, and makes the math differentiable
  • Closed-form solution exists (no iteration needed):
    • w₁ = Σ((xᵢ - x̄)(yᵢ - ȳ)) / Σ((xᵢ - x̄)²)
    • w₀ = ȳ - w₁·x̄
  • This is NOT gradient descent — it's a direct formula (works because the problem is convex)

MSE vs RMSE

  • MSE = (1/n) × Σ(yᵢ - ŷᵢ)² → average squared error
  • RMSE = √MSE → in the same units as y (e.g., "points")
  • RMSE of 5 means: on average, predictions are off by about 5 points

What sklearn Does Under the Hood

  • LinearRegression.fit(X, y) → computes w₁ and w₀ using the OLS formula
  • LinearRegression.predict(X) → applies ŷ = w₁·x + w₀
  • .coef_ → the slope(s)
  • .intercept_ → the intercept

9 · Common Pitfalls and Failure Modes

  • ⚠️ Fitting a line to non-linear data — always plot first; if the scatter curves, a line won't capture it
  • ⚠️ Outliers pulling the line — one extreme point can shift the slope dramatically
  • ⚠️ Extrapolation beyond data range — the model has no idea what happens outside the training range
  • ⚠️ Forgetting to reshape X for sklearn — sklearn expects 2D input: X.reshape(-1, 1)
  • ⚠️ Reporting only training MSE — always report test MSE (same lesson as Project 1)
  • ⚠️ Negative intercept confusion — a negative intercept doesn't mean negative scores; it's just where the line crosses y=0

10 · Scalability Considerations

  • OLS has a closed-form solution → O(n) for single feature — extremely fast
  • With 1 million rows → still instant (numpy handles it)
  • Memory: two arrays of 1M floats ≈ 16 MB — trivial
  • sklearn's LinearRegression uses numpy's optimized linear algebra → production-ready speed
  • Limitation: single feature only — for multiple features, see Project 9
  • For very large datasets (100M+ rows), even OLS is fast because it's a direct formula, not iterative

11 · Interpretability Considerations

  • ✅ Fully interpretable — "each extra hour of study adds ~X points"
  • ✅ Two numbers (slope + intercept) tell the entire story
  • ✅ The fitted line can be drawn on a scatter plot — visual proof
  • ✅ Stakeholders (teachers, students) can understand and act on it
  • ✅ Compared to Project 1's threshold model, this gives a continuous prediction — more informative
  • ⚠️ Interpretability is high because the model is simple — this will decrease as models get complex

12 · Complete Working Code

import numpy as np
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

# ─── Step 1: Create synthetic student data ───
np.random.seed(42)

hours = np.random.uniform(1, 10, size=25)           # 25 students, 1-10 hours
scores = 10 + 8 * hours + np.random.normal(0, 5, 25) # true relationship + noise
# True model: score ≈ 10 + 8 × hours (with noise)

print("=== Sample Data (first 8 students) ===")
for i in range(8):
    print(f"  Student {i+1}: Hours={hours[i]:.1f}  Score={scores[i]:.1f}")

# ─── Step 2: Train/Test Split (same concept from Project 1) ───
X_train, y_train = hours[:18], scores[:18]
X_test, y_test = hours[18:], scores[18:]
print(f"\nTrain: {len(X_train)} students | Test: {len(X_test)} students")

# ─── Step 3: Manual OLS Calculation ───
x_mean = np.mean(X_train)
y_mean = np.mean(y_train)

# Slope: w1 = Σ((xi - x̄)(yi - ȳ)) / Σ((xi - x̄)²)
numerator = np.sum((X_train - x_mean) * (y_train - y_mean))
denominator = np.sum((X_train - x_mean) ** 2)
slope_manual = numerator / denominator

# Intercept: w0 = ȳ - w1 × x̄
intercept_manual = y_mean - slope_manual * x_mean

print(f"\n=== Manual OLS ===")
print(f"  Slope (w1):     {slope_manual:.4f}")
print(f"  Intercept (w0): {intercept_manual:.4f}")
print(f"  Model: score = {slope_manual:.2f} × hours + {intercept_manual:.2f}")

# ─── Step 4: sklearn LinearRegression (verify) ───
model = LinearRegression()
model.fit(X_train.reshape(-1, 1), y_train)  # sklearn needs 2D input

print(f"\n=== sklearn LinearRegression ===")
print(f"  Slope (coef_):      {model.coef_[0]:.4f}")
print(f"  Intercept:          {model.intercept_:.4f}")
print(f"  Match manual? Slope: {np.isclose(slope_manual, model.coef_[0])}, "
      f"Intercept: {np.isclose(intercept_manual, model.intercept_)}")

# ─── Step 5: Predictions and Evaluation ───
y_train_pred = model.predict(X_train.reshape(-1, 1))
y_test_pred = model.predict(X_test.reshape(-1, 1))

train_mse = mean_squared_error(y_train, y_train_pred)
test_mse = mean_squared_error(y_test, y_test_pred)
train_rmse = np.sqrt(train_mse)
test_rmse = np.sqrt(test_mse)

print(f"\n=== Evaluation ===")
print(f"  Train RMSE: {train_rmse:.2f} points")
print(f"  Test RMSE:  {test_rmse:.2f} points")
print(f"  Interpretation: predictions are off by ~{test_rmse:.1f} points on average")

# ─── Step 6: Predictions vs Actuals (Test Set) ───
print(f"\n=== Test Predictions vs Actuals ===")
for i in range(len(X_test)):
    error = y_test_pred[i] - y_test[i]
    print(f"  Student: Hours={X_test[i]:.1f}  Predicted={y_test_pred[i]:.1f}  "
          f"Actual={y_test[i]:.1f}  Error={error:+.1f}")

# ─── Step 7: Visualization ───
fig, ax = plt.subplots(1, 1, figsize=(8, 5))
ax.scatter(X_train, y_train, color='#58a6ff', label='Training data', alpha=0.7)
ax.scatter(X_test, y_test, color='#f0883e', label='Test data', alpha=0.7)

# Fitted line
x_line = np.linspace(0, 11, 100)
y_line = model.predict(x_line.reshape(-1, 1))
ax.plot(x_line, y_line, color='#3fb950', linewidth=2, label=f'Fit: y={model.coef_[0]:.1f}x+{model.intercept_:.1f}')

ax.set_xlabel('Hours Studied')
ax.set_ylabel('Exam Score')
ax.set_title('Simple Linear Regression — Study Hours vs Score')
ax.legend()
ax.grid(True, alpha=0.2)
plt.tight_layout()
plt.savefig('project_02_plot.png', dpi=100, bbox_inches='tight')
print("\n✅ Plot saved as project_02_plot.png")
