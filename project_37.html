Project 37: Causal Inference — Does a Discount Actually Drive Sales?

Causal Inference Propensity Score Treatment Effect DoWhy Confounders

Difficulty: Hard  |  Builds on: Projects 30, 19

1 · Hypothetical Problem

  • Your e-commerce company sent a 20% discount email to some customers last month
  • Customers who received the discount spent $85 on average; those who didn't spent $65
  • Marketing claims: "The discount increased spending by $20!" — but is that causal?
  • Problem: the discount was sent to high-value customers (selection bias) — they would have spent more anyway
  • Goal → estimate the TRUE causal effect of the discount, controlling for confounders

2 · How to Approach It

  1. From Project 30: Bayesian thinking taught us to reason about uncertainty — causal inference adds the "why"
  2. From Project 19: handling imbalanced data — here the "imbalance" is between treated and control groups
  3. The fundamental problem: we can't observe the same customer both with AND without the discount
  4. Solution: find "similar" customers in both groups using propensity score matching
  5. Propensity score = P(receiving treatment | covariates) — estimated with logistic regression (Project 6)
  6. Match treated customers to control customers with similar propensity scores → estimate treatment effect

3 · How Can We Solve It

  1. Generate observational data with confounders (income, past purchases affect both treatment and outcome)
  2. Show the naive estimate is biased (simple difference in means)
  3. Estimate propensity scores using logistic regression
  4. Match treated to control using nearest-neighbor propensity score matching
  5. Compute the Average Treatment Effect on the Treated (ATT)
  6. Validate with a known true effect (since we generated the data)

4 · What Should Do

  • ✅ Identify confounders — variables that affect BOTH treatment assignment and outcome
  • ✅ Check covariate balance after matching — treated and control should look similar
  • ✅ Use multiple estimation methods and compare — if they agree, the result is more credible
  • ✅ Report ATT (effect on treated) vs ATE (effect on everyone) — they answer different questions
  • ✅ Do sensitivity analysis — how much unmeasured confounding would it take to nullify the result?

5 · What Should NOT Do

  • ❌ Do NOT equate correlation with causation — the naive difference is almost always biased
  • ❌ Do NOT forget unmeasured confounders — propensity scores only control for OBSERVED variables
  • ❌ Do NOT match on too many exact variables — the curse of dimensionality makes exact matching impossible
  • ❌ Do NOT ignore overlap — if some treated customers have no similar controls, matching fails
  • ❌ Do NOT use predictive models (Project 21, 23) for causal questions — prediction ≠ causation

6 · When to Use What

  • 🔹 Randomized experiment (A/B test) → gold standard for causation; use when possible (Project 30)
  • 🔹 Propensity score matching → when randomization isn't possible; observational data with measured confounders
  • 🔹 Inverse probability weighting (IPW) → alternative to matching; reweights samples by propensity
  • 🔹 Difference-in-differences → when you have before/after data for treated and control groups
  • 🔹 Instrumental variables → when there are unmeasured confounders but a valid instrument exists

7 · When NOT to Use

  • 🚫 When you can run a randomized experiment — it's always better than observational methods
  • 🚫 When key confounders are unmeasured — propensity scores can't fix what they can't see
  • 🚫 When there's no overlap between groups — can't match treated to controls if they're completely different
  • 🚫 When the causal structure is unknown — you need domain knowledge to identify confounders

8 · How It Only Works — Internal Mechanics

The Fundamental Problem of Causal Inference

  • For each customer, we observe ONE outcome: with discount OR without — never both
  • The unobserved outcome is the "counterfactual" — what WOULD have happened
  • Causal effect = Y(treated) - Y(untreated) — but we only see one of these per person
  • Solution: estimate the average effect across a population using matched groups

Propensity Score

  • e(X) = P(Treatment=1 | X) — probability of receiving treatment given covariates
  • Estimated using logistic regression: treatment ~ income + past_purchases + ...
  • Key property: conditioning on the propensity score balances covariates between groups
  • This reduces a multi-dimensional matching problem to a 1D problem

Matching

  • For each treated customer, find the control customer with the closest propensity score
  • The matched pair approximates a mini-experiment: similar customers, different treatments
  • ATT = average of (Y_treated - Y_matched_control) across all treated customers

Assumptions Required

  • Unconfoundedness: all confounders are measured and included — no hidden variables
  • Overlap (positivity): every customer has a non-zero chance of being in either group
  • SUTVA: one customer's treatment doesn't affect another's outcome
  • These assumptions are untestable — domain knowledge is essential

9 · Common Pitfalls and Failure Modes

  • ⚠️ Unmeasured confounders — the biggest threat; propensity scores only balance observed variables
  • ⚠️ Poor overlap — if propensity scores don't overlap between groups, matching is unreliable
  • ⚠️ Post-treatment variables — don't control for variables affected by the treatment itself
  • ⚠️ Matching without checking balance — always verify covariates are balanced after matching
  • ⚠️ Confusing prediction with causation — a model that predicts well doesn't identify causes

10 · Scalability Considerations

  • Propensity score estimation: logistic regression is O(n × p) — fast for moderate data
  • Nearest-neighbor matching: O(n × m) where n=treated, m=control — use KD-trees for speed
  • For millions of records, use approximate matching or IPW (no matching needed)
  • DoWhy framework handles the causal graph and estimation — but adds overhead for small problems

11 · Interpretability Considerations

  • ✅ ATT is highly interpretable: "The discount increased spending by $X for those who received it"
  • ✅ Propensity score distributions show how similar/different the groups are
  • ✅ Covariate balance plots show matching quality — stakeholders can verify fairness
  • ⚠️ The causal graph (DAG) requires domain expertise to construct and validate
  • ⚠️ Sensitivity analysis results can be hard to communicate — "how much hidden bias would change the conclusion"

12 · Complete Working Code

import numpy as np
import pandas as pd
from sklearn.linear_model import LogisticRegression
from sklearn.neighbors import NearestNeighbors
from scipy import stats
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

np.random.seed(42)

# ─── Step 1: Generate observational data with confounders ───
n = 2000
TRUE_EFFECT = 12.0  # The actual causal effect of the discount

# Confounders: income and past purchases
income = np.random.normal(50000, 15000, n)
past_purchases = np.random.poisson(5, n)
loyalty_score = np.random.uniform(0, 10, n)

# Treatment assignment (biased — high-value customers more likely to get discount)
propensity_true = 1 / (1 + np.exp(-(
    -2 + 0.00003 * income + 0.15 * past_purchases + 0.1 * loyalty_score
)))
treatment = np.random.binomial(1, propensity_true)

# Outcome: spending depends on confounders AND treatment
base_spending = 30 + 0.0005 * income + 3 * past_purchases + 2 * loyalty_score
noise = np.random.normal(0, 10, n)
spending = base_spending + TRUE_EFFECT * treatment + noise

df = pd.DataFrame({
    'income': income, 'past_purchases': past_purchases,
    'loyalty_score': loyalty_score, 'treatment': treatment, 'spending': spending
})

print("=== Observational Data ===")
print(f"  Total customers: {n}")
print(f"  Treated: {treatment.sum()} ({treatment.mean():.1%})")
print(f"  Control: {(1-treatment).sum()} ({1-treatment.mean():.1%})")
print(f"  TRUE causal effect: ${TRUE_EFFECT:.2f}")

# ─── Step 2: Naive estimate (biased) ───
naive_effect = df[df.treatment==1].spending.mean() - df[df.treatment==0].spending.mean()
print(f"\n=== Naive Estimate (Biased) ===")
print(f"  Mean spending (treated):  ${df[df.treatment==1].spending.mean():.2f}")
print(f"  Mean spending (control):  ${df[df.treatment==0].spending.mean():.2f}")
print(f"  Naive effect: ${naive_effect:.2f} (TRUE: ${TRUE_EFFECT:.2f})")
print(f"  Bias: ${naive_effect - TRUE_EFFECT:.2f} — confounders inflate the estimate!")

# ─── Step 3: Estimate propensity scores ───
X = df[['income', 'past_purchases', 'loyalty_score']]
y = df['treatment']

ps_model = LogisticRegression(max_iter=1000, random_state=42)
ps_model.fit(X, y)
df['propensity_score'] = ps_model.predict_proba(X)[:, 1]

print(f"\n=== Propensity Scores ===")
print(f"  Treated — mean PS: {df[df.treatment==1].propensity_score.mean():.3f}")
print(f"  Control — mean PS: {df[df.treatment==0].propensity_score.mean():.3f}")

# ─── Step 4: Propensity Score Matching (nearest neighbor) ───
treated = df[df.treatment == 1].copy()
control = df[df.treatment == 0].copy()

nn = NearestNeighbors(n_neighbors=1, metric='euclidean')
nn.fit(control[['propensity_score']])
distances, indices = nn.kneighbors(treated[['propensity_score']])

matched_control = control.iloc[indices.flatten()].copy()
matched_control.index = treated.index

# ATT: Average Treatment Effect on the Treated
att = (treated.spending.values - matched_control.spending.values).mean()
att_se = (treated.spending.values - matched_control.spending.values).std() / np.sqrt(len(treated))

print(f"\n=== Propensity Score Matching Results ===")
print(f"  Matched pairs: {len(treated)}")
print(f"  ATT (matched): ${att:.2f} ± ${1.96*att_se:.2f}")
print(f"  TRUE effect:    ${TRUE_EFFECT:.2f}")
print(f"  Bias reduction: ${abs(naive_effect - TRUE_EFFECT) - abs(att - TRUE_EFFECT):.2f}")

# ─── Step 5: Check covariate balance ───
print(f"\n=== Covariate Balance (Before vs After Matching) ===")
print(f"{'Variable':<18} {'Before (T-C)':<15} {'After (T-MC)':<15} {'Improved?'}")
for col in ['income', 'past_purchases', 'loyalty_score']:
    before_diff = treated[col].mean() - control[col].mean()
    after_diff = treated[col].mean() - matched_control[col].mean()
    improved = "✅" if abs(after_diff) < abs(before_diff) else "❌"
    print(f"  {col:<16} {before_diff:>12.2f}   {after_diff:>12.2f}   {improved}")

# ─── Step 6: IPW (Inverse Probability Weighting) as alternative ───
ps = df['propensity_score'].values
t = df['treatment'].values
y_obs = df['spending'].values

# IPW estimator for ATE
ipw_treated = np.sum(t * y_obs / ps) / np.sum(t / ps)
ipw_control = np.sum((1-t) * y_obs / (1-ps)) / np.sum((1-t) / (1-ps))
ipw_ate = ipw_treated - ipw_control

print(f"\n=== IPW Estimate ===")
print(f"  ATE (IPW): ${ipw_ate:.2f}")
print(f"  TRUE:      ${TRUE_EFFECT:.2f}")

# ─── Step 7: Summary comparison ───
print(f"\n=== Method Comparison ===")
print(f"  {'Method':<25} {'Estimate':>10} {'Bias':>10}")
print(f"  {'-'*47}")
print(f"  {'Naive (biased)':<25} ${naive_effect:>8.2f}  ${naive_effect-TRUE_EFFECT:>8.2f}")
print(f"  {'PS Matching (ATT)':<25} ${att:>8.2f}  ${att-TRUE_EFFECT:>8.2f}")
print(f"  {'IPW (ATE)':<25} ${ipw_ate:>8.2f}  ${ipw_ate-TRUE_EFFECT:>8.2f}")
print(f"  {'TRUE EFFECT':<25} ${TRUE_EFFECT:>8.2f}  ${'0.00':>8}")

# ─── Step 8: Visualization ───
fig, axes = plt.subplots(2, 2, figsize=(14, 12))

# Propensity score distributions
axes[0, 0].hist(df[df.treatment==1].propensity_score, bins=40, alpha=0.6,
                color='#f0883e', density=True, label='Treated')
axes[0, 0].hist(df[df.treatment==0].propensity_score, bins=40, alpha=0.6,
                color='#58a6ff', density=True, label='Control')
axes[0, 0].set_xlabel('Propensity Score')
axes[0, 0].set_ylabel('Density')
axes[0, 0].set_title('Propensity Score Distributions')
axes[0, 0].legend()
axes[0, 0].grid(True, alpha=0.2)

# Covariate balance
covariates = ['income', 'past_purchases', 'loyalty_score']
before_smd = [(treated[c].mean() - control[c].mean()) / df[c].std() for c in covariates]
after_smd = [(treated[c].mean() - matched_control[c].mean()) / df[c].std() for c in covariates]
y_pos = np.arange(len(covariates))
axes[0, 1].barh(y_pos - 0.15, np.abs(before_smd), 0.3, color='#f85149', alpha=0.7, label='Before matching')
axes[0, 1].barh(y_pos + 0.15, np.abs(after_smd), 0.3, color='#3fb950', alpha=0.7, label='After matching')
axes[0, 1].set_yticks(y_pos)
axes[0, 1].set_yticklabels(covariates)
axes[0, 1].set_xlabel('|Standardized Mean Difference|')
axes[0, 1].set_title('Covariate Balance')
axes[0, 1].axvline(0.1, color='white', linestyle='--', alpha=0.5, label='Threshold (0.1)')
axes[0, 1].legend(fontsize=8)
axes[0, 1].grid(True, alpha=0.2)

# Method comparison
methods = ['Naive', 'PS Match', 'IPW', 'TRUE']
estimates = [naive_effect, att, ipw_ate, TRUE_EFFECT]
colors = ['#f85149', '#3fb950', '#58a6ff', 'white']
axes[1, 0].bar(methods, estimates, color=colors, edgecolor='white', alpha=0.8)
axes[1, 0].axhline(TRUE_EFFECT, color='#f0883e', linestyle='--', linewidth=2, label=f'True effect: ${TRUE_EFFECT}')
axes[1, 0].set_ylabel('Estimated Effect ($)')
axes[1, 0].set_title('Causal Effect Estimates')
axes[1, 0].legend()
axes[1, 0].grid(True, alpha=0.2, axis='y')

# Matched pairs visualization
sample_idx = np.random.choice(len(treated), 20, replace=False)
for i, idx in enumerate(sample_idx):
    t_ps = treated.iloc[idx].propensity_score
    c_ps = matched_control.iloc[idx].propensity_score
    axes[1, 1].plot([0, 1], [t_ps, c_ps], color='#58a6ff', alpha=0.4, linewidth=0.8)
    axes[1, 1].scatter([0], [t_ps], color='#f0883e', s=20, zorder=5)
    axes[1, 1].scatter([1], [c_ps], color='#3fb950', s=20, zorder=5)
axes[1, 1].set_xticks([0, 1])
axes[1, 1].set_xticklabels(['Treated', 'Matched Control'])
axes[1, 1].set_ylabel('Propensity Score')
axes[1, 1].set_title('Sample Matched Pairs (20 shown)')
axes[1, 1].grid(True, alpha=0.2)

plt.tight_layout()
plt.savefig('project_37_causal_inference.png', dpi=100, bbox_inches='tight')
print(f"\n✅ Causal inference analysis saved as project_37_causal_inference.png")

13 · Key Takeaway + Bridge to Next Project

Key Takeaway

  • Correlation ≠ causation — naive comparisons are biased by confounders
  • Propensity score matching creates pseudo-experiments from observational data
  • Always check covariate balance after matching — unbalanced covariates mean biased estimates
  • Multiple methods (matching, IPW) should agree — disagreement signals problems
  • Unmeasured confounders remain the Achilles' heel — domain knowledge is essential
