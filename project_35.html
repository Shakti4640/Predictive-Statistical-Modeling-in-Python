Project 35: Multi-Label Classification — Tagging Articles With Many Labels

Multi-Label MultiOutputClassifier Label Powerset Hamming Loss Subset Accuracy

Difficulty: Hard  |  Builds on: Projects 12, 28

1 · Hypothetical Problem

  • You're building an auto-tagging system for a news platform with 8 topic categories
  • Each article can belong to multiple categories simultaneously — "Tech + Business," "Politics + Economy + International"
  • Dataset: 5,000 articles with TF-IDF features (Project 28) and binary label vectors
  • Standard multi-class classification (Project 12) forces exactly one label — that won't work here
  • Goal → predict the complete set of relevant tags for each article

2 · How to Approach It

  1. Recall from Project 12: decision trees classify into one of K classes — multi-class, not multi-label
  2. Multi-label treats each label as an independent binary classification problem (binary relevance)
  3. Or we can model label correlations — "articles about Tech are often also about Business"
  4. Standard accuracy doesn't work — we need metrics that handle partial correctness
  5. Use sklearn's MultiOutputClassifier and compare with classifier chains that capture label dependencies

3 · How Can We Solve It

  1. Generate synthetic multi-label data with correlated labels
  2. Implement binary relevance (one classifier per label) using MultiOutputClassifier
  3. Implement ClassifierChain to capture label dependencies
  4. Compare using multi-label metrics: hamming loss, subset accuracy, macro/micro F1
  5. Visualize label co-occurrence and per-label performance

4 · What Should Do

  • ✅ Use hamming loss as the primary metric — it measures the fraction of incorrectly predicted labels
  • ✅ Report both subset accuracy (exact match) and hamming loss — they tell different stories
  • ✅ Use ClassifierChain when labels are correlated — it feeds previous label predictions as features
  • ✅ Check label frequency distribution — rare labels need special handling (like class imbalance, Project 19)
  • ✅ Use per-label classification reports to identify which labels are hard to predict

5 · What Should NOT Do

  • ❌ Do NOT use standard accuracy — it requires ALL labels to be correct (too strict for multi-label)
  • ❌ Do NOT ignore label correlations — binary relevance treats labels independently, missing co-occurrence patterns
  • ❌ Do NOT use multi-class classifiers directly — they predict exactly one class, not a set of labels
  • ❌ Do NOT forget to handle rare labels — a label that appears in 2% of samples will be predicted as "absent" by default
  • ❌ Do NOT evaluate with a single metric — hamming loss, subset accuracy, and per-label F1 each reveal different issues

6 · When to Use What

  • 🔹 Binary Relevance (MultiOutputClassifier) → simple, fast, works when labels are independent
  • 🔹 ClassifierChain → when labels are correlated; chains predictions as features for subsequent labels
  • 🔹 Label Powerset → treats each unique label combination as a single class; captures all correlations but explodes with many labels
  • 🔹 Multi-class (Project 12) → when each sample has exactly one label — not multi-label
  • 🔹 Naive Bayes (Project 28) → fast base classifier for binary relevance on text data

7 · When NOT to Use

  • 🚫 When each sample has exactly one label — use multi-class classification (Project 12)
  • 🚫 Label Powerset with many labels — 2^L possible combinations; infeasible for L > 15
  • 🚫 When label space is huge (thousands of labels) — use extreme multi-label methods, not standard approaches
  • 🚫 When you need ranked labels, not binary — use learning-to-rank instead

8 · How It Only Works — Internal Mechanics

Multi-Label vs Multi-Class

  • Multi-class: y ∈ {0, 1, 2, ..., K-1} — exactly one class per sample
  • Multi-label: y ∈ {0, 1}^L — a binary vector of length L; multiple 1s allowed
  • Example: article about AI in healthcare → y = [1, 0, 1, 0, 0, 1, 0, 0] (Tech, Health, Science)

Binary Relevance

  • Train L independent binary classifiers — one per label
  • Classifier i predicts: "Is label i present?" (yes/no)
  • Simple and parallelizable — but ignores label correlations entirely
  • If "Tech" and "Business" always co-occur, binary relevance won't learn that

Classifier Chain

  • Train classifiers in a chain: classifier i gets original features + predictions from classifiers 1..i-1
  • This way, later classifiers can use earlier label predictions as additional features
  • Captures label dependencies: if classifier 1 predicts "Tech=1," classifier 2 for "Business" sees that
  • Order matters — try multiple random orderings and average (ensemble of chains)

Label Powerset

  • Treat each unique label combination as a single class
  • If labels are [Tech, Business, Health], possible classes: (0,0,0), (1,0,0), (0,1,0), (1,1,0), ...
  • Naturally captures all label correlations — but 2^L possible classes
  • Practical only for small L (≤10-12 labels) and when many combinations actually appear in data

Multi-Label Metrics

  • Hamming Loss: fraction of labels that are incorrectly predicted (lower is better)
  • Subset Accuracy: fraction of samples where ALL labels are correct (very strict)
  • Micro F1: pool all label predictions together, compute F1 — favors frequent labels
  • Macro F1: compute F1 per label, then average — treats all labels equally
  • Sample F1: compute F1 per sample, then average — measures per-sample quality

9 · Common Pitfalls and Failure Modes

  • ⚠️ Using accuracy instead of hamming loss — accuracy requires exact match of all labels; hamming loss gives partial credit
  • ⚠️ Ignoring rare labels — a label present in 1% of samples gets predicted as "absent" always; use class_weight or oversampling per label
  • ⚠️ Wrong chain order — ClassifierChain performance depends on label ordering; use ensemble of random orders
  • ⚠️ Label Powerset on many labels — with 20 labels, you'd have up to 1M classes; use binary relevance or chains instead
  • ⚠️ Not checking label correlations — if labels are truly independent, ClassifierChain adds complexity without benefit

10 · Scalability Considerations

  • Binary Relevance: trains L independent models — embarrassingly parallel, scales linearly with L
  • ClassifierChain: sequential by nature — L models trained one after another; harder to parallelize
  • Label Powerset: exponential in L — only practical for small label spaces
  • For extreme multi-label (thousands of labels), use specialized methods like FastXML or embedding-based approaches
  • TF-IDF features (Project 28) can be very high-dimensional — use sparse matrices and linear classifiers for speed

11 · Interpretability Considerations

  • ✅ Binary Relevance is the most interpretable — each label has its own model you can inspect independently
  • ✅ Per-label feature importance shows which words predict each topic (using text features from Project 28)
  • ⚠️ ClassifierChain is harder to interpret — predictions depend on the chain order and previous predictions
  • ⚠️ Label Powerset treats combinations as atomic classes — loses per-label interpretability
  • ⚠️ For stakeholders, show per-label confidence scores, not just binary predictions

12 · Complete Working Code

import numpy as np
from sklearn.datasets import make_multilabel_classification
from sklearn.model_selection import train_test_split
from sklearn.multioutput import MultiOutputClassifier, ClassifierChain
from sklearn.linear_model import LogisticRegression
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import (hamming_loss, accuracy_score, f1_score,
                              classification_report, multilabel_confusion_matrix)
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

# ─── Step 1: Generate multi-label dataset ───
np.random.seed(42)
X, Y = make_multilabel_classification(
    n_samples=5000, n_features=30, n_classes=8, n_labels=2.5,
    allow_unlabeled=False, random_state=42
)

label_names = ['Tech', 'Business', 'Politics', 'Health', 'Sports', 'Science', 'Entertainment', 'Economy']

X_train, X_test, Y_train, Y_test = train_test_split(
    X, Y, test_size=0.2, random_state=42)

print(f"=== Multi-Label Dataset ===")
print(f"  Samples: {X.shape[0]}, Features: {X.shape[1]}, Labels: {Y.shape[1]}")
print(f"  Avg labels per sample: {Y.sum(axis=1).mean():.1f}")
print(f"\n  Label frequencies:")
for i, name in enumerate(label_names):
    freq = Y[:, i].mean()
    print(f"    {name:<15}: {freq:.1%}")

# ─── Step 2: Binary Relevance (MultiOutputClassifier) ───
print(f"\n=== Strategy 1: Binary Relevance ===")
br_clf = MultiOutputClassifier(
    LogisticRegression(max_iter=1000, random_state=42), n_jobs=-1)
br_clf.fit(X_train, Y_train)
br_pred = br_clf.predict(X_test)

br_hamming = hamming_loss(Y_test, br_pred)
br_subset = accuracy_score(Y_test, br_pred)
br_micro_f1 = f1_score(Y_test, br_pred, average='micro')
br_macro_f1 = f1_score(Y_test, br_pred, average='macro')

print(f"  Hamming Loss:    {br_hamming:.4f}")
print(f"  Subset Accuracy: {br_subset:.4f}")
print(f"  Micro F1:        {br_micro_f1:.4f}")
print(f"  Macro F1:        {br_macro_f1:.4f}")

# ─── Step 3: Classifier Chain ───
print(f"\n=== Strategy 2: Classifier Chain ===")
cc_clf = ClassifierChain(
    LogisticRegression(max_iter=1000, random_state=42), order='random', random_state=42)
cc_clf.fit(X_train, Y_train)
cc_pred = cc_clf.predict(X_test)

cc_hamming = hamming_loss(Y_test, cc_pred)
cc_subset = accuracy_score(Y_test, cc_pred)
cc_micro_f1 = f1_score(Y_test, cc_pred, average='micro')
cc_macro_f1 = f1_score(Y_test, cc_pred, average='macro')

print(f"  Hamming Loss:    {cc_hamming:.4f}")
print(f"  Subset Accuracy: {cc_subset:.4f}")
print(f"  Micro F1:        {cc_micro_f1:.4f}")
print(f"  Macro F1:        {cc_macro_f1:.4f}")

# ─── Step 4: Ensemble of Classifier Chains ───
print(f"\n=== Strategy 3: Ensemble of 10 Classifier Chains ===")
chains = [ClassifierChain(LogisticRegression(max_iter=1000, random_state=42),
                          order='random', random_state=i)
          for i in range(10)]

for chain in chains:
    chain.fit(X_train, Y_train)

# Average predictions across chains
chain_preds = np.array([chain.predict(X_test) for chain in chains])
ensemble_pred = (chain_preds.mean(axis=0) >= 0.5).astype(int)

ens_hamming = hamming_loss(Y_test, ensemble_pred)
ens_subset = accuracy_score(Y_test, ensemble_pred)
ens_micro_f1 = f1_score(Y_test, ensemble_pred, average='micro')
ens_macro_f1 = f1_score(Y_test, ensemble_pred, average='macro')

print(f"  Hamming Loss:    {ens_hamming:.4f}")
print(f"  Subset Accuracy: {ens_subset:.4f}")
print(f"  Micro F1:        {ens_micro_f1:.4f}")
print(f"  Macro F1:        {ens_macro_f1:.4f}")

# ─── Step 5: Binary Relevance with Random Forest ───
print(f"\n=== Strategy 4: Binary Relevance (Random Forest) ===")
br_rf = MultiOutputClassifier(
    RandomForestClassifier(n_estimators=200, max_depth=10, random_state=42, n_jobs=-1))
br_rf.fit(X_train, Y_train)
rf_pred = br_rf.predict(X_test)

rf_hamming = hamming_loss(Y_test, rf_pred)
rf_subset = accuracy_score(Y_test, rf_pred)
rf_micro_f1 = f1_score(Y_test, rf_pred, average='micro')
rf_macro_f1 = f1_score(Y_test, rf_pred, average='macro')

print(f"  Hamming Loss:    {rf_hamming:.4f}")
print(f"  Subset Accuracy: {rf_subset:.4f}")
print(f"  Micro F1:        {rf_micro_f1:.4f}")
print(f"  Macro F1:        {rf_macro_f1:.4f}")

# ─── Step 6: Per-label performance ───
print(f"\n=== Per-Label F1 Scores (Best Model) ===")
best_pred = ensemble_pred
for i, name in enumerate(label_names):
    f1 = f1_score(Y_test[:, i], best_pred[:, i])
    support = Y_test[:, i].sum()
    print(f"  {name:<15}: F1 = {f1:.4f}  (support: {int(support)})")

# ─── Step 7: Visualization ───
fig, axes = plt.subplots(2, 2, figsize=(14, 12))

# Strategy comparison
strategies = ['Binary Rel\n(LogReg)', 'Classifier\nChain', 'Ensemble\nChains', 'Binary Rel\n(RF)']
hammings = [br_hamming, cc_hamming, ens_hamming, rf_hamming]
subsets = [br_subset, cc_subset, ens_subset, rf_subset]
micro_f1s = [br_micro_f1, cc_micro_f1, ens_micro_f1, rf_micro_f1]

x = np.arange(len(strategies))
width = 0.25
axes[0, 0].bar(x - width, micro_f1s, width, label='Micro F1', color='#58a6ff')
axes[0, 0].bar(x, subsets, width, label='Subset Acc', color='#3fb950')
axes[0, 0].bar(x + width, [1-h for h in hammings], width, label='1 - Hamming', color='#f0883e')
axes[0, 0].set_xticks(x)
axes[0, 0].set_xticklabels(strategies, fontsize=9)
axes[0, 0].set_ylabel('Score')
axes[0, 0].set_title('Strategy Comparison')
axes[0, 0].legend(fontsize=8)
axes[0, 0].grid(True, alpha=0.2, axis='y')

# Per-label F1
per_label_f1 = [f1_score(Y_test[:, i], best_pred[:, i]) for i in range(len(label_names))]
colors = ['#3fb950' if f > 0.6 else '#f0883e' if f > 0.4 else '#f85149' for f in per_label_f1]
axes[0, 1].barh(label_names, per_label_f1, color=colors, edgecolor='white', alpha=0.8)
axes[0, 1].set_xlabel('F1 Score')
axes[0, 1].set_title('Per-Label F1 (Ensemble Chains)')
axes[0, 1].grid(True, alpha=0.2, axis='x')

# Label co-occurrence matrix
cooccurrence = Y_train.T @ Y_train
np.fill_diagonal(cooccurrence, 0)
cooccurrence = cooccurrence / cooccurrence.max()
im = axes[1, 0].imshow(cooccurrence, cmap='YlOrRd')
axes[1, 0].set_xticks(range(len(label_names)))
axes[1, 0].set_yticks(range(len(label_names)))
axes[1, 0].set_xticklabels(label_names, rotation=45, ha='right', fontsize=8)
axes[1, 0].set_yticklabels(label_names, fontsize=8)
axes[1, 0].set_title('Label Co-occurrence')
plt.colorbar(im, ax=axes[1, 0])

# Label count distribution
label_counts = Y_test.sum(axis=1)
axes[1, 1].hist(label_counts, bins=range(int(label_counts.max()) + 2),
                color='#58a6ff', edgecolor='white', alpha=0.8, align='left')
axes[1, 1].set_xlabel('Number of Labels per Sample')
axes[1, 1].set_ylabel('Count')
axes[1, 1].set_title('Label Count Distribution')
axes[1, 1].grid(True, alpha=0.2, axis='y')

plt.tight_layout()
plt.savefig('project_35_multilabel.png', dpi=100, bbox_inches='tight')
print(f"\n✅ Multi-label analysis saved as project_35_multilabel.png")

Key Takeaway

  • Multi-label classification predicts a SET of labels per sample — fundamentally different from multi-class (Project 12)

  • Ensemble of chains with random orderings is the most robust approach
  • Use hamming loss (partial credit) and subset accuracy (exact match) together — they tell different stories
  • Rare labels need the same attention as minority classes in imbalanced classification (Project 19)
