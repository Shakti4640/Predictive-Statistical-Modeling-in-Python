Project 17: Predicting Energy Demand With Ridge Regression

Regularization Ridge (L2) alpha tuning coefficient shrinkage

Difficulty: Medium  |  Builds on: Projects 9, 15

1 · Hypothetical Problem

  • You work at a power utility company predicting daily energy demand
  • Features: temperature, humidity, wind speed, day of week, holiday flag, and 10 lagged demand values
  • With 15+ features, many correlated (temperature and humidity), OLS coefficients are unstable (Project 9)
  • Goal → use Ridge regression to stabilize coefficients and improve generalization

2 · How to Approach It

  1. Recall from Project 9: multicollinearity makes OLS coefficients large and unstable
  2. Ridge adds a penalty: Loss = MSE + α × Σ(wᵢ²)
  3. The penalty shrinks coefficients toward zero — prevents them from exploding
  4. α (alpha) controls the penalty strength — tune it with cross-validation (Project 15)

3 · How Can We Solve It

  1. Generate energy demand data with correlated features
  2. Show OLS coefficient instability with correlated features
  3. Fit Ridge with different alpha values
  4. Plot coefficient paths (how coefficients shrink as alpha increases)
  5. Use RidgeCV for automatic alpha selection

4 · What Should Do

  • ✅ Scale features before Ridge — the penalty treats all coefficients equally, so they must be on the same scale
  • ✅ Use RidgeCV or cross-validation to find the best alpha
  • ✅ Compare OLS vs Ridge coefficients — Ridge should be smaller and more stable
  • ✅ Plot coefficient paths to visualize the shrinkage effect

5 · What Should NOT Do

  • ❌ Do NOT use Ridge without scaling — features on larger scales get penalized more
  • ❌ Do NOT set alpha=0 — that's just OLS with no regularization
  • ❌ Do NOT set alpha too high — all coefficients shrink to near-zero (underfitting)
  • ❌ Do NOT expect Ridge to set coefficients exactly to zero — it shrinks but doesn't eliminate (that's Lasso, Project 18)

6 · When to Use What

  • 🔹 Ridge (L2) → when you have many correlated features and want to keep all of them
  • 🔹 OLS → when you have few features with low multicollinearity
  • 🔹 RidgeCV → automatic alpha selection via built-in cross-validation
  • 🔹 Coefficient paths → to visualize how regularization affects each feature

7 · When NOT to Use

  • 🚫 Ridge keeps ALL features — if you want automatic feature selection, use Lasso (Project 18)
  • 🚫 Ridge assumes a linear relationship — nonlinear patterns need different models
  • 🚫 With very few features and no multicollinearity, Ridge offers little benefit over OLS

8 · How It Only Works — Internal Mechanics

Ridge Regression Loss Function

  • Loss = Σ(yᵢ - ŷᵢ)² + α × Σ(wⱼ²)
  • First term: MSE (fit the data)
  • Second term: L2 penalty (keep coefficients small)
  • α balances fit vs simplicity

Why L2 Penalty Stabilizes Coefficients

  • With correlated features, OLS has many equally good solutions with large coefficients
  • The L2 penalty prefers the solution with the smallest coefficients
  • Mathematically: w = (XᵀX + αI)⁻¹Xᵀy — adding αI makes the matrix invertible
  • This is why Ridge "fixes" multicollinearity — it regularizes the ill-conditioned matrix

Alpha — The Regularization Strength

  • α = 0 → no penalty → pure OLS
  • α → ∞ → all coefficients → 0 → predicts the mean
  • Optimal α is found via cross-validation
  • RidgeCV tests multiple alphas and picks the best one automatically

Coefficient Shrinkage Path

  • As α increases, all coefficients shrink toward zero
  • Important features shrink slowly; unimportant ones shrink quickly
  • But NO coefficient reaches exactly zero — Ridge keeps all features

9 · Common Pitfalls and Failure Modes

  • ⚠️ Not scaling before Ridge — the penalty is applied equally to all coefficients
  • ⚠️ Using a single alpha without tuning — the default may not be optimal
  • ⚠️ Expecting feature selection — Ridge shrinks but never eliminates features
  • ⚠️ Comparing Ridge coefficients across different alpha values — they're not directly comparable

10 · Scalability Considerations

  • Ridge has a closed-form solution: O(np² + p³) — same as OLS
  • RidgeCV is efficient — it solves for multiple alphas simultaneously
  • For very large p (features), use SGD with L2 penalty
  • Ridge is one of the most computationally efficient regularized models

11 · Interpretability Considerations

  • ✅ Coefficients still have meaning: "each degree increase in temperature changes demand by X"
  • ✅ Coefficient paths show which features are most robust to regularization
  • ⚠️ Coefficients are biased (shrunk) — they underestimate the true effect
  • ⚠️ All features remain in the model — can't say "these features don't matter"

12 · Complete Working Code

import numpy as np
from sklearn.linear_model import Ridge, RidgeCV, LinearRegression
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
from sklearn.model_selection import cross_val_score, KFold
from sklearn.metrics import mean_squared_error
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

# ─── Step 1: Synthetic energy data with correlated features ───
np.random.seed(42)
n = 300

temperature = np.random.uniform(0, 40, n)
humidity = 80 - 0.8 * temperature + np.random.normal(0, 5, n)  # correlated!
wind = np.random.uniform(0, 30, n)
day_of_week = np.random.randint(0, 7, n)
holiday = np.random.choice([0, 1], n, p=[0.9, 0.1])

# 5 lagged demand values (correlated with each other)
lag1 = np.random.uniform(100, 500, n)
lag2 = lag1 * 0.9 + np.random.normal(0, 20, n)
lag3 = lag2 * 0.9 + np.random.normal(0, 20, n)
lag4 = lag3 * 0.9 + np.random.normal(0, 20, n)
lag5 = lag4 * 0.9 + np.random.normal(0, 20, n)

X = np.column_stack([temperature, humidity, wind, day_of_week, holiday,
                      lag1, lag2, lag3, lag4, lag5])
feature_names = ['temp', 'humidity', 'wind', 'day', 'holiday',
                 'lag1', 'lag2', 'lag3', 'lag4', 'lag5']

# True demand
demand = (200 + 5 * temperature - 2 * humidity + 1 * wind
          + 0.3 * lag1 + 0.1 * lag2
          + np.random.normal(0, 20, n))

# ─── Step 2: OLS vs Ridge coefficient comparison ───
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

ols = LinearRegression().fit(X_scaled, demand)
ridge_low = Ridge(alpha=0.1).fit(X_scaled, demand)
ridge_med = Ridge(alpha=10).fit(X_scaled, demand)
ridge_high = Ridge(alpha=1000).fit(X_scaled, demand)

print("=== Coefficient Comparison (scaled features) ===")
print(f"{'Feature':>10} {'OLS':>10} {'Ridge 0.1':>10} {'Ridge 10':>10} {'Ridge 1000':>10}")
print("-" * 55)
for i, name in enumerate(feature_names):
    print(f"{name:>10} {ols.coef_[i]:>10.3f} {ridge_low.coef_[i]:>10.3f} "
          f"{ridge_med.coef_[i]:>10.3f} {ridge_high.coef_[i]:>10.3f}")

# ─── Step 3: RidgeCV — automatic alpha selection ───
alphas = np.logspace(-2, 4, 50)
ridge_cv = RidgeCV(alphas=alphas, cv=5)
ridge_cv.fit(X_scaled, demand)
print(f"\n=== RidgeCV ===")
print(f"  Best alpha: {ridge_cv.alpha_:.4f}")
print(f"  R² score:   {ridge_cv.score(X_scaled, demand):.4f}")

# ─── Step 4: Cross-validated comparison ───
print(f"\n=== CV Comparison: OLS vs Ridge ===")
cv = KFold(n_splits=5, shuffle=True, random_state=42)

for name, model in [('OLS', Pipeline([('s', StandardScaler()), ('m', LinearRegression())])),
                     ('Ridge (α=best)', Pipeline([('s', StandardScaler()),
                                                   ('m', Ridge(alpha=ridge_cv.alpha_))]))]:
    scores = cross_val_score(model, X, demand, cv=cv,
                             scoring='neg_root_mean_squared_error')
    print(f"  {name:>16}: RMSE = {-scores.mean():.2f} ± {scores.std():.2f}")

# ─── Step 5: Coefficient path plot ───
alphas_path = np.logspace(-2, 5, 100)
coef_paths = []
for a in alphas_path:
    r = Ridge(alpha=a).fit(X_scaled, demand)
    coef_paths.append(r.coef_)
coef_paths = np.array(coef_paths)

fig, ax = plt.subplots(figsize=(10, 6))
for i, name in enumerate(feature_names):
    ax.plot(alphas_path, coef_paths[:, i], label=name)
ax.set_xscale('log')
ax.axvline(x=ridge_cv.alpha_, color='#f85149', linestyle='--', label=f'Best α={ridge_cv.alpha_:.2f}')
ax.set_xlabel('Alpha (regularization strength)')
ax.set_ylabel('Coefficient value')
ax.set_title('Ridge Coefficient Paths')
ax.legend(bbox_to_anchor=(1.05, 1), loc='upper left', fontsize=8)
ax.grid(True, alpha=0.2)
plt.tight_layout()
plt.savefig('project_17_ridge_paths.png', dpi=100, bbox_inches='tight')
print(f"\n✅ Coefficient paths saved as project_17_ridge_paths.png")
