Project 33: Calibrating Probability Outputs — Trust Your Model's Confidence

Difficulty: Hard  |  Builds on: Projects 10, 32

1 · Hypothetical Problem

  • You're building a medical screening system that flags patients for follow-up based on disease risk probability
  • Your stacking ensemble (Project 32) achieves 0.92 AUC — excellent discrimination
  • But when the model says "70% risk," only 45% of those patients actually have the disease
  • Doctors need trustworthy probabilities to make treatment decisions — not just rankings
  • Goal → calibrate the model so that predicted probabilities match observed frequencies

2 · How to Approach It

  1. Recall from Project 10: logistic regression outputs probabilities via the sigmoid — these are naturally well-calibrated
  2. Tree-based models and SVMs often produce poorly calibrated probabilities despite good AUC
  3. A reliability diagram (calibration curve) reveals the gap between predicted and actual probabilities
  4. Calibration methods (Platt scaling, isotonic regression) fix the mapping without changing the model's ranking
  5. Use CalibratedClassifierCV to apply calibration with proper cross-validation

3 · How Can We Solve It

  1. Train several classifiers and plot their reliability diagrams
  2. Measure calibration error (Brier score, Expected Calibration Error)
  3. Apply Platt scaling (sigmoid) and isotonic regression calibration
  4. Compare before/after calibration on reliability diagrams
  5. Show that calibration improves probability quality without hurting AUC

4 · What Should Do

  • ✅ Always check calibration when probabilities drive decisions — not just when AUC matters
  • ✅ Use reliability diagrams as the primary diagnostic — visual and intuitive
  • ✅ Use CalibratedClassifierCV with cv=5 — it handles the train/calibration split properly
  • ✅ Prefer isotonic regression when you have enough data (>1,000 samples) — more flexible than Platt
  • ✅ Report Brier score alongside AUC — Brier measures both discrimination AND calibration

5 · What Should NOT Do

  • ❌ Do NOT assume good AUC means good calibration — AUC only measures ranking, not probability quality
  • ❌ Do NOT calibrate on the training set — use a held-out calibration set or cross-validation
  • ❌ Do NOT use isotonic regression on tiny datasets — it overfits; use Platt (sigmoid) instead
  • ❌ Do NOT calibrate logistic regression — it's already well-calibrated by design (Project 10)
  • ❌ Do NOT expect calibration to improve AUC — it fixes probability quality, not ranking ability

6 · When to Use What

  • 🔹 Platt scaling (sigmoid) → when calibration curve is S-shaped; works well with small datasets; 2 parameters
  • 🔹 Isotonic regression → when calibration curve is non-monotonic or complex; needs more data; non-parametric
  • 🔹 No calibration needed → logistic regression, well-tuned neural networks (already calibrated)
  • 🔹 Calibration essential → Random Forest, SVM, GBM, stacking ensembles — all tend to be miscalibrated
  • 🔹 Temperature scaling → popular in deep learning; single parameter; similar to Platt but simpler

7 · When NOT to Use

  • 🚫 When you only care about ranking (AUC) — calibration doesn't change the order of predictions
  • 🚫 When your model is already well-calibrated — check the reliability diagram first
  • 🚫 When you have very few samples — isotonic regression will overfit; even Platt may be unreliable
  • 🚫 When the decision threshold is fixed — if you always threshold at 0.5, calibration doesn't change the outcome

8 · How It Only Works — Internal Mechanics

What Is Calibration?

  • A model is calibrated if: among all samples where it predicts P(y=1) = 0.7, exactly 70% are actually positive
  • Formally: P(Y=1 | f(X)=p) = p for all p ∈ [0, 1]
  • Calibration is about the absolute quality of probabilities, not their relative ordering
  • A perfectly calibrated model with AUC=0.6 is more useful for decision-making than an uncalibrated model with AUC=0.9

Reliability Diagram — How to Read It

  • Bin predictions into groups (e.g., 0.0-0.1, 0.1-0.2, ..., 0.9-1.0)
  • For each bin: x-axis = mean predicted probability, y-axis = actual fraction of positives
  • Perfect calibration = diagonal line (predicted matches actual)
  • Above diagonal = model is underconfident (predicts 0.3 but actual rate is 0.5)
  • Below diagonal = model is overconfident (predicts 0.8 but actual rate is 0.5)

Platt Scaling (Sigmoid Calibration)

  • Fits a logistic regression on the model's raw scores: P_calibrated = 1 / (1 + exp(-(a * f(x) + b)))
  • Only 2 parameters (a, b) — low risk of overfitting
  • Assumes the calibration curve is sigmoid-shaped — works well for SVMs and boosted models
  • Originally proposed by John Platt for calibrating SVM outputs

Isotonic Regression Calibration

  • Fits a non-decreasing step function mapping raw scores to calibrated probabilities
  • Non-parametric — can fix any shape of miscalibration
  • More flexible than Platt but needs more data to avoid overfitting
  • Preserves the ranking (monotonic) — AUC is unchanged after calibration

Brier Score and ECE

  • Brier score: BS = mean((p_predicted - y_actual)²) — lower is better; measures both calibration and discrimination
  • Expected Calibration Error (ECE): weighted average of |predicted - actual| across bins
  • Brier score decomposes into: calibration error + refinement (discrimination) + uncertainty
  • A model can have low Brier score with bad calibration if discrimination is excellent — check both

9 · Common Pitfalls and Failure Modes

  • ⚠️ Calibrating on training data — the model's training predictions are overconfident; always use held-out data
  • ⚠️ Isotonic on small data — with <500 calibration samples, isotonic regression creates a jagged, overfit mapping
  • ⚠️ Confusing AUC with calibration — AUC=0.95 doesn't mean probabilities are trustworthy
  • ⚠️ Too few bins in reliability diagram — 10 bins is standard; too few hides miscalibration, too many is noisy
  • ⚠️ Calibrating after threshold selection — calibrate first, then choose the threshold on calibrated probabilities

10 · Scalability Considerations

  • Calibration is cheap — Platt scaling fits 2 parameters; isotonic regression is O(n log n)
  • CalibratedClassifierCV with cv=5 retrains the base model 5 times — the bottleneck is the base model, not calibration
  • At inference, calibration adds negligible overhead — one sigmoid or one lookup in a step function
  • For very large datasets, method='prefit' lets you calibrate a pre-trained model without retraining
  • Calibration should be refreshed when the data distribution shifts — monitor with reliability diagrams in production

11 · Interpretability Considerations

  • ✅ Calibrated probabilities are directly interpretable: "This patient has a 73% risk" means something concrete
  • ✅ Reliability diagrams are intuitive for non-technical stakeholders — "the dots should be on the line"
  • ✅ Brier score is easy to explain: "average squared error of our probability predictions"
  • ⚠️ Calibration doesn't explain WHY the model predicts a certain probability — use SHAP (Project 27) for that
  • ⚠️ In medical/legal contexts, calibrated probabilities carry more weight than raw model scores

12 · Complete Working Code

import numpy as np
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split, StratifiedKFold
from sklearn.linear_model import LogisticRegression
from sklearn.ensemble import RandomForestClassifier, GradientBoostingClassifier
from sklearn.svm import SVC
from sklearn.calibration import CalibratedClassifierCV, calibration_curve
from sklearn.metrics import brier_score_loss, roc_auc_score, log_loss
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

# ─── Step 1: Dataset ───
np.random.seed(42)
X, y = make_classification(n_samples=8000, n_features=20,
                           n_informative=10, n_redundant=5,
                           flip_y=0.03, random_state=42)

X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, stratify=y, random_state=42)

print(f"=== Dataset ===")
print(f"  Train: {X_train.shape[0]}, Test: {X_test.shape[0]}")

# ─── Step 2: Train models with different calibration properties ───
models = {
    'Logistic Regression': LogisticRegression(max_iter=1000, random_state=42),
    'Random Forest': RandomForestClassifier(n_estimators=200, max_depth=10, random_state=42),
    'Gradient Boosting': GradientBoostingClassifier(n_estimators=200, learning_rate=0.1, random_state=42),
    'SVM (RBF)': SVC(probability=True, kernel='rbf', random_state=42),
}

print(f"\n=== Before Calibration ===")
print(f"{'Model':<25} {'AUC':>8} {'Brier':>8} {'Log Loss':>10}")
print(f"{'-'*55}")

uncalibrated_results = {}
for name, model in models.items():
    model.fit(X_train, y_train)
    probs = model.predict_proba(X_test)[:, 1]
    auc = roc_auc_score(y_test, probs)
    brier = brier_score_loss(y_test, probs)
    ll = log_loss(y_test, probs)
    uncalibrated_results[name] = {'probs': probs, 'auc': auc, 'brier': brier, 'logloss': ll}
    print(f"{name:<25} {auc:>8.4f} {brier:>8.4f} {ll:>10.4f}")

# ─── Step 3: Apply calibration ───
print(f"\n=== After Calibration (Platt Scaling) ===")
print(f"{'Model':<25} {'AUC':>8} {'Brier':>8} {'Log Loss':>10}")
print(f"{'-'*55}")

platt_results = {}
for name, model in models.items():
    cal_model = CalibratedClassifierCV(model, method='sigmoid', cv=5)
    cal_model.fit(X_train, y_train)
    probs = cal_model.predict_proba(X_test)[:, 1]
    auc = roc_auc_score(y_test, probs)
    brier = brier_score_loss(y_test, probs)
    ll = log_loss(y_test, probs)
    platt_results[name] = {'probs': probs, 'auc': auc, 'brier': brier, 'logloss': ll}
    print(f"{name:<25} {auc:>8.4f} {brier:>8.4f} {ll:>10.4f}")

print(f"\n=== After Calibration (Isotonic Regression) ===")
print(f"{'Model':<25} {'AUC':>8} {'Brier':>8} {'Log Loss':>10}")
print(f"{'-'*55}")

isotonic_results = {}
for name, model in models.items():
    cal_model = CalibratedClassifierCV(model, method='isotonic', cv=5)
    cal_model.fit(X_train, y_train)
    probs = cal_model.predict_proba(X_test)[:, 1]
    auc = roc_auc_score(y_test, probs)
    brier = brier_score_loss(y_test, probs)
    ll = log_loss(y_test, probs)
    isotonic_results[name] = {'probs': probs, 'auc': auc, 'brier': brier, 'logloss': ll}
    print(f"{name:<25} {auc:>8.4f} {brier:>8.4f} {ll:>10.4f}")

# ─── Step 4: Reliability diagrams ───
fig, axes = plt.subplots(2, 2, figsize=(14, 12))
axes = axes.flatten()

for idx, (name, _) in enumerate(models.items()):
    ax = axes[idx]

    # Uncalibrated
    prob_true_unc, prob_pred_unc = calibration_curve(
        y_test, uncalibrated_results[name]['probs'], n_bins=10, strategy='uniform')
    ax.plot(prob_pred_unc, prob_true_unc, 's-', color='#f85149', label='Uncalibrated', linewidth=2)

    # Platt
    prob_true_pl, prob_pred_pl = calibration_curve(
        y_test, platt_results[name]['probs'], n_bins=10, strategy='uniform')
    ax.plot(prob_pred_pl, prob_true_pl, 'o-', color='#58a6ff', label='Platt (sigmoid)', linewidth=2)

    # Isotonic
    prob_true_iso, prob_pred_iso = calibration_curve(
        y_test, isotonic_results[name]['probs'], n_bins=10, strategy='uniform')
    ax.plot(prob_pred_iso, prob_true_iso, '^-', color='#3fb950', label='Isotonic', linewidth=2)

    # Perfect calibration line
    ax.plot([0, 1], [0, 1], '--', color='#c9d1d9', alpha=0.5, label='Perfect')

    ax.set_xlabel('Mean Predicted Probability')
    ax.set_ylabel('Fraction of Positives')
    ax.set_title(f'{name}')
    ax.legend(loc='lower right', fontsize=8)
    ax.grid(True, alpha=0.2)
    ax.set_xlim(-0.05, 1.05)
    ax.set_ylim(-0.05, 1.05)

plt.suptitle('Reliability Diagrams — Before and After Calibration', fontsize=14, color='#58a6ff')
plt.tight_layout()
plt.savefig('project_33_calibration.png', dpi=100, bbox_inches='tight')
print(f"\n✅ Reliability diagrams saved as project_33_calibration.png")

# ─── Step 5: Brier score improvement summary ───
print(f"\n=== Brier Score Improvement Summary ===")
print(f"{'Model':<25} {'Before':>8} {'Platt':>8} {'Isotonic':>8} {'Best Δ':>8}")
print(f"{'-'*60}")
for name in models:
    before = uncalibrated_results[name]['brier']
    platt = platt_results[name]['brier']
    iso = isotonic_results[name]['brier']
    best_after = min(platt, iso)
    delta = before - best_after
    print(f"{name:<25} {before:>8.4f} {platt:>8.4f} {iso:>8.4f} {delta:>+8.4f}")

# ─── Step 6: ECE calculation ───
def expected_calibration_error(y_true, y_prob, n_bins=10):
    bin_edges = np.linspace(0, 1, n_bins + 1)
    ece = 0.0
    for i in range(n_bins):
        mask = (y_prob >= bin_edges[i]) & (y_prob < bin_edges[i+1])
        if mask.sum() == 0:
            continue
        bin_acc = y_true[mask].mean()
        bin_conf = y_prob[mask].mean()
        ece += mask.sum() * abs(bin_acc - bin_conf)
    return ece / len(y_true)

print(f"\n=== Expected Calibration Error (ECE) ===")
for name in models:
    ece_before = expected_calibration_error(y_test, uncalibrated_results[name]['probs'])
    ece_platt = expected_calibration_error(y_test, platt_results[name]['probs'])
    ece_iso = expected_calibration_error(y_test, isotonic_results[name]['probs'])
    print(f"  {name:<25} Before: {ece_before:.4f}  Platt: {ece_platt:.4f}  Isotonic: {ece_iso:.4f}")

  • Logistic regression (Project 10) is naturally well-calibrated; tree ensembles and SVMs usually need calibration
