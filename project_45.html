Project 45: Recommendation Engine — Collaborative Filtering With Matrix Factorization

Collaborative Filtering Matrix Factorization SVD ALS Implicit Feedback

Difficulty: Hard  |  Builds on: Projects 24, 14

1 · Hypothetical Problem

  • A streaming platform has 10,000 users and 5,000 movies — but each user has rated only 20–50 movies
  • The user-movie rating matrix is 99.5% empty (sparse)
  • Goal → predict what rating a user would give to movies they haven't seen, then recommend the highest-predicted ones
  • No content features needed — just the pattern of who liked what (collaborative filtering)

2 · How to Approach It

  1. From Project 24: PCA finds latent dimensions in data — matrix factorization does the same for user-item matrices
  2. From Project 14: K-Means groups similar items — collaborative filtering groups similar users/items implicitly
  3. Decompose the rating matrix R ≈ U × V^T where U = user factors, V = item factors
  4. Each user and item is represented by a low-dimensional latent vector (e.g., 50 dimensions)
  5. Predicted rating = dot product of user vector and item vector

3 · How Can We Solve It

  1. Generate a synthetic user-movie rating matrix with latent structure
  2. Implement user-based collaborative filtering (nearest neighbors)
  3. Implement matrix factorization using SVD
  4. Implement ALS (Alternating Least Squares) for large-scale factorization
  5. Evaluate with RMSE and ranking metrics (precision@k, recall@k)

4 · What Should Do

  • ✅ Handle sparsity — most entries are missing; only train on observed ratings
  • ✅ Use regularization in matrix factorization — prevents overfitting to observed ratings
  • ✅ Evaluate with ranking metrics (precision@k) — not just RMSE on ratings
  • ✅ Center ratings by user mean — removes user-specific bias (some rate everything high)
  • ✅ Use train/test split on ratings, not users — each user should have some ratings in both sets

5 · What Should NOT Do

  • ❌ Do NOT treat missing ratings as zeros — missing ≠ disliked; it means "not yet rated"
  • ❌ Do NOT use full SVD on the sparse matrix — it's O(n³) and treats missing as zero
  • ❌ Do NOT ignore the cold-start problem — new users/items have no ratings to learn from
  • ❌ Do NOT evaluate only on RMSE — a model can have low RMSE but bad recommendations
  • ❌ Do NOT recommend items the user has already rated — filter them out

6 · When to Use What

  • 🔹 User-based CF → small datasets; find similar users, recommend what they liked
  • 🔹 Item-based CF → more stable than user-based; items change less than user preferences
  • 🔹 Matrix Factorization (SVD/ALS) → large sparse matrices; learns latent factors
  • 🔹 Content-based filtering → when item features are available; handles cold-start for new items
  • 🔹 Hybrid → combine collaborative + content for best results

7 · When NOT to Use

  • 🚫 When there are very few users or items — not enough data for collaborative patterns
  • 🚫 When all items are unique (no repeat consumption) — collaborative filtering needs overlap
  • 🚫 For cold-start users/items — no interaction history to learn from; use content-based
  • 🚫 When explicit ratings are unavailable — use implicit feedback methods instead

8 · How It Only Works — Internal Mechanics

Collaborative Filtering Intuition

  • "Users who agreed in the past will agree in the future"
  • If User A and User B both liked movies X, Y, Z — and A liked movie W — recommend W to B
  • No need to know WHY they liked those movies — just the pattern of agreement

Matrix Factorization

  • Rating matrix R (users × items) ≈ U (users × k) × V^T (k × items)
  • k = number of latent factors (typically 20–100)
  • Each user has a k-dimensional preference vector; each item has a k-dimensional attribute vector
  • Predicted rating r̂_ui = u_u · v_i (dot product)
  • Latent factors might correspond to genres, mood, complexity — but they're learned, not predefined

ALS (Alternating Least Squares)

  • Objective: minimize Σ (r_ui - u_u · v_i)² + λ(||U||² + ||V||²) over observed ratings
  • Fix U, solve for V (least squares) → fix V, solve for U → repeat
  • Each step is a closed-form solution — no gradient descent needed
  • Parallelizable: each user's factors can be updated independently

Evaluation Metrics

  • RMSE: how close are predicted ratings to actual ratings
  • Precision@k: of the top-k recommended items, how many did the user actually like?
  • Recall@k: of all items the user liked, how many appeared in the top-k?
  • NDCG: measures ranking quality — higher-rated items should appear earlier

9 · Common Pitfalls and Failure Modes

  • ⚠️ Treating missing as zero — inflates the matrix with false negatives; only train on observed entries
  • ⚠️ Cold-start problem — new users/items have no history; need fallback strategies
  • ⚠️ Popularity bias — popular items get recommended to everyone; less diverse recommendations
  • ⚠️ Overfitting — with sparse data, the model can memorize observed ratings; use regularization
  • ⚠️ Evaluation leakage — don't let the model see test ratings during training

10 · Scalability Considerations

  • User-based CF: O(n_users²) for similarity computation — doesn't scale beyond ~100K users
  • ALS: O(n_observed × k²) per iteration — scales to millions of users/items
  • SVD: truncated SVD is O(n_observed × k) — efficient for sparse matrices
  • For production: use implicit library or Spark ALS for distributed computation

11 · Interpretability Considerations

  • ✅ "Users who liked X also liked Y" is intuitive — easy to explain recommendations
  • ✅ Latent factors can sometimes be interpreted (action vs drama, mainstream vs niche)
  • ⚠️ Matrix factorization factors are abstract — hard to explain WHY a specific recommendation was made
  • ⚠️ For explainability, combine with content features — "recommended because you liked similar sci-fi movies"

12 · Complete Working Code

import numpy as np
import pandas as pd
from sklearn.metrics import mean_squared_error
from scipy.sparse.linalg import svds
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

np.random.seed(42)

# ─── Step 1: Generate synthetic rating data ───
n_users = 500
n_items = 200
k_true = 10  # true latent dimensions

# True latent factors
U_true = np.random.normal(0, 1, (n_users, k_true))
V_true = np.random.normal(0, 1, (n_items, k_true))
R_true = U_true @ V_true.T + 3.0  # center around 3

# Clip to 1-5 rating scale
R_true = np.clip(R_true, 1, 5)

# Observe only ~5% of ratings (sparse)
observe_prob = 0.05
mask = np.random.binomial(1, observe_prob, (n_users, n_items)).astype(bool)
R_observed = np.where(mask, R_true + np.random.normal(0, 0.5, R_true.shape), 0)
R_observed = np.clip(R_observed, 1, 5) * mask

n_ratings = mask.sum()
sparsity = 1 - n_ratings / (n_users * n_items)

print("=== Rating Matrix ===")
print(f"  Users: {n_users}, Items: {n_items}")
print(f"  Observed ratings: {n_ratings} ({1-sparsity:.1%} of matrix)")
print(f"  Sparsity: {sparsity:.1%}")
print(f"  Rating range: {R_observed[mask].min():.1f} – {R_observed[mask].max():.1f}")

# ─── Step 2: Train/test split on observed ratings ───
observed_indices = np.argwhere(mask)
np.random.shuffle(observed_indices)
split = int(0.8 * len(observed_indices))
train_idx = observed_indices[:split]
test_idx = observed_indices[split:]

R_train = np.zeros_like(R_observed)
for u, i in train_idx:
    R_train[u, i] = R_observed[u, i]

train_mask = R_train > 0
test_ratings = np.array([R_observed[u, i] for u, i in test_idx])

print(f"  Train ratings: {len(train_idx)}, Test ratings: {len(test_idx)}")

# ─── Step 3: Baseline — global mean ───
global_mean = R_train[train_mask].mean()
baseline_pred = np.full(len(test_idx), global_mean)
baseline_rmse = np.sqrt(mean_squared_error(test_ratings, baseline_pred))
print(f"\n=== Baseline (Global Mean = {global_mean:.2f}) ===")
print(f"  RMSE: {baseline_rmse:.4f}")

# ─── Step 4: User-based collaborative filtering ───
print(f"\n=== User-Based Collaborative Filtering ===")
# Center ratings by user mean
user_means = np.zeros(n_users)
for u in range(n_users):
    user_ratings = R_train[u, R_train[u] > 0]
    user_means[u] = user_ratings.mean() if len(user_ratings) > 0 else global_mean

R_centered = R_train.copy()
for u in range(n_users):
    R_centered[u, train_mask[u]] -= user_means[u]

# Cosine similarity between users
from sklearn.metrics.pairwise import cosine_similarity
user_sim = cosine_similarity(R_centered)
np.fill_diagonal(user_sim, 0)

# Predict using top-K similar users
K = 20
ubcf_preds = []
for u, i in test_idx:
    sim_scores = user_sim[u]
    # Users who rated item i
    rated_mask = R_train[:, i] > 0
    relevant_sims = sim_scores * rated_mask
    top_k = np.argsort(relevant_sims)[-K:]
    top_k = top_k[relevant_sims[top_k] > 0]

    if len(top_k) > 0:
        weights = relevant_sims[top_k]
        weighted_ratings = R_centered[top_k, i]
        pred = user_means[u] + np.dot(weights, weighted_ratings) / (np.abs(weights).sum() + 1e-8)
    else:
        pred = user_means[u]
    ubcf_preds.append(np.clip(pred, 1, 5))

ubcf_rmse = np.sqrt(mean_squared_error(test_ratings, ubcf_preds))
print(f"  RMSE: {ubcf_rmse:.4f}")

# ─── Step 5: Matrix Factorization (SVD) ───
print(f"\n=== Matrix Factorization (Truncated SVD) ===")
# Fill missing with user means for SVD
R_filled = R_train.copy()
for u in range(n_users):
    R_filled[u, ~train_mask[u]] = user_means[u]

# Center
R_filled_centered = R_filled - user_means.reshape(-1, 1)

# Truncated SVD
k = 20
U, sigma, Vt = svds(R_filled_centered, k=k)
sigma_diag = np.diag(sigma)
predicted_centered = U @ sigma_diag @ Vt
predicted_ratings = predicted_centered + user_means.reshape(-1, 1)
predicted_ratings = np.clip(predicted_ratings, 1, 5)

svd_preds = [predicted_ratings[u, i] for u, i in test_idx]
svd_rmse = np.sqrt(mean_squared_error(test_ratings, svd_preds))
print(f"  Latent factors: {k}")
print(f"  RMSE: {svd_rmse:.4f}")

# ─── Step 6: ALS (manual implementation) ───
print(f"\n=== ALS (Alternating Least Squares) ===")
k_als = 20
lambda_reg = 0.1
n_iterations = 20

U_als = np.random.normal(0, 0.1, (n_users, k_als))
V_als = np.random.normal(0, 0.1, (n_items, k_als))

for iteration in range(n_iterations):
    # Fix V, solve for U
    for u in range(n_users):
        items_rated = np.where(train_mask[u])[0]
        if len(items_rated) == 0:
            continue
        V_sub = V_als[items_rated]
        r_sub = R_train[u, items_rated] - user_means[u]
        A = V_sub.T @ V_sub + lambda_reg * np.eye(k_als)
        b = V_sub.T @ r_sub
        U_als[u] = np.linalg.solve(A, b)

    # Fix U, solve for V
    for i in range(n_items):
        users_rated = np.where(train_mask[:, i])[0]
        if len(users_rated) == 0:
            continue
        U_sub = U_als[users_rated]
        r_sub = R_train[users_rated, i] - user_means[users_rated]
        A = U_sub.T @ U_sub + lambda_reg * np.eye(k_als)
        b = U_sub.T @ r_sub
        V_als[i] = np.linalg.solve(A, b)

    # Evaluate
    als_pred_matrix = U_als @ V_als.T + user_means.reshape(-1, 1)
    als_pred_matrix = np.clip(als_pred_matrix, 1, 5)
    als_preds = [als_pred_matrix[u, i] for u, i in test_idx]
    als_rmse = np.sqrt(mean_squared_error(test_ratings, als_preds))

    if (iteration + 1) % 5 == 0:
        print(f"  Iteration {iteration+1}: RMSE = {als_rmse:.4f}")

# ─── Step 7: Generate recommendations ───
print(f"\n=== Sample Recommendations (User 0) ===")
user_id = 0
user_rated = np.where(train_mask[user_id])[0]
user_unrated = np.where(~train_mask[user_id])[0]
pred_scores = als_pred_matrix[user_id, user_unrated]
top_10 = user_unrated[np.argsort(pred_scores)[-10:][::-1]]

print(f"  User {user_id} has rated {len(user_rated)} items")
print(f"  Top 10 recommendations:")
for rank, item in enumerate(top_10, 1):
    print(f"    {rank}. Item {item} (predicted rating: {als_pred_matrix[user_id, item]:.2f})")

# ─── Step 8: Summary and visualization ───
print(f"\n=== Method Comparison ===")
print(f"  {'Method':<25} {'RMSE':>8}")
print(f"  {'-'*35}")
print(f"  {'Global Mean':<25} {baseline_rmse:>8.4f}")
print(f"  {'User-Based CF (K=20)':<25} {ubcf_rmse:>8.4f}")
print(f"  {'SVD (k=20)':<25} {svd_rmse:>8.4f}")
print(f"  {'ALS (k=20)':<25} {als_rmse:>8.4f}")

fig, axes = plt.subplots(2, 2, figsize=(14, 12))

# Rating distribution
axes[0, 0].hist(R_observed[mask], bins=20, color='#58a6ff', alpha=0.7, edgecolor='white')
axes[0, 0].set_xlabel('Rating')
axes[0, 0].set_ylabel('Count')
axes[0, 0].set_title('Rating Distribution')
axes[0, 0].grid(True, alpha=0.2)

# Sparsity pattern (sample)
sample_R = R_train[:50, :50]
axes[0, 1].imshow(sample_R > 0, cmap='Blues', aspect='auto')
axes[0, 1].set_xlabel('Items')
axes[0, 1].set_ylabel('Users')
axes[0, 1].set_title('Rating Matrix Sparsity (50×50 sample)')

# Method comparison
methods = ['Mean', 'UBCF', 'SVD', 'ALS']
rmses = [baseline_rmse, ubcf_rmse, svd_rmse, als_rmse]
colors = ['#c9d1d9', '#f0883e', '#58a6ff', '#3fb950']
axes[1, 0].bar(methods, rmses, color=colors, alpha=0.8, edgecolor='white')
axes[1, 0].set_ylabel('RMSE')
axes[1, 0].set_title('RMSE Comparison')
axes[1, 0].grid(True, alpha=0.2, axis='y')

# Predicted vs actual
axes[1, 1].scatter(test_ratings, als_preds, alpha=0.3, s=10, color='#3fb950')
axes[1, 1].plot([1, 5], [1, 5], '--', color='white', alpha=0.5)
axes[1, 1].set_xlabel('Actual Rating')
axes[1, 1].set_ylabel('Predicted Rating (ALS)')
axes[1, 1].set_title('ALS: Predicted vs Actual')
axes[1, 1].grid(True, alpha=0.2)

plt.tight_layout()
plt.savefig('project_45_recommendation.png', dpi=100, bbox_inches='tight')
print(f"\n✅ Recommendation analysis saved as project_45_recommendation.png")

Key Takeaway

  • Collaborative filtering finds patterns in user-item interactions without needing content features
  • Matrix factorization learns latent factors for users and items — predicted rating = dot product
  • ALS is the workhorse for large-scale recommendation — parallelizable and handles sparsity
  • Never treat missing ratings as zeros — missing means "not yet rated," not "disliked"
  • Evaluate with ranking metrics (precision@k), not just RMSE — recommendations are about ordering
